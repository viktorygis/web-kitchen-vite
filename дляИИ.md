# Шаблон: инструкция по созданию страниц раздела

Этот файл — универсальный шаблон для разработчиков и ИИ-ассистентов. Используйте его при создании или правке страниц в любом разделе. Цель: единообразие, доступность и безопасность контента.

Репозиторий: **viktorygis/web-kitchen-vite**

---

## Краткие правила

- Имя файла: kebab-case, начинается с имени темы, например `async-await.html`, `functions-basics.html`.
- Страницы лежат в папках по разделу: `src/about-js-teoria/`, `src/about-css/`, `src/nodejs/` и т.д.
- Формат: HTML с `@@include` partial-ами (Vite + html-bundler).
- Код — только в блоке `div.code-block > pre > code`.
- Запрещено: `grid-column-*`, `div.example`, любые нестандартные обёртки вокруг `<pre><code>`.
- Заголовки: ровно один `<h1>` на страницу. Подзаголовки — `<h2>`, `<h3>`, `<h4>`.
- Meta description: уникален, 120–160 символов.
- Доступность: `skip-link`, `nav[role="navigation"][aria-label="..."]`, активная ссылка — `aria-current="page"`, `main[tabindex="-1"]`.

## Обязательная структура страницы

```html
<!DOCTYPE html>
<html lang="ru">
@@include('../partials/head.html', {
  "title": "Заголовок — краткое пояснение",
  "description": "Meta description 120–160 символов"
})

<body>
  <a class="skip-link" href="#main">Перейти к содержимому</a>

  <div class="wrapper">
    @@include('../partials/header.html', {})

    <main class="page" id="main" tabindex="-1">

      <nav class="nav-tabs" id="page-nav" role="navigation" aria-label="Навигация по разделам страницы">
        <a href="#intro" class="active" aria-current="page">Введение</a>
        <a href="#concepts">Основные понятия</a>
        <!-- остальные пункты -->
      </nav>

      <!-- секции -->

    </main>

    @@include('../partials/footer.html', {})
  </div>

  @@include('../partials/popup.html', {})
  @@include('../partials/js.html', {})
</body>

</html>
```

> **Важно:** `@@include` для `popup.html` и `js.html` должны идти **после** закрывающего `</div>` обёртки, но **до** `</body>`.

## Обязательное требование: определение в начале каждой секции

Каждая логическая секция и подпункт начинаются с короткого определения:

```html
<p class="note">
  <strong>Тема секции</strong> — краткое точное определение.<br />
  Дополнительное пояснение при необходимости.
</p>
```

- Определение — **перед** любым списком, таблицей, кодом, блоком примера.
- Длина: 1–2 предложения.
- Это улучшает сканируемость, доступность и SEO.

---

## Рекомендуемые секции

1. **Введение** — 1–3 абзаца с определением темы.
2. **Основные понятия** — каждый термин отдельным определением или списком `ul.list-ul`.
3. **Основной контент** — логические подразделы `h2/h3`, каждый начинается с определения.
4. **Примеры** — 5–15 строк с комментариями, каждый сопровождается определением.
5. **Практика / интерактив** — без `eval`, с `aria-live="polite"`.
6. **Ловушки / типичные ошибки** — `div.warning`, каждая проблема с определением.
7. **Советы / шаблоны** — `div.note-block`, каждый блок с определением.
8. **Лучшие практики** — `div.note-block`, каждый пункт с определением и примером кода.
9. **Итоги** — `div.note-block`, краткий список того, что изучили.

## Микро-структура подраздела

Для каждого понятия/метода/типа:

1. Определение — `p.note` — обязательно.
2. Особенности — `ul.list-ul` или `ol.list-ol`.
3. Примеры — `div.code-block`.
4. Применение — обычный `<p>`.
5. Ловушки — `div.warning` (если нужно).

Пример подраздела:

```html
<p class="note">
  <strong>Замыкания</strong> — функция вместе с лексическим окружением, в котором она была создана.<br />
  Используются для инкапсуляции состояния и создания приватных данных.
</p>

<ul class="list-ul">
  <li>Сохраняют состояние между вызовами.</li>
  <li>Могут приводить к утечкам памяти при неправильном использовании.</li>
</ul>

<div class="code-block">
  <pre><code>// Пример: простое замыкание
function makeCounter() {
  let count = 0;
  return function() { return ++count; };
}
const c = makeCounter();
console.log(c()); // 1
console.log(c()); // 2</code></pre>
</div>

<p>Применение: счётчики, фабрики функций, приватные переменные в модулях.</p>
```

## Стандартные классы и семантика

### Layout

| Класс | Описание |
|---|---|
| `main.page` | Главный контейнер, `id="main"`, `tabindex="-1"` |
| `section.section` | Логическая секция страницы |
| `div.container` | Ограничение ширины / внутренние отступы |
| `article.article-block` | Самостоятельный смысловой блок внутри секции |

### Навигация

```html
<nav class="nav-tabs" id="page-nav" role="navigation" aria-label="Навигация по разделам страницы">
  <a href="#intro" class="active" aria-current="page">Введение</a>
  <a href="#concepts">Основные понятия</a>
</nav>
```

- Активная ссылка: `class="active"` + `aria-current="page"`.

### Тексты

| Класс | Назначение |
|---|---|
| `p.note` | Определение / ключевой абзац (голубой фон, зелёный `strong`) |
| `p.note.note--info` | Совет / подсказка (жёлтый фон, красный `strong`) |
| `div.note-block` | Информационный блок (советы, итоги, рекомендации) |
| `div.warning` | Предупреждение / типичная ошибка (красный фон) |
| `div.summary` | Итоговый блок (синий фон) |

### Списки — три вида

```html
<!-- Список с маркерами (bullet •) -->
<ul class="list-ul">
  <li>Пункт один</li>
  <li><strong>Термин</strong> — определение термина прямо в li, без вложенных тегов-обёрток.</li>
  <li>С кодом: <code>response.ok</code> — true при статусах 200–299.</li>
</ul>

<!-- Список с нумерацией (1. 2. 3.) -->
<ol class="list-ol">
  <li>Первый шаг</li>
  <li>Второй шаг</li>
</ol>

<!-- Список с иконками/эмодзи -->
<ul class="list-icon">
  <li>✅ Можно делать так</li>
  <li>❌ Нельзя делать так</li>
</ul>
```

**Правила для списков:**
- Текст пункта пишется **прямо внутри `<li>`** — без дополнительных обёрток `<p>`, `<div>`, `<p class="note">`.
- `<strong>`, `<code>`, `<em>` внутри `<li>` — допустимы.
- `<p class="note">` внутри `<li>` — **запрещено**.
- Если нужно дать определение термину — пишите его в `<li>` через тире: `<li><strong>Термин</strong> — определение.</li>`.
- `p.note` используется только как самостоятельный блок **вне** списка — для определения секции или подраздела.
```

> `list-ul` — маркированный (•), `list-ol` — нумерованный (1.), `list-icon` — с эмодзи или SVG-иконкой.

### Таблицы

```html
<table class="table-note">
  <thead>
    <tr>
      <th>Колонка 1</th>
      <th>Колонка 2</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Значение</td>
      <td>Описание</td>
    </tr>
  </tbody>
</table>
```

### Изображения

```html
<!-- Изображение с подписью -->
<figure>
  <img class="note-img" src="/img/тема/название.png" alt="Краткое описание содержимого" />
  <figcaption class="note-caption">Рис. 1. Подпись к изображению</figcaption>
</figure>
```

**Классы изображений:**
- `img.note-img` — стандартное изображение (max-height: 400px, border, border-radius).
- `figcaption.note-caption` — подпись (14px, по центру, жирный).

**Правила:**
- `alt` — всегда. Декоративное → `alt=""`. Смысловое → краткое точное описание.
- Не дублируйте текст статьи в `alt`.
- Подпись в `figcaption` — короткая, не повторяет основной абзац.
- Важную информацию не прячьте только в картинку.

### Код

```html
### Код

- `div.code-block` содержит **только** `<pre><code>...</code></pre>` — ничего больше внутри.
- Заголовок к блоку кода (`<h4>`) ставится **снаружи и перед** `div.code-block`, а не внутри него.
- Запрещено вкладывать в `div.code-block` теги `<h4>`, `<p>`, `<div>` и любые другие обёртки.

```html
<!-- ✅ Правильно -->
<h4>Пример GET-запроса:</h4>
<div class="code-block">
  <pre><code>fetch('/api/data').then(r =&gt; r.json());</code></pre>
</div>

<!-- ❌ Неправильно -->
<div class="code-block">
  <h4>Пример GET-запроса:</h4>
  <pre><code>fetch('/api/data').then(r =&gt; r.json());</code></pre>
</div>
```
```

### Интерактив

```html
<button class="practice-btn" data-code="myKey">Запустить пример</button>

<div class="code-block">
  <pre><code id="output-result" aria-live="polite">// result</code></pre>
</div>
```

---

## Правила для кода и примеров

- Экранируйте спецсимволы в HTML-коде внутри `<code>`:
  - `<` → `&lt;`
  - `>` → `&gt;`
  - `&` → `&amp;`
- Комментарии в коде приветствуются (`// Пример:`, `// Хорошо`, `// Плохо`).
- Парные примеры «правильно / неправильно»:

```html
<div class="code-block">
  <pre><code>// ✅ Правильно:
const name = 'Ivan';

// ❌ Неправильно:
var n = 'Ivan';</code></pre>
</div>
```

- **Запрещено** запускать код через `eval`.

## Интерактивы и JS-паттерны

```js
const examples = {
  key1: () => 'result',
  asyncKey: async () => {
    return 'ok';
  }
};

document.addEventListener('click', async (e) => {
  const btn = e.target.closest('[data-code], .practice-btn');
  if (!btn) return;

  const key = btn.getAttribute('data-code') || btn.dataset.code;
  const fn = examples[key];
  if (!fn) return;

  try {
    const res = fn();
    const output = document.getElementById('output-result');
    if (res instanceof Promise) {
      output.textContent = 'Выполняется...';
      output.textContent = await res;
    } else {
      output.textContent = res;
    }
  } catch (err) {
    document.getElementById('output-result').textContent =
      'Ошибка: ' + (err && err.message ? err.message : String(err));
  }
});
```

- Результат всегда выводить в элемент с `aria-live="polite"`.
- Использовать `<button>`, а не `<a href="#">`.

## Accessibility

| Требование | Реализация |
|---|---|
| Skip link | `<a class="skip-link" href="#main">Перейти к содержимому</a>` |
| Навигация | `nav[role="navigation"][aria-label="Навигация по разделам страницы"]` |
| Активный пункт | `aria-current="page"` на текущей ссылке |
| Фокус на main | `id="main"` + `tabindex="-1"` |
| Динамический вывод | `aria-live="polite"` на элементе с результатом |

## Примеры «правильно / неправильно»

**✅ Правильно — код:**
```html
<div class="code-block">
  <pre><code>fetch('/api/data').then(r =&gt; r.json()).then(data =&gt; console.log(data));</code></pre>

<div class="example">

  </div>
</div>
```

**✅ Правильно — картинка:**

<img class="note-img" src="/img/js/closures.png" alt="Клиент-серверная архитектура" caption="подпись" />
<p class="note-caption">Рис. 1. Подпись</p>

## Чек-лист перед публикацией

- [ ] `title` + `meta description` уникальны, description 120–160 символов.
- [ ] `<h1>` только один, отражает тему страницы.
- [ ] Каждая секция начинается с `<p class="note"><strong>Тема</strong> — ...</p>`.
- [ ] Все `<pre><code>` — только внутри `div.code-block`.
- [ ] Спецсимволы `<`, `>`, `&` экранированы внутри `<code>`.
- [ ] Нет `grid-column-*`, `div.example` или других нестандартных обёрток.
- [ ] Все `id` секций совпадают с `href` в `nav-tabs`.
- [ ] Списки используют классы `list-ul`, `list-ol` или `list-icon`.
- [ ] Изображения — `img.note-img` внутри `<figure>` с `figcaption.note-caption`.
- [ ] Доступность: `skip-link`, `role="navigation"`, `aria-current`, `aria-live`.
- [ ] `@@include` для `popup.html` и `js.html` — после `</div>` обёртки, перед `</body>`.
