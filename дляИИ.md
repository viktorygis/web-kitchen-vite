# Инструкция по созданию страниц раздела «Теория JS» (формат .md)

Этот файл — стандарт для разработчиков и для ИИ-ассистентов. Используйте его как шаблон при добавлении или правке страниц в `/about-js-teoria/`. Всё — единообразно, доступно и безопасно.

## Краткие правила

- Имя файла: kebab-case, начинается с имени раздела (например `async-await.html`, `functions-basics.html`).
- Структура страницы: семантические секции + навигация якорями.
- Код: только в блоках:

```html
<div class="code-block">
  <pre><code>...</code></pre>
</div>
```

- Удалять: `grid-column-*`, `example`, любые кастомные обёртки вокруг `<pre><code>`.
- Один H1 на страницу, meta description уникален (120–160 символов).
- Доступность: skip-link, `role="navigation"`, `aria-current` на текущем пункте, `main tabindex="-1"`.

## Структура страницы (обязательные части)

### Head / meta (через partial)

```html
@@include('../partials/head.html', {
  "title": "Заголовок — краткое пояснение",
  "description": "Meta description 120–160 символов"
})
```

### Skip link + header

```html
<a class="skip-link" href="#main">Перейти к содержимому</a>
@@include('../partials/header.html', {})
```

### Main + nav

```html
<main class="page" id="main" tabindex="-1">
  <nav class="nav-tabs" id="page-nav" role="navigation" aria-label="Навигация по разделам страницы">
    <a href="#intro" class="active" aria-current="page">Введение</a>
    <a href="#concepts">Основные понятия</a>
    ...
  </nav>

  <!-- секции здесь -->
</main>
```

### Footer / scripts

```html
@@include('../partials/footer.html', {})
@@include('../partials/popup.html', {})
@@include('../partials/js.html', {})
```

## Рекомендуемые секции (по порядку)

1. Введение (H1) — 1–3 абзаца: цель страницы, чему научится читатель.
2. Основные понятия — ключевые термины (короткие определения).
3. Основной контент — логические подпункты (h2/h3).
4. Примеры — короткие (5–15 строк), с комментариями.
5. Практика / интерактив — безопасные демо (без `eval`).
6. Ловушки / типичные ошибки (warning).
7. Советы / шаблоны (note).
8. Лучшие практики.
9. Сводка / вывод.

### Микро-структура подраздела (внутри «Основного контента»)

Каждое понятие/тип/метод в «Основном контенте» рекомендуется раскрывать по устойчивому повторяемому паттерну:

1. **Особенности** — короткий список ключевых свойств понятия.
2. **Примеры** — рабочий код в `div.code-block`.
3. **Применение** — список реальных сценариев использования.

Такой паттерн упрощает сканирование страницы и делает все подпункты единообразными даже при разном объёме материала.

## Стандартные классы и их назначение

### Layout

- `main.page` — главный контейнер (`id="main"`, `tabindex="-1"`)
- `section.section` — логическая секция
- `div.container` — внутренний отступ/лимит ширины

### Навигация

- `nav.nav-tabs#page-nav[role="navigation"]` — якорная навигация по странице
- в ссылках: `class="active"` и `aria-current="page"` для текущего пункта

### Тексты

- `p.note` — определение / короткий параграф
- `p.note.note--info` — подсказка / дополнительная информация
- `div.note-block` — крупный информационный блок
- `div.warning` — предупреждение. Сюда обязательно выносить типичные ловушки и «исторические ошибки» языка (например, `typeof null === "object"`, путаница `null`/`undefined`), а не оставлять их в обычных параграфах внутри других секций.
- `ul.list-ul` — обычный список
- `table.table-note` — таблицы/сравнения. Рекомендуется также использовать как итоговую сводную таблицу в конце сложной темы с несколькими похожими сущностями (например, сравнение типов данных) — перед секцией «Ловушки» или «Сводка».

### Код

- `div.code-block` вокруг `<pre><code>`

### Интерактив

- кнопки: `<button class="practice-btn" data-code="key">Название</button>`
- результат: `<div class="code-block"><pre><code id="output-result">// result</code></pre></div>`
- demo inputs: используйте `aria-live="polite"` для результатов

## Формат определений (обязательно)

Каждое определение — в форме:

```html
<p class="note"><strong>Термин</strong> — краткое точное описание.<br />Дополнительная короткая фраза (если нужно).</p>
```

Пример:

```html
<p class="note"><strong>Стрелочные функции</strong> — компактный современный синтаксис для создания функций.<br />Они особенно удобны для коротких действий и колбэков.</p>
```

## Правила для кода и примеров

- Всегда экранировать спецсимволы: `<` → `&lt;`, `>` → `&gt;`, `&` → `&amp;`.
- Не использовать `grid-column-*`, `example` — оборачивать каждый пример только в `div.code-block`.
- Короткие комментарии внутри кода — приветствуются.
- Для тем с типичными ошибками или несколькими способами решения рекомендуется приводить парные примеры прямо в коде с комментариями `// Правильно:` и `// Неправильно:` (или аналогичными), а не только текстовым описанием рядом.
- Не выполнять код через `eval`; для запуска — использовать безопасные демо (jsonplaceholder и т.п.) или внешние песочницы.

## Интерактивы и общая JS-практика

Паттерн: объект `examples = { key: fn }`, делегирование кликов:

```js
document.addEventListener('click', async (e) => {
  const btn = e.target.closest('[data-code], .practice-btn');
  if (btn) {
    const key = btn.getAttribute('data-code') || btn.dataset.code;
    const fn = examples[key];
    if (!fn) return;
    try {
      const res = fn();
      if (res instanceof Promise) {
        document.getElementById('output-result').textContent = 'Выполняется...';
        document.getElementById('output-result').textContent = await res;
      } else {
        document.getElementById('output-result').textContent = res;
      }
    } catch (err) {
      document.getElementById('output-result').textContent = 'Ошибка: ' + (err && err.message ? err.message : String(err));
    }
  }
});
```

Демо-функции для практики могут подключаться отдельным скриптом на странице (через свой `<script>` или partial) — главное, чтобы кнопки имели рабочие обработчики и результат выводился в элемент с `aria-live="polite"`.

Плавная прокрутка для `#page-nav` реализуется в общем JS.

## Accessibility (A11y)

- Skip link: `<a class="skip-link" href="#main">Перейти к содержимому</a>`.
- `nav role="navigation" aria-label="Навигация по разделам страницы"`.
- На активной ссылке: `aria-current="page"`.
- `main` имеет `tabindex="-1"` для фокуса со skip-link.
- Динамический вывод — `aria-live="polite"` для результатов демо.
- Кнопки — элемент `<button>` (вместо ссылок с `href="#"`).

## SEO / meta

- Title и description через `head.html` partial.
- Description: 120–160 символов, уникален.
- H1 только один.
- Опционально: `rel="canonical"`, Open Graph и Twitter meta через partial.

## Примеры «правильно / неправильно»

**Правильно:**

```html
<div class="code-block">
  <pre><code>fetch('/api/data').then(r => r.json()).then(data => console.log(data));</code></pre>
</div>
```

**Неправильно:**

```html
<div class="example">
  <div class="grid-column-50">
    <pre><code>...</code></pre>
  </div>
</div>
```

→ Заменить на `div.code-block` без `example` и `grid-column-*`.
