## Краткие правила

| Правило              | Значение                                                             |
| -------------------- | -------------------------------------------------------------------- |
| **Имя файла**        | `kebab-case.html` (пример: `async-await.html`)                       |
| **Папка**            | По разделу: `src/about-js-teoria/`, `src/nodejs/`, `src/css/`        |
| **Формат**           | HTML + `@@include` partials (Vite)                                   |
| **Заголовки**        | Один `<h1>` на страницу, далее логичный порядок `<h2>/<h3>/<h4>`     |
| **Meta description** | 120–160 символов, уникален                                           |
| **Код**              | Только внутри `div.code-block > pre > code`                          |
| **Запрещено**        | `grid-column-*`, `div.example`, любые `<h*>` внутри `div.code-block` |

## Структура страницы

Рекомендуемая логика образовательной страницы:

- 📄 **Введение** — 1–3 абзаца с определением темы (`p.note` в начале).
- 📋 **Основные понятия** — каждый термин либо отдельным `p.note`, либо пунктом в `ul.list-ul`.
- 📚 **Основной контент** (`h2/h3`):
  - Определение (`p.note`).
  - Особенности (`ul.list-ul` или `ol.list-ol`).
  - Примеры (`div.code-block`).
  - Применение (обычный `<p>`).
- ⚠️ **Типичные ошибки / ловушки** — блоки `div.warning`.
- 💡 **Лучшие практики / советы** — блоки `div.note-block`.
- 📊 **Итоги** — `div.summary` или `div.note-block` с кратким списком выводов.
- ❓ **FAQ** — при необходимости, в конце страницы.

## HTML структура

### Шаблон страницы

```html
<!DOCTYPE html>
<html lang="ru">
  @@include('../partials/head.html', { "title": "Заголовок — краткое пояснение", "description": "Meta description 120–160 символов" })

  <body>
    <a class="skip-link" href="#main">Перейти к содержимому</a>

    <div class="wrapper">
      @@include('../partials/header.html', {})

      <main class="page" id="main" tabindex="-1">
        <nav class="nav-tabs" id="page-nav" role="navigation" aria-label="Навигация по разделам страницы">
          <a href="#intro" class="active" aria-current="page">Введение</a>
          <a href="#concepts">Основные понятия</a>
          <a href="#syntax">Синтаксис</a>
        </nav>

        <section class="section" id="intro">
          <div class="container">
            <h1>Заголовок страницы</h1>
            <!-- контент -->
          </div>
        </section>
      </main>

      @@include('../partials/footer.html', {})
    </div>

    @@include('../partials/popup.html', {}) @@include('../partials/js.html', {})
  </body>
</html>
```

**Важно:** `popup.html` и `js.html` подключаются **после** закрывающего `</div>` обёртки, но **до** `</body>`.

## Классы и элементы

### Определения в начале секции

Каждая логическая секция (и важный подпункт) начинается с определения:

```html
<p class="note">
  <strong>Название темы</strong>
  — краткое точное определение.
  <br />
  Дополнительное пояснение при необходимости.
</p>
```

- Определение идёт **перед** любым списком, таблицей, кодом или примером.
- Длина — 1–2 предложения.

### Текстовые блоки

| Класс            | Назначение                  | Пример                                                                      |
| ---------------- | --------------------------- | --------------------------------------------------------------------------- |
| `p.note`         | Определение, ключевой абзац | `<p class="note"><strong>Замыкание</strong> — функция с её окружением.</p>` |
| `div.note-block` | Советы, рекомендации, итоги | Блок с `<h3>` и `ul.list-ul` внутри                                         |
| `div.warning`    | Ошибки, предупреждения      | Красный фон, формулировка проблемы и решения                                |
| `div.summary`    | Итоговый блок               | Синий фон, краткий список результатов                                       |

### Списки — три вида

```html
<!-- Маркированный список -->
<ul class="list-ul">
  <li>Пункт один</li>
  <li>
    <strong>Термин</strong>
    — определение прямо в li.
  </li>
  <li>
    С кодом:
    <code>response.ok</code>
    — true при статусах 200–299.
  </li>
</ul>

<!-- Нумерованный список -->
<ol class="list-ol">
  <li>Первый шаг</li>
  <li>Второй шаг</li>
</ol>

<!-- Список с эмодзи / иконками -->
<ul class="list-icon">
  <li>✅ Можно делать так</li>
  <li>❌ Нельзя делать так</li>
</ul>
```

**Правила для списков:**

- Текст пункта пишется **прямо внутри `<li>`**, без вложенных `<p>` или `<div>`.
- Допустимы `<strong>`, `<code>`, `<em>` внутри `<li>`.
- `p.note` внутри `<li>` — **запрещено**.
- Если нужно дать определение термину — пишите его через тире:
  `<li><strong>Термин</strong> — определение.</li>`.

### Таблицы

Перед любой таблицей обязательно:

```html
<p class="note">
  <strong>Сравнение методов</strong>
  — преимущества и недостатки каждого подхода.
</p>
```

Сама таблица:

```html
<table class="table-note">
  <thead>
    <tr>
      <th>Метод</th>
      <th>Когда использовать</th>
      <th>Пример</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>
        <strong>forEach</strong>
        — перебирает элементы.
      </td>
      <td>Простой перебор без возврата.</td>
      <td><code>arr.forEach(x => console.log(x))</code></td>
    </tr>
  </tbody>
</table>
```

В ячейках обязательно короткое объяснение, а не просто термин.


## FAQ блок

### Назначение
**FAQ (Frequently Asked Questions)** — аккордеон с вопросами и ответами. Используется для размещения часто задаваемых вопросов в конце страницы или отдельной секции.

### Структура HTML

```html
<section class="section" id="название-faq">
  <div class="container">
    <h2>Часто задаваемые вопросы</h2>

    <div class="faq">

      <div class="faq__item">
        <button class="faq__question">
          Вопрос номер один?
        </button>
        <div class="faq__answer">
          <p>
            Ответ на вопрос. Может быть несколько абзацев или списков.
          </p>
        </div>
      </div>

      <div class="faq__item">
        <button class="faq__question">
          Вопрос номер два?
        </button>
        <div class="faq__answer">
          <p>
            Ответ на второй вопрос.
          </p>
        </div>
      </div>

    </div>

  </div>
</section>
```


### Изображения

```html
<figure>
  <img class="note-img" src="/img/theme/name.png" alt="Краткое описание содержимого" />
  <figcaption class="note-caption">Рис. 1. Подпись к изображению</figcaption>
</figure>
```

**Правила:**

- `alt` — всегда. Декоративное изображение → `alt=""`, смысловое → краткое описание.
- `figcaption` — короткая подпись, не дублирует текст статьи.
- Важную информацию не прятать только в картинку.

## Код и примеры

### Структура блока кода

```html
<!-- ✅ Правильно -->
<h4>Пример GET-запроса:</h4>
<div class="code-block">
  <pre><code>fetch('/api/data').then(r => r.json());</code></pre>
</div>

<!-- ❌ Неправильно -->
<div class="code-block">
  <h4>Заголовок внутри</h4>
  <pre><code>...</code></pre>
</div>
```

**Правила:**

- Заголовок (`<h4>`) — **снаружи**, перед `div.code-block`.
- `div.code-block` содержит **только** `<pre><code>...</code></pre>` — без дополнительных обёрток.
- Экранировать спецсимволы внутри `<code>`:
  - `<` → `&lt;`
  - `>` → `&gt;`
  - `&` → `&amp;`
- Комментарии в коде приветствуются (`// Пример`, `// Хорошо`, `// Плохо`).

### Парные примеры «правильно / неправильно»

```html
<div class="code-block">
  <pre><code>// ✅ Правильно:
const name = 'Ivan';

// ❌ Неправильно:
var n = 'Ivan';</code></pre>
</div>
```

### Интерактивные примеры

HTML:

```html
<button class="practice-btn" data-code="myKey">Запустить пример</button>
<div class="code-block">
  <pre><code id="output-result" aria-live="polite">// результат</code></pre>
</div>
```

JS-паттерн:

```js
const examples = {
  myKey: () => "результат работы",
  asyncKey: async () => await fetch("/api").then((r) => r.json()),
};
```

Результат интерактивов выводится в элемент с `aria-live="polite"`.

### ID секций

Формат: `[тема]-[подтема]`.

**Примеры:**

```text
js-intro, js-syntax, js-closures
node-http, node-middleware
css-grid, css-flexbox
```

**Запрещено:**

```text
section1, block2, test
```

## Доступность

| Требование         | Реализация                                                                                           |
| ------------------ | ---------------------------------------------------------------------------------------------------- |
| Skip link          | `<a class="skip-link" href="#main">Перейти к содержимому</a>`                                        |
| Навигация          | `<nav class="nav-tabs" id="page-nav" role="navigation" aria-label="Навигация по разделам страницы">` |
| Активная ссылка    | `aria-current="page"` на текущей ссылке + `class="active"`                                           |
| Фокус на main      | `id="main"` + `tabindex="-1"`                                                                        |
| Динамический вывод | `aria-live="polite"` на элементе результата                                                          |

## FAQ по гайду

**Q:** Обязательно ли определение перед каждой подсекцией?
**A:** Да. Каждый `h2`, `h3` и логический блок должны начинаться с `p.note`.

**Q:** Можно ли использовать свои CSS-классы?
**A:** Нет. Используем только стандартные классы: `note`, `note-block`, `warning`, `summary`, `list-ul`, `list-ol`, `list-icon`, `code-block`, `table-note`.

**Q:** Максимум строк в примере кода?
**A:** 15–20 строк. Если больше — разбить на несколько примеров или вынести в отдельный файл.

**Q:** Где регистрировать интерактивные примеры?
**A:** В общем файле `src/js/examples.js` или в профильном partial для раздела (по структуре проекта).

**Q:** Можно ли вставлять видео или GIF?
**A:** Видео — по согласованию с командой. GIF — только как декоративные иллюстрации, не как основной источник информации.

## Чек-лист перед публикацией

- [ ] `title` и `meta description` уникальны, description 120–160 символов.
- [ ] `<h1>` только один и отражает тему страницы.
- [ ] Каждая секция начинается с
       `p.note` с `<strong>Тема</strong> — кратким определением`.
- [ ] Весь код — только внутри `div.code-block > pre > code`.
- [ ] Спецсимволы `<`, `>` и `&` экранированы внутри `<code>`.
- [ ] Нет `grid-column-*`, `div.example` и других нестандартных обёрток.
- [ ] Все `id` секций совпадают с `href` в `nav-tabs`.
- [ ] Списки используют классы `list-ul`, `list-ol` или `list-icon`.
- [ ] Изображения — в `<figure>` с `img.note-img` и `figcaption.note-caption`.
- [ ] Доступность: `skip-link`, `role="navigation"`, `aria-current`, `aria-live`.
- [ ] `@@include` для `popup.html` и `js.html` — после `</div>` обёртки, до `</body>`.
- [ ] Страница протестирована локально через `npm run dev`.

## Процесс создания страницы

1. Создать файл `src/[раздел]/новая-страница.html` в нужной папке.
2. Скопировать HTML-шаблон из раздела [HTML структура](#html-структура).
3. Заполнить контент согласно типу страницы (Tutorial / Theory / Reference).
4. Проверить локально: `npm run dev`.
5. Обновить навигацию раздела в `partials/subnav.html` при необходимости.
6. Создать PR с понятным описанием изменений.
7. После ревью — выполнить merge и проверить страницу на проде.
