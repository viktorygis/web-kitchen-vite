React Context

React Context — способ передать данные через дерево компонентов, минуя props.
Удобно для тем, авторизации, глобальных настроек.

Основы Context


const ThemeContext = React.createContext('light');
createContext создает объект для передачи значения.

Provider и Consumer


<ThemeContext.Provider value="dark">
  <App />
</ThemeContext.Provider>
Provider задает значение для всех дочерних компонентов.

useContext — получение значения


const theme = React.useContext(ThemeContext);
useContext — хук для доступа к значению контекста.

Практика: переключение темы


<div id="root"></div>
<script type="text/babel">
const ThemeContext = React.createContext();

function ThemeSwitcher() {
  const theme = React.useContext(ThemeContext);
  return <div>Текущая тема: {theme}</div>;
}

function App() {
  const [theme, setTheme] = React.useState('light');
  return (
    <ThemeContext.Provider value={theme}>
      <button onClick={() => setTheme(t => t === 'light' ? 'dark' : 'light')}>Сменить тему</button>
      <ThemeSwitcher />
    </ThemeContext.Provider>
  );
}
const root = ReactDOM.createRoot(document.getElementById('root'));
root.render(<App />);
</script>

Советы и типичные ошибки

  • Не используйте Context для каждого пропа — только для глобальных данных.
  • Обновление значения вызывает перерисовку всех потребителей.
  • Выносите Provider как можно выше в дереве компонентов.

Лучшие практики

  • Создавайте отдельные контексты для разных типов данных.
  • Документируйте, что хранится в каждого Context.
  • Используйте мемоизацию для предотвращения лишних рендеров.

Примеры использования Context

Контекст авторизации


const AuthContext = React.createContext();
<AuthContext.Provider value={user}>...</AuthContext.Provider>
const user = React.useContext(AuthContext);

Множественные контексты


<ThemeContext.Provider value="dark">
  <AuthContext.Provider value={user}>
    <App />
  </AuthContext.Provider>
</ThemeContext.Provider>
