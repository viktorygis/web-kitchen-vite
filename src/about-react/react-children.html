Children и композиция компонентов в React

Композиция — основной способ организации интерфейса в React.
props.children — позволяет передавать контент (JSX, текст, другие компоненты) внутрь компонента.
Благодаря композиции можно строить гибкие и переиспользуемые UI-блоки.

Основы props.children


function Wrapper(props) {
  return <div className="wrap">{props.children}</div>;
}
<Wrapper>Контент внутри!</Wrapper>
props.children — всё, что написано между открывающим и закрывающим тегом компонента.

Композиция компонентов


function Card({children}) {
  return <div className="card">{children}</div>;
}
function App() {
  return (
    <Card>
      <h3>Заголовок</h3>
      <p>Текст карточки</p>
    </Card>
  );
}
Можно передавать любое количество дочерних элементов, в том числе другие компоненты и JSX.

Render Props — функция как дочерний элемент


function List({ renderItem, items }) {
  return <ul>{items.map(renderItem)}</ul>;
}
function App() {
  const products = ['Ноутбук', 'Смартфон', 'Наушники'];
  return (
    <List
      items={products}
      renderItem={(item, i) => <li key={i}>{item}</li>}
    />
  );
}
Render Prop — паттерн, когда компонент принимает функцию для рендера содержимого.

Практика: создаём Layout с children


<div id="root"></div>
<script type="text/babel">
function Layout({children}) {
  return (
    <div style={{
      border: "2px solid #3366d4",
      padding: "14px",
      borderRadius: "8px"
    }}>
      <header><h2>Шапка сайта</h2></header>
      <main>{children}</main>
      <footer>© 2025</footer>
    </div>
  );
}
function App() {
  return (
    <Layout>
      <p>Это контент внутри макета!</p>
      <button>Кнопка</button>
    </Layout>
  );
}
const root = ReactDOM.createRoot(document.getElementById('root'));
root.render(<App />);
</script>

Советы и типичные ошибки

  • Всегда используйте props.children для передачи вложенного контента!
  • Композиция = переиспользуемость. Делайте компоненты максимально гибкими.
  • Для сложных случаев используйте React.cloneElement или render props .
  • Передавайте не только JSX, но и функции, массивы, даже другие компоненты!

Лучшие практики

  • Документируйте children (PropTypes, JSDoc).
  • Используйте композицию вместо наследования — это философия React!
  • Не злоупотребляйте сложными вложенными структурами — следите за читаемостью.
  • Разделяйте ответственность: layout-компоненты для структуры, UI-компоненты для отображения.

Примеры использования children и composition

Передача компонента как children


function Alert({children}) {
  return <div className="alert">{children}</div>;
}
<Alert><strong>Внимание!</strong> Что-то случилось.</Alert>

Композиция: вложенные компоненты


function Panel({children}) {
  return <section className="panel">{children}</section>;
}
function App() {
  return (
    <Panel>
      <h3>Заголовок</h3>
      <p>Описание</p>
    </Panel>
  );
}

Render Prop для кастомного рендера


function CustomList({items, renderItem}) {
  return <ul>{items.map(renderItem)}</ul>;
}
<CustomList
  items={[1,2,3]}
  renderItem={item => <li key={item}>{item * 2}</li>}
/>
