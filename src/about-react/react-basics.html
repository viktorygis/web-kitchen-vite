Основы React

React — это библиотека для создания пользовательских интерфейсов. Она позволяет строить интерфейсы из небольших переиспользуемых частей — компонентов.
Основы React это JSX, props, state и ключи (key).

JSX, props, state и ключи - это фундаментальные концепции React, которые позволяют создавать динамические и интерактивные веб-приложения.

Приложения на React собираются из компонентов. Компонент — это часть пользовательского интерфейса, у которого есть собственная логика и внешность. Компонент может быть маленьким, как кнопка, или большим, как целая страница.

React-компоненты — это функции JavaScript, которые возвращают разметку: 


function MyButton() {
  return (
    
  );
}

Основные понятия

  • JSX — синтаксис для описания интерфейса внутри JavaScript. JSX (JavaScript XML) позволяет писать HTML-подобную разметку прямо в JS-коде.
  • Компонент - это функция или класс, который возвращает JSX-разметку для отображения на странице.
  • Props - свойства компонентов, передаваемые от родителя к дочернему компоненту.
  • State - внутреннее состояние компонента, которое может изменяться со временем.
  • Ключи (key) - уникальные идентификаторы для элементов в списках, помогающие React эффективно обновлять DOM.
  • Условный рендеринг - отображение элементов на основе условий.
  • Списки - вывод массивов данных с помощью метода map().
  • Лучшие практики - рекомендации по написанию чистого и поддерживаемого кода в React.

JSX: разметка внутри JS

JSX - это синтаксис, позволяющий писать HTML-подобную разметку внутри JavaScript-кода. JSX упрощает создание компонентов и делает код более читаемым. Он не обязателен, но большинство проектов на React предпочитают его использовать из-за удобства.

JSX строже HTML. Вам нужно закрывать теги вроде <br />. Ваш компонент также не может возвращать несколько JSX-тегов. Их нужно будет обернуть внутрь общего родителя, например, <div>...</div> или пустую обёртку вида <>...</>

Как работает JSX

  • JSX похож на HTML, но это всё ещё JavaScript.
  • JSX компилируется в вызовы React.createElement(). (где создаются элементы React)
  • JSX позволяет вставлять выражения JavaScript внутри фигурных скобок {}.

Пример JSX

function Hello() {
  return <h1>Привет, React!</h1>;
}
Объяснение кода:
  • Компонент Hello возвращает JSX-разметку.
  • JSX похож на HTML, но внутри JavaScript.

🚀 JSX облегчает создание компонентов и улучшает читаемость кода!

Сравнение JSX и HTML

// JSX
const element = <div className="container">Привет, мир!</div>;
// HTML
const element = '<div class="container">Привет, мир!</div>';
Объяснение кода:
  • В JSX используется className вместо class, так как class — это зарезервированное слово в JavaScript.
  • JSX позволяет вставлять выражения JavaScript внутри фигурных скобок {}.

✨ JSX сочетает в себе мощь JavaScript и удобство HTML!

Стилизация компонентов в React

В React есть несколько способов добавления стилей к компонентам. Вы можете использовать встроенные стили, CSS-классы или CSS-in-JS библиотеки.

CSS-классы

В React CSS-классы объявляются с помощью className. Оно работает аналогично HTML-атрибуту class:

function ColoredText() {
  return <h2 className="blue-text">Этот текст синий</h2>;
}
Объяснение кода:
  • Используем атрибут className для назначения CSS-класса элементу.
  • CSS-класс blue-text должен быть определён в вашем CSS-файле.

🎨 CSS-классы позволяют легко переиспользовать стили в разных компонентах!

Встроенные стили

function StyledButton() {
  const buttonStyle = {
    backgroundColor: 'blue',
    color: 'white',
    padding: '10px 20px',
    border: 'none',
    borderRadius: '5px'
  };
  return <button style={buttonStyle}>Кнопка</button>;
}
Объяснение кода:
  • Создаём объект buttonStyle с CSS-свойствами в camelCase.
  • Применяем стили к элементу с помощью атрибута style.

CSS-in-JS библиотеки

CSS-in-JS библиотеки, такие как styled-components или Emotion, позволяют писать CSS прямо в JavaScript. Они обеспечивают изоляцию стилей и динамическое создание классов.

🎨 Встроенные стили позволяют быстро и удобно стилизовать компоненты!

Отображение данных в JSX

В React вы можете легко отображать данные внутри JSX-разметки, используя фигурные скобки {}. Это позволяет вставлять переменные, выражения и результаты функций прямо в разметку.

Пример отображения данных

function UserGreeting(props) {
  return <h2>Привет, {props.user.name}!</h2>;
}
const user = { name: 'Аня' };
const root = ReactDOM.createRoot(document.getElementById('data-display-demo'));
root.render(<UserGreeting user={user} />);
Объяснение кода:
  • Компонент UserGreeting принимает prop user.
  • Внутри JSX используем фигурные скобки {} для вставки значения props.user.name.
  • Создаём объект user с именем и передаём его в компонент.

📊 Фигурные скобки позволяют вставлять JavaScript-выражения в JSX-разметку!

Props: свойства компонентов

Props - это способ передачи данных от родительского компонента к дочернему. Props делают компоненты гибкими и переиспользуемыми.

Как работают props

  • Props передаются как атрибуты в JSX.
  • Внутри компонента доступны через объект props.

Пример передачи props

function Welcome(props) {
  return <h2>Привет, {props.name}!</h2>;
}

// Использование:
<Welcome name="Аня" />
Объяснение кода:
  • Компонент Welcome принимает prop name.
  • При использовании компонента передаём значение "Аня".

🎯 Props позволяют создавать динамические и переиспользуемые компоненты!

State: внутреннее состояние компонента

State - это данные, которые принадлежат конкретному компоненту и могут изменяться со временем. State позволяет создавать интерактивные компоненты.

Как работает state

  • State создаётся с помощью хука useState.
  • Изменение state вызывает повторный рендер компонента.

Пример использования state

function Counter() {
  const [count, setCount] = React.useState(0);
  return (
    <div>
      <p>Счётчик: {count}</p>
      <button onClick={() => setCount(count + 1)}>Увеличить</button>
    </div>
  );
}
Объяснение кода:
  • Используем хук useState для создания state count с начальным значением 0.
  • При нажатии на кнопку вызывается функция setCount, которая увеличивает значение count.
  • Компонент автоматически перерисовывается с новым значением счётчика.

⚡ State позволяет создавать динамические и интерактивные компоненты!

Условный рендеринг

Условный рендеринг позволяет отображать разные элементы в зависимости от состояния компонента. Это делает интерфейсы более динамичными и адаптивными.

Как работает условный рендеринг

  • Используйте JavaScript-выражения внутри JSX для условий.
  • Часто используются тернарный оператор ? и логический оператор &&.

Пример условного рендеринга

function Greeting(props) {
  return (
    props.isLoggedIn
      ? <p>Добро пожаловать!</p>
      : <p>Пожалуйста, войдите.</p>
  );
}
Объяснение кода:
  • Компонент Greeting принимает prop isLoggedIn.
  • Если isLoggedIn истинно, отображается приветственное сообщение.
  • Если ложно, отображается приглашение войти.

🔄 Условный рендеринг позволяет создавать динамические интерфейсы!

Списки и ключи

Списки позволяют отображать массивы данных в виде элементов интерфейса. В React для этого часто используется метод map(). Ключи (key) помогают React эффективно обновлять и перерисовывать списки при изменениях.

Как работать со списками и ключами

  • Используйте метод map() для создания списка элементов из массива.
  • Каждому элементу списка обязательно назначайте уникальный key.

Пример списка с ключами

function UserList(props) {
  return (
    <ul>
      {props.users.map(user =>
        <li key={user}>{user}</li>
      )}
    </ul>
  );
}
// Использование:
<UserList users={["Аня", "Макс", "Иван"]} />
Объяснение кода:
  • Компонент UserList принимает prop users, который является массивом имён.
  • Используем метод map() для создания элемента <li> для каждого имени.
  • Каждому элементу списка назначаем уникальный key, равный имени пользователя.

📋 Правильное использование ключей улучшает производительность при работе со списками!

Практика: мини-приложение на React

Давайте создадим простое React-приложение, которое демонстрирует использование JSX, props, state, условного рендеринга и списков с ключами.

Описание приложения

  • Компонент App содержит состояние для имени пользователя, списка пользователей и видимости приветственного сообщения.
  • Есть поле ввода для изменения имени пользователя.
  • Отображается приветствие с именем пользователя.
  • Выводится список пользователей.
  • Есть кнопка для переключения видимости приветственного сообщения.

Код приложения

<div id="root"></div>
<script type="text/babel">
function App() {
  const [name, setName] = React.useState("Аня");
  const [users, setUsers] = React.useState(["Аня", "Макс", "Иван"]);
  const [isVisible, setIsVisible] = React.useState(true);
  return (
    <div>
      <input value={name} onChange={e => setName(e.target.value)} placeholder="Ваше имя" />
      <h2>Привет, {name}!</h2>
      <ul>
        {users.map(user => <li key={user}>{user}</li>)}
      </ul>
      <button onClick={() => setIsVisible(v => !v)}>Показать/скрыть приветствие</button>
      {isVisible && <p>Это условное сообщение!</p>}
    </div>
  );
}
const root = ReactDOM.createRoot(document.getElementById('root'));
root.render(<App />);
</script>
Объяснение кода:
  • Используем хук useState для создания трёх состояний: name, users и isVisible.
  • Поле ввода позволяет изменять имя пользователя, обновляя состояние name.
  • Приветствие отображает текущее значение name.
  • Список пользователей создаётся с помощью метода map(), где каждому элементу назначается уникальный key.
  • Кнопка переключает состояние isVisible, что влияет на отображение условного сообщения.

Интерактивный пример

Советы и типичные ошибки

  • Не забывайте импортировать React и ReactDOM в файлы, где используете JSX.
  • Всегда назначайте уникальные ключи (key) при рендеринге списков.
  • Избегайте прямого изменения state; используйте функции обновления состояния.
  • Используйте условный рендеринг для управления отображением элементов.
  • Компоненты должны быть маленькими и выполнять одну функцию.
💡 Следуйте этим советам, чтобы писать чистый и эффективный код в React!

Лучшие практики

  • Структурируйте код: отделяйте компоненты по функциям и файлам.
  • Используйте props для передачи данных, state — для локального состояния.
  • Делайте компоненты маленькими и переиспользуемыми.
  • Документируйте пропсы и state компонентов.
Создавайте читаемые, масштабируемые компоненты. Следуйте документации React и практикуйте современные подходы!

Реальные примеры использования JSX, props и state в React

Пример 1. Приветствие с prop

В этом примере мы создадим компонент Greeting, который принимает prop name и отображает приветственное сообщение.

function Greeting(props) {
  return <h2>Добро пожаловать, {props.name}!</h2>;
}
const root = ReactDOM.createRoot(document.getElementById('greeting-demo'));
root.render(<Greeting name="Вика" />);
Объяснение кода:
  • Компонент Greeting принимает prop name.
  • При использовании компонента передаём значение "Вика".
  • Компонент отображает приветственное сообщение с именем.

Пример 2. Счётчик со state

В этом примере мы создадим компонент Counter, который использует state для отслеживания значения счётчика и позволяет увеличивать его по нажатию кнопки.

function Counter() {
  const [count, setCount] = React.useState(0);
  return (
    <div>
      <p>Счётчик: {count}</p>
      <button onClick={() => setCount(count + 1)}>Увеличить</button>
    </div>
  );
}
const root = ReactDOM.createRoot(document.getElementById('counter-demo'));
root.render(<Counter />);
Объяснение кода:
  • Используем хук useState для создания state count с начальным значением 0.
  • При нажатии на кнопку вызывается функция setCount, которая увеличивает значение count.
  • Компонент автоматически перерисовывается с новым значением счётчика.

Пример 3. Список пользователей с key

В этом примере мы создадим компонент UserList, который принимает массив имён пользователей через prop и отображает их в виде списка. Каждому элементу списка назначается уникальный ключ (key).

function UserList(props) {
  return (
    <ul>
      {props.users.map(user => <li key={user}>{user}</li>)}
    </ul>
  );
}
const root = ReactDOM.createRoot(document.getElementById('userlist-demo'));
root.render(<UserList users={["Аня", "Макс", "Иван"]} />);
Объяснение кода:
  • Компонент UserList принимает prop users, который является массивом имён.
  • Используем метод map() для создания элемента <li> для каждого имени.
  • Каждому элементу списка назначаем уникальный key, равный имени пользователя.
