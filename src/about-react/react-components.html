Компоненты в React

Компонент — основная строительная единица React.
Это функция (или класс), возвращающая JSX.
Всё приложение состоит из компонентов!

Виды компонентов

Компоненты делятся на 2 группы по синтаксису

  • Функциональный компонент — обычная функция JS. (Рекомендуется)
  • Классовый компонент — class extends React.Component (Устарело)

Пример: Функциональный компонент


function HelloFunc() {
  return <h1>Привет! (Функция)</h1>;
}

Пример: Классовый компонент


class HelloClass extends React.Component {
  render() {
    return <h1>Привет! (Класс)</h1>;
  }
}

По роли в приложении

  • Контейнеры (умные, Smart/Container Components) — отвечают за логику, работу с данными, состояние, могут подключаться к API или store.
    Пример: компонент, который делает fetch данных и передает их вниз.
  • Презентационные (глупые, Dumb/Presentational Components) — отвечают только за отображение, получают данные через props.
    Пример: компонент Button, который просто отображает кнопку.

Контейнер:


function UsersContainer() {
  const [users, setUsers] = React.useState([]);
  React.useEffect(() => {
    fetch("https://reqres.in/api/users?per_page=2")
      .then(r => r.json())
      .then(json => setUsers(json.data));
  }, []);
  return <UserList users={users} />;
}
function UserList({ users }) {
  return <ul>{users.map(u => <li key={u.id}>{u.first_name}</li>)}</ul>;
}

Презентационный:


function Button({children, onClick}) {
  return <button onClick={onClick}>{children}</button>;
}

По уровню абстракции

  • Высокоуровневые — крупные блоки (Layout, Page, App, Form).
  • Низкоуровневые — маленькие переиспользуемые кирпичики (Button, Input, Icon).

Высокоуровневый:


function Page() {
  return (
    <div>
      <Header />
      <Main />
      <Footer />
    </div>
  );
}

Низкоуровневый:


function Icon() {
  return <span role="img" aria-label="star">⭐</span>;
}

По переиспользованию

  • Переиспользуемые — универсальные элементы (Modal, Dropdown, Card, Loader).
  • Специализированные — используются только в одном месте (например, форма регистрации).

Переиспользуемый:


function Modal({ open, children }) {
  if (!open) return null;
  return <div className="modal">{children}</div>;
}

Специализированный:


function RegisterForm() {
  // ... только для регистрации
  return <form>...</form>;
}

По способу управления состоянием

  • Контролируемые (Controlled) — состояние передается через props, обычно для form-элементов.
  • Неконтролируемые (Uncontrolled) — состояние внутри DOM (через ref).

Контролируемый:


function ControlledInput() {
  const [val, setVal] = React.useState("");
  return <input value={val} onChange={e => setVal(e.target.value)} />;
}

Неконтролируемый:


function UncontrolledInput() {
  const ref = React.useRef();
  return (
    <React.Fragment>
      <input ref={ref} />
      <button onClick={() => alert(ref.current.value)}>Показать значение</button>
    </React.Fragment>
  );
}

Другие классификации

  • Pure Components — оптимизированы, не рендерятся без необходимости (React.memo, PureComponent).
  • HOC (Higher-Order Components) — функции, которые принимают компонент и возвращают новый (например, withRouter).
  • Render Props — компоненты, принимающие функцию как child (например, <Mouse>{fn} ).
  • Hooks-based — используют хуки (почти все современные функциональные компоненты).
Любой компонент может относиться сразу к нескольким категориям!
Функциональный презентационный контролируемый компонент — обычная практика.

Создание компонента


function Welcome(props) {
  return <h2>Добро пожаловать, {props.name}!</h2>;
}
Имя компонента — всегда с большой буквы!

Композиция компонентов


function App() {
  return (
    <div>
      <Header />
      <Content />
      <Footer />
    </div>
  );
}
Можно вкладывать компоненты друг в друга — это и есть композиция.

Практика: свой компонент


function Card(props) {
  return <div style={{border:"1px solid #ccc", padding:10}}>
    <h3>{props.title}</h3>
    <div>{props.children}</div>
  </div>;
}
function App() {
  return <Card title="Заголовок">Контент внутри карточки!</Card>;
}

Советы и типичные ошибки

  • Всегда называйте компоненты с большой буквы.
  • Один компонент — одна задача.
  • Используйте props для передачи данных.

Лучшие практики

  • Делите приложение на маленькие переиспользуемые компоненты.
  • Не пишите логику вне компонента, если она зависит от props/state.
  • Документируйте интерфейс компонента.

Примеры компонентов

Счетчик как компонент


function Counter() {
  const [n, setN] = React.useState(0);
  return <button onClick={() => setN(n+1)}>{n}</button>;
}
