Мемоизация и оптимизация в React

Мемоизация — техника сохранения результата вычислений для избежания лишних рендеров.
В React используются React.memo , useMemo , useCallback для оптимизации производительности.

React.memo — мемоизированный компонент


const MyComp = React.memo(function MyComp(props) {
  // компонент будет перерисовываться только при изменении props
  return <div>{props.value}</div>;
});
React.memo предотвращает лишний рендер, если props не изменились.

useMemo — мемоизация вычислений


function Expensive({n}) {
  const result = React.useMemo(() => compute(n), [n]);
  return <div>Результат: {result}</div>;
}
useMemo сохраняет результат вычисления, если зависимости не изменились.

useCallback — мемоизация функций


const handleClick = React.useCallback(() => alert('Клик!'), []);
useCallback сохраняет ссылку на функцию, если зависимости не изменились.

Практика: мемоизированный список


<div id="root"></div>
<script type="text/babel">
const List = React.memo(function List({ items }) {
  console.log('List render');
  return <ul>{items.map(i => <li key={i}>{i}</li>)}</ul>;
});
function App() {
  const [count, setCount] = React.useState(0);
  const items = React.useMemo(() => [1,2,3], []);
  return (
    <div>
      <button onClick={() => setCount(c => c + 1)}>{count}</button>
      <List items={items} />
    </div>
  );
}
const root = ReactDOM.createRoot(document.getElementById('root'));
root.render(<App />);
</script>

Советы и типичные ошибки

  • Не используйте мемоизацию без необходимости: это не всегда ускоряет, иногда мешает.
  • Следите за зависимостями в useMemo/useCallback!
  • React.memo работает только для компонентов с props.
  • Проверяйте работу мемоизации через console.log.

Лучшие практики

  • Мемоизируйте только тяжелые вычисления или крупные компоненты.
  • Не мемоизируйте простые функции и значения без необходимости.
  • Документируйте, зачем и что мемоизируется.
