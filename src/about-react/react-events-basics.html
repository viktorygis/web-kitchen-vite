События в React

События — основной способ реагировать на действия пользователя.
В React обработка событий похожа на обычный JS, но есть отличия!
Всё — через camelCase ( onClick , onChange ), обработчики — функции.
На этой странице — всё для старта с событиями!

  • React использует свои синтетические события, которые оборачивают стандартные события браузера для кроссбраузерности.
  • События в React именуются в стиле camelCase : onClick , onChange , onSubmit и т.д.
  • Обработчики событий — это функции , которые вы передаете в JSX.

Основы событий в React

  • Все события — это camelCase: onClick , onChange , onSubmit .
  • Передавайте функцию-обработчик: <button onClick={fn}> .
  • Обработчик получает объект события 💡 event .
  • ✅ Не вызывайте обработчик сразу: onClick={handleClick} — правильно, onClick={handleClick()} — ошибка!

Обработчики событий


function Clicker() {
  function handleClick() {
    alert("Клик!");
  }
  return <button onClick={handleClick}>Нажми меня</button>;
}
👍 Пояснение: Обработчик — обычная функция.
Не вызывайте её напрямую: onClick={handleClick} , не onClick={handleClick()} !

Объект события

В обработчик автоматически передается объект события event . Он содержит всю информацию о событии (например, введённый текст или координаты клика). 


function InputLogger() {
  function handleChange(event) {
    console.log(event.target.value);
  }
  return <input onChange={handleChange} />;
}
📑 Пояснение: event.target.value — текущее значение инпута.
Выводится в консоль при каждом изменении.

Практика: Кликер и инпут


<div id="events-demo"></div>
<script type="text/babel">
function Demo() {
  const [count, setCount] = React.useState(0);
  const [text, setText] = React.useState("");
  return (
    <div>
      <button onClick={() => setCount(count + 1)}>Клик: {count}</button>
      <br/>
      <input onChange={e => setText(e.target.value)} placeholder="Пиши сюда!"/>
      <p>Вы ввели: {text}</p>
    </div>
  );
}
const eventsRoot = ReactDOM.createRoot(document.getElementById('events-demo'));
eventsRoot.render(<Demo />);
</script>
✅ Пояснение:
  • Клик по кнопке увеличивает счётчик ( onClick ).
  • Ввод в поле обновляет текст ( onChange ).

Советы и типичные ошибки

  • ✔ Не вызывайте обработчик в JSX: onClick={fn} — правильно, onClick={fn()} — ошибка!
  • ✔ Используйте event для доступа к значению инпута: event.target.value .
  • ✔ Для передачи параметров — используйте стрелочные функции: onClick={() => fn(arg)} .
  • ⚠ Не пишите тяжелую логику прямо в JSX — выносите обработчики отдельно.
Частая ошибка: onClick={fn()} вызовет функцию сразу при рендере, а не по клику!

Лучшие практики

  • 💡 Выносите обработчики в отдельные функции, чтобы код был чище.
  • 💡 Давайте обработчикам осмысленные имена: handleClick , handleChange .
  • 💡 Минимизируйте логику внутри JSX.
  • 💡 Используйте хуки только внутри компонентов.
💡 Совет: Старайтесь не использовать анонимные функции в props, если это не нужно для передачи параметров — это поможет избежать лишних перерисовок.

Примеры событий в React

Пример 1: Кнопка

Пояснение: 👈 Кнопка с обработчиком onClick , который вызывает alert . 

function Button() {
  return <button onClick={() => alert("Нажато!")}>Жми</button>;
}

Пример 2: Инпут

Пояснение: 📑 Инпут с обработчиком onChange , который выводит значение в консоль. 

function InputDemo() {
  return <input onChange={e => console.log(e.target.value)} />;
}
