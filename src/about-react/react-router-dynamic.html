Динамические маршруты и параметры в React Router

Динамические маршруты позволяют передавать переменные части URL (например, id пользователя или товара) для отображения конкретного контента.

Параметры маршрута


<Route path="/user/:id" element={<User />} />
:id — параметр маршрута. Можно использовать несколько параметров!

Извлечение параметров внутри компонента


import { useParams } from 'react-router-dom';
function User() {
  const { id } = useParams();
  return <h2>Пользователь с id: {id}</h2>;
}
useParams() — хук для получения параметров из URL.

Динамические ссылки с параметрами


<Link to={`/user/${user.id}`}>Профиль</Link>
Используйте шаблонные строки для подстановки id!

Практика: список пользователей с переходом по id


/* App.js */
import { BrowserRouter, Routes, Route, Link, useParams } from 'react-router-dom';

function User() {
  const { id } = useParams();
  return <h2>Пользователь: {id}</h2>;
}

function App() {
  const users = [{id:1, name:'Аня'}, {id:2, name:'Иван'}, {id:3, name:'Мария'}];
  return (
    <BrowserRouter>
      <ul>
        {users.map(u => (
          <li key={u.id}>
            <Link to={`/user/${u.id}`}>{u.name}</Link>
          </li>
        ))}
      </ul>
      <Routes>
        <Route path="/user/:id" element={<User />} />
      </Routes>
    </BrowserRouter>
  );
}

Советы и типичные ошибки

  • Передавайте параметры только через path маршрута.
  • Проверяйте, что параметр существует, чтобы избежать ошибок.
  • Проверяйте типы передаваемых параметров.

Лучшие практики

  • Давайте осмысленные имена параметрам маршрута.
  • Используйте useParams для извлечения данных из URL.
  • Добавляйте обработку 404 для несуществующих id.
