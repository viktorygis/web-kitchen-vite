Снипеты (Snippets)

Снипеты — это шаблоны кода, которые можно вставлять по короткой команде. Экономят кучу времени!

Как добавить свой снипет:

  1. Открой командную палитру Ctrl+Shift+P (или Cmd+Shift+P на Mac).
  2. Введи snippets и выбери Preferences: Configure User Snippets .
  3. Выбери язык или New Global Snippets file .
  4. Добавь свой снипет по примеру: 
    "Print to console": {
    "prefix": "log",
    "body": ["console.log('$1');"],
    "description": "Вывести в консоль"
}

Теперь просто напиши log и нажми Tab — снипет подставится!

Настройки и кастомизация

Где править настройки?

  • Через UI: Файл → Настройки → Параметры ( Ctrl+, ).
  • Вручную: settings.json — ищи команду Preferences: Open Settings (JSON) .

Полезные параметры:

{
    //Настройка шрифта
    "editor.fontSize": 16,
    //Настройка отступов в 2 пробела
    "editor.tabSize": 2,
    //Автозакрытие скобок и кавычек
    "files.trimTrailingWhitespace": true,
    //Автосохранение файлов
    "editor.formatOnSave": true,
    //Автосохранение при смене фокуса
    "files.autoSave": "onFocusChange"
}

  • One Dark Pro

Переход между темами:

  • Ctrl+K Ctrl+T — быстрое меню выбора темы.

Prettier и автоформатирование: решение проблем с переносами и тегами

Часто после установки Prettier и нажатия Ctrl + Alt + F (или Format Document ), появляется проблема:
Простые HTML-теги, такие как <b> , <i>, <span> и строки после них переносятся на новые строки.
Пример нежелательного форматирования: 

<b>
  Привет!
</b>

Ожидалось: <b>Привет!</b>

Это выглядит неаккуратно и мешает писать компактный HTML или разметку email.

Почему так происходит?

  • Prettier по умолчанию всегда переносит содержимое на новую строку после тега, если считает, что это "блок".
  • В HTML настройка htmlWhitespaceSensitivity влияет на расстановку пробелов и переносов.

Как исправить?

Нужно указать Prettier, что чувствительность к пробелам и переносам в HTML должна быть strict (или ignore ), чтобы сохранять компактный формат .

  1. Открой .prettierrc или settings.json в проекте.
  2. Добавь/измени параметры: 
    {
  // для .prettierrc
  "htmlWhitespaceSensitivity": "ignore",
  "printWidth": 120,
  "proseWrap": "preserve"
}
    {
  //для VSCode settings.json (если глобально)
		//ignore - отключает избыточные переносы для обычных тегов
  "prettier.htmlWhitespaceSensitivity": "ignore",
		//120 символов до переноса
  "prettier.printWidth": 120,
		//preserve - не разбивать текст на новые строки
  "prettier.proseWrap": "preserve"
}
  3. htmlWhitespaceSensitivity: "ignore" — отключает избыточные переносы для обычных тегов.
  4. printWidth: 120 — строки будут дольше не разбиваться.
  5. proseWrap: "preserve" — не разбивать текст на новые строки.
  • Для jsx/tsx: htmlWhitespaceSensitivity тоже помогает если вы пишете компоненты с html.
  • Только для простых тегов: старайтесь использовать их без переносов в исходнике или вручную правьте место переноса.

Проверить форматтер по умолчанию в VSCode:

  1. Открой настройки ( Ctrl + , Ctrl и запятая), введи default formatter .
  2. Убедись, что выбран Prettier - Code formatter .

Важные команды и быстрый доступ:

  • Format Document : Shift+Alt+F (или Ctrl+Alt+F , если так назначено)
  • Restart Prettier Server: для устранения глюков с автосохранением
Иногда можно отключить автоматическое форматирование для некоторых типов файлов через: 
"[html]": { "editor.formatOnSave": false }

Ещё полезные Prettier-настройки для html:

{
   //Использовать одинарные кавычки
    "prettier.singleQuote": true,
				//Отступ в 2 пробела
    "prettier.tabWidth": 2,
				//Использовать пробелы вместо табов
    "prettier.endOfLine": "lf"
}

Иногда помогают плагины:

  • Например, Prettier Plugin for HTML , если нужен совсем иной стиль.

Все настройки Prettier для VS Code — полный список

  • prettier.printWidth
    Максимальная длина строки.
    80 (по умолчанию). Увеличьте для длинных строк, например: 120 .
  • prettier.tabWidth
    Кол-во пробелов на один уровень отступа.
    2 — стандарт для JS/HTML.
  • prettier.useTabs
    Использовать табы ( \t ) вместо пробелов для отступов.
  • prettier.semi
    Ставить точку с запятой в конце строки.
    true — ставим (по умолчанию).
  • prettier.singleQuote
    Одинарные кавычки вместо двойных для строк.
    true — проектный стандарт во многих командах.
  • prettier.quoteProps
    Как ставить кавычки для свойств объектов:
    "as-needed" | "consistent" | "preserve"
    "as-needed" - только когда нужно.
    "consistent" - если одно свойство в кавычках, все в кавычках.
    "preserve" - сохранять как в исходнике.
  • prettier.jsxSingleQuote
    Использовать одинарные кавычки в JSX.
    true для <div className='foo'>.
    false по умолчанию (двойные кавычки).
  • prettier.trailingComma
    Запятая после последнего элемента:
    "none" , "es5" , "all"
  • prettier.bracketSpacing
    Пробелы внутри скобок объектов.
    true — { a: 1 } ; false — {a: 1}
  • prettier.jsxBracketSameLine
    Переносить закрывающую скобку JSX
    false — на новую строку, true — на ту же строку.
  • prettier.arrowParens
    Всегда ли оборачивать параметры стрелочных функций в скобки.
    "always" | "avoid"
  • prettier.proseWrap
    Разбивать строки в Markdown.
    "always" , "never" , "preserve"
  • prettier.endOfLine
    Символ конца строки:
    "auto" , "lf" , "crlf" , "cr"
  • prettier.htmlWhitespaceSensitivity
    Чувствительность к пробелам/переносам для HTML.
    "css" , "strict" , "ignore"
  • prettier.vueIndentScriptAndStyle
    Добавлять отступ для кода в <script>/<style> в Vue-файлах.
  • prettier.embeddedLanguageFormatting
    Форматировать код, встроенный в шаблоны (например, JS в HTML).
  • prettier.requireConfig
    Форматировать только если найден config-файл Prettier.
    false — форматирует всегда.
  • prettier.ignorePath
    Определяет путь к файлу .prettierignore для игнорирования файлов при форматировании.
  • prettier.useEditorConfig
    Учитывать .editorconfig проекта во время форматирования.
  • prettier.withNodeModules
    Форматировать файлы в node_modules .
Большинство этих настроек удобно прописывать в settings.json или .prettierrc .
Не используйте слишком жёсткие ограничения — выбирайте командный стиль!

Пример конфига для settings.json VS Code: 

{
  "prettier.printWidth": 120,
  "prettier.tabWidth": 2,
  "prettier.useTabs": false,
  "prettier.semi": true,
  "prettier.singleQuote": true,
  "prettier.trailingComma": "es5",
  "prettier.bracketSpacing": true,
  "prettier.jsxBracketSameLine": false,
  "prettier.arrowParens": "always",
  "prettier.proseWrap": "preserve",
  "prettier.endOfLine": "lf",
  "prettier.htmlWhitespaceSensitivity": "ignore",
  "prettier.vueIndentScriptAndStyle": true
}

Beautify — что это за расширение для VSCode?

Beautify — популярное расширение для Visual Studio Code, предназначенное для форматирования и "украшения" (от англ. beautify — приводить к красоте) HTML, CSS, JavaScript, JSON и других текстовых файлов.

Что делает Beautify?

  • Расставляет отступы, пробелы, переносы строк в соответствии с вашими настройками.
  • Делает код более читаемым, особенно если он был «свален в кучу» без форматирования.
  • Позволяет гибко управлять правилами форматирования (например, разрешать компактные теги или длинные строки).
  • Может работать там, где Prettier делает переносы «по-своему» (например, в email-верстке).

Для чего полезно?

  • HTML: Beautify позволяет настроить переносы строк, компактизировать инлайн-разметку (например, ...) и избежать избыточного разбивания по строкам.
  • CSS/SCSS: Упорядочивает правила, делает автосортировку, форматирует вложенные блоки.
  • JavaScript: Делает читаемым неаккуратный JS-код, но для продакшн-кода часто лучше подходит Prettier.

Как установить?

  1. Откройте боковую панель расширений Ctrl+Shift+X.
  2. Найдите "Beautify" (автор HookyQR).
  3. Установите и перезапустите VSCode (если требуется).

Как использовать?

  • Быстрое форматирование:
    Shift+Alt+F или через контекстное меню "Format Document".
  • Через командную палитру:
    Ctrl+Shift+P → "Beautify File" или "Beautify Selection".

Где настраивать?

  • Файл настроек: .jsbeautifyrc в корне проекта
  • Или прямо в settings.json VSCode.
  • Примеры параметров: 
    {
  "beautify.language": {
    "html": ["htm", "html", "handlebars"],
    "css": ["css", "scss"],
    "js": ["js", "json", "ts"]
  },
  "beautify.config": {
    "indent_size": 2,
    "max_preserve_newlines": 2,
    "wrap_line_length": 0,
    "end_with_newline": false
  }
}
Beautify — хорошая альтернатива Prettier, когда нужен контроль над компактностью HTML, email-версткой или нестандартным форматированием!
Совместим с Prettier, но не рекомендуется использовать вместе в одном проекте — может возникнуть конфликт стилей.

Работа с проектами

  • Открыть папку как проект: Файл → Открыть папку...
  • Работа с рабочими пространствами: через .code-workspace файлы — удобно для монореп или нескольких папок.
  • Терминал прямо в редакторе: Ctrl+`
  • Открывай несколько окон для разных проектов.
  • Интеграция с Git: вкладка Source Control, Ctrl+Shift+G .

Быстрый старт:

  1. Открой папку проекта.
  2. Установи нужные расширения.
  3. Создай .vscode/settings.json для индивидуальных настроек проекта.

Полезные горячие клавиши:

  • Множественный курсор: Alt + Click или Ctrl + Alt + Down/Up

Как сохранить настройки VSCode (снипеты, расширения, форматтеры) при переустановке или удалении

1. Встроенная синхронизация (Settings Sync)

  • Включение:
    Ctrl+Shift+P → Turn On Settings Sync
    Или через профиль пользователя (иконка в правом нижнем углу) — Включить синхронизацию
  • Что сохраняется:
    настройки редактора ( settings.json ), снипеты, расширения, форматтеры, горячие клавиши, темы
  • Хранение: настройки синхронизируются через ваш аккаунт Microsoft или GitHub (облако)
  • Как восстановить: после новой установки VSCode авторизуйтесь в том же аккаунте — все подтянется автоматически!

2. Ручное сохранение файлов VSCode

  • Копируйте папку профиля:
    Windows: %APPDATA%\Code\
    Mac: ~/Library/Application Support/Code/
    Linux: ~/.config/Code/
  • Что копировать:
    • User/settings.json — настройки редактора
    • User/snippets/ — ваши снипеты
    • User/keybindings.json — комбинации клавиш
    • extensions/ — установленные расширения (см. следующий пункт)

3. Бэкап и восстановление расширений списком

Выполните в терминале:

code --list-extensions > my-extensions.txt

Чтобы восстановить:

cat my-extensions.txt | xargs -L 1 code --install-extension

4. Форматтеры и конфиги (.prettierrc, .eslintrc)

  • Лучше хранить внутри проекта/репозитория — так всё всегда под рукой.
  • Глобальные конфиги тоже можно скопировать из профиля пользователя.
Рекомендация:
Самый надежный и удобный способ — Settings Sync в VSCode.
В остальных случаях делайте резервные копии папки профиля и держите проектные конфиги в репозитории!

Часто задаваемые вопросы и ошибки

  • Ошибка: settings.json невалиден — проверь синтаксис!
  • Ошибка: Расширение конфликтует с другим — отключи лишние.
  • Ошибка Prettier: Лишние переносы после тега . — Исправь htmlWhitespaceSensitivity , см. раздел Prettier.
  • Вопрос: Как сбросить все настройки?
    — Открой settings.json и удали параметры, либо используй команду Reset Settings.
  • Вопрос:Как синхронизировать настройки?
    — Включи Settings Sync : Ctrl+Shift+P → Turn On Settings Sync.
  • Вопрос: Где хранятся снипеты?
    — В папке User/snippets .
  • Вопрос: Как временно отключить автоформатирование?
    — "editor.formatOnSave": false для нужного языка или глобально.
