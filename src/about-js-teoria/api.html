Работа с API в JavaScript

API (Application Programming Interface) - это интерфейс, который позволяет программам взаимодействовать друг с другом. Веб-приложения часто используют API для получения данных с серверов, отправки запросов и выполнения различных операций.

API — это "мостик" между твоим приложением и внешними данными. А с помощью функции fetch в JavaScript ты можешь легко отправлять запросы и получать данные.

💡 Что такое API?

  • API — это набор правил и протоколов, позволяющих приложениям обмениваться данными.
  • С помощью API можно получать данные с серверов, отправлять запросы и выполнять действия.
  • В JavaScript для работы с API часто используется функция fetch для отправки HTTP-запросов и получения ответов.
  • API чаще всего возвращает данные в формате JSON, который легко преобразуется в объекты JavaScript.
  • API может требовать авторизации, для чего используется токен доступа.

Для чего нужно API?

  • Интеграция с внешними сервисами (платежные системы, карты, соцсети и др.).
  • Получение актуальных данных с серверов (погода, новости и др.).
  • Отправка данных на сервер (формы обратной связи, комментарии и др.).
  • Автоматизация процессов (обновление данных, синхронизация).
  • Расширение функциональности приложений без необходимости писать весь код с нуля.

Основные понятия

  • API — интерфейс для взаимодействия между программами.
  • REST — архитектурный стиль для построения API.
  • fetch — встроенная функция JavaScript для запросов к серверу.
  • JSON — популярный формат передачи данных в API.
  • Запрос (request) и ответ (response) — основные компоненты общения с API.
  • Токен авторизации — ключ для доступа к защищённым API.
  • Эндпоинт — конкретный URL для обращения к функции API.
  • response.ok — свойство ответа, показывающее успешность запроса.

fetch

fetch это встроенная функция JavaScript, которая позволяет отправлять запросы на сервер и получать данные.

📜 Синтаксис fetch

fetch("URL")
// Преобразование данных в JSON
.then((response) => response.json())

// Работа с полученными данными
.then((data) => {
   console.log(data);
})

// Обработка ошибок
.catch((error) => {
   console.error("Произошла ошибка:", error);
});

Как это работает?

  • fetch("URL") — отправляет запрос по указанному адресу (например, к API).
  • .then((response) => response.json()) — преобразует ответ сервера из формата JSON в JavaScript-объект.
  • .then((data) => {...}) — получает готовые данные и работает с ними.
  • .catch((error) => {...}) — обрабатывает ошибки, если что-то пошло не так (например, сервер недоступен).

⚠ Важно:

● fetch возвращает промис, поэтому его результат нужно обрабатывать через .then или async/await

● Если сервер возвращает ошибку (например, 404) , fetch сам по себе не считает это ошибкой — статус нужно проверять вручную

JSON — это текстовый формат, который выглядит как объект JavaScript.

Как обработать JSON с помощью fetch?

  1. Сначала отправляем запрос с помощью fetch
  2. Используем метод .json() для преобразования данных из JSON в объект JavaScript
  3. Работаем с данными, как с обычным объектом
  4. Если сервер вернёт ошибку, обработаем её в блоке catch 
fetch("https://jsonplaceholder.typicode.com/users/1")

// Преобразуем JSON в объект
.then((response) => response.json())

// Работаем с полученными данными
.then((data) => {

//Выводим имя и email пользователя
   console.log(`Имя: ${data.name}`);
   console.log(`Email: ${data.email}`);
})

// Обрабатываем ошибки
.catch((error) => {
   console.error("Ошибка:", error.message);
});

💬 Что здесь происходит?

  • Отправляем GET-запрос к API , получаем объект пользователя.
  • response.json() преобразует ответ от сервера в JavaScript -объект.
  • Выводим имя и email пользователя.
  • Обработка ошибок: если сервер вернёт не JSON, .json() выдаст ошибку — используем .catch() .

⚠ Важно:
JSON — это текст, его нужно преобразовать в объект с помощью .json() .
Если сервер возвращает не JSON , метод .json() выдаст ошибку — не забудь добавить обработку ошибок с помощью .catch() .

Обработка ошибок

При работе с API важно правильно обрабатывать ошибки, чтобы приложение не ломалось при сбоях.

Иногда запрос может завершиться с ошибкой (например, сервер недоступен или URL неправильный) . Если мы не обработаем такие ситуации, наш код может сломаться.

Как это сделать?

  • Используй свойство response.ok , чтобы проверить, успешен ли запрос.
    Если значение false , значит, произошла ошибка.
  • Обрабатывай ошибки с помощью .catch()
    Этот метод ловит любые проблемы, которые возникают во время выполнения запроса.
  • Если сервер вернёт ошибку (например, 404) , но мы не проверим response.ok , fetch успешно выполнится, но данные будут недоступны!

📜 Пример обработки ошибок

//Отправляем запрос к API
fetch("https://jsonplaceholder.typicode.com/invalid-url")

//Проверяем статус ответа
.then((response) => {

//Проверяем, успешен ли запрос c помощью response.ok
   if (!response.ok) {

      //Если нет, выбрасываем ошибку с кодом статуса
      throw new Error("Ошибка: " + response.status);
   }

   //Преобразуем ответ в JSON
   return response.json();
})

//Работаем с полученными данными
.then((data) => {
   //Выводим данные в консоль
   console.log("Данные:", data);
})

//Обрабатываем ошибки
.catch((error) => {
   console.error("Произошла ошибка:", error.message);
});
  • Проверяем response.ok — если запрос неудачен, выбрасываем ошибку.
  • Все ошибки попадают в .catch() .
  • Если сервер вернёт ошибку (например, 404), но мы не проверим response.ok , fetch успешно выполнится, но данные будут недоступны!

⚠ Важно:
Проверяйте response.ok — если запрос неудачен, выбрасывайте ошибку.
Все ошибки должны попадать в блок .catch() , чтобы приложение не ломалось.

Отправка данных с помощью POST

POST — это метод HTTP-запроса, который используется для отправки данных на сервер.

📜 Пример отправки данных


//Отправляем POST-запрос на сервер
fetch("https://jsonplaceholder.typicode.com/posts", {
    //Указываем метод запроса
    method: "POST",
    //Заголовки запроса
    headers: {
        //Формат данных, которые мы отправляем
        "Content-Type": "application/json",
    },
    //Данные, которые мы отправляем на сервер
    //Преобразуем объект в JSON-строку
    body: JSON.stringify({
        //Заголовок нового поста
        title: "Новый пост",
        //Содержимое поста
        body: "Это содержимое поста",
        //ID пользователя, к которому относится пост
        userId: 1,
    }),
})
    //Преобразуем ответ сервера в объект
    .then((response) => response.json())
    //Обрабатываем ответ
    .then((data) => {
        //Выводим данные в консоль
        console.log("Успешно отправлено:", data);
    })
    //Обрабатываем ошибки
    .catch((error) => {
        console.error("Ошибка:", error.message);
    });

🔍 Что происходит в этом коде?

  • Метод fetch : Мы отправляем запрос по адресу https://jsonplaceholder.typicode.com/posts .
  • method: "POST" Указываем метод POST , чтобы отправить данные (по умолчанию используется GET , который только получает данные).
  • headers: Указываем заголовок Content-Type: application/json , чтобы сервер понимал, что мы отправляем данные в формате JSON
  • body: Передаём данные в виде объекта, который нужно преобразовать в строку с помощью JSON.stringify()
  • Ответ от сервера: Сервер возвращает ответ (например, успешно созданный объект), который мы преобразуем в JSON с помощью .json() и выводим в консоль.

⚠ Важно:
При отправке данных всегда указывайте заголовок Content-Type: application/json , если данные в формате JSON.
Используйте JSON.stringify() для преобразования объекта в строку JSON.

Практика

Интерактивная практика: получение пользователя по ID

Введите ID пользователя от 1 до 10 и получите информацию с помощью API.
Это эмуляция работы с fetch: данные берутся с публичного тестового сервера.

Пример 1: Получение пользователя по ID

Код примера:


	<div class="demo-box">
	    <label>Введите ID пользователя:</label>
	    <input type="number" id="userIdDemo" min="1" max="10" placeholder="например, 1" />
	    <button class="demo-btn" onclick="getUserDemo()">Показать пользователя</button>
	    <div class="demo-result" id="userDemoResult"></div>
	</div>
	

	// Получение пользователя по ID
	function getUserDemo() {
	    const userId = document.getElementById("userIdDemo").value;
	    const resultBlock = document.getElementById("userDemoResult");
	    if (!userId || userId < 1 || userId > 10) {
	        resultBlock.textContent = "Пожалуйста, введите ID от 1 до 10.";
	        return;
	    }
	    resultBlock.textContent = "Загружаем данные...";
	    fetch(`https://jsonplaceholder.typicode.com/users/${userId}`)
	        .then((response) => {
	            if (!response.ok) throw new Error("Ошибка: " + response.status);
	            return response.json();
	        })
	        .then((data) => {
	            resultBlock.innerHTML = `
	                Имя: ${data.name}

	                Email: ${data.email}

	                Город: ${data.address.city}
	            `;
	        })
	        .catch((error) => {
	            resultBlock.textContent = "Ошибка запроса: " + error.message;
	        });
	}

Пример 2: Получение списка всех пользователей

Описание: Получаем список всех пользователей с сервера и выводим их имена и email.

Пример 4: Получение комментариев к посту

Описание: Получаем список комментариев к посту по его ID и выводим имя, email и текст комментария.

Код примера:


	<div class="demo-box">
	    <label>Введите ID поста:</label>
	    <input type="number" id="postCommentsDemo" min="1" max="100" placeholder="например, 1" />
	    <button class="demo-btn" onclick="getPostCommentsDemo()">Показать комментарии</button>
	    <div class="demo-result" id="postCommentsDemoResult"></div>
	</div>
	

	// Получение комментариев к посту по ID
	function getPostCommentsDemo() {
	    const postId = document.getElementById("postCommentsDemo").value;
	    const resultBlock = document.getElementById("postCommentsDemoResult");
	    if (!postId || postId < 1 || postId > 100) {
	        resultBlock.textContent = "Пожалуйста, введите ID поста от 1 до 100.";
	        return;
	    }
	    resultBlock.textContent = "Загружаем данные...";
	    fetch(`https://jsonplaceholder.typicode.com/comments?postId=${postId}`)
	        .then((response) => {
	            if (!response.ok) throw new Error("Ошибка: " + response.status);
	            return response.json();
	        })
	        .then((comments) => {
	            if (!comments.length) {
	                resultBlock.innerHTML = "Комментариев не найдено.";
	                return;
	            }
	            resultBlock.innerHTML = "Комментарии к посту:
" + comments.map(c => `${c.name} (${c.email})
${c.body}
`).join("
");
	        })
	        .catch((error) => {
	            resultBlock.textContent = "Ошибка запроса: " + error.message;
	        });
	}

Пример 5: Получение информации о компании пользователя

Описание: Получаем информацию о компании пользователя по его ID и выводим название, слоган и вид деятельности.

Код примера:


	<div class="demo-box">
	    <label>Введите ID пользователя:</label>
	    <input type="number" id="userCompanyDemo" min="1" max="10" placeholder="например, 1" />
	    <button class="demo-btn" onclick="getUserCompanyDemo()">Показать компанию пользователя</button>
	    <div class="demo-result" id="userCompanyDemoResult"></div>
	</div>
	

	// Получение информации о компании пользователя
	function getUserCompanyDemo() {
	    const userId = document.getElementById("userCompanyDemo").value;
	    const resultBlock = document.getElementById("userCompanyDemoResult");
	    if (!userId || userId < 1 || userId > 10) {
	        resultBlock.textContent = "Пожалуйста, введите ID от 1 до 10.";
	        return;
	    }
	    resultBlock.textContent = "Загружаем данные...";
	    fetch(`https://jsonplaceholder.typicode.com/users/${userId}`)
	        .then((response) => {
	            if (!response.ok) throw new Error("Ошибка: " + response.status);
	            return response.json();
	        })
	        .then((data) => {
	            if (!data.company) {
	                resultBlock.innerHTML = "Компания не найдена.";
	                return;
	            }
	            resultBlock.innerHTML = `
	                Компания: ${data.company.name}

	                Слоган: ${data.company.catchPhrase}

	                Вид деятельности: ${data.company.bs}
	            `;
	        })
	        .catch((error) => {
	            resultBlock.textContent = "Ошибка запроса: " + error.message;
	        });
	}

Все примеры используют публичное API jsonplaceholder.typicode.com , удобно для учебных целей.
Вы можете копировать код и менять id/кнопки для реальных задач!

Советы и типичные ошибки

  • fetch возвращает промис, поэтому его результат нужно обрабатывать через .then или async/await .
  • Если сервер возвращает ошибку (например, 404) , fetch сам по себе не считает это ошибкой — статус response.ok нужно проверять вручную.
  • JSON — это текст, его нужно преобразовать в объект с помощью .json() .
  • Если сервер возвращает не JSON , метод .json() выдаст ошибку — не забудь добавить обработку ошибок с помощью .catch() .
Частая ошибка — забытый или неверный токен, неправильный Content-Type заголовок, отсутствие проверки response.ok .

Лучшие практики

  • Используйте переменные окружения для хранения токенов и ключей.
  • Не публикуйте приватные ключи и токены в открытом репозитории.
  • Документируйте работу с API для вашей команды.
  • Добавляйте обработку ошибок и логирование в интеграции с API.
  • Проверяйте статус ответа response.ok и обрабатывайте ошибки корректно.
Соблюдение лучших практик помогает обезопасить данные и упростить поддержку интеграций.
