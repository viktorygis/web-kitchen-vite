Циклы в JavaScript

Циклы — конструкции, которые повторяют выполнение блока кода, пока выполняется условие или определённое число раз.
Используются для перебора коллекций, повторных вычислений и автоматизации однотипных операций.

Основные типы циклов:

  • while — повторяет, пока условие истинно (проверка до итерации).
  • do...while — выполняет тело как минимум один раз (проверка после итерации).
  • for — универсальная форма с явным началом, условием и шагом.

Также есть for...of, for...in — рассматриваются отдельно (в темах про массивы и объекты).

Основные понятия цикла

  • Итерация — одно выполнение тела цикла.
  • Условие — выражение, определяющее продолжать или закончить цикл.
  • Счётчик — переменная, изменяющаяся на каждой итерации (например, i++).
  • Тело цикла — код, который повторяется.
  • Выход — момент, когда условие перестаёт быть истиной или выполнен break.

Общий принцип:

Инициализация → Проверка условия → (true?) Выполнение тела → Шаг → Проверка → ... → (false?) Выход

Цикл while — с предусловием

Цикл while — это «повторяй, пока условие истинно».
Полезен, когда ты не знаешь заранее, сколько итераций будет, но точно знаешь условие для остановки.

Синтаксис:

while (условие) {
  // тело цикла
}

Базовый пример:

let i = 0;              // 1. Инициализация счётчика
while (i < 7) {         // 2. Проверка условия перед каждой итерацией
  console.log("Повторение:", i); // 3. Тело (выполняется, если условие true)
  i++;                  // 4. Шаг — обязательно, иначе бесконечный цикл
}

Вывод в консоль:

Повторение: 0
Повторение: 1
Повторение: 2
Повторение: 3
Повторение: 4
Повторение: 5
Повторение: 6

Число 7 не выводится, потому что условие i < 7 становится ложным при i === 7.

Разбор по шагам:

  1. Инициализация: i = 0.
  2. Проверка условия: i < 7 → true → выполняем тело.
  3. Выполнение тела: вывод "Повторение: 0".
  4. Изменение счётчика: i++ → i = 1.
  5. Повтор: снова проверяем условие — пока true, цикл продолжается.
  6. Завершение: при i = 7 условие false → выходим.

Когда использовать:

  • Количество повторов заранее неизвестно.
  • Ожидаешь «сигнал остановки» (например, ввод пустой строки).
  • Нужна реакция на внешние изменения состояния.

Пример с пользовательским вводом:

let line;
while ((line = prompt("Введите что-нибудь (пусто — выход)")) !== "") {
  console.log("Введено:", line);
}
console.log("Завершено.");

Типичные ошибки:

  • Нет изменения счётчика: условие никогда не станет ложным → бесконечный цикл.
  • Слишком сложное условие: вынеси части во временные переменные.
  • Неожиданное приведение типов: сравнение строки с числом ("7" < 10 → true).

Цикл do...while — с постусловием

Цикл do...while похож на while, но есть ключевое отличие:
тело выполняется минимум один раз, даже если условие изначально ложно.

Синтаксис:

do {
  // тело цикла (выполнится хотя бы один раз)
} while (условие);

Базовый пример:

let i = 0;
do {
  console.log("Повторение:", i);
  i++;
} while (i < 5);

Вывод в консоль:

Повторение: 0
Повторение: 1
Повторение: 2
Повторение: 3
Повторение: 4

Почему «хотя бы один раз»?

let n = 10;
do {
  console.log("Это выведется даже при ложном условии");
} while (n < 0);

// Вывод: "Это выведется даже при ложном условии"

Здесь тело выполнится один раз, хотя n < 0 уже изначально false.

Практический пример (валидация ввод��):

let value;
do {
  value = prompt("Введите число больше 10 (Отмена - выход)", "");
  if (value === null) {
    console.log("Отменено пользователем.");
    break;
  }
} while (+value <= 10);

if (value !== null) {
  console.log("Вы ввели:", value);
}

Когда использовать:

  • Нужно выполнить действие хотя бы один раз (меню, ввод, первичная инициализация).
  • Проверка условия логически следует после выполнения шага.
  • Условие зависит от результата тела (например, пользовательского ввода).

Типичные ошибки:

  • Забыл выход: условие всегда true ��� бесконечный цикл.
  • Неправильное условие: проверяется после тела — иногда «лишняя» итерация.
  • Преобразование типов: prompt() возвращает строку — используй +value или Number(value).

Цикл for — универсальный

Цикл for — это «компактный повторитель»: запускает один и тот же блок кода заданное число раз.
Вся логика (старт, условие, изменение счётчика) сосредоточена в одной строке.

Синтаксис:

for (инициализация; условие; шаг) {
  // тело цикла
}
  • Инициализация — выполняется один раз перед первой проверкой (обычно объявление счётчика: let i = 0).
  • Условие — проверяется перед каждой итерацией. Если false — выходим.
  • Шаг — выполняется после тела каждой итерации (часто i++).
  • Тело — код, который повторяется.

Разбор базового примера:

for (let i = 0; i < 5; i++) {
  console.log("Повторение:", i);
}

Что происходит по шагам:

  1. let i = 0 — инициализация (один раз).
  2. Проверка i < 5 → true → выполняем тело → вывод Повторение: 0.
  3. i++ → i = 1, снова проверка условия → и так далее.
  4. Когда i становится 5, условие i < 5 ложно → цикл завершается.

Части цикла (таблица):

Часть Когда выполняется Пример
Инициализация Перед первой проверкой условия let i = 0
Условие Перед каждой итерацией i < 5
Тело Если условие true console.log(i)
Шаг После тела, перед следующей проверкой i++

Варианты и гибкость:

// 1. Пропуск инициализации (счётчик объявлен заранее)
let k = 0;
for (; k < 3; k++) console.log(k);

// 2. Пропуск шага
for (let j = 0; j < 3;) {
  console.log(j);
  j++;
}

// 3. Несколько переменных
for (let a = 0, b = 5; a < b; a++, b--) {
  console.log(a, b);
}

// 4. Обратный отсчёт
for (let n = 5; n > 0; n--) {
  console.log(n);
}

// 5. Бесконечный цикл (нужен break)
for (;;) {
  break;
}

Когда использовать for:

  • Заранее известно количество повторов.
  • Нужен индекс (позиция в массиве).
  • Нужно компактно описать управление счётчиком.

Типичные ошибки:

  • Бесконечный цикл: забыли изменить счётчик (i++).
  • Off-by-one: i <= arr.length вместо i < arr.length.
  • Неподходящий тип: использование var вместо let (протекает за пределы цикла).
  • Лишний индекс: когда индекс не нужен — рассмотри for...of.

Управление потоком: break и continue

break — выход из цикла

break — немедленно прерывает цикл и переходит на строку после него. 

for (let i = 0; i < 10; i++) {
  if (i === 5) {
    console.log("Нашли 5, выходим");
    break;
  }
  console.log(i);
}
// Вывод: 0, 1, 2, 3, 4, "Нашли 5, выходим"

continue — пропуск итерации

continue — пропускает остаток текущей итерации и переходит к следующей. 

for (let i = 0; i < 5; i++) {
  if (i === 2) {
    console.log("Пропускаем 2");
    continue;
  }
  console.log(i);
}
// Вывод: 0, 1, "Пропускаем 2", 3, 4

Пример: поиск первого чётного числа

for (let i = 1; i <= 10; i++) {
  if (i % 2 === 1) {
    continue; // пропускаем нечётные
  }
  console.log("Чётное:", i);
  if (i === 4) break; // выходим при нахождении 4
}

Сравнение break и continue:

Оператор Действие Продолжение
break Полностью выходит из цикла Переходит после цикла
continue Пропускает остаток текущей итерации Переходит к следующей итерации

Совет: инвертирование условия

Частые continue иногда ухудшают читаемость — попробуй инвертировать условие:

// Вместо:
for (let x of arr) {
  if (!x.active) continue;
  process(x);
}

// Можно:
for (let x of arr) {
  if (x.active) {
    process(x);
  }
}

Бесконечные циклы

Бесконечный цикл — это цикл, у которого условие завершения никогда не становится ложным (или вообще отсутствует).
Полезен, когда требуется «постоянный цикл ожидания» с явным выходом внутри. 

while (true) {
  console.log("Этот цикл никогда не закончится сам!");
  // Нужен break (или return) внутри при определённом условии
}
Никогда не оставляй бесконечный цикл без выхода — это «повисание» интерфейса и нагрузка на CPU.

Как избежать непреднамеренного бесконечного цикла:

  • Изменяй состояние: счётчик / данные должны двигать условие к завершению.
  • Проверяй условие: может ли оно стать false вообще?
  • Используй break: когда наступает «условие выхода».
  • Ограничивай попытки: добавь «страховку» (макс. число итераций).

Осознанный бесконечный цикл с защитой:

let attempts = 0;
while (true) {
  attempts++;
  if (Math.random() > 0.85) {
    console.log("Успех на попытке", attempts);
    break;
  }
  if (attempts > 1000) {
    console.log("Принудительный выход (лимит)");
    break;
  }
}

Другие способы бесконечного цикла:

// for с пустыми частями
for (;;) {
  if (someCondition) break;
}

// do...while с true
do {
  if (someCondition) break;
} while (true);

Сравнение типов циклов

Цикл Когда использовать Проверка условия Мин. итераций Пример сценария
while Неизвестно число повторов заранее Перед каждой итерацией 0 Чтение до пустой строки
do...while Нужно выполнить минимум 1 раз После итерации 1 Меню ввода, валидация
for Число повторов известно / индекс важен Перед каждой итерацией 0 Перебор первых N элементов

Лучшие практики

  • Избегай «магических чисел» — выноси пределы в переменные/константы.
  • Используй for...of для итерации по массивам, когда индекс не нужен.
  • Используй методы массива (map, filter, reduce) для декларативной логики.
  • Преждевременный break вместо лишних флагов увеличивает читаемость.
  • Не изменяй длину массива (например, push/splice) внутри обычного for.
  • Старайся, чтобы тело цикла было небольшим — вынеси сложную логику в функцию.

Пример хорошего стиля:

// Хорошо: ясная цель, чистый код
for (let i = 0; i < items.length; i++) {
  const item = items[i];
  if (!item.valid) continue;

  processItem(item);
}

// Плохо: вложенность, трудно читать
for (let i = 0; i < items.length; i++) {
  if (items[i].valid) {
    if (items[i].active) {
      if (items[i].priority > 5) {
        // глубокая вложенность
      }
    }
  }
}

Ловушки

  • Бесконечный цикл: 
    // ОШИБКА
let i = 0;
while (i < 10) {
  console.log(i);
  // забыли i++
}
  • Off-by-one ошибка в for: 
    // ОШИБКА
let arr = [1, 2, 3];
for (let i = 0; i <= arr.length; i++) {
  console.log(arr[i]); // undefined на последней итерации
}

// ПРАВИЛЬНО
for (let i = 0; i < arr.length; i++) {
  console.log(arr[i]);
}
  • Модификация массива в цикле: 
    // ОШИБКА
let arr = [1, 2, 3, 4, 5];
for (let i = 0; i < arr.length; i++) {
  if (arr[i] === 3) {
    arr.splice(i, 1); // удаляем элемент
    // но i не пересчитывается, индексы сбиваются
  }
}
  • Область видимости i (var vs let): 
    // С var: i доступна везде
for (var i = 0; i < 3; i++) {}
console.log(i); // 3 (утечка!)

// С let: i видна только в цикле
for (let i = 0; i < 3; i++) {}
console.log(i); // ReferenceError
  • Использование for...in для массива: приводит к перебору «лишних» ключей / не гарантирует порядок.

Итоги

  • while — когда заранее неизвестно число итераций.
  • do...while — гарантированно одна итерация.
  • for — когда нужно явное управление счётчиком.
  • break и continue управляют потоком внутри цикла.
  • Бесконечные циклы нужны редко — всегда добавляй условие выхода.
  • Современные методы массивов часто читаемее ручных циклов.
  • Используй let, а не var для счётчика в for.
  • Избегай глубокой вложенности — вынеси сложную логику в функцию.
