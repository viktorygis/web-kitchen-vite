Объекты в JavaScript

Объекты — это составные типы данных, представляющие собой коллекции пар "ключ-значение". Они используются для хранения сложных структур данных и являются основой объектно-ориентированного программирования в JavaScript.

Объекты — это как коробки, в которых можно хранить несколько разных вещей (значений). Вместо того, чтобы хранить переменные по отдельности, объекты помогают нам группировать их и делать код более организованным. 🗂️

Объекты используются для хранения коллекций различных значений и более сложных сущностей. В JavaScript объекты используются очень часто, это одна из основ языка.

Объект может быть создан с помощью фигурных скобок {…} с необязательным списком свойств.
Свойство – это пара «ключ: значение», где ключ – это строка (также называемая «именем свойства»), а значение может быть чем угодно.

Пример объекта:

const user = {
  name: 'Андрей',
  age: 30
};

Мы можем представить объект в виде ящика с подписанными папками. Каждый элемент данных хранится в своей папке, на которой написан ключ. По ключу папку легко найти, удалить или добавить в неё что-либо.

Зачем нужны объекты?

  • Хранение связанных данных в одной структуре
  • Создание сложных сущностей с методами и свойствами
  • Использование в объектно-ориентированном программировании
  • Упрощение передачи данных между функциями и модулями

Ключи в объекте должны быть уникальными

Если в объекте указать несколько свойств с одинаковым ключом, то будет использовано последнее указанное значение.

let car = {
  model: "Toyota",
  year: 2020
};

console.log(car); // { model: 'Toyota', year: 2020 }

car.model = "Honda"; // Изменяем значение свойства
console.log(car); // { model: 'Honda', year: 2020 }

Способы создания объектов

1. Литерал объекта (рекомендуется)

Литерал объекта - самый распространённый и удобный способ создания объектов. Позволяет создавать объекты с заданными свойствами и методами в компактном виде.

Синтаксис:

const obj = {
  key: 'value',
  method() { /* ... */ }
};

При использовании литерального синтаксиса {...} мы сразу можем поместить в объект несколько свойств в виде пар «ключ: значение». У каждого свойства есть ключ (также называемый «имя» или «идентификатор»). После имени свойства следует двоеточие ":", и затем указывается значение свойства. Если в объекте несколько свойств, то они перечисляются через запятую.

2. Конструктор Object

Конструктор Object - функция, которая создаёт новый объект. Обычно его не используют для создания объектов, так как есть более удобные способы.

Синтаксис:

const obj = new Object();
obj.key = 'value';

3. Object.create()

Object.create() - создаёт новый объект с указанным прототипом. Полезен для создания объектов с наследованием.

Синтаксис:

const prototype = { a: 1 };
const obj = Object.create(prototype);

4. Конструктор класса (ES6)

Конструктор класса - современный синтаксис для создания объектов и работы с наследованием. Позволяет определять шаблоны для создания похожих объектов.

Синтаксис:

class Person {
  constructor(name) {
    this.name = name;
  }
}
const obj = new Person('Андрей');

5. Объект как константа

const user = {} - Объект, объявленный как константа, может быть изменён. Только сама переменная защищена от переприсваивания.

Пример:

const user = {
  name: "John"
};

user.name = "Pete"; // Работает

alert(user.name); // Pete

// Но это не работает:
user = {}; // Ошибка!

Объявление const защищает от изменений только саму переменную user, а не её содержимое. user хранит ссылку на объект. Эта ссылка не может быть изменена. Но содержимое объекта менять можно.

Сравнение способов создания объектов

Название Способ Описание Когда использовать
Литерал объекта {…} Создание объекта с заданными свойствами и методами. Для простых объектов с известными свойствами.
Конструктор Object new Object() Создание пустого объекта, свойства добавляются позже. Редко используется, когда нужен пустой объект.
Object.create() Object.create(proto) Создание объекта с указанным прототипом для наследования. Когда нужно создать объект с определённым прототипом.
Конструктор класса class MyClass {…} Определение шаблона для создания похожих объектов с наследованием. Для сложных структур и объектов с поведением (методами).

Работа со свойствами объектов

1. Доступ к свойствам

Точечная нотация

Для обращения к свойствам используется запись «через точку»:

const person = { name: 'Андрей' };

// Точечная нотация
console.log(person.name); // 'Андрей'

Скобочная нотация

Для свойств, имена которых состоят из нескольких слов или содержат специальные символы:

let user = { "likes birds": true };
// Скобочная нотация
console.log(user['likes birds']); // true

Для динамических ключей

const person = { name: 'Андрей' };
const key = 'name';
console.log(person[key]); // 'Андрей'

2. Добавление и изменение свойств

const car = { model: 'Tesla' };

// Добавление свойства
car.year = 2023;

// Изменение свойства
car.model = 'Model S';

console.log(car);
// { model: 'Model S', year: 2023 }

3. Удаление свойств

Для удаления свойства используется оператор delete:

const obj = { a: 1, b: 2, c: 3 };

delete obj.b;
console.log(obj); // { a: 1, c: 3 }

// Проверка существования
console.log('b' in obj); // false

4. Короткие свойства (ES6)

Короткие свойства - это синтаксис, позволяющий создавать свойства объекта, используя только имя переменной, если имя свойства совпадает с именем переменной. Это сокращает запись и улучшает читаемость кода.

const name = 'Андрей';
const age = 30;

// Вместо { name: name, age: age }
const person = { name, age };

console.log(person); // { name: 'Андрей', age: 30 }

5. Оператор in

Оператор in — проверяет наличие свойства в объекте, включая унаследованные свойства.

Синтаксис:

propertyName in object

Основные особенности:

  • Проверяет собственные и унаследованные свойства
  • Лучше использовать, чем сравнение с undefined
  • Возвращает boolean: true или false

Пример проверки свойства:

const car = { model: 'Tesla' };
console.log('model' in car); // true
console.log('year' in car); // false

Пример: суммирование зарплат

let salaries = {
  John: 100,
  Ann: 160,
  Pete: 130
};

let sum = 0;

for (let key in salaries) {
  sum += salaries[key];
}
console.log(sum); // 390

Пример: умножение числовых свойств

let menu = {
  width: 200,
  height: 300,
  title: "My menu"
};

console.log(menu); // До: {width: 200, height: 300, title: 'My menu'}

function multiplyNumeric(obj) {
  for (let key in obj) {
    if (typeof obj[key] === "number") {
      obj[key] *= 2;
    }
  }
}

multiplyNumeric(menu);
console.log(menu); // После: {width: 400, height: 600, title: 'My menu'}

Методы объектов

Метод объекта — это функция, которая хранится как свойство объекта и предназначена для выполнения действий с данными этого объекта.

Методы вызываются с помощью точки (.) после имени объекта, а затем скобок (), как обычная функция. Внутри метода можно использовать ключевое слово this для доступа к другим свойствам того же объекта.

Пример простого метода:

const person = {
  name: "Иван",
  greet: function() {
    console.log("Меня зовут " + this.name);
  }
};

person.greet(); // Выведет: Меня зовут Иван

Пример объекта с несколькими методами:

const person = {
  name: "Иван",
  age: 25,
  greet: function() {
    console.log("Меня зовут " + this.name);
  },
  showAge: function() {
    console.log("Мне " + this.age + " лет");
  }
};

person.greet(); // Меня зовут Иван
person.showAge(); // Мне 25 лет

Современный синтаксис методов (ES6):

В современном JavaScript можно использовать сокращённый синтаксис для методов:

const person = {
  name: "Иван",
  age: 25,

  // Сокращённый синтаксис (без function и двоеточия)
  greet() {
    console.log("Меня зовут " + this.name);
  },

  showAge() {
    console.log("Мне " + this.age + " лет");
  }
};

person.greet(); // Меня зовут Иван

Особенности методов объектов:

  • Метод имеет доступ к свойствам объекта через this
  • Можно добавлять и удалять методы динамически
  • Методы могут вызывать другие методы того же объекта
  • Методы могут возвращать значения
  • Стрелочные функции как методы работают иначе (иной контекст this)

Встроенные методы Object

Встроенные методы Object — это функции, которые JavaScript предоставляет для работы с объектами. Они позволяют получать ключи, значения, пары ключ-значение, копировать объекты, защищать от изменений и многое другое.

1. Object.keys()

Object.keys() — возвращает массив ключей объекта.

Особенности:

  • Возвращает только перечисляемые свойства
  • Не возвращает унаследованные свойства
  • Порядок ключей сохраняется

Синтаксис:

Object.keys(obj)

Пример:

const person = {
  name: "Андрей",
  age: 30
};
const keys = Object.keys(person);
console.log(keys); // ['name', 'age']

2. Object.values()

Object.values() — возвращает массив значений объекта.

Особенности:

  • Порядок значений соответствует Object.keys()
  • ES2017+ (не поддерживается в старых браузерах)
  • Возвращает только собственные свойства

Синтаксис:

Object.values(obj)

Пример:

const person = { name: 'Андрей', age: 30 };
const values = Object.values(person);
console.log(values); // ['Андрей', 30]

3. Object.entries()

Object.entries() — возвращает массив пар [ключ, значение].

Особенности:

  • Удобен для преобразования объектов в Map
  • Используется для итерация по объекту
  • ES2017+

Синтаксис:

Object.entries(obj)

Пример:

const person = {
  name: "Андрей",
  age: 30
};
const entries = Object.entries(person);
console.log(entries);
// [['name', 'Андрей'], ['age', 30]]

4. Object.assign()

Object.assign() — копирует свойства из объектов-источников в целевой объект.

Особенности:

  • Изменяет целевой объект (мутирующий метод)
  • Не копирует унаследованные свойства
  • Поверхностное копирование (не глубокое)
  • Возвращает целевой объект

Синтаксис:

Object.assign(target, ...sources)

Пример: объединение объектов

const target = { a: 1 };
const source = { b: 2 };

Object.assign(target, source);
console.log(target); // { a: 1, b: 2 }

5. Object.freeze()

Object.freeze() — замораживает объект, предотвращая любые изменения.

Особенности:

  • Нельзя добавлять/удалять/изменять свойства
  • В строгом режиме попытки изменения вызывают ошибку
  • Не рекурсивен (вложенные объекты не замораживаются)
  • Возвращает замороженный объект

Синтаксис:

Object.freeze(obj)

Пример:

const person = Object.freeze({ name: 'Андрей' });

person.name = 'Иван'; // Не сработает
console.log(person.name); // 'Андрей'

// Проверка
console.log(Object.isFrozen(person)); // true

6. Object.seal()

Object.seal() — запрещает добавление и удаление свойств, но позволяет менять значения существующих свойств.

Особенности:

  • Можно менять существующие свойства
  • Нельзя добавлять новые свойства
  • Нельзя удалять свойства
  • Менее строгий, чем Object.freeze()

Пример:

const obj = Object.seal({ a: 1 });

obj.a = 2; // Работает
console.log(obj.a); // 2

obj.b = 3; // Не добавит
console.log(obj); // { a: 2 }

console.log(Object.isSealed(obj)); // true

7. Object.hasOwn() (ES2022)

Object.hasOwn() — проверяет наличие собственного свойства объекта (новый стандарт вместо hasOwnProperty).

Особенности:

  • Проверяет только собственные свойства (не унаследованные)
  • Безопаснее, чем hasOwnProperty
  • ES2022+

Пример:

const obj = { a: 1 };

console.log(Object.hasOwn(obj, 'a')); // true
console.log(Object.hasOwn(obj, 'b')); // false
console.log(Object.hasOwn(obj, 'toString')); // false
// результат появится здесь

Лучшие практики работы с объектами

1. Предпочитайте литерал объекта конструктору Object

✅ Хорошо:

const user = {
  name: 'Андрей',
  age: 30
};

❌ Плохо:

const user = new Object();
user.name = 'Андрей';
user.age = 30;

2. Используйте const для объектов по умолчанию

// ✅ Хорошо - const позволяет менять содержимое
const user = { name: 'Андрей' };
user.name = 'Иван'; // OK

// ❌ Плохо - пересваивание переменной
let config = { debug: true };
config = { debug: false }; // Лишнее переприсваивание

3. Используйте деструктуризацию для доступа к свойствам

✅ Хорошо:

const user = { name: 'Андрей', age: 30, city: 'Москва' };
const { name, age } = user;
console.log(name, age);

❌ Плохо:

const user = { name: 'Андрей', age: 30, city: 'Москва' };
console.log(user.name, user.age, user.city);

4. Используйте Object.keys()/values()/entries() для итерации

const obj = { a: 1, b: 2, c: 3 };

// ✅ Хорошо - современный подход
for (const [key, value] of Object.entries(obj)) {
  console.log(key, value);
}

// ⚠️ Приемлемо - цикл for...in
for (const key in obj) {
  if (Object.hasOwn(obj, key)) {
    console.log(key, obj[key]);
  }
}

5. Проверяйте существование свойств правильно

const obj = { prop: undefined };

// ✅ Хорошо - проверяет само наличие свойства
console.log(Object.hasOwn(obj, 'prop')); // true

// ❌ Плохо - undefined может быть значением
console.log(obj.prop !== undefined); // false (неправильно!)

6. Избегайте поверхностного копирования для вложенных объектов

const original = {
  user: { name: 'Андрей' },
  settings: { theme: 'dark' }
};

// ❌ Плохо - поверхностное копирование
const shallow = Object.assign({}, original);
shallow.user.name = 'Иван'; // Изменилась оригинальная ссылка!

// ✅ Хорошо - глубокое копирование
const deep = JSON.parse(JSON.stringify(original));
deep.user.name = 'Иван'; // original не изменился

7. Используйте Object.freeze() для константных данных

const APP_CONFIG = Object.freeze({
  API_URL: 'https://api.example.com',
  TIMEOUT: 5000,
  DEBUG: false
});

// ❌ Это не сработает (ошибка в строгом режиме или молча игнорируется)
APP_CONFIG.API_URL = 'http://another.com';

8. Используйте опциональную цепочку для безопасного доступа

const user = {
  profile: {
    name: 'Андрей'
  }
};

// ✅ Хорошо - опциональная цепочка (ES2020)
console.log(user?.profile?.name); // 'Андрей'
console.log(user?.contacts?.email); // undefined

// ❌ Плохо - может вызвать ошибку
console.log(user.contacts.email); // TypeError!

Типичные ошибки и ловушки

❌ Ошибка 1: Поверхностное копирование вложенных объектов

const obj1 = { data: { value: 10 } };
const obj2 = { ...obj1 }; // Spread operator

obj2.data.value = 20;
console.log(obj1.data.value); // 20 - изменился оригинал!

// ✅ Решение: используйте глубокое копирование
const obj3 = JSON.parse(JSON.stringify(obj1));
obj3.data.value = 30;
console.log(obj1.data.value); // 20 - оригинал не изменился

❌ Ошибка 2: Сравнение объектов по значению вместо по ссылке

const obj1 = { a: 1 };
const obj2 = { a: 1 };

console.log(obj1 === obj2); // false - разные ссылки!
console.log(obj1 == obj2); // false

// ✅ Правильное сравнение значений:
const isEqual = JSON.stringify(obj1) === JSON.stringify(obj2);
console.log(isEqual); // true

❌ Ошибка 3: Использование this без контекста в методах

const user = {
  name: 'Андрей',
  greet: function() {
    console.log(this.name);
  }
};

const greet = user.greet;
greet(); // undefined - потеря контекста this!

// ✅ Решение 1: используйте стрелочные функции
const user2 = {
  name: 'Андрей',
  greet: () => {
    console.log(this.name); // this сохраняется
  }
};

// ✅ Решение 2: используйте bind
const boundGreet = user.greet.bind(user);
boundGreet(); // 'Андрей'

❌ Ошибка 4: Забывчивость о порядке свойств в цикле for...in

const obj = { b: 2, a: 1, c: 3 };

// for...in может перебирать свойства в непредсказуемом порядке
for (const key in obj) {
  console.log(key); // Порядок не гарантирован!
}

// ✅ Решение: используйте Object.entries() с известным порядком
for (const [key, value] of Object.entries(obj)) {
  console.log(key, value); // Порядок гарантирован
}

❌ Ошибка 5: Проверка существования свойства через undefined

const obj = { prop: undefined };

// ❌ Плохо - возвращает false, хотя свойство существует
if (obj.prop !== undefined) {
  console.log('Свойство существует');
} // Не выполнится

// ✅ Хорошо - проверяет само наличие свойства
if (Object.hasOwn(obj, 'prop')) {
  console.log('Свойство существует');
} // Выполнится

❌ Ошибка 6: Забывчивость о неизменяемости массивов в Object.freeze()

const obj = Object.freeze({
  arr: [1, 2, 3]
});

obj.arr = [4, 5, 6]; // Не сработает
obj.arr[0] = 999; // Но это СРАБОТАЕТ!

console.log(obj.arr); // [999, 2, 3]

// ✅ Решение: заморозьте и вложенные массивы
function deepFreeze(obj) {
  Object.freeze(obj);
  Object.values(obj).forEach(val => {
    if (typeof val === 'object' && val !== null) {
      deepFreeze(val);
    }
  });
}

const frozen = deepFreeze({ arr: [1, 2, 3] });
frozen.arr[0] = 999; // TypeError в строгом режиме

❌ Ошибка 7: Итерация по скрытым свойствам объекта

const parent = { inheritedProp: 'inherited' };
const child = Object.create(parent);
child.ownProp = 'own';

// ❌ Плохо - включит унаследованные свойства
for (const key in child) {
  console.log(key); // ownProp, inheritedProp
}

// ✅ Хорошо - только собственные свойства
for (const key in child) {
  if (Object.hasOwn(child, key)) {
    console.log(key); // только ownProp
  }
}

Продвинутые техники работы с объектами

1. Деструктуризация объектов (ES6)

Деструктуризация объектов — синтаксис, позволяющий извлекать свойства из объекта и присваивать их переменным в одной строке. 

const user = {
  id: 1,
  name: 'Андрей',
  address: {
    city: 'Москва'
  }
};

// Базовый вариант
const { name, id } = user;

// С переименованием (псевдонимом)
const { name: userName, id: userId } = user;

// Вложенная деструктуризация
const { address: { city } } = user;

// С значениями по умолчанию
const { age = 30 } = user;

console.log(userName, city, age);
// 'Андрей', 'Москва', 30

2. Опциональная цепочка (ES2020)

Опциональная цепочка — оператор ?. позволяет безопасно получать доступ к вложенным свойствам и методам, даже если промежуточные значения равны null или undefined. 

const user = {
  profile: {
    name: 'Андрей'
  }
};

// Безопасный доступ к вложенным свойствам
console.log(user?.profile?.name); // 'Андрей'
console.log(user?.contacts?.email); // undefined

// Безопасный вызов методов
user.greet?.(); // Не вызовет ошибку, если метода нет

// С объединением нулевого значения (??)
const email = user?.email ?? 'no-email@example.com';
console.log(email); // 'no-email@example.com'

3. Object.defineProperty()

Object.defineProperty() — метод для определения новых или изменения существующих свойств объекта с полным контролем через дескрипторы.

Дескрипторы свойства:

  • value — значение свойства
  • writable — можно ли менять значение (true/false)
  • enumerable — перечисляется ли в for...in (true/false)
  • configurable — можно ли менять дескриптор (true/false)
  • get — функция-геттер
  • set — функция-сеттер
const obj = {};

// Определение свойства "только для чтения"
Object.defineProperty(obj, 'readOnly', {
  value: 42,
  writable: false,
  enumerable: true,
  configurable: false
});

obj.readOnly = 100; // Не изменится (в строгом режиме - ошибка)
console.log(obj.readOnly); // 42

// Определение скрытого свойства
Object.defineProperty(obj, 'hidden', {
  value: 'secret',
  enumerable: false
});

console.log(Object.keys(obj)); // ['readOnly'] - hidden не включено

4. Геттеры и сеттеры

Геттеры и сеттеры — специальные функции, которые автоматически вызываются при доступе и присваивании значения свойству. 

const user = {
  firstName: 'Андрей',
  lastName: 'Иванов',

  // Геттер
  get fullName() {
    return `${this.firstName} ${this.lastName}`;
  },

  // Сеттер
  set fullName(value) {
    [this.firstName, this.lastName] = value.split(' ');
  }
};

console.log(user.fullName); // 'Андрей Иванов'
user.fullName = 'Петр Сидоров';
console.log(user.firstName); // 'Петр'
console.log(user.lastName); // 'Сидоров'

5. Proxy объекты (ES6)

Proxy — объект, который позволяет перехватывать и переопределять базовые операции над другим объектом. 

const user = { name: 'Андрей', age: 30 };

const handler = {
  get(target, prop) {
    console.log(`Доступ к свойству: ${prop}`);
    return target[prop];
  },
  set(target, prop, value) {
    console.log(`Установка ${prop} = ${value}`);
    target[prop] = value;
  }
};

const proxy = new Proxy(user, handler);

console.log(proxy.name); // "Доступ к свойству: name" → 'Андрей'
proxy.age = 31; // "Установка age = 31"

6. Spread оператор для объектов (ES2018)

Spread оператор — ... копирует свойства объекта в новый объект (поверхностное копирование). 

const obj1 = { a: 1, b: 2 };
const obj2 = { c: 3 };

// Объединение объектов
const merged = { ...obj1, ...obj2 };
console.log(merged); // { a: 1, b: 2, c: 3 }

// Переопределение свойств
const updated = { ...obj1, b: 99 };
console.log(updated); // { a: 1, b: 99 }

// Клонирование (поверхностное)
const clone = { ...obj1 };
console.log(clone); // { a: 1, b: 2 }

Итоги и выводы

Ключевые моменты об объектах:

  • Основа JavaScript — объекты являются фундаментальной частью языка и используются везде
  • Ссылочный тип — объекты передаются по ссылке, а не по значению
  • Динамические — свойства можно добавлять, менять и удалять во время выполнения
  • Гибкие — значениями могут быть любые типы данных, включая функции (методы)
  • Мощные методы — Object.keys(), Object.entries(), Object.assign() и другие упрощают работу

Когда использовать разные способы создания:

Способ Когда использовать
Литерал {} В 99% случаев - для простых и сложных объектов
Классы (class) Когда нужны множественные экземпляры с методами и наследованием
Object.create() Для специфического контроля над прототипом
new Object() Редко используется, не рекомендуется

Практические советы:

  • ✅ Используйте const для объектов - вы можете менять их содержимое
  • ✅ Применяйте деструктуризацию для упрощения кода
  • ✅ Используйте опциональную цепочку ?. для безопасного доступа
  • ✅ Предпочитайте Object.entries() циклам for...in
  • ✅ Помните о поверхностном копировании при использовании Object.assign()
  • ❌ Избегайте прямого доступа через точку к неизвестным свойствам
  • ❌ Не забывайте проверять существование свойств перед использованием
  • ❌ Не полагайтесь на порядок свойств в цикле for...in
