Асинхронность в JavaScript: промисы и async/await

JavaScript — однопоточный язык, но с помощью промисов и async/await можно удобно и безопасно работать с асинхронными операциями, такими как запросы к серверу, таймеры и обработка данных.

Асинхронность в JavaScript — это способ выполнять операции, не блокируя основной поток выполнения. Это позволяет создавать отзывчивые приложения, которые могут обрабатывать события, такие как клики и ввод данных, в то время как другие задачи выполняются в фоновом режиме.

Примеры использования

  • Загрузка данных с сервера без блокировки интерфейса.
  • Выполнение длительных операций, таких как обработка изображений или видео.
  • Работа с таймерами и событиями, например, анимация или задержка выполнения кода.

Асинхронность позволяет JavaScript быть более эффективным и отзывчивым, особенно в веб-приложениях, где важно быстро реагировать на действия пользователя.

Основные понятия

  • Promise — объект, представляющий результат асинхронной операции.
    Promise может находиться в одном из трёх состояний: pending (ожидание), fulfilled (успех) или rejected (ошибка).
  • .then() — метод для обработки успешного результата промиса.
    then() позволяет указать функцию, которая будет выполнена, когда промис завершится успешно.
  • .catch() — метод для обработки ошибок промиса.
    catch() позволяет указать функцию, которая будет выполнена, если промис завершится с ошибкой.
  • async/await — синтаксис для написания асинхронного кода, похожего на последовательный.
    async обозначает асинхронную функцию, а await позволяет "дождаться" завершения промиса перед выполнением следующей строки кода.

Promise

Promise — это объект, который представляет асинхронную операцию и позволяет обрабатывать её результат или ошибку.

  • У промиса три состояния: pending (ожидание), fulfilled (успех), rejected (ошибка).
  • Для завершения промиса вызываются функции resolve (успех) или reject (ошибка).
  • Промисы позволяют избежать "адских колбеков" и делают код более читаемым.
  • Промисы можно цепочкой обрабатывать с помощью .then() и .catch() .

Пример кода:

const myPromise = new Promise((resolve, reject) => {
  // Асинхронный код
});

Пример pending:

// Асинхронная операция, которая ещё не завершена
const myPromise = new Promise((resolve, reject) => {

// Промис находится в состоянии pending
console.log("Промис в состоянии pending");

// Здесь можно выполнять асинхронные действия, например, запрос к	серверу
setTimeout(() => {
   resolve("Операция завершена успешно!");
}, 2000);

// После завершения асинхронной операции вызываем resolve или reject
resolve("Результат операции");
reject("Произошла ошибка.");
});

Пример fulfilled:

// Промис успешно завершён
const myPromise = new Promise((resolve, reject) => {

// Асинхронная операция завершена успешно
console.log("Промис в состоянии fulfilled");

resolve("Операция завершена успешно!");

// Здесь можно выполнять действия с результатом
console.log("Результат операции:", "Операция завершена успешно!");

// После успешного завершения вызываем resolve
resolve("Результат операции");
reject("Произошла ошибка.");
});

Пример rejected:

// Промис завершён с ошибкой
const myPromise = new Promise((resolve, reject) => {

// Асинхронная операция завершена с ошибкой
console.log("Промис в состоянии rejected");
reject("Произошла ошибка.");

// Здесь можно выполнять действия с ошибкой
console.error("Ошибка операции:", "Произошла ошибка.");

// После завершения с ошибкой вызываем reject
reject("Ошибка операции");
resolve("Результат операции");
});
Промисы позволяют писать асинхронный код, который легко читать и поддерживать. Они помогают избежать "адских колбеков" и делают обработку ошибок более структурированной.

.then()

.then() - это метод промиса, который позволяет обработать успешный результат асинхронной операции. Он принимает функцию обратного вызова, которая будет выполнена, когда промис завершится успешно.

then() позволяет писать асинхронный код в виде цепочек вызовов, что делает его более читаемым и понятным.

  • .then() позволяет обработать успешный результат промиса.
  • Внутри .then() можно выполнять любые действия с полученными данными.
  • Можно цепочкой вызывать .then() для последовательной обработки результатов.
  • Если нужно обработать ошибку, используйте .catch() после .then() .
  • Если нужно выполнить действия независимо от результата промиса, используйте .finally() .
  • Метод .then() возвращает новый промис, что позволяет строить цепочки вызовов.

Пример:

myPromise
  .then((result) => {
    console.log(result);
  });

Пример:

//myPromise — это промис, который может завершиться успешно или с ошибкой
//new Promise создаёт новый промис
const myPromise = new Promise((resolve) => {
setTimeout(() => resolve("Операция завершена успешно!"), 2000);
});
myPromise.then((result) => {
console.log(result); // "Операция завершена успешно!"
});

.catch()

.catch() — это метод промиса, который позволяет обработать ошибки или отклонения промиса. Он принимает функцию обратного вызова, которая будет выполнена, если промис завершится с ошибкой.

catch() позволяет централизованно обрабатывать ошибки в цепочке промисов, что делает код более чистым и понятным.

  • .catch() используется для обработки ошибок или отклонения промиса.
  • Внутри .catch() обычно выводится ошибка или выполняется альтернативная логика.
  • Если промис завершился с ошибкой, управление передаётся в блок .catch() .
  • Если нужно выполнить действия независимо от результата промиса, используйте .finally() .
  • Метод .catch() возвращает новый промис, что позволяет строить цепочки вызовов.

Пример:

myPromise
  .catch((error) => {
    console.error(error);
  });

Пример:

//myPromise — это промис, который может завершиться с ошибкой
const myPromise = new Promise((resolve, reject) => {

// Асинхронная операция, которая может завершиться с ошибкой
// Например, запрос к серверу или таймер
setTimeout(() => {
   reject("Произошла ошибка.");
}, 2000);
});

myPromise
//Если промис завершился успешно, выводим результат
.then((result) => {
   console.log(result); // "Операция завершена успешно!"
	})

//Если промис завершился с ошибкой, выводим сообщение об ошибке
.catch((error) => {
   console.error(error); // "Произошла ошибка."
});
Метод .catch() позволяет обрабатывать ошибки в цепочке промисов, что делает код более чистым и понятным. Он помогает избежать "адских колбеков" и упрощает обработку ошибок.

async/await

async/await — это синтаксический сахар для работы с промисами, который позволяет писать асинхронный код в виде последовательных операций. С помощью async можно объявить асинхронную функцию, а await позволяет "дождаться" завершения промиса перед выполнением следующей строки кода.

async/await делает асинхронный код более похожим на синхронный, что упрощает его чтение и понимание. Это позволяет избежать "адских колбеков" и делает обработку ошибок более структурированной.

  • async обозначает асинхронную функцию.
  • await позволяет "дождаться" завершения промиса перед переходом к следующей строке.
  • Ошибки обрабатываются через try/catch внутри асинхронной функции.
  • Асинхронные функции возвращают промис, что позволяет использовать .then() и .catch() для обработки результатов.
  • Асинхронные функции можно вызывать как обычные функции, но они всегда возвращают промис.

Пример:

async function fetchData() {
  const result = await new Promise((resolve) => {
    setTimeout(() => resolve("Данные загружены!"), 2000);
  });
  console.log(result);
}

Пример:

async function fetchData() {
  try {
    // Создаём промис, который через 2 секунды завершится с ошибкой
    const result = await new Promise((resolve, reject) => {
      setTimeout(() => {
        reject("Ошибка: данные не найдены!");
      }, 2000);
    });

    // Если промис завершился успешно, выводим результат
    console.log(result);
  } catch (error) {
    // Если промис завершился с ошибкой, выводим ошибку
    console.error(error); // "Ошибка: данные не найдены!"
  }
}

Советы и типичные ошибки

  • Используйте try/catch для обработки ошибок в async функциях.
  • Не используйте await вне async функции — будет синтаксическая ошибка.
  • Если цепочка промисов длинная — рассмотрите .then для последовательной обработки.
Важно: всегда обрабатывайте ошибки промисов, чтобы приложение не "падало" неожиданно.

Лучшие практики

  • Старайтесь писать асинхронный код с async/await для лучшей читаемости.
  • Обрабатывайте ошибки через try/catch или .catch .
  • Не смешивайте then/catch и async/await в одном фрагменте кода — выберите один стиль.
  • Документируйте асинхронные функции и их возвращаемые значения.
