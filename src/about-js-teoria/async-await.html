Асинхронность в JavaScript: промисы, callbacks и async/await

JavaScript — однопоточный язык, но с помощью асинхронных механизмов (callbacks, promises, async/await) он эффективно обрабатывает операции ввода/вывода, сетевые запросы и таймеры, не блокируя интерфейс.

На этой странице — понятия, корректные примеры и рекомендации по работе с асинхронностью.

Основные понятия

  • Callback — функция, переданная в другую функцию и вызываемая позже, после завершения операции.
  • Promise — объект, представляющий результат асинхронной операции; состояния: pending, fulfilled, rejected.
  • .then / .catch / .finally — методы промиса для обработки результата, ошибок и завершающих действий.
  • async / await — синтаксический сахар над промисами: async помечает функцию, await приостанавливает исполнение внутри async до разрешения промиса.
  • Параллельность — выполнение нескольких асинхронных задач одновременно (Promise.all, Promise.allSettled и т.д.).

Callbacks

Callback — самый простой способ уведомить код о завершении операции. Минус: при нескольких вложенных асинхциях код становится трудно читаемым ("callback hell").

function doWork(cb) {
  setTimeout(() => cb(null, "result"), 100);
}

doWork((err, res) => {
  if (err) console.error(err);
  else console.log(res); // "result"
});

Если цепочка асинхронных действий растёт — лучше перейти на промисы или async/await.

Promise

Promise — это объект, который представляет будущее значение асинхронной операции.

Создание промиса

const p = new Promise((resolve, reject) => {
  // Асинхронная операция
  setTimeout(() => resolve("OK"), 1000);
  // в случае ошибки:
  // reject(new Error("fail"));
});

Состояния

  • pending — ожидает выполнения.
  • fulfilled — успешно завершён.
  • rejected — завершён с ошибкой.

Промисы позволяют строить цепочки обработки результатов и централизовать обработку ошибок.

.then / .catch / .finally

Методы промиса: .then обрабатывает успешный результат, .catch — ошибки, .finally выполняется в любом случае после завершения.

fetch('/api/data')
  .then(res => res.json())
  .then(data => {
    console.log('data', data);
  })
  .catch(err => {
    console.error('Ошибка:', err);
  })
  .finally(() => {
    console.log('Завершено');
  });

Важно: .catch поймает ошибки из любой предыдущей части цепочки.

async / await

async/await — делает асинхронный код более похожим на синхронный; await можно использовать только внутри async-функции.

async function fetchData() {
  try {
    const res = await fetch('/api/data');
    const json = await res.json();
    return json;
  } catch (err) {
    console.error('Ошибка:', err);
    throw err;
  }
}

async-функция всегда возвращает промис; если внутри функции бросается ошибка — внешний промис отклоняется.

Параллельно / Последовательно

Чтобы выполнить задачи параллельно и дождаться всех результатов, используйте Promise.all; для последовательного выполнения применяйте await по очереди внутри async-функции.

// Последовательно
async function runSequentially() {
  await f();
  await g();
}

// Параллельно
async function runParallel() {
  const [r1, r2] = await Promise.all([f(), g()]);
}

Promise.all прерывается при первом reject; если важны все результаты — используйте Promise.allSettled.

Обработка ошибок

Обрабатывайте ошибки в промисах с помощью .catch или внутри async-функций через try/catch — незахваченные ошибки приводят к непредсказуемому поведению.

async function safe() {
  try {
    const data = await fetchData();
    console.log(data);
  } catch (err) {
    console.error('Ошибка:', err);
  }
}

Примеры

// Пример: промис, который либо resolve, либо reject
const myPromise = new Promise((resolve, reject) => {
  const success = Math.random() > 0.5;
  if (success) resolve('Успех');
  else reject(new Error('Ошибка'));
});

myPromise
  .then(res => console.log(res))
  .catch(err => console.error(err.message));

Промис может завершиться успешно или с ошибкой — обработка через .then/.catch.

// Пример: async/await
async function demo() {
  const result = await new Promise(r => setTimeout(() => r('Данные загружены!'), 1000));
  console.log(result);
}
demo();

await делает код читаемее и удобнее для последовательной обработки.

// Пример: Promise.all и Promise.allSettled
const wait = ms => new Promise(r => setTimeout(() => r(ms), ms));

Promise.all([wait(50), wait(100), wait(150)])
  .then(results => console.log('All:', results));

Promise.allSettled([wait(50), Promise.reject('err')])
  .then(results => console.log('Settled:', results));

Promise.all возвращает массив результатов или отклоняется при первой ошибке; allSettled возвращает статусы всех задач.

Практика

Попробуйте запустить примеры в окружении (консоли браузера или в песочнице). В учебных материалах лучше избегать выполнения eval — используйте безопасные демонстрации.

// Самопроверка: функция, возвращающая промис
function wait(ms) {
  return new Promise(resolve => setTimeout(() => resolve(ms), ms));
}

wait(200).then(ms => console.log('Ждали', ms, 'мс'));

Советы и типичные ошибки

  • Не используйте top-level await в окружениях, где он не поддерживается — используйте async-функции.
  • Не забывайте возвращать промис из функции, если хотите строить цепочки (.then).
  • Не смешивайте стили (.then/.catch и async/await) в одном фрагменте — это ухудшает читаемость.
  • Для параллелизма используйте Promise.all или специализированные методы, а не последовательный await в цикле.

Лучшие практики

  • Предпочитайте async/await — за читаемость; .then удобен для простых цепочек.
  • Обрабатывайте ошибки везде, где возможны сбои (через catch или try/catch).
  • Документируйте, что функция возвращает промис и какие ошибки возможны.
  • Используйте Promise.allSettled, если важны результаты всех задач, даже при ошибках.
