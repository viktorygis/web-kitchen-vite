Преобразование типов в JavaScript

Преобразование типов — автоматическое или явное изменение типа значения.
JavaScript использует неявное преобразование в операциях и предоставляет функции для явного преобразования.

Основные виды преобразований

  • Строковое преобразование — String()
  • Численное преобразование — Number()
  • Логическое преобразование — Boolean()

Когда происходит преобразование

  • При математических операциях
  • При сравнении значений
  • При выводе данных
  • В логических контекстах
  • При конкатенации строк

Строковое преобразование

String(value) — преобразует значение в строку.
Происходит автоматически при конкатенации строк или выводе в консоль.

Синтаксис

String(value)

Где value — любое значение для преобразования.

Примеры преобразования в строку:

String(true);      // "true"
String(null);      // "null"
String(123);       // "123"
String(undefined); // "undefined"
String([1, 2, 3]); // "1,2,3"

Особенности строкового преобразования

  • Работает для всех типов данных
  • Для объектов вызывает метод toString()
  • Для массивов возвращает элементы через запятую
  • null и undefined преобразуются в строки "null" и "undefined"

Оператор + для конкатенации

Оператор + выполняет конкатенацию (объединение строк), если хотя бы один операнд является строкой: 

let result1 = "Hello, " + 123;     // "Hello, 123"
let result2 = "Age: " + true;      // "Age: true"
let result3 = "Count: " + undefined; // "Count: undefined"

Практическое применение

  • Формирование строковых сообщений для вывода
  • Подготовка данных для отправки на сервер
  • Создание URL-адресов с параметрами
  • Логирование информации

Численное преобразование

Number(value) — преобразует значение в число.
Происходит автоматически в математических операциях и сравнениях.

Синтаксис

Number(value)

Где value — любое значение для преобразования.

Примеры преобразования в число:

Number("123");    // 123
Number("123z");   // NaN
Number(true);     // 1
Number(false);    // 0
Number(null);     // 0
Number(undefined);// NaN

Правила преобразования в число

Значение Результат Примечание
undefined NaN Не число
null 0 Специальное случай
true / false 1 / 0 Логические значения
Пустая строка "" 0 Специальное правило
Пробельные символы " " 0 Обрезаются и преобразуются
Нечисловая строка "abc" NaN Не может быть преобразована
Строка с числом " 123 " 123 Пробелы обрезаются

Альтернативные методы преобразования в число

parseInt("100px");    // 100 (целое число)
parseFloat("12.5em"); // 12.5 (дробное число)
+"123";               // 123 (унарный плюс)
Number("123");        // 123 (явное преобразование)

⚠️ Важно о NaN

console.log(Number("abc"));   // NaN
console.log(typeof NaN);      // "number" (парадокс!)
console.log(NaN === NaN);     // false (не сравнивается сам с собой)
console.log(isNaN(NaN));      // true (правильный способ проверки)
console.log(Number.isNaN(NaN));// true (более надёжный способ)

Логическое преобразование

Boolean(value) — преобразует значение в логический тип.
Происходит автоматически в условиях (if, while) и логических операциях (&&, ||, !).

Синтаксис

Boolean(value)

Где value — любое значение для преобразования.

Примеры преобразования в логический тип:

Boolean(1);       // true
Boolean(0);       // false
Boolean("text");  // true
Boolean("");      // false
Boolean(null);    // false
Boolean([]);      // true (объект!

Правила преобразования в логический тип

Значение Результат Категория
0 false Falsy значения
"" (пустая строка) false
null false
undefined false
NaN false
Любое число ≠ 0 (например, 1, -1, 0.5) true Truthy значения
Любая непустая строка (например, "0", " ") true
Объект (включая пустой {}) true
Массив (включая пустой []) true
Функция true

Важные особенности логического преобразования

  • Только 5 значений преобразуются в false: 0, "", null, undefined, NaN
  • Строка "0" преобразуется в true (не пустая строка)
  • Пробелы в строке — true (не пустая строка)
  • Все объекты преобразуются в true, даже пустые
  • Пустые массивы [] преобразуются в true (это объект!)
  • Функции преобразуются в true

⚠️ Частые ошибки с пустыми массивами

let arr = [];
Boolean(arr);  // true (это ловушка!)
if (arr) console.log("true"); // выведет "true"

// Правильная проверка на пустой массив:
if (arr.length === 0) {
  console.log("массив пуст");
}

Значение String() Number() Boolean()
undefined "undefined" NaN false

Лучшие практики

✅ Используйте явное преобразование для ясности

// Хорошо - явное преобразование
const age = Number("25");
const message = String(true);
const isActive = Boolean(1);

// Менее ясно - неявное преобразование
const age2 = +"25";
let message2 = true + "";
let isActive2 = !!"text";

✅ Выбирайте правильный метод преобразования в число

// Для чистого числового преобразования:
Number("123");    // 123
+"123";           // 123 (унарный плюс)

// Для извлечения чисел из строк:
parseInt("100px");    // 100
parseFloat("3.14em"); // 3.14

// Для проверки валидности:
Number.isNaN(NaN);    // true (надёжнее чем isNaN())
Number.isInteger(5);  // true

✅ Проверяйте значения перед преобразованием

// Хорошо - проверка перед использованием
const userInput = "123";
if (userInput !== "") {
  const num = Number(userInput);
  if (!Number.isNaN(num)) {
    console.log(num);
  }
}

// Или проще:
const num = parseInt(userInput, 10);
if (Number.isNaN(num)) {
  console.log("Некорректное число");
}

✅ Будьте осторожны с логическими преобразованиями

// Помните о Falsy значениях:
const values = [0, "", null, undefined, NaN, false];

values.forEach(value => {
  if (!value) {
    console.log(`${value} это falsy`);
  }
});

// Явная проверка часто лучше:
if (arr.length === 0) {
  // пустой массив
}

if (str === "") {
  // пустая строка
}

✅ Используйте строгое сравнение (===)

// ❌ Нестрогое сравнение вызывает неявное преобразование
0 == false;        // true
"" == false;       // true
null == undefined; // true

// ✅ Строгое сравнение избегает преобразований
0 === false;        // false
"" === false;       // false
null === undefined; // false

Типичные ошибки

❌ Забывают о NaN в числовом преобразовании

// Проблема:
const result = Number("abc"); // NaN
if (result > 0) {
  console.log("положительное число"); // не выполнится
}

// Правильно:
if (!Number.isNaN(result)) {
  if (result > 0) {
    console.log("положительное число");
  }
}

❌ Путают пустой массив с false

// Ошибка:
if ([]) {
  console.log("это true!"); // выведет
}

// Правильно проверить:
if (Array.isArray(arr) && arr.length === 0) {
  console.log("пустой массив");
}

❌ Используют нестрогое сравнение с преобразованиями

// Неожиданные результаты:
"5" == 5;     // true (преобразуется!)
"5" == true;  // false
0 == false;   // true
"" == false;  // true

// Правильно - используйте ===
"5" === 5;    // false
"5" === "5";  // true

❌ Неправильно проверяют на null/undefined

// Ошибка - также поймёт 0, "", false
if (!value) {
  console.log("что-то пошло не так");
}

// Правильно:
if (value === null || value === undefined) {
  console.log("нет значения");
}

// Или современный способ:
if (value ?? false) {
  console.log("есть значение");
}

❌ Забывают о пробельных символах

// Может быть неожиданно:
Boolean("   ");     // true (пробелы это символы)
Number("   ");      // 0 (пробелы обрезаются)
Boolean(Number("  ")); // false

// Если нужна проверка на пусто/пробелы:
if (str.trim() === "") {
  console.log("строка пуста или только пробелы");
}

❌ Используют parseInt без второго параметра

// Опасно:
parseInt("10");      // 10 (но может быть 8, если "010")
parseInt("010");     // может быть 10 или 8

// Правильно - всегда указывайте систему счисления:
parseInt("10", 10);  // 10 (десятичная)
parseInt("010", 10); // 10 (явно десятичная)
parseInt("FF", 16);  // 255 (шестнадцатеричная)

Итоги

Ключевые моменты

  • JavaScript выполняет как явное, так и неявное преобразование типов
  • Явное преобразование делает код понятнее и безопаснее
  • Есть только 5 falsy значений: 0, "", null, undefined, NaN
  • NaN — это число, но не равно самому себе
  • Пустые массивы и объекты преобразуются в true
  • Строгое сравнение (===) избегает неожиданных преобразований

Рекомендации

  1. Используйте явное преобразование для ясности: String(), Number(), Boolean()
  2. Всегда используйте === вместо ==
  3. Проверяйте значения перед преобразованием в число
  4. Помните о особых случаях: NaN, пустые массивы, null vs undefined
  5. Для parseInt указывайте систему счисления: parseInt(str, 10)
  6. Используйте Number.isNaN() вместо isNaN()
