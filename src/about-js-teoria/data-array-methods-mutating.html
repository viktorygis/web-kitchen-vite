Мутирующие методы массивов

Мутирующие методы — это методы, которые изменяют исходный массив "на месте", а не создают новый. Они напрямую модифицируют содержимое массива и его длину.

⚠️ Важно о мутации

Мутирующие методы полезны для производительности (избегают создания копий), но могут привести к ошибкам, если:

  • Массив используется в других местах кода (например, в React/Vue props)
  • Вы случайно изменили массив, который не планировали менять
  • Код работает с замороженными или запечатанными объектами

💡 Для безопасности: Если нужна немутирующая версия, используйте spread оператор [...arr] для копирования перед мутирующим методом, или выбирайте специальные немутирующие методы (например, toSorted(), toReversed() в ES2023+).

push() — Добавление в конец

push() - метод добавляет один или несколько элементов в конец массива. Является мутирующим методом.

Синтаксис:

arr.push(element1, ..., elementN)

Параметры:

  • один или несколько элементов, которые будут добавлены в конец массива

Возвращает:

  • Новую длину массива (после добавления элементов)

Описание:

  • Изменяет исходный массив — отдельно копировать результат не нужно
  • Временная сложность: O(1) — эффективнее, чем unshift()
  • Если нужен немутирующий аналог, используйте spread: [...arr, element]

Пример:

let vegetables = ["помидор", "картофель"];
const newLength = vegetables.push("морковь");

console.log(vegetables); // ["помидор", "картофель", "морковь"]
console.log(newLength);  // 3

⚠️ Частые ошибки

1. Путают возвращаемое значение с массивом

Метод возвращает число (новую длину), а не обновлённый массив:

let result = arr.push(item);
console.log(result); // число, а НЕ массив!

2. Передают массив, ожидая его "слияния" с исходным

push() добавляет переданный массив как один вложенный элемент, а не разворачивает его:

const arr = [1, 2];
arr.push([3, 4]);
console.log(arr); // [1, 2, [3, 4]] — НЕ [1, 2, 3, 4]!

// Чтобы объединить массивы, используйте spread или concat:
arr.push(...[3, 4]);      // через spread
// или
const merged = arr.concat([3, 4]); // немутирующий вариант

3. push() с очень большим количеством аргументов через spread

Распаковка огромного массива в аргументы функции может превысить лимит стека вызовов:

const target = [];
const hugeArray = new Array(200000).fill(0);

target.push(...hugeArray);
// RangeError: Maximum call stack size exceeded

// Решение: используйте concat()
const merged = target.concat(hugeArray);

💡 Когда использовать

  • Накопление результатов в цикле: results.push(item)
  • Реализация стека (LIFO) в паре с pop()
  • Добавление элементов в массив состояния React/Vue (правильно: setItems(prev => [...prev, newItem]))

pop() — Удаление с конца

pop() - метод удаляет последний элемент массива и возвращает его. Является мутирующим методом.

Синтаксис:

arr.pop()

Параметры:

  • Метод не принимает параметров

Возвращает:

  • Удаленный элемент массива (или undefined, если массив пуст)

Описание:

  • Уменьшает length массива на 1
  • Временная сложность: O(1)
  • Часто используется в реализации стека (LIFO)

Пример:

const fruits = ['яблоко', 'груша', 'апельсин'];
const lastFruit = fruits.pop();
console.log(lastFruit); // 'апельсин'
console.log(fruits); // ['яблоко', 'груша']

// Пустой массив
const empty = [];
const removed = empty.pop();
console.log(removed); // undefined

⚠️ Частые ошибки

1. Путают pop() с shift()

pop() удаляет элемент с конца, а shift() — с начала.

2. Используют pop() в условии цикла с "ложными" значениями

Если в массиве есть 0, '', null, цикл while (arr.pop()) остановится на них:

const queue = [1, 2, 0, 3];

// Неправильно:
while (queue.pop()) { console.log('обработка'); }
// Цикл остановится на 0

// Правильно:
while (queue.length > 0) {
  const item = queue.pop();
  console.log('обработка', item);
}

3. Считают, что const запрещает мутацию

const запрещает переприсваивание, но не изменение содержимого массива:

const arr = [1, 2, 3];
arr.pop(); // работает
console.log(arr); // [1, 2]

💡 Когда использовать

  • Реализация стека (LIFO) с push()
  • История действий для "Undo"
  • Обработка элементов с конца небольшого массива

unshift() — Добавление в начало

unshift() - метод добавляет один или несколько элементов в начало массива. Является мутирующим методом.

Синтаксис:

arr.unshift(element1, ..., elementN)

Параметры:

  • element1, ..., elementN - элементы для добавления в начало массива

Возвращает:

  • Новую длину массива

Описание:

  • Сдвигает все существующие элементы массива
  • Временная сложность: O(n) - менее эффективен чем push из-за переиндексации

Пример:

const nums = [2, 3];
nums.unshift(0, 1);
console.log(nums); // [0, 1, 2, 3]

⚠️ Частые ошибки

1. Используют unshift() в цикле для больших массивов

Каждый вызов сдвигает все элементы — O(n). В цикле это становится O(n²):

// Медленно:
const result = [];
for (const item of hugeArray) {
  result.unshift(item); // O(n) на каждой итерации
}

// Быстрее: push() + reverse()
const result2 = [];
for (const item of hugeArray) {
  result2.push(item);
}
result2.reverse();

2. Передают массив как один элемент вместо разворачивания

const arr = [3, 4];
arr.unshift([1, 2]); // [[1, 2], 3, 4] — не то!

// Используйте spread:
arr.unshift(...[1, 2]); // [1, 2, 3, 4]

💡 Когда использовать

  • Добавление в начало небольшого массива (где порядок важен)
  • Для больших массивов рассмотрите другие структуры данных
  • Немутирующий аналог: [newItem, ...arr]

shift() — Удаление с начала

shift() - метод удаляет первый элемент массива и возвращает его. Является мутирующим методом.

Синтаксис:

arr.shift()

Параметры:

  • Метод не принимает параметров

Возвращает:

  • Удаленный элемент массива (или undefined, если массив пуст)

Описание:

  • Сдвигает все последующие элементы на одну позицию назад
  • Временная сложность: O(n)
  • Используется в реализации очереди (queue)

Пример:

const queue = ['Аня', 'Борис', 'Вика'];
const first = queue.shift();
console.log(first); // 'Аня'
console.log(queue); // ['Борис', 'Вика']

⚠️ Частые ошибки

1. Путают shift() с pop()

shift() удаляет с начала, pop() — с конца.

2. Используют shift() в цикле для больших массивов

Каждый вызов сдвигает все элементы — O(n). Лучше используйте индекс:

// Медленно:
while (queue.length > 0) {
  const item = queue.shift(); // O(n) каждый раз
}

// Быстрее:
let index = 0;
while (index < queue.length) {
  const item = queue[index];
  index++;
}

💡 Когда использовать

  • Реализация очереди (FIFO) — обработка в порядке поступления
  • На больших данных используйте индекс вместо shift()
  • Немутирующий аналог: arr.slice(1)

splice() — Универсальное добавление/удаление

splice() - метод добавляет и/или удаляет элементы в любом месте массива. Является мутирующим методом.

Синтаксис:

arr.splice(start, deleteCount, item1, ..., itemN)

Параметры:

  • start - индекс, с которого начинаются изменения
  • deleteCount - количество элементов для удаления (необязательно)
  • item1, ..., itemN - элементы для вставки (необязательно)

Возвращает:

  • Массив удаленных элементов (пустой массив, если ничего не удалено)

Примеры:

// Удаление
const arr1 = [1, 2, 3, 4];
arr1.splice(1, 2); // [2, 3]
// arr1: [1, 4]

// Вставка
const arr2 = ['a', 'd'];
arr2.splice(1, 0, 'b', 'c'); // []
// arr2: ['a', 'b', 'c', 'd']

// Замена
const arr3 = [10, 20, 30];
arr3.splice(1, 1, 'XX'); // [20]
// arr3: [10, 'XX', 30]

⚠️ Частые ошибки

1. Путают splice() со slice()

splice() мутирует, slice() — нет:

const arr = [1, 2, 3, 4];
arr.slice(1, 3);  // [2, 3] — не изменяет arr
arr.splice(1, 2); // [2, 3] — изменяет arr!

2. Второй параметр как конечный индекс

В splice(start, deleteCount) второй параметр — это количество, не индекс:

const arr = [1, 2, 3, 4, 5];
arr.splice(1, 3); // удалит 3 элемента: [2, 3, 4]
console.log(arr); // [1, 5]

3. Забывают, что deleteCount можно не указывать

Без deleteCount удаляются ВСЕ элементы до конца:

const arr = [1, 2, 3, 4, 5];
arr.splice(2); // удаляет ВСЕ от 2 до конца
console.log(arr); // [1, 2]

💡 Когда использовать

  • Удаление по индексу: arr.splice(index, 1)
  • Вставка в середину: arr.splice(index, 0, newItem)
  • Замена: arr.splice(index, 1, newItem)
  • Немутирующий аналог: комбинация slice() или метод toSpliced()

sort() — Сортировка

sort() - метод сортирует элементы массива на месте. Является мутирующим методом.

Синтаксис:

arr.sort([compareFunction])

Параметры:

  • compareFunction - функция сравнения (необязательно). Возвращает: < 0 (a перед b), > 0 (b перед a), 0 (порядок не важен)

Возвращает:

  • Отсортированный массив (тот же самый, изменённый на месте)

Примеры:

// Сортировка строк
const names = ['Мария', 'Анна', 'Иван'];
names.sort(); // ['Анна', 'Иван', 'Мария']

// Сортировка чисел (возрастание)
const nums = [10, 2, 5];
nums.sort((a, b) => a - b); // [2, 5, 10]

// Сортировка чисел (убывание)
nums.sort((a, b) => b - a); // [10, 5, 2]

// Сортировка объектов
const users = [
  { name: 'Аня', age: 25 },
  { name: 'Петя', age: 30 }
];
users.sort((a, b) => a.age - b.age);

⚠️ Частые ошибки

1. Случайная мутация массива

sort() изменяет исходный массив — опасно, если он используется в React/Vue props:

// Неправильно:
function render(items) {
  return items.sort((a, b) => a - b);
}

// Правильно:
function render(items) {
  return [...items].sort((a, b) => a - b);
}

2. Компаратор возвращает boolean вместо числа

// Неправильно:
nums.sort((a, b) => a > b); // может дать неверный результат

// Правильно:
nums.sort((a, b) => a - b);

3. Забывают о лексикографической сортировке чисел

Без компаратора числа сортируются как строки:

const nums = [10, 1, 5];
nums.sort(); // [1, 10, 5] — лексикографически!

nums.sort((a, b) => a - b); // [1, 5, 10] — числово

💡 Когда использовать

  • Упорядочивание данных для отображения
  • Подготовка перед бинарным поиском
  • Если нужен немутирующий аналог: используйте [...arr].sort() или toSorted() (ES2023+)

reverse() — Разворот

reverse() - метод переворачивает порядок элементов массива на месте. Является мутирующим методом.

Синтаксис:

arr.reverse()

Параметры:

  • Метод не принимает параметров

Возвращает:

  • Перевернутый массив (тот же самый, изменённый на месте)

Пример:

const letters = ['a', 'b', 'c'];
letters.reverse();
console.log(letters); // ['c', 'b', 'a']

⚠️ Частые ошибки

1. Пытаются вызвать на строке

У строк нет метода reverse():

'hello'.reverse(); // TypeError

// Правильно для строк:
'hello'.split('').reverse().join(''); // 'olleh'

2. Не ожидают мутацию исходного массива

const original = [1, 2, 3];
const ref = original;

ref.reverse();
console.log(original); // [3, 2, 1] — изменился!

// Если нужна копия:
const reversed = [...original].reverse();

💡 Когда использовать

  • Обратный хронологический порядок (последние сообщения сверху)
  • Проверка на палиндром
  • Немутирующий аналог: [...arr].reverse() или toReversed() (ES2023+)

fill() — Заполнение

fill() - метод заполняет элементы массива одинаковым значением. Является мутирующим методом.

Синтаксис:

arr.fill(value, [start], [end])

Параметры:

  • value - значение для заполнения
  • start - индекс начала (необязательно, по умолчанию 0)
  • end - индекс конца, не включается (необязательно, по умолчанию arr.length)

Возвращает:

  • Изменённый массив (тот же самый)

Примеры:

// Заполнение всего массива
const arr1 = new Array(3).fill(0);
// [0, 0, 0]

// Частичное заполнение
const arr2 = [1, 2, 3, 4, 5];
arr2.fill('x', 1, 4);
// [1, 'x', 'x', 'x', 5]

⚠️ Частые ошибки

1. Заполнение объектом или массивом

ВСЕ элементы ссылаются на ОДИН и тот же объект:

const matrix = new Array(3).fill([]);
matrix[0].push(1);
console.log(matrix); // [[1], [1], [1]] — все изменились!

// Правильно:
const matrix2 = Array.from({ length: 3 }, () => []);
matrix2[0].push(1);
console.log(matrix2); // [[1], [], []]

2. Путают end как включающий индекс

Как и в slice(), end не включается:

const arr = [1, 2, 3, 4, 5];
arr.fill('x', 1, 4);
console.log(arr); // [1, 'x', 'x', 'x', 5] — индекс 4 не изменился

💡 Когда использовать

  • Инициализация: new Array(n).fill(0)
  • Обнуление счётчиков
  • Создание пустого игрового поля из примитивов
  • Для независимых объектов: используйте Array.from()

Метод Описание Синтаксис Возвращает Сложность
push() Добавляет элементы в конец arr.push(...items) Новая длина O(1)
pop() Удаляет последний элемент arr.pop() Удалённый элемент O(1)
unshift() Добавляет элементы в начало arr.unshift(...items) Новая длина O(n)
shift() Удаляет первый элемент arr.shift() Удалённый элемент O(n)
splice() Удаляет/вставляет в любом месте arr.splice(i, count, ...items) Удалённые элементы O(n)
sort() Сортирует элементы arr.sort([fn]) Сам массив O(n log n)
reverse() Переворачивает порядок arr.reverse() Сам массив O(n)
fill() Заполняет одинаковым значением arr.fill(val, start, end) Сам массив O(n)
