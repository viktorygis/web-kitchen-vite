Математические операторы и округление в JavaScript

Математические операции — это основные арифметические операторы, способы округления, работа с числами и парсинг строк, генерация случайных чисел и решение проблем точности с числами с плавающей точкой.

Что вы узнаете

  • Базовые арифметические операторы: + - * / % **
  • Унарные операторы: +x (приведение к числу), -x (смена знака)
  • Методы округления: Math.floor, Math.ceil, Math.round, Math.trunc
  • Форматирование чисел методом toFixed()
  • Парсинг строк в числа: Number(), parseInt(), parseFloat()
  • Генерация случайных чисел с Math.random()
  • Решение проблем точности с плавающей точкой

Арифметические операторы

JavaScript поддерживает все базовые операции. Деление всегда возвращает число с плавающей точкой. Порядок операций важен для правильного результата.

Оператор Название Пример Результат
+ Сложение 3 + 2 5
- Вычитание 5 - 2 3
* Умножение 2 * 3 6
/ Деление 7 / 2 3.5
% Остаток от деления 7 % 2 1
** Возведение в степень 2 ** 5 32
+x Унарный плюс (приведение к числу) +'42' 42 (число)
-x Смена знака -10 -10

Примеры арифметических операций:

// Базовые операции
console.log(3 + 2);    // 5
console.log(5 - 2);    // 3
console.log(2 * 3);    // 6
console.log(7 / 2);    // 3.5
console.log(7 % 2);    // 1
console.log(2 ** 5);   // 32

// Унарные операторы
console.log(+'42');    // 42 (приведение строки к числу)
console.log(-10);      // -10 (смена знака)

Особенности операторов

  • Оператор + со строками выполняет конкатенацию: '1' + 2 = '12'
  • Унарный + приводит значение к числу: +'1.5' = 1.5
  • Остаток % — знак совпадает со знаком делимого (первого операнда)
  • Деление на ноль возвращает Infinity: 5 / 0 = Infinity

Приоритет операций

Порядок выполнения операций (от выше к ниже):

  1. () — скобки
  2. ** — возведение в степень
  3. * / % — умножение, деление, остаток
  4. + - — сложение, вычитание

Пример приоритета операций:

// Без скобок - умножение выполняется первым
let a = 5 + 2 * 3;      // 11
let b = 10 - 2 ** 2;    // 6

// Со скобками - другой порядок
let c = (5 + 2) * 3;    // 21
let d = (10 - 2) ** 2;  // 64

Округление чисел

Округление — преобразование числа с десятичной частью в целое. JavaScript предоставляет четыре функции из объекта Math и метод toFixed() для форматирования.

Функция Описание Примеры Демо
Math.floor(x) Вниз до целого Math.floor(3.9) = 3
Math.ceil(x) Вверх до целого Math.ceil(3.1) = 4
Math.round(x) До ближайшего целого Math.round(3.5) = 4
Math.trunc(x) Отбрасывает дробную часть Math.trunc(-3.9) = -3
num.toFixed(n) Форматирует с n знаками (1.005).toFixed(2) = '1.01'

Примеры округления:

// Math.floor - вниз
console.log(Math.floor(3.9));   // 3
console.log(Math.floor(-3.1));  // -4

// Math.ceil - вверх
console.log(Math.ceil(3.1));    // 4
console.log(Math.ceil(-3.9));   // -3

// Math.round - к ближайшему
console.log(Math.round(3.5));   // 4
console.log(Math.round(3.4));   // 3

// Math.trunc - отбросить дробь
console.log(Math.trunc(3.9));   // 3
console.log(Math.trunc(-3.9));  // -3

// toFixed - форматирование
console.log((1.005).toFixed(2));     // '1.01' (строка!)
console.log((3.14159).toFixed(2));   // '3.14'

Важные моменты

  • Метод toFixed() возвращает строку — используйте Number(x.toFixed(n)) для числа
  • Для округления до n знаков после точки: Math.round(x * 10**n) / 10**n
  • Math.round(0.5) → 1, но Math.round(-0.5) → 0
  • Math.floor и Math.ceil работают противоположно с отрицательными числами

Округление до n знаков после точки:

// Способ 1: toFixed (проще, но возвращает строку)
const price1 = Number((19.99123).toFixed(2));  // 19.99

// Способ 2: Math.round (более гибкий)
const price2 = Math.round(19.99123 * 100) / 100;  // 19.99

// Функция для удобства
function roundTo(value, decimals) {
  return Math.round(value * 10 ** decimals) / 10 ** decimals;
}

console.log(roundTo(3.14159, 2));  // 3.14

Парсинг строк и проверка чисел

Парсинг строк в числа

Функция Описание Пример
Number(str) Преобразует в число (строгое) Number('42') = 42
parseInt(str, radix) Парсит целое число с основанием parseInt('101', 2) = 5
parseFloat(str) Парсит число с точкой parseFloat('3.14px') = 3.14

Примеры парсинга:

// Number - строгое преобразование
console.log(Number('42'));       // 42
console.log(Number('3.14'));     // 3.14
console.log(Number('42px'));     // NaN

// parseInt - парсинг целых чисел
console.log(parseInt('42'));     // 42
console.log(parseInt('101', 2)); // 5 (двоичная система)
console.log(parseInt('FF', 16)); // 255 (шестнадцатеричная)

// parseFloat - парсинг чисел с точкой
console.log(parseFloat('3.14'));      // 3.14
console.log(parseFloat('3.14px'));    // 3.14 (px игнорируется)

Проверка чисел

Функция Описание Пример
Number.isNaN(value) Строгая проверка на NaN Number.isNaN(NaN) = true
Number.isFinite(value) Проверяет конечное число Number.isFinite(42) = true
Number.isInteger(value) Проверяет целое число Number.isInteger(42) = true

Примеры проверок:

// Number.isNaN - строгая проверка
console.log(Number.isNaN(NaN));        // true
console.log(Number.isNaN('foo'));      // false (не приводит тип)

// Старая функция isNaN (нестрогая, не рекомендуется)
console.log(isNaN('foo'));             // true (приводит тип)

// Number.isFinite
console.log(Number.isFinite(42));      // true
console.log(Number.isFinite(Infinity)); // false
console.log(Number.isFinite('42'));    // false (не приводит тип)

// Number.isInteger
console.log(Number.isInteger(42));     // true
console.log(Number.isInteger(42.0));   // true
console.log(Number.isInteger(3.14));   // false

Разделители разрядов и системы счисления

Числовые литералы:

// Разделители для читаемости
const million = 1_000_000;    // 1000000
const billion = 1_000_000_000; // 1000000000

// Двоичная система (0b)
const binary = 0b1010;        // 10

// Восьмеричная система (0o)
const octal = 0o77;           // 63

// Шестнадцатеричная система (0x)
const hex = 0xFF;             // 255
const color = 0x00FF00;       // 65280

Рекомендации

  • Используйте Number.isNaN() и Number.isFinite() — они не приводят тип
  • Избегайте старой функции isNaN() — она приводит значение к числу
  • Используйте parseInt() с явным указанием основания: parseInt(str, 10)
  • Для разделения больших чисел используйте подчеркивание: 1_000_000

Случайные числа

Math.random() возвращает случайное число в диапазоне [0, 1) (включая 0, не включая 1). На его основе можно строить случайные числа в любом диапазоне.

Базовая функция Math.random():

// Возвращает число от 0 до 1 (не включая 1)
console.log(Math.random());     // 0.123456789
console.log(Math.random());     // 0.987654321

Формулы для различных диапазонов

Случайное целое число в диапазоне [min, max]:

// Формула
Math.floor(min + Math.random() * (max - min + 1))

// Пример: случайное число от 1 до 6 (кубик)
const dice = Math.floor(1 + Math.random() * 6);
console.log(dice);  // 1, 2, 3, 4, 5 или 6

Случайное вещественное число в диапазоне [min, max):

// Формула
min + Math.random() * (max - min)

// Пример: случайное число от 5 до 10
const random = 5 + Math.random() * 5;
console.log(random);  // 5.123 или 7.456 или 9.999

Удобные функции:

// Случайное целое число
function randomInt(min, max) {
  return Math.floor(min + Math.random() * (max - min + 1));
}

// Случайное вещественное число
function randomFloat(min, max) {
  return min + Math.random() * (max - min);
}

console.log(randomInt(1, 6));      // 1-6
console.log(randomFloat(5, 10));   // 5.xxx-9.xxx

Практические примеры

Случайный элемент из массива:

const items = ['apple', 'banana', 'cherry'];
const random = items[Math.floor(Math.random() * items.length)];
console.log(random);  // один из трёх элементов

// Функция для удобства
function randomItem(arr) {
  return arr[Math.floor(Math.random() * arr.length)];
}

console.log(randomItem(['A', 'B', 'C']));  // A, B или C

Случайный цвет:

// Случайный RGB цвет
function randomColor() {
  const r = Math.floor(Math.random() * 256);
  const g = Math.floor(Math.random() * 256);
  const b = Math.floor(Math.random() * 256);
  return `rgb(${r}, ${g}, ${b})`;
}

console.log(randomColor());  // rgb(123, 45, 200)

// Случайный hex цвет
function randomHexColor() {
  return '#' + Math.floor(Math.random() * 16777215).toString(16).padStart(6, '0');
}

console.log(randomHexColor());  // #3a7b2f

Точность и ошибки округления

Проблема точности возникает потому, что JavaScript использует двоичное представление чисел (IEEE 754). Некоторые десятичные числа не могут быть представлены точно в двоичной системе. Например: 0.1 + 0.2 !== 0.3

Демонстрация проблемы:

// Проблема точности
console.log(0.1 + 0.2);           // 0.30000000000000004
console.log(0.1 + 0.2 === 0.3);   // false

// Другие примеры
console.log(0.1 + 0.2 + 0.3);     // 0.6000000000000001
console.log(1 - 0.9);             // 0.09999999999999998

Решения

Способ 1: Округление до n знаков

function roundTo(value, decimals) {
  return Math.round(value * 10 ** decimals) / 10 ** decimals;
}

const sum = 0.1 + 0.2;
const rounded = roundTo(sum, 2);
console.log(rounded);        // 0.3
console.log(rounded === 0.3); // true

Способ 2: Сравнение с допуском (EPSILON)

// Функция для сравнения с допуском
function nearlyEqual(a, b, epsilon = Number.EPSILON) {
  return Math.abs(a - b) < epsilon;
}

const sum = 0.1 + 0.2;
console.log(nearlyEqual(sum, 0.3));        // true
console.log(nearlyEqual(sum, 0.3, 0.01));  // true

// Number.EPSILON — наименьшая разница между 1 и числом, большим 1
console.log(Number.EPSILON);  // 2.220446049250313e-16

Способ 3: Работа с целыми числами (для денег)

// Храните деньги в центах/копейках
const price1 = 10;  // 10 центов
const price2 = 20;  // 20 центов
const total = price1 + price2;  // 30 центов
const dollarAmount = total / 100;  // 0.30 долларов

// Или используйте Decimal для точной арифметики
// Но это требует отдельной библиотеки

⚠️ Типичные ошибки

  • ❌ Прямое сравнение с ===: 0.1 + 0.2 === 0.3 (false!)
  • ❌ Доверие к количеству знаков: (0.1 + 0.2).toFixed(1) может вернуть '0.3', но это строка
  • ✅ Использование округления или сравнения с допуском
  • ✅ Для денег: работайте с целыми числами (центы, копейки)

Полезные математические функции

Стандартные методы объекта Math

Функция Описание Пример
Math.abs(x) Абсолютное значение Math.abs(-5) = 5
Math.max(...values) Максимальное значение Math.max(1, 4, 2) = 4
Math.min(...values) Минимальное значение Math.min(1, 4, 2) = 1
Math.pow(x, y) x в степени y Math.pow(3, 4) = 81
Math.sqrt(x) Квадратный корень Math.sqrt(16) = 4
Math.sign(x) Знак числа (-1, 0 или 1) Math.sign(-10) = -1
Math.hypot(x, y) Гипотенуза (длина вектора) Math.hypot(3, 4) = 5

Примеры использования:

// Абсолютное значение
console.log(Math.abs(-5));    // 5
console.log(Math.abs(-3.14)); // 3.14

// Максимум и минимум
console.log(Math.max(1, 4, 2));      // 4
console.log(Math.min(1, 4, 2));      // 1
console.log(Math.max(...[1, 4, 2])); // 4

// Степень
console.log(Math.pow(2, 8));  // 256
console.log(2 ** 8);          // 256 (современный синтаксис)

// Корень
console.log(Math.sqrt(16));   // 4
console.log(Math.sqrt(2));    // 1.4142...

// Знак
console.log(Math.sign(10));   // 1
console.log(Math.sign(-10));  // -1
console.log(Math.sign(0));    // 0

// Гипотенуза (расстояние от точки до начала координат)
console.log(Math.hypot(3, 4)); // 5

Полезные вспомогательные функции

Зажатие значения (clamp):

// Функция для ограничения значения в диапазоне [min, max]
function clamp(value, min, max) {
  return Math.min(Math.max(value, min), max);
}

console.log(clamp(50, 0, 100));   // 50
console.log(clamp(120, 0, 100));  // 100
console.log(clamp(-5, 0, 100));   // 0

// Полезно для ограничения координат, громкости, прогресса
const volume = clamp(150, 0, 100);  // 100

Линейная интерполяция (lerp):

// Интерполяция между двумя значениями
function lerp(a, b, t) {
  return a + (b - a) * t;
}

console.log(lerp(0, 100, 0));    // 0
console.log(lerp(0, 100, 0.5));  // 50
console.log(lerp(0, 100, 1));    // 100

// Полезно для анимаций и переходов
const startColor = 0;
const endColor = 255;
const progress = 0.3;  // 30%
const currentColor = lerp(startColor, endColor, progress);  // 76.5

Нормализация значения в диапазон [0, 1]:

// Преобразовать значение из одного диапазона в [0, 1]
function normalize(value, min, max) {
  return (value - min) / (max - min);
}

console.log(normalize(50, 0, 100));   // 0.5
console.log(normalize(75, 0, 100));   // 0.75
console.log(normalize(0, 0, 100));    // 0

// Обратная операция (денормализация)
function denormalize(normalized, min, max) {
  return normalized * (max - min) + min;
}

console.log(denormalize(0.5, 0, 100));   // 50

Лучшие практики

✅ Используйте правильный метод округления

// ❌ Плохо - непредсказуемо
const price = (19.99 * 3).toFixed(2);

// ✅ Хорошо - контролируемо
function roundPrice(amount) {
  return Math.round(amount * 100) / 100;
}

✅ Для денег используйте целые числа

// ❌ Плохо - проблемы с точностью
const amount = 19.99 + 0.01;  // 20 или 19.99999?

// ✅ Хорошо - работайте в центах/копейках
const amountCents = 1999 + 1;  // 2000 центов = $20.00
const dollars = amountCents / 100;  // 20

✅ Используйте Number.isNaN() вместо isNaN()

// ❌ Плохо - приводит типы
if (isNaN(value)) {  // isNaN('foo') = true!
  // ...
}

// ✅ Хорошо - без приведения типов
if (Number.isNaN(value)) {
  // ...
}

✅ Явно указывайте основание в parseInt()

// ❌ Плохо - основание определяется автоматически
const num = parseInt('010');  // 8 (восьмеричная) или 10 (десятичная)?

// ✅ Хорошо - явно указано
const num = parseInt('010', 10);  // 10 (десятичная)

✅ Используйте скобки для читаемости

// Менее читаемо
const result = 5 + 2 * 3 - 1;

// Более читаемо
const result = (5 + (2 * 3)) - 1;
const result = 5 + (2 * 3) - 1;

✅ Используйте разделители для больших чисел

// Менее читаемо
const population = 1000000;

// Более читаемо
const population = 1_000_000;

⚠️ Частые ошибки

  • ❌ Прямое сравнение вещественных чисел с ===
  • ❌ Использование toFixed() для вычислений (она возвращает строку)
  • ❌ Предположение о точности двоичной арифметики
  • ❌ Забывание о NaN !== NaN
  • ❌ Деление на ноль без проверки
