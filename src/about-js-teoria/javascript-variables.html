Переменные

Переменные — именованные хранилища для данных в JavaScript.
Они используются для хранения информации о товарах, сообщениях, пользователях и любых других данных.

Основные понятия

  • Именованное хранилище данных
  • Могут изменяться (кроме констант)
  • Имеют область видимости (scope)
  • Могут содержать любые типы данных

Где используются переменные

  • Хранение промежуточных результатов вычислений
  • Работа с пользовательскими данными
  • Управление состоянием приложений
  • Сохранение конфигураций и констант

Объявление переменных

Объявление переменной — создание именованного хранилища для данных.
В JavaScript используются три ключевых слова: let, const и var.

Простое объявление:

let message = 'Hello!'; // объявление с инициализацией
let age; // объявление без значения

Множественное объявление:

let user = 'John',
    age = 25,
    isAdmin = false;

(Примечание: рекомендуется объявлять по одной переменной на строку для читаемости)

Особенности объявления

Ключевое слово Область видимости Можно изменять Hoisting Когда использовать
let Блочная Да Без инициализации По умолчанию для изменяемых переменных
const Блочная Нет Без инициализации Для констант и неизменяемых ссылок
var Функциональная Да С инициализацией (undefined) Только для поддержки старого кода

Типы переменных

Изменяемые переменные (let)

Переменные объявленные с let могут быть изменены в любой момент:

let counter = 0;
counter = 1; // значение можно изменить
let price;
price = 100; // можно присвоить позже
price = 99.99; // и изменить снова

Константы (const)

Константы не могут быть переопределены, но их свойства могут быть изменены:

const PI = 3.14159;
// PI = 3.1415; // ❌ Ошибка: присвоение константе

const user = { name: 'John' };
user.name = 'Pete'; // ✅ Можно, изменяется свойство объекта
// user = {}; // ❌ Ошибка: переопределение константы

Переменные старого стиля (var)

var — это способ объявления переменных в старом JavaScript (до ES6). Его отличия от let и const:

Область видимости var (функциональная):

function example() {
  if (true) {
    var x = 10;
  }
  console.log(x); // 10 (переменная видна за пределами блока if!)
}

function example2() {
  if (true) {
    let y = 10;
  }
  console.log(y); // ❌ ReferenceError: y is not defined
}

Hoisting (поднятие) с var:

console.log(a); // undefined (не ошибка!)
var a = 5;

// Это эквивалентно:
var a; // объявление "поднимается" наверх
console.log(a); // undefined
a = 5;

// С let это даёт ошибку:
console.log(b); // ❌ ReferenceError
let b = 5;

Переобъявление var:

var x = 1;
var x = 2; // ✅ Можно переобъявить
console.log(x); // 2

let y = 1;
let y = 2; // ❌ SyntaxError: Identifier 'y' has already been declared

var в цикле:

// С var - проблема!
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
// Выведет: 3, 3, 3 (i изменяется до выполнения callback)

// С let - правильно
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
// Выведет: 0, 1, 2 (каждый i находится в своём блоке)

Сравнение let, const и var

Характеристика let const var
Область видимости Блочная { } Блочная { } Функциональная
Можно изменять ✅ Да ❌ Нет ✅ Да
Можно переобъявлять ❌ Нет ❌ Нет ✅ Да
Hoisting Temporal Dead Zone Temporal Dead Zone Инициализирован как undefined
Используется сейчас ✅ Да ✅ Да ❌ Нет (устарела)

Когда что использовать

  • const — по умолчанию, если значение не меняется
  • let — если значение будет изменяться
  • var — ❌ НИКОГДА в новом коде, только в старых проектах

⚠️ Почему var считается плохой практикой

  • Функциональная область видимости создаёт баги
  • Hoisting с undefined приводит к непредсказуемому поведению
  • Можно случайно переобъявить переменную
  • Трудно отследить, где переменная была изменена
  • Modern JavaScript (ES6+) предоставляет let и const специально для избежания этих проблем

Именование переменных

Имя переменной должно четко отражать её назначение.
В JavaScript используют camelCase для переменных и UPPER_CASE для констант.

Правила именования

  • ✅ Может содержать буквы, цифры, $, _
  • ❌ Не может начинаться с цифры
  • ❌ Пробелы запрещены
  • ✅ Регистр имеет значение (myVar ≠ myvar)
  • ❌ Нельзя использовать зарезервированные слова

Примеры имен переменных:

let userName; // ✅ хорошее имя
let shoppingCart; // ✅ хорошее имя
let 123abc; // ❌ плохое (начинается с цифры)
let let; // ❌ плохое (ключевое слово)
let _private; // ✅ приватная переменная
let $value; // ✅ допустимо, но не рекомендуется

Рекомендации по именованию

  1. Используйте понятные, описательные имена
  2. Избегайте сокращений (используйте userData вместо data)
  3. Для булевых переменных используйте префиксы: is, has, can
    • isActive — активен ли элемент
    • hasPermission — есть ли разрешение
    • canDelete — можно ли удалить
  4. Для констант используйте UPPER_CASE с подчеркиваниями
    • MAX_SIZE
    • API_URL
    • DEFAULT_TIMEOUT

Присвоение значений

Присвоение — это процесс установки или изменения значения переменной.
В JavaScript используется оператор = (равно) для присвоения.

Простое присвоение:

let message;
message = 'Hello!'; // присвоение значения переменной

let age = 30; // объявление с одновременным присвоением

Множественное присвоение:

let a, b, c;
a = b = c = 0; // все переменные получают значение 0

// Операция выполняется справа налево:
// 1. c присваивается 0
// 2. b присваивается значение c (0)
// 3. a присваивается значение b (0)

Особенности присвоения

  • Присвоение возвращает присвоенное значение
  • Можно использовать в выражениях: let x = (a = 5) + 2; // x будет 7
  • При множественном присвоении выполняется справа налево
  • Приоритет присвоения очень низкий

Обновление значений

Обновление — это процесс изменения значения переменной после её объявления.
Мы можем обновлять переменные let так часто, как захотим.

Примеры обновления значений:

let status = "Сплю";
console.log(status); // "Сплю"

status = "А теперь не сплю😢";
console.log(status); // "А теперь не сплю😢"

status = "Еду на работу😭";
console.log(status); // "Еду на работу😭"

Обновление числовых переменных:

let count = 10;
count = count + 5; // увеличение на 5
console.log(count); // 15

count = count - 3; // уменьшение на 3
console.log(count); // 12

count = count * 2; // умножение на 2
console.log(count); // 24

Математические операции

Математические операции используются для выполнения вычислений с переменными.
JavaScript поддерживает стандартные арифметические операторы.

Основные арифметические операторы

Оператор Название Пример Результат
+ Сложение 5 + 3 8
- Вычитание 5 - 3 2
* Умножение 5 * 3 15
/ Деление 15 / 3 5
% Остаток от деления 17 % 5 2
** Возведение в степень 2 ** 3 8

Примеры математических операций:

let counter = 5;
counter = counter + 1; // увеличение на 1
console.log(counter); // 6

let a = 10;
let b = 3;
console.log(a + b); // 13
console.log(a - b); // 7
console.log(a * b); // 30
console.log(a / b); // 3.333...
console.log(a % b); // 1
console.log(a ** 2); // 100

Оператор + для конкатенации строк

Оператор + используется не только для сложения чисел, но и для объединения строк:

Конкатенация строк с оператором +:

let firstName = "John";
let lastName = "Doe";
let fullName = firstName + " " + lastName;
console.log(fullName); // "John Doe"

let age = 25;
let message = "Мне " + age + " лет!";
console.log(message); // "Мне 25 лет!"

⚠️ Важно: Тип приоритета при использовании +

console.log(5 + 3); // 8 (числа складываются)
console.log("5" + 3); // "53" (строковая конкатенация)
console.log(5 + "3"); // "53" (строковая конкатенация)
console.log("5" + "3"); // "53" (строковая конкатенация)

Сокращённые операторы присвоения

let x = 10;
x += 5; // то же самое, что x = x + 5
console.log(x); // 15

x -= 3; // то же самое, что x = x - 3
console.log(x); // 12

x *= 2; // то же самое, что x = x * 2
console.log(x); // 24

x /= 4; // то же самое, что x = x / 4
console.log(x); // 6

Лучшие практики

✅ Используйте const по умолчанию

// Хорошо
const PI = 3.14159;
const MAX_USERS = 100;

// Используйте let только когда нужно изменять
let userCount = 0;
userCount++;

✅ Давайте понятные имена переменным

// Хорошо
const userAge = 25;
const isAdmin = true;
const emailAddress = "user@example.com";

// Плохо
const ua = 25;
const ia = true;
const ea = "user@example.com";

✅ Инициализируйте переменные сразу

// Хорошо
const name = "John";
const age = 25;

// Избегайте
let name;
let age;
name = "John";
age = 25;

✅ Группируйте связанные переменные

// Хорошо - данные пользователя вместе
const userName = "John";
const userEmail = "john@example.com";
const userAge = 25;

// Или используйте объект
const user = {
  name: "John",
  email: "john@example.com",
  age: 25
};

✅ Никогда не используйте var

// ❌ Плохо - избегайте var
var oldStyle = "не используйте это";

// ✅ Хорошо - используйте const и let
const newStyle = "используйте это";
let mutableValue = "или это если нужно менять";

Типичные ошибки

❌ Забыли присвоить значение

let message; // переменная объявлена, но не инициализирована
console.log(message); // undefined

// Правильно:
let message = "Hello";
console.log(message); // "Hello"

❌ Попытка изменить const

const PI = 3.14;
PI = 3.1415; // ❌ TypeError: Assignment to constant variable

// Правильно:
let radius = 5;
radius = 10; // ✅ OK для let

❌ Использование var вместо let/const

// Плохо - использование var (старый стандарт)
var x = 5;
if (true) {
  var x = 10;
}
console.log(x); // 10 (неожиданный результат!)

// Хорошо - использование let
let y = 5;
if (true) {
  let y = 10;
}
console.log(y); // 5 (блочная видимость работает правильно)

❌ Неправильное имя переменной

let 123name; // ❌ Ошибка: начинается с цифры
let my-var; // ❌ Ошибка: содержит дефис
let let; // ❌ Ошибка: ключевое слово

// Правильно:
let name123; // ✅ OK
let my_var; // ✅ OK
let myVar; // ✅ OK (camelCase)

❌ Строковая конкатенация вместо сложения

let a = "5";
let b = "3";
console.log(a + b); // "53" (строковая конкатенация, а не сложение!)

// Правильно:
let x = 5;
let y = 3;
console.log(x + y); // 8 (математическое сложение)

// Или преобразуйте строки:
console.log(Number(a) + Number(b)); // 8

❌ Забыли обновить переменную

let score = 50;
score + 10; // ❌ Действие выполнено, но результат не сохранён
console.log(score); // 50 (значение не изменилось)

// Правильно:
score = score + 10; // ✅ Результат сохранён
console.log(score); // 60

// Или используйте сокращённый оператор:
score += 10;
console.log(score); // 70

❌ Проблемы с var в циклах и блоках

// Проблема с var
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
// Выведет: 3, 3, 3 (неожиданно!)

// Решение с let
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
// Выведет: 0, 1, 2 (правильно!)

Итоги

Что мы изучили

  • Переменные — это именованные хранилища для данных
  • Три способа объявления: let, const, var
  • const по умолчанию, let для изменяемых данных, var только для совместимости
  • Правильное именование улучшает читаемость кода
  • Математические операции используются для вычислений и объединения данных
  • Сокращённые операторы делают код более компактным

Ключевые правила

  1. Используйте const для значений, которые не меняются
  2. Используйте let для переменных, которые будут изменяться
  3. ❌ Никогда не используйте var в новом коде
  4. Давайте понятные, описательные имена переменным
  5. Инициализируйте переменные при объявлении
  6. Помните о типах данных при использовании оператора +
  7. Блочная область видимости let и const предпочтительнее функциональной области var
