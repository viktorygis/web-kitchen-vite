Строгий режим — "use strict"

Строгий режим — режим выполнения JavaScript, который применяет более строгие правила и ограничения.
Был введен в ECMAScript 5 (2009) для устранения проблем обратной совместимости и улучшения безопасности кода.

Основные цели строгого режима

  • Устранение "небрежных" и опасных ошибок
  • Предотвращение использования устаревших и нестабильных возможностей
  • Упрощение оптимизации кода для JavaScript движков
  • Подготовка к будущим версиям языка

Где применяется

  • В отдельных скриптах
  • В отдельных функциях
  • Автоматически в модулях (ES6)
  • Автоматически в классах (ES6)

⚠️ Важно

Строгий режим является односторонней операцией — нельзя отключить его после активации на определённом уровне. Он оставляет код работающим в более строгих условиях, что помогает избежать ошибок.

Использование "use strict"

Директива "use strict" — специальная строка, которая активирует строгий режим.
Должна быть первой исполняемой строкой в скрипте или функции (могут быть комментарии выше).

Глобальное применение

Активирует строгий режим для всего скрипта:

"use strict";

// Весь код ниже будет в строгом режиме
let x = 10;
console.log(x);

function myFunction() {
  // Также в строгом режиме
  let y = 20;
}

Локальное применение

Активирует строгий режим только для конкретной функции:

// Этот код НЕ в строгом режиме
function normalFunction() {
  x = 10; // Создаёт глобальную переменную (разрешено)
}

// Этот код В строгом режиме
function strictFunction() {
  'use strict';
  y = 20; // ReferenceError: y is not defined
}

normalFunction();    // Работает
strictFunction();    // Ошибка!

Активация в разных контекстах

Контекст Способ активации Примечание
Скрипт "use strict"; первой строкой Включает для всего файла
Функция 'use strict'; первой строкой функции Включает только для функции
Модуль (ES6) Автоматически включен Не требует директивы
Класс (ES6) Автоматически включен Все методы в строгом режиме
Консоль браузера Нужно обернуть в IIFE (function() { 'use strict'; ... })()

Активация в консоли браузера (Developer Tools):

// Нужно обернуть в IIFE
(function() {
  'use strict';
  x = 10; // ReferenceError
})();

// Или с использованием блока
{
  'use strict';
  y = 20; // ReferenceError
}

Важные моменты при использовании

  • Директива должна быть первой исполняемой строкой (комментарии разрешены)
  • Использование "use strict"; (с двойными кавычками) или 'use strict'; (с одинарными) — оба варианта работают
  • Регистр имеет значение: только "use strict" или 'use strict' работают
  • Директива — это всего лишь строка, поэтому старые браузеры просто её игнорируют

Особенности строгого режима

1. Переменные должны быть объявлены

Нельзя использовать переменные без объявления через var, let или const:

// БЕЗ strict
x = 10; // Создаёт глобальную переменную

// СО strict
'use strict';
x = 10; // ReferenceError: x is not defined

// Правильно
let x = 10; // OK
const y = 20; // OK
var z = 30; // OK

2. Удаление переменных запрещено

Нельзя удалять переменные, объявленные через var, let или const:

// БЕЗ strict
let x = 10;
delete x; // undefined в non-strict, false в результате

// СО strict
'use strict';
let x = 10;
delete x; // SyntaxError: Delete of an unqualified identifier in strict mode

3. Параметры функции должны быть уникальными

Запрещены дублирующиеся имена параметров:

// БЕЗ strict
function sum(a, a) {
  return a + a; // Используется второй параметр
}
sum(1, 2); // 4

// СО strict
'use strict';
function sum(a, a) { // SyntaxError: Duplicate parameter name not allowed here
  return a + a;
}

4. Свойства объектов не могут быть удалены так же

const obj = {};
Object.defineProperty(obj, 'x', { value: 10, configurable: false });

// БЕЗ strict
delete obj.x; // false (молчаливо игнорируется)

// СО strict
'use strict';
delete obj.x; // TypeError: Cannot delete property 'x' of #<Object>

5. Запрещено использование eval()

eval() не создаёт переменные в локальной области видимости:

// БЕЗ strict
eval('var x = 10');
console.log(x); // 10 (глобальная переменная создана)

// СО strict
'use strict';
eval('var x = 10');
console.log(typeof x); // undefined (x локальна для eval)

6. Запрещено использование with

Конструкция with запрещена в строгом режиме:

// БЕЗ strict
const obj = { x: 10 };
with (obj) {
  console.log(x); // 10 (работает)
}

// СО strict
'use strict';
with (obj) { // SyntaxError: Strict mode code may not include a with statement
  console.log(x);
}

7. Запрещены восьмеричные литералы

Синтаксис старых восьмеричных чисел запрещён:

// БЕЗ strict
let num = 010; // 8 (восьмеричное число)

// СО strict
'use strict';
let num = 010; // SyntaxError: Octal literals are not allowed in strict mode

8. Свойства arguments неизменяемы

function test(a) {
  // БЕЗ strict
  arguments[0] = 20;
  console.log(a); // 20 (привязаны вместе)

  // СО strict
  'use strict';
  arguments[0] = 30;
  console.log(a); // Остаёт 1 (независимы)
}
test(1);

Изменения в работе this

В строгом режиме поведение this существенно отличается от обычного режима.
Это одно из самых важных изменений.

this в функциях без объекта

// БЕЗ strict
function greet() {
  console.log(this); // window (или global в Node.js)
}
greet();

// СО strict
'use strict';
function greet() {
  console.log(this); // undefined
}
greet(); // this = undefined

this при вызове через call/apply

function test(value) {
  console.log(this, value);
}

// БЕЗ strict
test.call(null, 'a');      // window, 'a'
test.apply(null, ['b']);   // window, 'b'

// СО strict
'use strict';
test.call(null, 'a');      // null, 'a'
test.apply(null, ['b']);   // null, 'b'
test.call(undefined, 'c'); // undefined, 'c'

this в методах объекта

const obj = {
  value: 42,
  getValue: function() {
    console.log(this.value);
  }
};

// Работает одинаково в обоих режимах
obj.getValue(); // 42 (this = obj)

// Проблема при присваивании
const fn = obj.getValue;
fn(); // БЕЗ strict: window.value (undefined)
      // СО strict: Error: Cannot read property 'value' of undefined

Сравнение: Strict vs Non-Strict

Возможность Без strict Со strict
Необъявленные переменные ✅ Создаёт глобальные ❌ ReferenceError
Удаление переменных ❌ false ❌ SyntaxError
Дублирующиеся параметры ✅ Разрешены ❌ SyntaxError
eval() ✅ Создаёт в глобальной области ✅ Создаёт локально
with statement ✅ Разрешен ❌ SyntaxError
Восьмеричные литералы ✅ 010 = 8 ❌ SyntaxError
this в функции ✅ window/global ✅ undefined
arguments привязаны к параметрам ✅ Да ❌ Нет

Практическое сравнение:

// === БЕЗ STRICT ===
x = 10; // Создаёт window.x
function test() { return this; }
test() === window; // true

// === СО STRICT ===
'use strict';
x = 10; // ReferenceError
function test() { return this; }
test() === undefined; // true

Лучшие практики

✅ Используйте для новых проектов

// Всегда добавляйте в начало нового файла
"use strict";

// Весь код вашего приложения
const app = {
  // ...
};

✅ Используйте в модулях ES6

// Модули автоматически в строгом режиме
// Не нужно писать "use strict"
export function myFunction() {
  // Уже в строгом режиме
  x = 10; // ReferenceError
}

✅ Используйте для отдельных функций в старых проектах

// Переходите постепенно
function newFunction() {
  'use strict';
  // Только эта функция в строгом режиме
  let x = 10;
}

function oldFunction() {
  // Может быть в старом стиле
  // Постепенно переводите
}

✅ Тестируйте код в строгом режиме

// Разработка
const app = {
  init: function() {
    'use strict';
    // Проверяйте весь код в строгом режиме
    // Это поможет найти проблемы рано
  }
};

✅ В современных фреймворках он уже включен

// React, Vue, Angular и другие фреймворки
// автоматически используют "use strict"
// в своих сборщиках (webpack, vite и т.д.)

// Вам просто нужно писать код безопасно
const Component = () => {
  // Уже в строгом режиме через сборщик
  return <div>Hello</div>;
};

Типичные ошибки

❌ "use strict" не первая строка

// Ошибка - директива не сработает
console.log('something');
"use strict"; // Не активируется!

// Правильно - первой строкой
"use strict";
console.log('something');

❌ Забывают активировать в новых файлах

// Плохо - можно случайно создать глобальные переменные
function init() {
  data = []; // Создаёт глобальную переменную!
}

// Хорошо
"use strict";
function init() {
  const data = []; // Ошибка: data не определена
}

❌ Неправильное написание директивы

// ❌ Не работает - неправильный регистр
"USE STRICT"; // Просто строка, не директива

// ❌ Не работает - дополнительные слова
"use strict mode"; // Просто строка

// ✅ Работает
"use strict";
'use strict';

// ✅ Работает (если первая)
// комментарий
"use strict";

❌ Неправильно обрабатывают this в callback-функциях

// Ошибка
'use strict';
const obj = {
  value: 10,
  process: function(callback) {
    callback(); // this будет undefined!
  }
};

// Правильно - используйте arrow-функции или bind
obj.process(() => {
  console.log(this.value); // this из внешней области
});

// Или bind
obj.process(function() {
  console.log(this.value);
}.bind(obj));

❌ Ожидают, что strict активируется глобально в concat-файлах

// file1.js
"use strict";
// ...

// file2.js
// Это файл НЕ в строгом режиме!
// "use strict" не передаётся между файлами

// Решение - добавьте в каждый файл или используйте сборщик
"use strict";
// ...

❌ Удаляют необходимые переменные

// Ошибка
'use strict';
let x = 10;
delete x; // TypeError: Cannot delete property 'x'

// Правильно - просто переприсваивайте
x = null; // или undefined
delete obj.property; // OK для свойств объектов

❌ Смешивают strict и non-strict функции неправильно

// Опасно - разные режимы
'use strict';
function strictFn() {
  x = 10; // ReferenceError
}

function nonStrictFn() { // ❌ Будет в strict режиме!
  x = 10; // ReferenceError
}

// Функции наследуют режим оборачивающего скрипта

Итоги

Ключевые моменты

  • "use strict" — директива для активации строгого режима
  • Должна быть первой исполняемой строкой в скрипте или функции
  • Запрещает необъявленные переменные
  • Запрещает удаление обычных переменных
  • Запрещает дублирующиеся параметры функции
  • Изменяет поведение this (становится undefined вместо window)
  • eval() работает локально, а не глобально
  • Модули и классы ES6 автоматически в строгом режиме

Рекомендации

  1. ✅ Используйте "use strict" для всех новых проектов
  2. ✅ Добавляйте в начало файла или функции
  3. ✅ Проверяйте конфликты с существующим кодом
  4. ✅ Используйте в консоли при отладке
  5. ✅ Помните о изменении поведения this
  6. ✅ Не беспокойтесь в ES6 модулях и классах
  7. ❌ Не удаляйте его из старого кода без проверки
  8. ❌ Не ожидайте его активации через concat файлов
