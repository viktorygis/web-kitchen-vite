Операторы нулевого слияния и присваивания

Операторы ?? и ??= позволяют удобно работать с null и undefined. Они часто используются для задания значений по умолчанию и безопасного присваивания.

Основные операторы

  • Нулевого слияние: ?? — возвращает первое определённое значение (≠ null/undefined)
  • Нулевое присваивание: ??= — присваивает, если переменная null или undefined

?? — оператор нулевого слияния

Оператор ?? возвращает первое значение, которое не null и не undefined.

Примеры:

let user;
let defaultName = "Аноним";
console.log(user ?? defaultName); // "Аноним"
// Возвращает первое определённое значение

let a = null, b = undefined, c = "Текст";
console.log(a ?? b ?? c ?? "Пусто"); // "Текст"
// Возвращает первое определённое значение

Применение для значений по умолчанию:

function getName(name) {
  return name ?? "Гость";
}

console.log(getName());        // "Гость"
console.log(getName("Вася")); // "Вася"

Сравнение ?? и ||

Разница: || возвращает первое "истинное" значение, а ?? — первое определённое (≠ null/undefined).

Как они работают

  • || использует проверку на "истинность" (truthy/falsy)
  • ?? — проверяет только на null и undefined
  • || считает 0, "", false, NaN "ложными"
  • ?? считает эти значения определёнными и вернёт их

Примеры:

let height = 0;

// Используя ||
console.log(height || 100); // 100
// (0 считается ложным)

// Используя ??
console.log(height ?? 100); // 0
// (0 — определённое значение)

Другие примеры разницы:

let count = 0;
let timeout = 0;
let name = "";

// С ||
console.log(count || 10);      // 10 (потеряли 0)
console.log(timeout || 5000);  // 5000 (потеряли 0)
console.log(name || "гость");  // "гость" (потеряли "")

// С ??
console.log(count ?? 10);      // 0 (сохранили значение)
console.log(timeout ?? 5000);  // 0 (сохранили значение)
console.log(name ?? "гость");  // "" (сохранили значение)

null default default
undefined default default

false default false
NaN default NaN
"text" "text" "text"

??= — оператор нулевого присваивания

Оператор ??= присваивает новое значение только если переменная null или undefined.

Примеры:

// Пример 1: переменная равна null
let userAge = null;
userAge ??= 18;
console.log(userAge); // 18

// Пример 2: переменная имеет значение
let userAge = 25;
userAge ??= 30;
console.log(userAge); // 25 (не изменилась)

// Пример 3: переменная undefined
let score;
score ??= 0;
console.log(score); // 0

// Пример 4: переменная имеет 0
let timeout = 0;
timeout ??= 5000;
console.log(timeout); // 0 (не изменилась, т.к. 0 ≠ null/undefined)

Когда использовать ??=

  • Инициализация переменных только если они не определены
  • Сохранение значений 0, "", false
  • Безопасное присваивание параметров конфига

Приоритет оператора ??

У ?? низкий приоритет. Используйте скобки для корректных вычислений и избежания ошибок.

Примеры с приоритетом:

// Правильно - со скобками
let area = (height ?? 100) * (width ?? 50);

// Правильно - простое выражение
let x = a ?? b ?? c;

// Вычисления с логическими операторами
let result = (condition && value) ?? fallback;

⚠️ Важно: смешивание операторов

?? нельзя смешивать с || и && без скобок!

// ❌ Ошибка - синтаксическая ошибка
let x = a && b ?? c;

// ✅ Правильно - со скобками
let x = (a && b) ?? c;

// ✅ Правильно - целиком в скобках
let x = a && (b ?? c);

Ловушки и рекомендации

⚠️ Основные ловушки

  • Не смешивайте ?? с ||/&& без скобок — получите синтаксическую ошибку
  • Помните о разнице: ?? проверяет только null/undefined, || проверяет falsy
  • Используйте ?? для значений по умолчанию, если важны 0 или ""
  • ??= удобно для инициализации переменных, но только если они действительно "пустые"
  • Не переусложняйте логику множественными ?? — лучше использовать объекты и деструктуризацию

✅ Лучшие практики

  • Выбирайте ?? если нужно сохранить 0 и ""
  • Выбирайте || если нужно проверить на любое falsy значение
  • Всегда используйте скобки при комбинировании логических операторов
  • Предпочитайте ??= присваиванию с условием if (x === null || x === undefined)
