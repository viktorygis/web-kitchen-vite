Конструкция switch в JavaScript

Конструкция switch — альтернатива длинным цепочкам if...else if.
Сравнивает одно выражение с несколькими вариантами и выполняет соответствующий блок кода.

С помощью switch ты можешь:

  • Проверять одно значение против множества вариантов.
  • Писать более читаемый код вместо глубокой вложенности.
  • Группировать несколько случаев в один блок кода.
  • Обрабатывать значения перечисляемых типов (дни, статусы, команды).

Как работает switch

Алгоритм работы:
1. Вычисляется выражение в switch(...)
2. Результат сравнивается с каждым case (строгое сравнение ===)
3. При совпадении выполняется код этого case
4. break завершает выполнение
5. Если совпадений нет — выполняется default

Диаграмма выполнения

Выражение (switch)
    ↓
Проверка: case 1? НЕТ
    ↓
Проверка: case 2? ДА
    ↓
Выполнение кода case 2
    ↓
break → Выход из switch

Синтаксис switch

Базовая структура:

switch (выражение) {
  case значение1:
    // код для значение1
    break;

  case значение2:
    // код для значение2
    break;

  default:
    // код, если нет совпадений
}

Ключевые элементы:

  • switch — ключевое слово для начала конструкции.
  • case — вариант для сравнения с выражением.
  • break — прерывает выполнение и выходит из switch.
  • default — блок по умолчанию (если нет совпадений). Необязателен.

Простой пример

Определение дня недели по числовому значению: 

let day = 3;
let dayName;

switch (day) {
  case 1:
    dayName = "Понедельник";
    break;
  case 2:
    dayName = "Вторник";
    break;
  case 3:
    dayName = "Среда";
    break;
  case 4:
    dayName = "Четверг";
    break;
  case 5:
    dayName = "Пятница";
    break;
  case 6:
    dayName = "Суббота";
    break;
  case 7:
    dayName = "Воскресенье";
    break;
  default:
    dayName = "Неверный день";
}

console.log(dayName); // "Среда"

При day = 3 сравнивается:

  • case 1: → 3 === 1 → false, пропускаем.
  • case 2: → 3 === 2 → false, пропускаем.
  • case 3: → 3 === 3 → true, выполняем код, break.

Группировка case

Один блок кода можно выполнить для нескольких значений. Просто перечисли несколько case подряд. 

let grade = 'B';
let result;

switch (grade) {
  case 'A':
  case 'B':
    result = "Отлично!";
    break;

  case 'C':
    result = "Хорошо";
    break;

  case 'D':
  case 'E':
    result = "Нужно подтянуть знания";
    break;

  default:
    result = "Неизвестная оценка";
}

console.log(result); // "Отлично!"

Код "Отлично!" выполнится как для grade = 'A', так и для grade = 'B'.

Когда использовать: Когда несколько значений должны привести к одному результату.

Fall-through: проваливание без break

Если забыть break, выполнение "проваливается" в следующий case. Обычно это ошибка, но иногда используется намеренно. 

let number = 2;
let message = "";

switch (number) {
  case 1:
    message += "Один ";
  case 2:
    message += "Два ";
  case 3:
    message += "Три";
    break;
  default:
    message = "Неизвестное число";
}

console.log(message); // "Два Три"

Что произойдёт:

  • case 1: → 2 === 1 → false, пропускаем.
  • case 2: → 2 === 2 → true, добавляем "Два ".
  • Нет break, выполнение продолжается!
  • case 3: → выполняем, добавляем "Три", затем break.
  • Результат: "Два Три"
Осторожно! Fall-through — частая причина ошибок. Всегда добавляй break в конце case, если ты не хочешь этого поведения!
Если намеренно опускаешь break, оставь комментарий: // fall-through

Строгое сравнение (===)

Switch всегда использует строгое сравнение (===). Тип и значение должны совпадать полностью. Типы не приводятся. 

let value = "5";

switch (value) {
  case 5:     // НЕ сработает ("5" !== 5)
    console.log("Число 5");
    break;
  case "5":   // Сработает
    console.log("Строка '5'");
    break;
  default:
    console.log("Другое");
}

// Вывод: "Строка '5'"

Примеры строгого сравнения:

  • 5 === "5" → false (число vs строка)
  • 1 === true → false (число vs булево)
  • 0 === false → false (число vs булево)
  • "" === 0 → false (строка vs число)

Выражения в case

В case можно использовать выражения, а не только константы. Хитрый способ: переводи в switch true и проверяй выражения. 

let a = 10;
let b = 5;

switch (true) {
  case (a + b > 20):
    console.log("Сумма больше 20");
    break;
  case (a * b === 50):
    console.log("Произведение равно 50");
    break;
  case (a > b):
    console.log("a больше b");
    break;
  default:
    console.log("Другое");
}

// Вывод: "Произведение равно 50"
Как это работает:
1. В switch передаём true
2. Каждый case содержит выражение (возвращает true/false)
3. Сравнивается: true === (результат выражения)
4. Когда выражение вернёт true — совпадение найдено!

Работа с разными типами данных

Switch может работать с любыми типами данных: числа, строки, булевы значения, null, undefined и даже объекты.

Пример с булевыми значениями:

let value = true;

switch (value) {
  case true:
    console.log("Истина");
    break;
  case false:
    console.log("Ложь");
    break;
  default:
    console.log("Не булево значение");
}

// Вывод: "Истина"

Пример со строками:

let command = "start";

switch (command) {
  case "start":
    console.log("Запуск...");
    break;
  case "stop":
    console.log("Остановка...");
    break;
  case "pause":
    console.log("Пауза...");
    break;
  default:
    console.log("Неизвестная команда");
}

// Вывод: "Запуск..."

Пример с null/undefined:

let value = null;

switch (value) {
  case null:
    console.log("Значение null");
    break;
  case undefined:
    console.log("Значение undefined");
    break;
  default:
    console.log("Другое значение");
}

// Вывод: "Значение null"

Сравнение switch и if...else

Критерий switch if...else
Читаемость Выше при 3+ условиях Выше при 2-3 условиях
Сравнение Только строгое (===) Любые условия (>, <, &&, ||)
Множество ветвей Удобно для 5+ вариантов Становится громоздким
Группировка условий Естественная через перечисление case Через || и &&
Диапазоны значений Неудобно (нужны выражения) Естественно (10 < x && x < 20)
Производительность Может быть выше при многих case Одинакова

Когда использовать switch?

  • Одно выражение сравнивается с 3+ вариантами.
  • Типичные значения: дни недели, статусы, команды, коды ошибок.
  • Когда нужна группировка нескольких case в один блок.

Когда использовать if...else?

  • Проверка диапазонов значений (10 < x < 20).
  • Сложные условия с логическими операторами.
  • 2-3 простых условия.
  • Когда нужны разные типы сравнения.

Лучшие практики

  • Всегда добавляй break: В конце каждого case, если ты не хочешь fall-through.
  • Включай default: Даже если не ожидаешь других значений — для отловки ошибок.
  • Группируй case: Если несколько значений приводят к одному результату.
  • Комментируй fall-through: Если намеренно опускаешь break, пиши // fall-through.
  • Избегай сложных выражений: Switch теряет читаемость со сложной логикой в case.
  • Используй для простых проверок: Если нужна сложная логика — используй if/else.
  • Соблюдай стиль отступов: Для улучшения читаемости кода.

Пример хорошего стиля:

switch (status) {
  case "pending":
    handlePending();
    break;

  case "success":
  case "completed":
    handleSuccess();
    break;

  case "error":
  case "failed":
    handleError();
    break;

  default:
    console.warn("Unknown status:", status);
}

Ловушки

  • Забыл break: Выполнение "проваливается" в следующий case. Частая ошибка! 
    // ОШИБКА
switch (x) {
  case 1:
    console.log("Один"); // забыли break!
  case 2:
    console.log("Два");
    break;
}
// При x=1: выведет "Один" И "Два"
  • Путаница с типами: 5 !== "5". Switch использует ===. 
    // ОШИБКА
let x = "5";
switch (x) {
  case 5: // не сработает
    console.log("Число");
    break;
}
  • Case с объектами: Объекты сравниваются по ссылке, не по содержимому. 
    // ОШИБКА
let obj1 = {x: 1};
let obj2 = {x: 1};

switch (obj1) {
  case obj2: // false, разные ссылки
    console.log("Совпадение");
}
  • Отсутствие default: Если значение не совпадает ни с одним case — ничего не произойдёт. Добавь default для отловки ошибок.
  • Сложная логика в case: Switch теряет преимущество читаемости.

Полезные шаблоны

1. Карта действий (switch как диспетчер)

function handleCommand(cmd) {
  switch (cmd) {
    case "help":
      showHelp();
      break;
    case "start":
      startApp();
      break;
    case "stop":
      stopApp();
      break;
    default:
      console.error("Unknown command:", cmd);
  }
}

2. Группировка с условной логикой

let role = "admin";

switch (role) {
  case "admin":
  case "moderator":
    console.log("Доступ к панели управления");
    break;

  case "user":
    console.log("Доступ к профилю");
    break;

  case "guest":
    console.log("Ограниченный доступ");
    break;
}

3. Switch с возвращением значения

function getStatusMessage(code) {
  switch (code) {
    case 200:
      return "OK";
    case 404:
      return "Not Found";
    case 500:
      return "Server Error";
    default:
      return "Unknown Status";
  }
}

4. Switch(true) для выражений

let age = 25;

switch (true) {
  case age < 13:
    console.log("Дети");
    break;
  case age < 18:
    console.log("Подростки");
    break;
  case age < 65:
    console.log("Взрослые");
    break;
  default:
    console.log("Пенсионеры");
}

5. Объект вместо switch для большей гибкости

// Альтернатива switch для простых отображений
const actions = {
  start: () => console.log("Запуск"),
  stop: () => console.log("Остановка"),
  pause: () => console.log("Пауза")
};

let cmd = "start";
actions[cmd]?.(); // Вызовет действие или ничего

Итоги

  • switch — альтернатива цепочкам if...else if.
  • Используй для сравнения одного выражения с несколькими вариантами.
  • Всегда добавляй break в конце case (если не нужен fall-through).
  • switch использует строгое сравнение (===), типы не приводятся.
  • Можно группировать несколько case для одного блока кода.
  • Fall-through (без break) — частая ошибка, комментируй намеренное использование.
  • Для выражений используй switch (true) с условиями в case.
  • Для диапазонов и сложной логики используй if...else.
  • Всегда включай default для обработки неожиданных значений.
  • При 3+ вариантов switch часто более читаемый, чем if/else.
