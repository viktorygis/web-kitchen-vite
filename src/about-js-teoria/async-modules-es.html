Модули в JavaScript (ES Modules)

Модули позволяют разбивать код на независимые части, которые легко переиспользовать, поддерживать и тестировать.
Это делает разработку крупных приложений удобнее, а код — чище и понятнее.

Модули — одна из самых крутых и полезных фишек JavaScript!
Они помогают организовать код, работать в команде и использовать повторно нужные функции в разных местах.

Зачем это нужно?

Модули нужны для:

  • Организации кода: модули помогают структурировать проект и разделять его на логические части.
  • Повторного использования: один и тот же модуль можно использовать в разных частях приложения.
  • Упрощения тестирования: модули можно тестировать независимо друг от друга.
  • Работа в команде: модули можно легко передавать между участниками команды и использовать в разных проектах.

Основные понятия

  • Модуль — это файл, который содержит код (например, функции, классы), который можно использовать в других файлах.
  • Экспорт (export) — позволяет сделать функции, переменные или классы доступными в других файлах.
  • Импорт (import) — позволяет использовать экспортированные значения из другого файла.
  • type="module" — атрибут, который указывает браузеру, что данный скрипт является модулем.
  • Путь к модулю указывается явно (например, ./math.js ).

Экспорт

Чтобы сделать функции или переменные доступными в других модулях, их нужно экспортировать.

📜 Как экспортировать?

1️⃣ Экспорт прямо при объявлении:

// math.js
export const pi = 3.14;
export function square(x) {
  return x * x;
}

🔹 Такой способ удобен, если нужно экспортировать отдельные функции или переменные.

2️⃣ Экспорт в конце файла:

// math.js
const pi = 3.14;
function square(x) {
  return x * x;
}

export { pi, square };

🔹 Такой подход удобен, когда нужно экспортировать сразу несколько значений.

⚠ Обрати внимание: Без export переменные и функции будут доступны только в этом файле!
Имена экспортируемых значений должны совпадать в export и import

Импорт

Чтобы использовать функции или переменные из других модулей, их нужно импортировать.

📜 Как импортировать?

1️⃣ Именованный импорт:

// main.js
import { add, PI } from './math.js';

console.log(add(2, 3)); // 5
console.log(PI); // 3.14

🔹 Такой способ удобен, если нужно импортировать несколько значений.

2️⃣ Импорт всего модуля:

// main.js
import * as math from './math.js';

console.log(math.add(2, 3)); // 5
console.log(math.PI); // 3.14

🔹 Такой подход удобен, когда нужно импортировать все экспортированные значения из модуля.

Подключение модулей в HTML?

Чтобы использовать модули в браузере, нужно подключить их через тег <script> .

📜 Пример подключения:

<script type="module" src="main.js"></script>

🔹 Обратите внимание на атрибут type="module" — он указывает, что скрипт является модулем.

Примеры

Пример 1

В этом примере мы создадим простой модуль для математических операций.

Подключение модуля

Для подключения модуля в HTML, используйте следующий код:

<script type="module" src="main.js"></script>

export модулей

Для экспорта значений из модуля, используйте ключевое слово export : 

// math.js
	export function add(a, b) {
	   return a + b;
	}

	export const PI = 3.14;

import модулей

Для импорта значений из модуля, используйте ключевое слово import : 

// main.js
	import { add, PI } from './math.js';

	console.log(add(2, 3)); // 5
	console.log(PI); // 3.14

Пример 2

Создадим модуль с функцией приветствия и импортируем её в основной файл.

export функции приветствия

// greet.js
	export function greet(name) {
	    return `Привет, ${name}!`;
	}

import функции в main.js

// main.js
	import { greet } from './greet.js';

	console.log(greet("Вася")); // Привет, Вася!

Пример 3

Импорт всего содержимого модуля как объекта и использование его функций и констант.

Экспорт нескольких значений

// utils.js
	export function multiply(a, b) {
	    return a * b;
	}

	export function subtract(a, b) {
	    return a - b;
	}

	export const VERSION = "1.0";

Импорт всего содержимого модуля

// main.js
	import * as utils from './utils.js';

	console.log(utils.multiply(3, 5)); // 15
	console.log(utils.subtract(10, 4)); // 6
	console.log(utils.VERSION); // 1.0
