Взаимодействие с пользователем

Модальные окна — простой способ взаимодействия с пользователем в браузере.
JavaScript предоставляет три встроенные функции: alert, prompt и confirm.

Общие особенности модальных окон

  • Блокируют выполнение скрипта до взаимодействия пользователя
  • Останавливают рендеринг страницы
  • Внешний вид зависит от браузера и операционной системы
  • Не являются частью DOM (не могут быть стилизованы)

Когда использовать

  • ✅ Быстрое прототипирование
  • ✅ Отладка кода
  • ✅ Срочные предупреждения и ошибки
  • ❌ НЕ для повседневного взаимодействия (используйте пользовательские диалоги)
  • ❌ НЕ для важных операций (пользователи часто нажимают OK автоматически)

⚠️ Важно

Модальные окна блокируют всю страницу и раздражают пользователей. Используйте их только когда это действительно необходимо. Для лучшего пользовательского опыта создавайте собственные диалоги с HTML, CSS и JavaScript.

Функция alert()

alert() — показывает сообщение пользователю и ждёт, пока пользователь нажмёт кнопку «ОК».
Самая простая функция для вывода уведомлений.

Синтаксис

alert(message);

Где message — текст или значение для отображения (будет преобразовано в строку).

Примеры использования alert():

alert("Добро пожаловать на наш сайт!");

let name = "John";
alert("Привет, " + name);

alert("Ошибка: неверный пароль");

let count = 42;
alert(count);  // преобразуется в "42"

Особенности alert()

  • Окно имеет только одну кнопку OK
  • Возвращает undefined
  • Текст автоматически преобразуется в строку
  • Блокирует выполнение кода до нажатия OK
  • Пользователь не может взаимодействовать с страницей, пока окно открыто

Практическое применение

  • Вывод важных уведомлений
  • Отображение сообщений об ошибках
  • Информирование о результатах операций
  • Отладка — быстрая проверка значений

Практический пример с alert():

function saveData() {
  // Сохраняем данные
  const result = saveToServer();

  if (result.success) {
    alert("Данные успешно сохранены!");
  } else {
    alert("Ошибка при сохранении: " + result.error);
  }
}

// Использование
saveData();

✅ Совет по форматированию

Используйте шаблонные строки для более читаемого форматирования:

// Вместо:
alert("Привет, " + name + "! Ваш ID: " + id);

// Лучше:
alert(`Привет, ${name}! Ваш ID: ${id}`);

Функция prompt()

prompt() — отображает диалоговое окно с запросом на ввод текста.
Позволяет получить информацию от пользователя через клавиатуру.

Синтаксис

result = prompt(title, [default]);

title — текст вопроса для отображения
default — начальное значение в поле ввода (необязательно)
Возвращает: строка (если ОК) или null (если Отмена)

Примеры использования prompt():

let name = prompt("Как вас зовут?", "Гость");
console.log(name); // введённое имя или "Гость" по умолчанию

let age = prompt("Сколько вам лет?");
console.log(age); // введённый возраст или null если отмена

let color = prompt("Ваш любимый цвет?", "синий");
if (color !== null) {
  alert(`Вы выбрали: ${color}`);
}

Особенности prompt()

  • Возвращает строку (даже если введено число)
  • Возвращает null при нажатии Отмена или Escape
  • Второй параметр (default) показывается в поле ввода
  • ВСЕГДА проверяйте результат на null перед использованием
  • В IE всегда рекомендуется указывать второй параметр

✅ Правильная проверка результата:

let input = prompt("Введите число:", "10");

// ВСЕГДА проверяйте на null первым
if (input === null) {
  console.log("Пользователь отменил ввод");
} else if (input === "") {
  console.log("Пользователь ничего не ввёл");
} else {
  console.log("Пользователь ввёл:", input);
}

Практический пример — валидация ввода в цикле:

let age;
while (true) {
  age = prompt("Сколько вам лет?", "18");

  // Пользователь нажал Отмена
  if (age === null) {
    alert("Опрос отменён");
    break;
  }

  // Проверяем, является ли это числом
  if (!isNaN(age) && age !== "") {
    alert(`Ваш возраст: ${age} лет`);
    break;
  }

  // Просим ввести заново
  alert("Пожалуйста, введите корректное число!");
}

Практическое применение

  • Запрос имени пользователя
  • Ввод числовых значений (цены, количество и т.д.)
  • Простые вопросы и ответы
  • Временная авторизация

⚠️ Преобразование типов

prompt() всегда возвращает строку! Если нужны числа, преобразуйте:

let input = prompt("Введите число:");
let number = Number(input); // или parseInt(input, 10)
let num2 = +input;          // или унарный плюс

console.log(typeof input);  // "string"
console.log(typeof number); // "number"

Функция confirm()

confirm() — отображает диалоговое окно с вопросом и двумя кнопками: OK и Отмена.
Используется для получения ответа "Да" или "Нет" от пользователя.

Синтаксис

result = confirm(question);

question — текст вопроса
Возвращает: true (если ОК) или false (если Отмена)

Примеры использования confirm():

let isAdult = confirm("Вам есть 18 лет?");
console.log(isAdult); // true или false

let shouldDelete = confirm("Вы уверены, что хотите удалить файл?");
if (shouldDelete) {
  // Удаляем файл
  console.log("Файл удалён");
} else {
  console.log("Удаление отменено");
}

Особенности confirm()

  • Возвращает логическое значение: true или false
  • Нельзя изменить текст кнопок (всегда OK и Отмена)
  • Часто используется в цепочках условий if/else
  • Удобен для принятия бинарных решений

Практические примеры с confirm():

// Пример 1: Подтверждение удаления
function deleteFile(fileName) {
  if (confirm(`Удалить файл "${fileName}"?`)) {
    // Выполняем удаление
    console.log("Файл удалён");
  } else {
    console.log("Удаление отменено");
  }
}

// Пример 2: Проверка согласия
function submitForm() {
  if (confirm("Вы согласны с условиями использования?")) {
    // Отправляем форму
    console.log("Форма отправлена");
  } else {
    alert("Вы должны согласиться с условиями");
  }
}

// Пример 3: Цепочка условий
function performAction() {
  if (!confirm("Вы действительно хотите это сделать?")) {
    alert("Действие отменено");
    return;
  }

  if (!confirm("Это необратимое действие. Продолжить?")) {
    alert("Действие отменено");
    return;
  }

  // Выполняем действие
  console.log("Действие выполнено");
}

Практическое применение

  • Подтверждение опасных операций (удаление, выход)
  • Согласие с условиями и политиками
  • Выбор между двумя вариантами
  • Проверка намерений пользователя

Сравнение функций

Функция Кнопки Возвращает Использование
alert() ОК undefined Информирование пользователя
prompt() ОК, Отмена, поле ввода Строка или null Получение текстовой информации
confirm() ОК, Отмена true или false Получение ответа да/нет

Практическое сравнение — комплексный пример:

// Комбинация всех трёх функций
function register() {
  // ALERT - информирование
  alert("Добро пожаловать на страницу регистрации!");

  // PROMPT - получение информации
  let name = prompt("Ваше имя?", "");
  if (name === null) return;

  // CONFIRM - получение подтверждения
  if (!confirm(`Подтвердить регистрацию для ${name}?`)) {
    alert("Регистрация отменена");
    return;
  }

  // ALERT - уведомление о результате
  alert(`Добро пожаловать, ${name}!`);
}

register();

Лучшие практики

✅ Всегда проверяйте результат prompt() на null

// Хорошо
let input = prompt("Введите число:");
if (input !== null) {
  let number = Number(input);
  console.log(number);
}

// Плохо - не проверяем на null
let input2 = prompt("Введите число:");
let number2 = Number(input2); // ошибка если null

✅ Используйте шаблонные строки для форматирования

// Хорошо
let name = "John";
alert(`Привет, ${name}!`);

// Менее читаемо
alert("Привет, " + name + "!");

✅ Преобразуйте данные из prompt() если нужны числа

// Хорошо
let age = prompt("Возраст:");
if (age !== null) {
  age = Number(age); // или parseInt(age, 10)
  console.log(typeof age); // "number"
}

// Плохо - остаётся строкой
let age2 = prompt("Возраст:");
console.log(typeof age2); // "string"

✅ Давайте понятные сообщения и вопросы

// Хорошо - ясные вопросы
alert("Профиль успешно обновлён!");
let name = prompt("Как вас зовут?", "Гость");
if (confirm("Вы уверены, что хотите выйти?")) { ... }

// Плохо - неясные сообщения
alert("OK");
let input = prompt("?", "");
if (confirm("Y/N?")) { ... }

✅ Для важных операций используйте пользовательские диалоги

// Не используйте для критичного:
// confirm("Удалить всю базу данных?");

// Лучше создайте свой диалог:
// <div class="custom-dialog">
//   <h3>Вы уверены?</h3>
//   <p>Это действие необратимо!</p>
//   <button id="confirm-btn">Удалить</button>
//   <button id="cancel-btn">Отмена</button>
// </div>

✅ Предоставляйте значения по умолчанию в prompt()

// Хорошо - есть значение по умолчанию
let name = prompt("Имя:", "Гость");

// Менее удобно - нет подсказки
let name2 = prompt("Имя:");

// Хорошо - пустая строка если очень нужно
let customValue = prompt("Введите значение:", "");

Типичные ошибки

❌ Не проверяют результат prompt() на null

// Ошибка
let name = prompt("Имя:");
alert("Привет, " + name); // выведет "Привет, null" если отмена

// Правильно
let name = prompt("Имя:");
if (name !== null) {
  alert("Привет, " + name);
}

❌ Забывают преобразовать строку из prompt() в число

// Ошибка - остаётся строкой
let age = prompt("Возраст:");
if (age > 18) { // сравнивает как строки!
  console.log("Взрослый");
}

// Правильно
let age = prompt("Возраст:");
if (age !== null && Number(age) > 18) {
  console.log("Взрослый");
}

❌ Используют много alert/confirm/prompt подряд

// Плохо - раздражает пользователя
alert("Добро пожаловать!");
alert("Заполните профиль");
alert("Прочитайте документ");
alert("Согласитесь с условиями");

// Лучше - один диалог или использовать page-based UI
// Или группировать вопросы в одном диалоге

❌ Сравнивают prompt() результат неправильно

// Ошибка - пустая строка не равна null
let value = prompt("Введите:");
if (value === null) {
  // Если пользователь оставил пусто и нажал OK,
  // сюда не попадёт (value будет "")
}

// Правильно
if (value === null) {
  console.log("Отмена");
} else if (value === "") {
  console.log("Пусто");
} else {
  console.log("Введено:", value);
}

❌ Используют модальные окна для критичных операций

// Ошибка - пользователи часто нажимают OK автоматически
if (confirm("Удалить все файлы?")) {
  deleteAllFiles(); // Очень опасно!
}

// Лучше - используйте пользовательский диалог
// с явным вводом подтверждения:
// Введите "УДАЛИТЬ" для подтверждения

❌ Не проверяют пустые строки в prompt()

// Ошибка
let name = prompt("Имя:", "");
console.log(name.length); // ошибка если нажали OK без ввода

// Правильно
let name = prompt("Имя:", "");
if (name !== null && name.trim() !== "") {
  console.log("Имя введено:", name);
} else {
  alert("Пожалуйста, введите имя");
}

❌ Зависают в бесконечном цикле с prompt()

// Ошибка - цикл может зависнуть
while (true) {
  let value = prompt("Введите число:");
  // Если нажать Отмена, value = null
  // и цикл не завершится
}

// Правильно
while (true) {
  let value = prompt("Введите число:");
  if (value === null) break; // выход при отмене
  if (!isNaN(value)) break; // выход при корректном числе
}

Итоги

Ключевые моменты

  • alert() — показывает сообщение, возвращает undefined
  • prompt() — получает текст, возвращает строку или null
  • confirm() — получает да/нет, возвращает true или false
  • Все три функции блокируют выполнение кода
  • ВСЕГДА проверяйте результат prompt() на null
  • ВСЕГДА преобразуйте числа из prompt() в Number()

Рекомендации

  1. Используйте эти функции для простой отладки и прототипирования
  2. Для финального приложения создавайте пользовательские диалоги
  3. Всегда проверяйте результаты перед использованием
  4. Используйте шаблонные строки для форматирования
  5. Не используйте модальные окна для критичных операций
  6. Преобразуйте типы данных если необходимо
  7. Помните о гранулярности — не показывайте слишком много диалогов
  8. Предоставляйте значения по умолчанию в prompt()

Когда использовать альтернативы

  • Вместо множественных alert/confirm — используйте пользовательские диалоги
  • Вместо модальных окон для форм — используйте HTML формы
  • Вместо alert для ошибок — используйте console.error() или встроенные системы логирования
  • Для лучшего UX — создавайте собственные компоненты диалогов
