Условные конструкции в JavaScript

Условные конструкции позволяют выполнять различные блоки кода в зависимости от логического результата выражения.
Базовый инструмент — if ,
дополнения: else , else if
и тернарный оператор ?: .

С помощью if, else if и else ты можешь:

  • Проверять условия (например, если пользователь вошёл в систему).
  • Выполнять разные действия в зависимости от результата.
  • Делать код более гибким и умным.

Как работают условные конструкции

Идея проста: код выполняется только если заданное условие истинно ( true ).
Если условие ложно ( false ), можно выполнить альтернативный блок ( else ) или проверить следующее условие через else if .

Алгоритм выполнения

  1. if проверяет условие. Если оно truthy — выполняется его блок, цепочка завершается.
  2. Если условие ложно — переход к следующему else if (если есть) и проверка нового условия.
  3. Если ни одно условие не прошло — выполняется else (если он есть).
  4. Если else отсутствует — просто ничего не выполняется.

Пример с несколькими ветками

let age = 17;

if (age >= 18) {                       // 1) Проверяем первое условие
  console.log("Ты совершеннолетний!");  // Выполнится если age >= 18
} else if (age >= 16) {                 // 2) Проверяем следующее
  console.log("Ты почти взрослый!");    // Выполнится если age >= 16 и предыдущее условие не прошло
} else {                                // 3) Иначе
  console.log("Ты ещё молод!");
}

Вывод в консоль при age = 17:

Ты почти взрослый!

Почему вывелось именно это?

  • age >= 18 → false , блок пропускается.
  • age >= 16 → true , выполняется только этот блок.
  • else уже не рассматривается (цепочка остановлена).

if — базовая проверка

if — Выполняет блок кода, если условие даёт true после приведения к логическому типу.

Синтаксис:

if (условие) {
	  // код выполняется, если условие truthy
	}

Пример:

let age = 18;
if (age >= 18) {
   console.log("Доступ открыт");
}
Стили: Всегда используй фигурные скобки, даже если одна строка — это снижает ошибки при добавлении новых строк.

else — альтернативная ветка

else — Исполняется, когда условие в if ложно. 

if (age >= 18) {
  console.log("Взрослый");
} else {
  console.log("Несовершеннолетний");
}

else if — последовательные проверки

else if — Позволяет проверить серию взаимно исключающих условий. 

let year = 2025;

if (year < 2025) {
  console.log("Слишком рано");
} else if (year > 2025) {
  console.log("Слишком поздно");
} else {
  console.log("Точно!");
}

Когда заменить на switch ?

Если сравнивается одно и то же значение с набором констант — switch улучшает читаемость.

Тернарный оператор ?:

Используется для вычисления и возврата одного из двух значений. Это выражение, а не инструкция.

Синтаксис:

let result = условие ? значениеЕслиTrue : значениеЕслиFalse;

Пример:

let age = 20;
let label = age >= 18 ? "Взрослый" : "Младше 18";
console.log(label);
Не используйте тернарный оператор для выполнения «побочных эффектов» (например, разных console.log ), если это ухудшает читаемость.

Вложенные тернарные операторы

Можно писать цепочку для множественного выбора, но легко ухудшить читаемость.

let score = 73;
let grade =
  score >= 90 ? "A" :
  score >= 75 ? "B" :
  score >= 60 ? "C" :
  "D";

console.log(grade);
Совет: Если более 2–3 уровней вложенности — лучше заменить на if / else if или объект-словарь.

Лучшие практики

  • Ясная логика: Предпочитай простые условия и «ранний выход» вместо глубокой вложенности.
  • Guard clauses: Сначала отбрасывай невалидные случаи, затем основной сценарий.
  • Скобки всегда: Избегай стиля if (x) doSomething() .
  • Не переиспользуй тернарный оператор: Он для выбора значения, а не структурного управления.
  • Избегай присваивания в условиях: if (x = y) → почти всегда ошибка (используй линтер).

Пример guard-подхода:

function process(user) {
  if (!user) return;
  if (!user.active) return;
  // основной код
  console.log("Обработка пользователя");
}

Сравнение if и ?:

Аспект if / else ?: (тернарный)
Тип конструкции Инструкция Выражение (возвращает значение)
Количество ветвей До скольких угодно ( else if ) 2 (true / false)
Читаемость при сложной логике Хорошая Падает с вложенностью
Лучшее применение Ветвление кода Присвоение значения
Side-effects (побочные эффекты) Обычны Нежелательны в обеих ветках

Ловушки

  • Присваивание вместо сравнения: if (x = 5) → всегда truthy (если 5 ≠ 0). Используй === .
  • Скрытые приведения: if ("0") → true (строка не пустая).
  • Лишние тернарные: (cond ? true : false) → просто !!cond .
  • Дублирование условий: Не повторяй одну и ту же проверку в разных ветках.
  • Неиспользуемый else: Иногда лучше ранний return вместо «пирамиды» вложенных if.

Полезные шаблоны

1. Словарь вместо цепочки if

const roles = {
  admin: "Панель администратора",
  user: "Личный кабинет",
  guest: "Гостевой доступ"
};

function getRoleLabel(r) {
  return roles[r] || "Неизвестно";
}

2. Условное выполнение через && (для побочных эффектов осторожно)

isLoggedIn && showDashboard();

3. Значение по умолчанию через || (для null/undefined лучше ??)

let timeout = userTimeout ?? 3000; // если userTimeout null или undefined

4. Ранний выход (guard)

function handle(data) {
  if (!data) return;
  if (data.error) return;
  // основной путь
}

Итоги

  • if / else if / else — основа ветвления кода.
  • ?: — компактный выбор значения, не злоупотреблять вложенностью.
  • Используй guard-подход для уменьшения вложенности.
  • Выбирай switch , когда много сравнений с одной переменной.
  • Избегай скрытых приведений и присваиваний в условиях.
