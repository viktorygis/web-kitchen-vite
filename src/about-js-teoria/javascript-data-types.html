Типы данных в JavaScript

JavaScript — динамически типизированный язык с 8 основными типами данных.
Переменные могут менять тип в процессе выполнения программы.

Классификация типов:

  • 7 примитивных типов
  • 1 объектный тип
  • Специальные значения (null, undefined)

Особенности:

  • Автоматическое преобразование типов
  • Гибкость в работе с данными
  • Нестрогая типизация

Примитивные типы данных

Примитивы — простые неизменяемые значения, которые хранятся непосредственно в переменной.
При присваивании создается копия значения.

Тип Описание Пример typeof
number Целые и дробные числа (±2 53 -1) let n = 3.14; "number"
bigint Целые числа произвольной длины const big = 123n; "bigint"
string Текст в кавычках let s = "текст"; "string"
boolean Логические true/false let b = true; "boolean"
null Специальное "ничего" let x = null; "object" (ошибка)
undefined Неприсвоенное значение let u; "undefined"
symbol Уникальные идентификаторы const sym = Symbol(); "symbol"

Особенности примитивов:

  • Неизменяемы (immutable)
  • Сравниваются по значению
  • Занимают фиксированный объем памяти

Объектные типы

Объекты — сложные структуры данных, которые могут содержать коллекции свойств и методов.
При присваивании передается ссылка на объект.

Основные объектные типы:

  • Обычные объекты {}
  • Массивы []
  • Функции function(){}
  • Даты new Date()
  • Регулярные выражения /pattern/

Пример объекта:

const user = {
  name: "Иван",
  age: 30,
  isAdmin: false
};

Особенности объектов:

  • Изменяемы (mutable)
  • Сравниваются по ссылке
  • Могут содержать методы
  • Имеют прототипы

Строки (String)

Строки — используются для хранения и работы с текстовыми данными.

Основные особенности:

  • Заключаются в кавычки: '', "" или ``
  • Поддерживают интерполяцию через ${} в обратных кавычках
  • Имеют множество полезных методов (length, toUpperCase и др.)

Примеры использования:

// Объявление строк
let name = "Вика";
let message = `Привет, ${name}!`; // интерполяция

// Методы строк
console.log(name.length); // 4
console.log(message.toUpperCase()); // "ПРИВЕТ, ВИКА!"

Где применяются:

  • Хранение имен, адресов, текстовых данных
  • Формирование сообщений для пользователей
  • Работа с URL и путями к файлам
  • Валидация ввода в формах

Числа (Number)

Числа — используются для математических операций и хранения числовых данных.

Особенности:

  • Могут быть целыми и дробными
  • Поддерживают специальные значения: Infinity, -Infinity, NaN
  • Имеют методы для работы (toFixed, toString и др.)

Примеры:

let age = 38;
let price = 99.99;
let result = 10 / 0; // Infinity

// Математические операции
let total = price * 1.2; // НДС
let rounded = total.toFixed(2); // "119.99"

Применение:

  • Расчеты в финансовых приложениях
  • Работа с датами и временем (timestamp)
  • Графики и диаграммы
  • Игровая логика (очки, здоровье)

Логические значения (Boolean)

Boolean — это тип данных, который может принимать только два значения: true (истина) и false (ложь).
Эти значения называются булевыми по имени математика Джорджа Буля.

  • true означает, что утверждение истинно.
  • false означает, что утверждение ложно.

Особенности:

  • Используются в условиях и сравнениях
  • Любое значение можно преобразовать к boolean
  • Ложные значения: false, 0, "", null, undefined, NaN

Примеры:

let isMom = true;
let hasDiscount = false;

// Использование в условиях
if (isMom) {
  console.log("Это мама");
}

// Преобразование типов
console.log(Boolean(1)); // true
console.log(!!"text"); // true (двойное отрицание)

// Сравнения
let isEqual = (5 == 5);  // Это будет true, потому что 5 равно 5
let isGreater = (5 > 3);  // Это будет true, потому что 5 больше 3
let isLess = (5 < 3);     // Это будет false, потому что 5 не меньше 3

// Логическое отрицание
let isRaining = true;
console.log(!isRaining);  // false

let isSunny = false;
console.log(!isSunny);    // true

// Пример с переменными
// Мы можем сохранить отрицание в другой переменной
const isMorning = true;
const isEvening = !isMorning;
console.log(isEvening); // false

Применение:

  • Управление потоком выполнения (if/else)
  • Флаги состояния (isLoading, isActive)
  • Валидация данных (isValid)
  • Переключатели (darkMode, notificationsEnabled)

💡 Запомни: true и false — это не строки, а именно логические значения. Их можно использовать в условиях и операторах.

Undefined

Undefined — означает, что переменная объявлена, но не инициализирована.

Особенности:

  • Возвращается при обращении к несуществующему свойству
  • Функция возвращает undefined, если нет return
  • Не следует явно присваивать undefined

Примеры:

let x;
console.log(x); // undefined

function noReturn() {}
console.log(noReturn()); // undefined

let obj = {};
console.log(obj.property); // undefined

Применение:

  • Обнаружение неинициализированных переменных
  • Проверка наличия свойств объектов
  • Аргументы функций по умолчанию

Null

Null — специальное значение, представляющее "ничего" или "пусто".

Особенности:

  • Должен присваиваться явно
  • Используется для очистки значений
  • typeof null возвращает "object" (историческая ошибка)

Примеры:

let y = null; // явное присвоение

// Очистка ссылки
let data = getData();
data = null; // освобождаем память

// Проверка
if (y === null) {
  console.log("Значение отсутствует");
}

Применение:

  • Явное указание на отсутствие значения
  • Очистка ссылок на объекты
  • Работа с DOM (элемент не найден)
  • Инициализация переменных перед использованием

Массивы (Array)

Массивы — это упорядоченные коллекции значений, которые могут содержать элементы разных типов.

Синтаксис:

Если обычная переменная хранит одно значение, то массив позволяет хранить несколько в квадратных скобках [ ], разделяя элементы запятыми.

let array = [element1, element2, element3];

Примеры:

let fruits = ["яблоко", "банан", "вишня"];
let numbers = [1, 2, 3, 4];

Может хранить разные типы данных:

Массивы могут содержать элементы разных типов, включая числа, строки, объекты и даже другие массивы. 

let mixed = [1, "текст", true, null, undefined, { name: "Иван" }];
let simple = ["hello", 25];

Создание пустого массива:

Пустой массив можно создать с помощью литерала [] или конструктора new Array().
Потом можно добавлять элементы с помощью методов массива, таких как push(). 

let guestList = []; // Пустой массив
guestList.push("Леха", "Лепеха"); // Добавление элементов
console.log(guestList); // ["Леха", "Лепеха"]

Переприсвоение массива:

Если переменная объявлена с помощью let, ей можно присвоить совершенно новый массив целиком. 

let foods = ["овощи", "фрукты"];
foods = ["чипсики", "энергетики"]; // Переменной присвоен новый массив
console.log(foods); // Выведет ["чипсики", "энергетики"]

Индексы:

Индексы — это числовые позиции элементов в массиве, начиная с нуля. Первый элемент имеет индекс 0, второй — 1 и так далее.
Для доступа к элементу массива используйте квадратные скобки с индексом. 

let fruits = ["яблоко", "банан", "вишня"];
console.log(fruits[0]); // "яблоко"
console.log(fruits[1]); // "банан"
console.log(fruits[2]); // "вишня"

Обращение к несуществующему элементу:

Если обратиться к индексу, который не существует, вернется undefined.

let items = ["ручка", "ластик", "тетрадь"];

console.log(items[1]); // "ластик" ✅ (существует)
console.log(items[5]); // undefined ❌ (такого индекса нет)

Массив через const:

Даже если массив объявлен с помощью const, его элементы можно изменять, добавлять и удалять. Однако нельзя переназначить саму переменную массива.

Assignment to constant variable. — ошибка, возникающая при попытке переназначить переменную, объявленную с помощью const.

const numbers = [1, 2, 3];
numbers[1] = 42; // Изменяем второй элемент
console.log(numbers); // [1, 42, 3]

// Попытка переназначить массив вызовет ошибку
// numbers = [4, 5, 6]; // ❌ Ошибка: Assignment to constant variable.

Замена элемента по индексу:

Чтобы заменить любой элемент массива, нужно просто указать его индекс и новое значение. 

let fruits = ["яблоко", "банан", "арбузик"];
fruits[1] = "клубничка"; // Меняем "банан" на "клубничку"
console.log(fruits); // ["яблоко", "клубничка", "арбузик"]

Свойство .length

Свойство .length возвращает количество элементов в массиве. 

let fruits = ["яблоко", "банан", "арбузик"];
console.log(fruits.length); // 3

length - 1

Чтобы получить индекс последнего элемента массива, нужно использовать length - 1. 

let fruits = ["яблоко", "банан", "арбузик"];
console.log(fruits[fruits.length - 1]); // "арбузик"

// Изменяем последний элемент массива
let numbers = [213, 44, 95, 23, 71, 85, 12];
numbers[numbers.length - 1] = 3;
console.log(numbers);

Удаление всех элементов массива

Чтобы удалить все элементы массива, можно присвоить ему пустой массив. 

let fruits = ["яблоко", "банан", "арбузик"];
fruits = []; // Теперь массив пустой
console.log(fruits); // []

Полная замена массива

Чтобы заменить весь массив, можно присвоить переменной новый массив. 

let fruits = ["яблоко", "банан", "арбузик"];
fruits = ["груша", "апельсин"]; // Теперь массив содержит новые элементы
console.log(fruits); // ["груша", "апельсин"]

Проверка типов данных

Оператор typeof - используется для определения типа данных переменной или выражения.

Пример:

console.log(typeof 42);           // "number"
console.log(typeof "текст");       // "string"
console.log(typeof true);          // "boolean"
console.log(typeof undefined);     // "undefined"
console.log(typeof null);          // "object" (историческая ошибка)
console.log(typeof {});            // "object"
console.log(typeof []);            // "object" (массивы - объекты)
console.log(typeof function(){}); // "function", хотя и является объектом

Зачем нужна проверка типов:

  • Для отладки и предотвращения ошибок
  • Для динамической обработки данных
  • Для проверки входных данных в функциях
  • Для обеспечения совместимости с различными типами данных

Проверка типа данных помогает избежать ошибок в программе. Например, если мы ожидаем число, но получаем строку, это может привести к неправильным результатам.
💡 typeof помогает убедиться, что данные имеют нужный тип. Создадим переменную isActive и проверим её тип.

let isActive = true;
console.log(typeof isActive); // "boolean"

Проверка на null:

Важно: typeof null возвращает "object", что является исторической ошибкой.
Так что, несмотря на то что null — это отдельный тип данных, typeof null всё равно возвращает "object" 

let value = null;
console.log(typeof value); // "object"

// Для проверки на null используйте строгое сравнение:
if (value === null) {
  console.log("Это null");
}

Проверка на undefined:

Важно: undefined — это отдельный тип данных, который указывает на то, что переменная была объявлена, но не инициализирована.
typeof undefined возвращает "undefined" 

let value;
console.log(typeof value); // "undefined"

// Для проверки на undefined используйте строгое сравнение:
if (value === undefined) {
  console.log("Это undefined");
}

Тип данных null используется для явного указания, что переменная не имеет значения. Это не то же самое, что undefined — null означает отсутствие значения, в то время как undefined указывает, что значение ещё не присвоено.

Создадим переменную user и присвоим ей значение null, чтобы когда данные станут доступны, можно заменить null на объект с данными.

let user = null;
console.log(user); // Выведет null

// Позже, когда данные станут доступны:
user = { name: "Иван", age: 30 };
console.log(user); // Выведет объект { name: "Иван", age: 30 }

Таблица типов данных:

Выражение Результат typeof Примечание

typeof "текст" "string"
typeof true "boolean"
typeof undefined "undefined"
typeof null "object" Историческая ошибка
typeof {} "object"
typeof [] "object" Для массивов используйте Array.isArray()

Рекомендации и примеры проверки типов:

1. Проверка null:

const maybeNull = null;

// Правильный способ:
if (maybeNull === null) {
  console.log('Это null');
}

// Неправильно (вернет 'object'):
console.log(typeof maybeNull);

2. Проверка массивов:

const arr = [1, 2, 3];
const obj = {};

// Правильный способ:
console.log(Array.isArray(arr)); // true
console.log(Array.isArray(obj)); // false

// Неправильно (оба вернут 'object'):
console.log(typeof arr);
console.log(typeof obj);

3. Проверка объектов:

const date = new Date();
const user = { name: 'John' };

// Проверка конкретного типа:
console.log(date instanceof Date); // true
console.log(user instanceof Object); // true

// Проверка на обычный объект (не массив, не null):
function isPlainObject(value) {
  return typeof value === 'object'
    && value !== null
    && !Array.isArray(value);
}

console.log(isPlainObject(user)); // true
console.log(isPlainObject(date)); // false

4. Другие полезные проверки:

// Проверка числа (учитывает NaN):
function isNumber(value) {
  return typeof value === 'number' && !isNaN(value);
}

// Проверка функции:
const fn = () => {};
console.log(typeof fn === 'function'); // true

// Проверка undefined:
let x;
console.log(typeof x === 'undefined'); // true
console.log(x === undefined); // тоже работает

Важные замечания:

  • typeof null возвращает 'object' - это известная ошибка в языке
  • Для проверки массивов всегда используйте Array.isArray()
  • instanceof не работает с примитивами и между фреймами
  • Для надежной проверки типов в сложных случаях используйте библиотеки типа lodash
