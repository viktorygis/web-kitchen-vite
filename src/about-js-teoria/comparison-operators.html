Операторы сравнения в JavaScript

Операторы сравнения - это специальные символы или комбинации символов, которые используются для сравнения двух значений. Они возвращают логическое значение true или false в зависимости от результата сравнения.

Вот основные операторы:

  • == — нестрогое равенство, Сравнивает значения без учета их типа.
  • === — строгое равенство, Сравнивает значения и их типы.
  • != — нестрогое неравенство, Возвращает true, если значения не равны.
  • !== — строгое неравенство, Возвращает true, если значения или их типы разные.
  • > — больше, Проверяет, больше ли одно число другого.
  • < — меньше, Проверяет, меньше ли одно число другого.
  • >= — больше или равно, Проверяет, больше или равно ли одно число другому.
  • <= — меньше или равно, Проверяет, меньше или равно ли одно число другому.

Приведение типов - это процесс преобразования значения из одного типа данных в другой. Например, строка может быть преобразована в число, или логическое значение может быть преобразовано в строку. В JavaScript приведение типов может происходить автоматически (неявное) или вручную (явное).

Примеры:

console.log('5' === 5);       // false (разные типы)
console.log(null == undefined); // true (специальное правило)
console.log(null === undefined); // false (разные типы)
console.log('2' > '15');      // true (лексикографически)

== — нестрогое равенство

== — нестрогое равенство - сравнивает значения с приведением типов. Истинно, если значения равны после приведения.

Основные трансформации:

  • '5' == 5 → true (строка → число)
  • true == 1 , false == 0 → true
  • '' == 0 , ' ' == 0 → true
  • null == undefined → true (уникальное правило)
  • '0' == false → true
  • [] == '' → true , [] == 0 → true
  • NaN == NaN → false

Примеры:

// Лучше избегать без явной причины
console.log('5' == 5);        // true
console.log(false == 0);      // true
console.log([] == '');        // true
console.log([] == 0);         // true
console.log([0] == 0);        // true

=== — строгое равенство

=== — строгое равенство - сравнивает значения без приведения типов. Истинно, если значения и типы одинаковы.

Ключевые моменты:

  • '5' === 5 → false
  • 0 === false → false
  • NaN === NaN → false (используйте Number.isNaN() )

Примеры:

console.log('5' === 5);  // false
console.log(5 === 5);    // true
console.log(Number.isNaN(NaN)); // true

!= — нестрогое неравенство

!= — нестрогое неравенство - проверяет неравенство, но без учета типа данных. Истинно, если значения не равны после приведения типов.

Типичные результаты:

  • 5 != '5' → false
  • null != undefined → false
  • '' != 0 → false

Примеры:

console.log(5 != '5');          // false
console.log(null != undefined); // false
console.log('' != 0);           // false

!== — строгое неравенство

!== — строгое неравенство - проверяет неравенство с учетом типа данных. Истинно, если значения или их типы различны.

Особенности:

  • 5 !== '5' → true
  • 5 !== 5 → false
  • NaN !== NaN → true

Примеры:

console.log(5 !== '5'); // true
console.log(5 !== 5);  // false
console.log(NaN !== NaN); // true

// Пример с переменными
let fruit1="яблоко";
let fruit2="яблоко";
console.log(fruit1 !== fruit2); // false (оба значения и типы одинаковы)

let fruit3="банан";
console.log(fruit1 !== fruit3); // true (значения разные)

Реляционные операторы

Реляционные операторы - это операторы, которые сравнивают два значения и возвращают логическое значение true или false в зависимости от результата сравнения. Они включают: >, <, >=, <=.

Оператор > используется для проверки, больше ли одно значение другого. Он возвращает true, если первое значение больше второго.

А оператор < проверяет, меньше ли одно значение другого. Он возвращает true, если первое значение меньше второго.

Примеры:

console.log(10 > 5);  // true
console.log(3 > 7);   // false

console.log(3 < 5);   // true
console.log(10 < 5);  // false

Оператор >= проверяет, больше ли ИЛИ равно одно значение другому. Он возвращает true, если первое значение больше или равно второму, и на наоборот с оператором <= меньше ли ИЛИ равно одно значение другому.

Примеры:

console.log(5 >= 3);   // true
console.log(3 >= 5);   //false
console.log(10 >= 10); //true

console.log(3 <= 5);  //true
console.log(5 <= 3);  //false
console.log(10 <= 10); //true

Примеры:

let apples = 5;
let oranges = 4;
let comparison = apples >= oranges;
console.log("Больше или равно ли яблок апельсинам? " + comparison); //Больше или равно ли яблок апельсинам? true

Сравнение строк

Строки сравниваются в JavaScript по кодам символов, то есть символы имеют свой уникальный код в таблице Unicode, и JavaScript использует эти коды для сравнения.

Примеры:

  • 'abc' < 'abd' → true - сравниваются посимвольно, и 'c' имеет меньший код, чем 'd'.
  • '2' > '15' → true - сравниваются посимвольно, и '2' имеет больший код, чем '1'.
  • 'apple' < 'apricot' - сравниваются посимвольно, и 'l' имеет меньший код, чем 'r'. → true

Unicode - это стандарт кодирования символов, который присваивает уникальный числовой код каждому символу из различных языков и систем письма. В JavaScript строки сравниваются на основе этих кодов, что позволяет корректно сравнивать символы из разных алфавитов.
Например, код символа 'A' равен 65, а код символа 'a' равен 97. Поэтому при сравнении строк JavaScript использует эти коды для определения порядка символов.

Регистр букв влияет на сравнение!

Регистр букв влияет на сравнение! Символы в верхнем регистре считаются меньше, чем в нижнем, потому что их кодовые точки в таблице Unicode меньше.

// Пример сравнения строк с разным регистром
console.log("Apple" < "apple"); // true (A имеет меньший код, чем a)
console.log("banana"<"apple");// true

Сравнение ==

При использовании оператора == строки сравниваются посимвольно, и результат будет true только если строки идентичны, но регистр не учитывается.

console.log("Hello" == "Hello"); // true
console.log("Hello" == "hello"); // false (разный регистр)
console.log("banana"=="apple");// false (разные строки)

Сравнение ===

При использовании оператора === строки сравниваются посимвольно, и результат будет true только если строки идентичны и имеют одинаковый регистр.

console.log("Hello" === "Hello"); // true
console.log("Hello" === "hello"); // false (разный регистр)
console.log("banana"==="apple");// false (разные строки)

Сравнение строки с числом

При сравнении строки с числом, JavaScript пытается преобразовать строку в число. Если строка не может быть преобразована в число, результат будет false.

console.log("5" > 3); // true (строка "5" преобразуется в число 5)
console.log("abc" > 3); // false (строка "abc" не может быть преобразована в число)
console.log("2abc" > 3); // false (строка "2abc" не может быть преобразована в число)

⚠ Запомни: строки, содержащие числа, могут быть приведены к числовому типу, но строки, не содержащие чисел, будут преобразованы в NaN, что всегда сравнивается как false

Если первая строка короче, то она считается "меньше"

console.log("app" < "apple"); // true (первая строка короче)
console.log("app" > "apple"); // false (первая строка короче)

!== - строгое неравенство

Оператор !== проверяет, не равны ли строки с учетом регистра. Если строки различаются хотя бы одним символом или регистром, результат будет true. Чтобы проверить, что строка не равна другой строке, мы используем оператор строгого неравенства 

console.log("Hello" !== "hello"); // true (разный регистр)
console.log("banana" !== "apple");// true (разные строки)

null и undefined

Имеют специальное правило равенства между собой и ведут себя по-разному при приведении к числам.

Правила:

  • null == undefined → true
  • null === undefined → false
  • null в числовом сравнении → 0
  • undefined → NaN ⇒ сравнения >, <, >=, <= → false

Примеры:

console.log(null == undefined); // true
console.log(null >= 0);         // true (null -> 0)
console.log(null == 0);         // false
console.log(undefined > 0);     // false

NaN

NaN не равно ничему, даже самому себе.

Проверки:

  • NaN === NaN → false
  • Number.isNaN(NaN) → true
  • isNaN('foo') → true (нестрогое приведение)

Примеры:

console.log(NaN === NaN);        // false
console.log(Number.isNaN(NaN));  // true
console.log(isNaN('foo'));       // true

Ловушки и рекомендации

  • Предпочитайте === / !== вместо == / != .
  • null и undefined — равны только друг другу.
  • Используйте Number.isNaN() для проверки NaN.
  • Строки сравниваются лексикографически: '2' > '15' → true .
  • Пустые массивы могут вводить в заблуждение: [] == '' , [] == 0 .
