Введение в JavaScript

JavaScript — это язык программирования, который был создан для "оживления" веб-страниц. Скрипты на JavaScript встраиваются в HTML и выполняются автоматически при загрузке страниц в браузере.

💡 Почему JavaScript?

  • Полная интеграция с HTML/CSS
  • Простота для базовых задач
  • Поддержка всеми браузерами
  • Кроссплатформенность
  • Работает на клиенте и сервере

Альтернативные языки: TypeScript, CoffeeScript, Dart транспилируются в JavaScript, предлагая дополнительные возможности, такие как строгая типизация.

Основные факты о JavaScript

🏛️ Независимость от Java

Первоначально язык называлась "LiveScript". Была переименована в JavaScript в попытке воспользоваться популярностью Java. Сегодня это независимый язык со своей спецификацией ECMAScript.

⚙️ Исполнение кода

JavaScript выполняется в разных окружениях:

  • В браузере: Chrome (V8), Firefox (SpiderMonkey), Safari (JavaScriptCore), Edge (V8)
  • На сервере: Node.js, Deno
  • В приложениях: Electron, React Native

🔒 Безопасность в браузере

JavaScript ограничен в своих возможностях для защиты пользователей:

  • ❌ Не может произвольно читать/писать файлы на жёсткий диск
  • ❌ Не имеет доступа к другим вкладкам (политика "одинакового источника")
  • ✅ Может изменять DOM, отправлять запросы, работать с cookies

Возможности и ограничения

✅ Возможности в браузере ❌ Ограничения
Изменение HTML и CSS (DOM, CSSOM) Ограниченный доступ к файловой системе
Реагирование на действия пользователя (События) Политика "одинакового источника"
Сетевые запросы (AJAX, Fetch, WebSocket) Нельзя выполнять системные команды
Работа с cookies и localStorage Ограниченный доступ к оборудованию
Работа с геолокацией (с разрешения)

Характеристики JavaScript

Характеристика Описание
Интерпретируемый язык Код выполняется построчно, без предварительной компиляции
Динамическая типизация Тип переменной определяется во время выполнения, может меняться
Прототипное наследование Объекты наследуют от других объектов через прототипы
Функции первого класса Функции — это значения, их можно передавать как аргументы
Поддержка OOP Объектно-ориентированное программирование с классами (ES6+)
Асинхронность Поддержка callbacks, Promises, async/await

Тег <script>

Тег <script> используется для вставки JavaScript кода в HTML документ. Может содержать код непосредственно или ссылаться на внешний файл.

Встроенный скрипт

Код выполняется автоматически при загрузке страницы:

<script>
  alert('Привет, мир!');
</script>

Внешний скрипт

Рекомендуется для структуры и переиспользования:

<script src="/path/to/script.js"></script>

Атрибуты тега script

Атрибут Описание Значение Статус
src Путь к внешнему файлу JavaScript URL файла ✅ Актуально
type MIME тип скрипта "text/javascript" (по умолчанию) ⚠️ Не требуется
async Загружать скрипт асинхронно (не блокирует парсинг HTML) Boolean ✅ Актуально
defer Выполнить скрипт после загрузки DOM Boolean ✅ Актуально
language Язык скрипта (устарело) "JavaScript" ❌ Устарел

💡 Лучшая практика размещения

Скрипты обычно помещают в конец <body> или используют атрибут defer в <head> для оптимизации загрузки страницы.

Структура кода JavaScript

Инструкции

Инструкции — это команды, которые выполняют действия. Каждая инструкция обычно заканчивается точкой с запятой. 

alert('Привет');
alert('Мир');

Точка с запятой

Хотя JavaScript часто автоматически добавляет точки с запятой (автоматическая вставка ASI), их явное использование предотвращает ошибки:

⚠️ Проблема без точки с запятой

let a = 1
let b = 2
[a, b] = [b, a] // Может быть неправильно интерпретировано!

✅ Правильное использование

let a = 1;
let b = 2;
[a, b] = [b, a]; // Работает корректно

Рекомендации по структуре

  • ✅ Всегда используйте точки с запятой для завершения инструкций
  • ✅ Разделяйте логические блоки пустыми строками
  • ✅ Соблюдайте единый стиль кодирования (используйте линтеры: ESLint)
  • ✅ Используйте правильный отступ (2 или 4 пробела, или табуляция)

Комментарии в JavaScript

Комментарии — это части кода, которые игнорируются интерпретатором. Служат для объяснения и документирования работы кода.

Однострочные комментарии

// Это однострочный комментарий
let x = 5; // Комментарий после кода

Многострочные комментарии

/*
  Это многострочный
  комментарий для объяснения
  более сложного кода
*/
let y = 10;

JSDoc комментарии

Специальный формат для документирования функций и параметров:

/**
 * Возвращает сумму двух чисел
 * @param {number} a Первое число
 * @param {number} b Второе число
 * @returns {number} Сумма a и b
 * @example
 * sum(2, 3); // 5
 */
function sum(a, b) {
  return a + b;
}

Правила хороших комментариев

Правило Описание Пример
"Почему", не "Что" Объясняйте причину, а не буквальное содержание кода // Используем Map для O(1) поиска
Избегайте дублирования Не повторяйте то, что видно из кода Не пишите: // Увеличиваем x на 1
JSDoc для функций Документируйте параметры и возвращаемые значения @param, @returns, @example
Обновляйте при изменениях Комментарии должны быть актуальны Если код изменился — обновите комментарий

❌ Примеры плохих комментариев

// Это переменная
let x = 5;

// Увеличиваем на 1
x++;

// Выводим x
console.log(x);

✅ Примеры хороших комментариев

// Кэшируем результаты для оптимизации
const cache = new Map();

// Используем -1 для обозначения "не найдено"
const notFound = -1;

Спецификация ECMAScript

ECMA-262 — официальная спецификация, содержащая полную и формализованную информацию о JavaScript. Определяет сам язык программирования на самом глубоком уровне.

📚 О спецификации

Спецификация может показаться сложной из-за формального стиля изложения. Это не для ежедневного использования, но если вам нужна самая достоверная информация — это правильное место.

История версий

Версия Год Особенности
ES5 2009 Strict mode, JSON, Array methods (map, filter, reduce)
ES6 (ES2015) 2015 Classes, Arrow functions, const/let, Promises, Modules
ES2016+ 2016+ Ежегодные обновления: async/await, Symbols, Proxy, etc.

Где найти спецификацию

  • Официальный текст: tc39.es/ecma262
  • Предложения новых возможностей: github.com/tc39/proposals
  • Stage процесс: Предложения проходят 5 этапов (Stage 0-4), Stage 3 — практически готовые функции

Лучшие практики

✅ Используйте внешние файлы для JavaScript

Встроенный скрипт усложняет поддержку и кэширование:

// Хорошо:
<script src="/js/main.js"></script>

// Плохо:
<script>
  // весь код здесь
</script>

✅ Размещайте скрипты правильно

Скрипты в конце body или с атрибутом defer:

<!-- Вариант 1: В конце body -->
<body>
  <content></content>
  <script src="/js/main.js"></script>
</body>

<!-- Вариант 2: В head с defer -->
<head>
  <script src="/js/main.js" defer></script>
</head>

✅ Используйте точки с запятой

Явные точки с запятой предотвращают ошибки автоматической вставки:

// Хорошо:
let x = 5;
console.log(x);

// Может привести к ошибкам:
let x = 5
console.log(x)

✅ Комментируйте сложный код

Объясняйте "почему", а не "что":

// Хорошо — объясняет причину:
// Используем Map для быстрого поиска O(1)
const userMap = new Map();

// Плохо — описывает очевидное:
// Создаём карту пользователей
const userMap = new Map();

❌ Типичные ошибки новичков

  • Вставлять все скрипты в <head> без defer/async
  • Полагаться на автоматическую вставку точек с запятой
  • Писать слишком много комментариев для очевидного кода
  • Использовать встроенные скрипты для крупного функционала
  • Забывать об экранировании HTML символов в коде
