Функции-выражения

Функция-выражение — это когда функция создаётся как часть выражения (например, присваивается переменной или передаётся как аргумент). Это отличается от объявления функции (function declaration).

К чему это полезно?

  • Функции можно хранить в переменных и передавать как значения
  • Можно создавать анонимные функции для одноразового использования
  • Позволяет создавать замыкания и модульные структуры

Function declaration vs function expression

Объявления функций "поднимаются" (hoisted) — их можно вызвать до объявления. Выражения функций видимы только после создания.

// Declaration (hoisted)
function greet() {
  return "Hi";
}
console.log(greet()); // "Hi"

// Expression (not hoisted in the same way)
const say = function() {
  return "Hello";
};
console.log(say()); // "Hello"

Named и Anonymous function expressions

// Anonymous
const anon = function() {
  console.log("I'm anonymous");
};

// Named (полезно для отладки и рекурсии)
const factorial = function fact(n) {
  return n <= 1 ? 1 : n * fact(n - 1);
};
console.log(factorial(5)); // 120

IIFE (Immediately Invoked Function Expression)

Одноразовая функция, выполняемая сразу после создания — полезна для локализации переменных.

(function() {
  const secret = "только в этой функции";
  console.log("IIFE runs:", secret);
})();

Функции как значения

Функции можно передавать в другие функции, возвращать из функций и сохранять в структурах данных.

function map(arr, fn) {
  const res = [];
  for (let i = 0; i < arr.length; i++) res.push(fn(arr[i]));
  return res;
}

const doubled = map([1,2,3], x => x * 2);
console.log(doubled); // [2,4,6]

Примеры

Пример: колбэк

function doAsync(task, cb) {
  setTimeout(() => {
    cb(task + " done");
  }, 100);
}

doAsync("job", (res) => console.log("Callback got:", res));

Пример: возвращаемая функция

function makeCounter() {
  let count = 0;
  return function() {
    return ++count;
  };
}
const c = makeCounter();
console.log(c()); // 1
console.log(c()); // 2

Ловушки и советы

⚠️ Частые ошибки

  • Ожидание hoisting у function expression — вызов до присвоения даст ошибку
  • Анонимные функции сложнее дебажить (стек вызовов без имени)
  • Не перепутайте контекст this в function expression vs arrow function
