Итерационные методы массивов

Итерационные методы — это методы, которые проходят по каждому элементу массива и выполняют над ними операции. Они полезны для обработки и анализа больших наборов данных.

🔄 Как работают итерационные методы

  • forEach() — просто выполняет функцию для каждого элемента
  • every() — проверяет, ВСЕ ли элементы удовлетворяют условию
  • some() — проверяет, ХОТЯ БЫ ОДИН элемент удовлетворяет условию
  • reduce() — аккумулирует значения в один результат

💡 Важно: Итерационные методы НЕ создают новый массив (кроме reduce, который возвращает одно значение). Используйте map() и filter(), если нужен новый массив.

forEach() — Итерация по элементам

forEach() - метод выполняет функцию один раз для каждого элемента массива. Не возвращает новый массив и не изменяет исходный (если функция не мутирует).

Синтаксис:

arr.forEach(callback(element, index, array))

Параметры:

  • callback - функция для выполнения на каждой итерации

Возвращает:

  • undefined (не возвращает ничего полезного)

Примеры:

const fruits = ['apple', 'banana', 'orange'];

// Просто вывести каждый элемент
fruits.forEach(fruit => {
  console.log(fruit);
});
// apple
// banana
// orange

// С индексом
fruits.forEach((fruit, index) => {
  console.log(`${index}: ${fruit}`);
});
// 0: apple
// 1: banana
// 2: orange

// Практический пример: накопление в переменную (плохо!)
let sum = 0;
[1, 2, 3, 4].forEach(num => {
  sum += num;
});
console.log(sum); // 10

⚠️ Частые ошибки

1. Используют forEach() когда нужен map()

forEach() не возвращает новый массив:

// Неправильно — результат не сохранится:
const doubled = arr.forEach(x => x * 2); // undefined

// Правильно:
const doubled = arr.map(x => x * 2);

2. Пытаются использовать break или continue

В forEach() нельзя использовать break/continue:

// Неправильно:
arr.forEach(x => {
  if (x === 2) break; // SyntaxError!
});

// Правильно — используйте for или find():
for (const x of arr) {
  if (x === 2) break; // работает
}

3. Изменяют массив внутри forEach()

Это может привести к неожиданному поведению:

const arr = [1, 2, 3];
arr.forEach((x, i) => {
  if (x === 2) arr.splice(i, 1); // опасно!
});

💡 Когда использовать

  • Выполнение побочных эффектов: логирование, обновление DOM, API запросы
  • Простая итерация без трансформации данных
  • Если нужен новый массив — используйте map()
  • Если нужна фильтрация — используйте filter()

every() и some() — Проверка условия

every() - проверяет, ВСЕ ЛИ элементы удовлетворяют условию. some() - проверяет, ХОТЯ БЫ ОДИН элемент удовлетворяет условию.

Синтаксис:

arr.every(callback(element, index, array))
arr.some(callback(element, index, array))

Возвращает:

  • true или false

Примеры:

const nums = [2, 4, 6, 8];

// every() — все ли чётные?
console.log(nums.every(x => x % 2 === 0)); // true

// some() — есть ли хотя бы одно чётное?
console.log(nums.some(x => x % 2 === 0)); // true

const mixed = [1, 2, 3, 5];

// every() — все ли чётные?
console.log(mixed.every(x => x % 2 === 0)); // false

// some() — есть ли хотя бы одно чётное?
console.log(mixed.some(x => x % 2 === 0)); // true (есть 2)

// Практический пример: валидация
const users = [
  { name: 'Аня', age: 25 },
  { name: 'Петя', age: 30 },
  { name: 'Виктор', age: 28 }
];

// Все ли совершеннолетние?
const allAdult = users.every(u => u.age >= 18); // true

// Есть ли несовершеннолетние?
const hasMinor = users.some(u => u.age < 18); // false

⚠️ Частые ошибки

1. Путают логику every() и some()

const arr = [2, 4, 5, 6];

arr.every(x => x % 2 === 0); // false (не все чётные)
arr.some(x => x % 2 === 0);  // true (есть чётные)

2. Забывают о пустых массивах

Пустой массив для every() возвращает true (vacuous truth), для some() — false:

const empty = [];

empty.every(x => x > 10); // true — технически верно (нет элементов, которые не подходят)
empty.some(x => x > 10);  // false — нет элементов, которые подходят

3. Используют неправильное условие

// Неправильно — без return:
arr.every(x => { x > 0 }); // false (return пропущен)

// Правильно:
arr.every(x => x > 0);

💡 Когда использовать

  • every() — все ли элементы валидны? Все ли права установлены?
  • some() — есть ли ошибки? Есть ли совпадений?
  • Валидация формы: fields.every(f => f.valid)
  • Проверка прав: user.permissions.some(p => p === 'admin')

reduce() и reduceRight() — Аккумуляция

reduce() - метод аккумулирует значения массива в один результат, вызывая функцию для каждого элемента. Это один из самых мощных методов массивов.

Синтаксис:

arr.reduce(callback(accumulator, element, index, array), [initialValue])
arr.reduceRight(callback(accumulator, element, index, array), [initialValue])

Параметры:

  • callback - функция, которая получает аккумулятор и текущий элемент
  • initialValue - начальное значение аккумулятора (необязательно)

Возвращает:

  • Одно значение — результат аккумуляции

Примеры:

// Сумма всех чисел
const nums = [1, 2, 3, 4, 5];
const sum = nums.reduce((acc, x) => acc + x, 0);
console.log(sum); // 15

// Произведение
const product = nums.reduce((acc, x) => acc * x, 1);
console.log(product); // 120

// Подсчёт элементов по типу
const items = ['a', 'b', 'a', 'c', 'b', 'a'];
const count = items.reduce((acc, item) => {
  acc[item] = (acc[item] || 0) + 1;
  return acc;
}, {});
console.log(count); // { a: 3, b: 2, c: 1 }

// Объединение массивов массивов
const arr = [[1, 2], [3, 4], [5, 6]];
const flat = arr.reduce((acc, row) => acc.concat(row), []);
console.log(flat); // [1, 2, 3, 4, 5, 6]

// Преобразование массива в объект
const users = [
  { id: 1, name: 'Аня' },
  { id: 2, name: 'Петя' }
];
const userMap = users.reduce((acc, user) => {
  acc[user.id] = user;
  return acc;
}, {});
console.log(userMap); // { 1: { id: 1, name: 'Аня' }, ... }

// reduceRight() — справа налево
const arr2 = [1, 2, 3, 4];
arr2.reduce((acc, x) => acc - x, 0);      // 0 - 1 - 2 - 3 - 4 = -10
arr2.reduceRight((acc, x) => acc - x, 0); // 0 - 4 - 3 - 2 - 1 = -10

⚠️ Частые ошибки

1. Забывают initialValue

Без начального значения первый элемент используется как аккумулятор:

const nums = [1, 2, 3];

// Без initialValue:
nums.reduce((acc, x) => acc + x); // 1 + 2 + 3 = 6

// С initialValue:
nums.reduce((acc, x) => acc + x, 0); // 0 + 1 + 2 + 3 = 6

// Разница — на пустом массиве:
[].reduce((acc, x) => acc + x); // TypeError!
[].reduce((acc, x) => acc + x, 0); // 0

2. Забывают return аккумулятор

Callback ДОЛЖЕН возвращать обновленный аккумулятор:

// Неправильно:
const result = nums.reduce((acc, x) => { acc + x }); // undefined!

// Правильно:
const result = nums.reduce((acc, x) => acc + x, 0);

3. Мутируют аккумулятор

Хотя это работает, лучше создавать новый объект/массив:

// Работает, но опасно:
const result = arr.reduce((acc, x) => {
  acc.push(x * 2);
  return acc;
}, []);

// Лучше — использовать map() + filter():
const result = arr.filter(x => x > 5).map(x => x * 2);

4. Используют reduce() для всего подряд

Reduce мощный, но часто код более читаемо с map/filter:

// Сложно читать:
arr.reduce((acc, x) => x > 2 ? [...acc, x * 2] : acc, []);

// Читабельнее:
arr.filter(x => x > 2).map(x => x * 2);

💡 Когда использовать

  • Суммирование/произведение: nums.reduce((a, x) => a + x, 0)
  • Подсчёт элементов: arr.reduce((acc, x) => ({ ...acc, [x]: (acc[x] || 0) + 1 }), {})
  • Преобразование массива в объект по ID
  • Выравнивание вложенных массивов (хотя есть flat())
  • Если код становится сложным — рассмотрите map/filter вместо reduce
