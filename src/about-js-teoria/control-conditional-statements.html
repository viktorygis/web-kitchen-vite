Условные конструкции в JavaScript

Условные конструкции позволяют выполнять различные блоки кода в зависимости от результата проверки условия.
Базовые инструменты: if, else, else if и тернарный оператор ?:.

С помощью условных конструкций ты можешь:

  • Проверять условия (например, вошёл ли пользователь в систему).
  • Выполнять разные действия в зависимости от результата.
  • Делать программы более гибкими и интеллектуальными.
  • Принимать решения на основе значений переменных.

Как работают условные конструкции

Основная идея: код выполняется только если условие истинно (true).
Если условие ложно (false), можно выполнить альтернативный блок (else) или проверить следующее условие (else if).

Алгоритм выполнения

  1. if проверяет условие. Если оно truthy — выполняется его блок, цепочка завершается.
  2. Если условие ложно — переход к следующему else if (если есть).
  3. Если ни одно условие не прошло — выполняется else (если он есть).
  4. Если else отсутствует — просто ничего не выполняется.

Пример с несколькими ветками

let age = 17;

if (age >= 18) {                       // 1) Проверяем первое условие
  console.log("Ты совершеннолетний!");  // Выполнится если age >= 18
} else if (age >= 16) {                 // 2) Проверяем следующее условие
  console.log("Ты почти взрослый!");    // Выполнится если age >= 16
} else {                                // 3) Иначе
  console.log("Ты ещё молод!");
}

Вывод в консоль при age = 17:

Ты почти взрослый!

Почему вывелось именно это?

  • age >= 18 → false, первый блок пропускается.
  • age >= 16 → true, выполняется только этот блок.
  • else не рассматривается (цепочка уже завершена).

if — базовая проверка

if — выполняет блок кода, если условие даёт true после приведения к логическому типу.

Синтаксис:

if (условие) {
  // код выполняется, если условие truthy
}

Пример:

let age = 18;
if (age >= 18) {
   console.log("Доступ открыт");
}
Лучшая практика: Всегда используй фигурные скобки {}, даже если блок состоит из одной строки. Это снижает риск ошибок при добавлении новых строк кода.

else — альтернативная ветка

else — выполняется, когда условие в if ложно. Используется для обработки противоположного случая. 

if (age >= 18) {
  console.log("Взрослый");
} else {
  console.log("Несовершеннолетний");
}

Всегда выполняется ровно один из двух блоков: либо if, либо else.

else if — последовательные проверки

else if — позволяет проверить серию взаимно исключающих условий. Выполняется, если предыдущее if было ложно, а его условие истинно. 

let year = 2025;

if (year < 2025) {
  console.log("Слишком рано");
} else if (year > 2025) {
  console.log("Слишком поздно");
} else {
  console.log("Точно!");
}

Когда использовать else if?

  • Когда нужно проверить несколько различных условий.
  • Для категоризации значений (например, оценки: A, B, C, D).
  • Если условия взаимно исключают друг друга.

Альтернатива: switch

Если сравнивается одно и то же значение с набором констант, switch улучшает читаемость и производительность.

Тернарный оператор ?:

Тернарный оператор — компактный способ выбрать одно из двух значений на основе условия.
Это выражение (возвращает значение), а не инструкция.

Синтаксис:

let result = условие ? значениеЕслиTrue : значениеЕслиFalse;

Пример:

let age = 20;
let label = age >= 18 ? "Взрослый" : "Младше 18";
console.log(label); // "Взрослый"

Сравнение с if/else:

// Вариант 1: if/else
let label;
if (age >= 18) {
  label = "Взрослый";
} else {
  label = "Младше 18";
}

// Вариант 2: тернарный оператор (компактнее)
let label = age >= 18 ? "Взрослый" : "Младше 18";
Осторожно: Не используй тернарный оператор для выполнения побочных эффектов, если это ухудшает читаемость. Например, избегай: condition ? console.log("A") : console.log("B") — используй обычный if/else.

Вложенные тернарные операторы

Можно цеплять несколько тернарных операторов для множественного выбора, но это быстро снижает читаемость. 

let score = 73;
let grade =
  score >= 90 ? "A" :
  score >= 75 ? "B" :
  score >= 60 ? "C" :
  "D";

console.log(grade); // "B"
Совет: Если более 2–3 уровней вложенности — лучше заменить на if / else if или объект-словарь. 
// Лучше: использовать объект
const grades = {
  90: "A",
  75: "B",
  60: "C"
};

let grade = "D";
for (let threshold in grades) {
  if (score >= threshold) {
    grade = grades[threshold];
    break;
  }
}

Сравнение if/else и ?:

Аспект if / else ?: (тернарный)
Тип конструкции Инструкция Выражение (возвращает значение)
Количество ветвей Множество (есть else if) Только 2 (true / false)
Читаемость при сложной логике Хорошая Падает с вложенностью
Лучшее применение Ветвление кода, побочные эффекты Присвоение значения
Может вернуть значение Нет (нужен return) Да (сразу)

Лучшие практики

  • Ясная логика: Предпочитай простые условия и «ранний выход» вместо глубокой вложенности.
  • Guard clauses: Проверяй невалидные случаи в начале и возвращайся рано, затем основной сценарий.
  • Всегда используй скобки: Избегай стиля if (x) doSomething() без {}.
  • Используй === вместо ==: === проверяет тип и значение, избегая неожиданных приведений.
  • Тернарный оператор для значений: Используй ?: для присвоения значения, а не для управления потоком.

Пример guard-подхода:

function process(user) {
  // Проверяем невалидные случаи в начале
  if (!user) return;
  if (!user.active) return;

  // Основной код
  console.log("Обработка пользователя:", user.name);
}

Ловушки

  • Присваивание вместо сравнения: if (x = 5) → присваивает 5 и всегда true. Используй if (x === 5).
  • Скрытые приведения типов: if ("0") → true (строка не пустая). Используй if (value === 0).
  • Путаница с falsy значениями: 0, "", null, undefined, NaN, false — все они falsy. Будь явным: if (count === 0).
  • Лишние тернарные: (cond ? true : false) → просто cond или !!cond.
  • Дублирование условий: Не повторяй одну и ту же проверку в разных ветках — выдели в переменную.

Полезные шаблоны

1. Словарь вместо цепочки if/else if

const roles = {
  admin: "Панель администратора",
  user: "Личный кабинет",
  guest: "Гостевой доступ"
};

function getRoleLabel(role) {
  return roles[role] || "Неизвестно";
}

2. Условное выполнение через && (для побочных эффектов осторожно)

isLoggedIn && showDashboard();
// Эквивалент:
if (isLoggedIn) showDashboard();

3. Значение по умолчанию через ?? (для null/undefined)

let timeout = userTimeout ?? 3000;
// Если userTimeout null или undefined, используем 3000
// Отличие от ||: ?? не трактует 0 как falsy

4. Ранний выход (guard clause)

function handle(data) {
  if (!data) return;
  if (data.error) return;
  // основной путь
  console.log("Успешно обработано");
}

5. Проверка множественных условий

// Плохо: глубокая вложенность
if (user) {
  if (user.active) {
    if (user.premium) {
      // действие
    }
  }
}

// Хорошо: guard clause
if (!user || !user.active || !user.premium) return;
// действие

Итоги

  • if / else if / else — основа ветвления кода.
  • ?: — компактный выбор значения между двумя вариантами.
  • Используй guard-подход для уменьшения вложенности и улучшения читаемости.
  • Выбирай switch, когда много сравнений одной переменной с константами.
  • Используй === вместо == для избежания неожиданных приведений типов.
  • Запомни falsy значения: 0, "", null, undefined, NaN, false.
  • Тернарный оператор для значений, if/else для побочных эффектов.
