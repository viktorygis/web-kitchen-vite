Базовые операторы в JavaScript

Операторы — символы или ключевые слова, которые выполняют операции над значениями (операндами).
В JavaScript операторы делятся на категории по количеству операндов и типу операций.

Классификация операторов

  • Унарные — работают с одним операндом (-x, ++x, !x)
  • Бинарные — работают с двумя операндами (x + y, x > y)
  • Тернарные — работают с тремя операндами (a ? b : c)

Основные категории операторов

  • Арифметические — математические операции
  • Присваивания — сохранение значений в переменные
  • Сравнения — сравнение значений
  • Логические — логические операции
  • Побитовые — операции на уровне битов
  • Типов — проверка и преобразование типов

⚠️ Приоритет операций

Порядок выполнения операций важен! Используйте скобки для явного указания порядка:

// Без скобок - умножение выполнится первым
console.log(5 + 2 * 3); // 11

// Со скобками - другой результат
console.log((5 + 2) * 3); // 21

Математические операторы

Арифметические операторы — выполняют математические операции с числами.
JavaScript поддерживает все основные арифметические операции.

Оператор Название Описание Пример Результат
+ Сложение Складывает два числа 5 + 3 8
- Вычитание Вычитает второе число из первого 5 - 3 2
* Умножение Умножает два числа 5 * 3 15
/ Деление Делит первое число на второе 15 / 3 5
% Остаток от деления (modulo) Возвращает остаток от деления 17 % 5 2
** Возведение в степень Возводит число в степень 2 ** 3 8

Примеры математических операций:

// Сложение
let a = 10;
let b = 40;
console.log(a + b); // 50

// Вычитание
console.log(b - a); // 30

// Умножение
console.log(a * b); // 400

// Деление
console.log(b / a); // 4

// Остаток от деления
console.log(b % a); // 0
console.log(40 % 7); // 5

// Возведение в степень
console.log(2 ** 3); // 8
console.log(5 ** 2); // 25

Особенности математических операторов

  • Деление / всегда возвращает число (даже для целочисленного деления)
  • Остаток % сохраняет знак делимого (первого операнда)
  • Оператор + выполняет конкатенацию строк, если хотя бы один операнд — строка
  • Остаток от деления полезен для проверки делимости (чётные/нечётные числа)
  • Все операторы работают с преобразованием типов (числовым преобразованием)

Приоритет операций (от высшего к низшему)

  1. () — скобки
  2. ** — возведение в степень
  3. *, /, % — умножение, деление, остаток
  4. +, - — сложение, вычитание

Пример с приоритетами операций:

// Без скобок
let result1 = 5 + 2 * 3;     // 11 (умножение выполняется первым)
let result2 = 10 - 2 ** 2;   // 6 (степень имеет приоритет)

// Со скобками
let result3 = (5 + 2) * 3;   // 21 (скобки меняют порядок)
let result4 = (10 - 2) ** 2; // 64 (скобки меняют порядок)

console.log(result1, result2, result3, result4); // 11, 6, 21, 64

✅ Совет: Используйте унарный плюс для преобразования в число

let str = "10";
let num = +str; // 10 (преобразуется в число)

console.log(typeof str);  // "string"
console.log(typeof num);  // "number"

Операторы присваивания

Операторы присваивания — сохраняют значения в переменные.
В JavaScript есть базовый оператор = и составные операторы присваивания.

Оператор Название Пример Эквивалент
= Присваивание x = 5 -
+= Сложение и присваивание x += 2 x = x + 2
-= Вычитание и присваивание x -= 1 x = x - 1
*= Умножение и присваивание x *= 3 x = x * 3
/= Деление и присваивание x /= 2 x = x / 2
%= Остаток и присваивание x %= 5 x = x % 5
**= Степень и присваивание x **= 2 x = x ** 2

Примеры операторов присваивания:

let x = 10;

// Базовое присваивание
x = 5; // x = 5

// Составные операторы
x += 2; // x = 7 (то же, что x = x + 2)
x -= 1; // x = 6 (то же, что x = x - 1)
x *= 2; // x = 12 (то же, что x = x * 2)
x /= 3; // x = 4 (то же, что x = x / 3)
x %= 3; // x = 1 (то же, что x = x % 3)

console.log(x); // 1

Особенности присваивания

  • Присваивание возвращает присвоенное значение
  • Можно использовать множественное присваивание: a = b = c = 5
  • Составные операторы выполняют операцию и присваивание в одно действие
  • Составные операторы работают с любыми бинарными операторами
  • Присваивание ассоциативно справа налево

Цепочка присваиваний:

// Все переменные получат значение 5
let a, b, c;
a = b = c = 5;

console.log(a); // 5
console.log(b); // 5
console.log(c); // 5

// Выполняется справа налево:
// 1. c = 5
// 2. b = (результат: 5)
// 3. a = (результат: 5)

✅ Совет: Составные операторы для краткости

// Менее читаемо
x = x + 10;
y = y * 2;

// Более читаемо и кратко
x += 10;
y *= 2;

Операторы инкремента и декремента

Инкремент (++) увеличивает значение на 1, декремент (--) уменьшает на 1.
Эти операторы имеют две формы: префиксную и постфиксную.

Форма Оператор Описание Пример
Префиксный инкремент ++x Увеличивает значение и возвращает новое значение let x = 5; let y = ++x; // x = 6, y = 6
Постфиксный инкремент x++ Увеличивает значение и возвращает старое значение let x = 5; let y = x++; // x = 6, y = 5
Префиксный декремент --x Уменьшает значение и возвращает новое значение let x = 5; let y = --x; // x = 4, y = 4
Постфиксный декремент x-- Уменьшает значение и возвращает старое значение let x = 5; let y = x--; // x = 4, y = 5

Различие между префиксной и постфиксной формой:

// Префиксная форма - возвращает новое значение
let a = 5;
let b = ++a;
console.log(a); // 6
console.log(b); // 6

// Постфиксная форма - возвращает старое значение
let c = 5;
let d = c++;
console.log(c); // 6
console.log(d); // 5

Использование в циклах:

// Инкремент в цикле
for (let i = 0; i < 5; i++) {
  console.log(i); // 0, 1, 2, 3, 4
}

// Декремент в цикле
for (let i = 5; i > 0; i--) {
  console.log(i); // 5, 4, 3, 2, 1
}

Важные моменты

  • Инкремент и декремент работают только с переменными (левое значение)
  • Постфиксная форма менее эффективна (создаёт копию старого значения)
  • В большинстве случаев префиксная форма предпочтительнее
  • В циклах разница между ++i и i++ не имеет значения
  • Используйте += 1 для большей читаемости в некоторых случаях

⚠️ Избегайте использования в сложных выражениях

// Плохо - непредсказуемо
let x = 5;
let result = x++ + ++x;
console.log(result); // 12 (может быть непонятно)

// Лучше - ясно
let x = 5;
x++;
let result = x + x;
console.log(result); // 12 (понятнее)

Операторы сравнения

Операторы сравнения — сравнивают два значения и возвращают логический результат (true или false).
Часто используются в условиях и циклах.

Оператор Название Описание Пример Результат
== Равенство С преобразованием типов 5 == "5" true
=== Строгое равенство Без преобразования типов 5 === "5" false
!= Неравенство С преобразованием типов 5 != "5" false
!== Строгое неравенство Без преобразования типов 5 !== "5" true
> Больше Проверяет больше ли первое значение 5 > 3 true
< Меньше Проверяет меньше ли первое значение 5 < 3 false
>= Больше или равно Проверяет больше или равно 5 >= 5 true
<= Меньше или равно Проверяет меньше или равно 5 <= 5 true

Примеры операторов сравнения:

// Нестрогое равенство (с преобразованием)
console.log(5 == "5");    // true
console.log(0 == false);  // true
console.log(null == undefined); // true

// Строгое равенство (без преобразования)
console.log(5 === "5");   // false
console.log(0 === false); // false

// Операторы больше/меньше
console.log(10 > 5);      // true
console.log(5 < 10);      // true
console.log(5 >= 5);      // true
console.log(5 <= 5);      // true

// Неравенство
console.log(5 != "5");    // false
console.log(5 !== "5");   // true

Важные моменты

  • Всегда используйте === и !== для избежания непредсказуемых преобразований
  • Нестрогое равенство == выполняет неявное преобразование типов
  • Сравнение строк происходит лексикографически (по алфавиту)
  • При сравнении чисел со строками, строка преобразуется в число
  • null == undefined возвращает true, но null === undefined возвращает false

⚠️ Опасности нестрогого сравнения

// Неожиданные результаты с ==
console.log("" == 0);        // true
console.log("0" == 0);       // true
console.log([] == 0);        // true
console.log([] == "");       // true
console.log([] == ![]);      // true (парадокс!)

// Строгое сравнение === работает предсказуемо
console.log("" === 0);       // false
console.log("0" === 0);      // false
console.log([] === 0);       // false

Лучшие практики

✅ Всегда используйте === вместо ==

// ❌ Плохо
if (age == "18") {
  console.log("Совершеннолетний");
}

// ✅ Хорошо
if (age === 18) {
  console.log("Совершеннолетний");
}

✅ Используйте скобки для явного порядка операций

// Менее читаемо
let result = a + b * c - d / e;

// Более читаемо
let result = (a + (b * c)) - (d / e);
let result = a + (b * c) - (d / e);

✅ Используйте составные операторы для краткости

// Более многословно
counter = counter + 1;
price = price * discount;

// Более кратко и понятно
counter += 1;
price *= discount;

✅ Избегайте побочных эффектов в инкременте

// Плохо - непредсказуемо
let array = [1, 2, 3];
let x = 0;
array[x++] = x++; // Какой результат?

// Хорошо - ясно
let array = [1, 2, 3];
let x = 0;
array[x] = x + 1;
x += 2;

✅ Преобразуйте типы явно для ясности

// Неявное преобразование (может быть непонятно)
let str = "10";
let num = str * 1; // 10

// Явное преобразование (ясно)
let str = "10";
let num = Number(str); // 10
let num2 = parseInt(str, 10); // 10

✅ Используйте префиксный инкремент в циклах

// Хорошо
for (let i = 0; i < 10; ++i) {
  console.log(i);
}

// Тоже хорошо (в JavaScript разницы нет)
for (let i = 0; i < 10; i++) {
  console.log(i);
}

Типичные ошибки

❌ Использование == вместо ===

// Ошибка - неожиданный результат
if (user_age == "18") { // true если age = 18 (число)
  console.log("Взрослый");
}

// Правильно
if (user_age === 18) { // true только если age = 18 (число)
  console.log("Взрослый");
}

❌ Забывают скобки для порядка операций

// Ошибка - неправильный результат
let price = 100;
let tax = 0.1;
let discountPercent = 0.2;
let total = price * (1 - discountPercent) + price * tax; // Правильно
// Но без скобок была бы ошибка:
let wrong = price * 1 - discountPercent + price * tax;

❌ Путают ++i и i++ в составных выражениях

// Ошибка - получаются разные значения
let x = 0;
let result1 = x++; // result1 = 0, x = 1
console.log(result1); // 0

let y = 0;
let result2 = ++y; // result2 = 1, y = 1
console.log(result2); // 1

// Правильно - явно указывайте намерение
x++;
console.log(x);

❌ Используют инкремент с побочными эффектами

// Плохо - непредсказуемо
let i = 5;
let arr = [1, 2, 3];
arr[i++] = i++; // Какой индекс? Какое значение?

// Хорошо - явно
let i = 5;
let arr = [1, 2, 3];
arr[i] = i + 1;
i++;
i++;

❌ Забывают о преобразовании типов

// Ошибка - строка остаётся строкой
let str = "10";
let result = str + 5; // "105" (конкатенация, не сложение)

// Правильно - явно преобразуйте
let str = "10";
let result = Number(str) + 5; // 15

❌ Используют составные операторы неправильно

// Ошибка
let x = 10;
x =+ 5; // Это унарный плюс! x = 5, а не 15

// Правильно
let x = 10;
x += 5; // x = 15

❌ Забывают о приоритете операций

// Ошибка - неправильный результат
let result = 2 + 3 * 4; // 14, не 20 (умножение первым)

// Правильно - используйте скобки
let result = (2 + 3) * 4; // 20

Итоги

Ключевые моменты

  • Операторы выполняют операции над значениями (операндами)
  • Арифметические операторы: +, -, *, /, %, **
  • Операторы присваивания: =, +=, -=, *=, /=, %=, **=
  • Инкремент и декремент: ++, -- (префиксные и постфиксные)
  • Операторы сравнения: ==, ===, !=, !==, >, <, >=, <=
  • Приоритет операций влияет на результат вычислений

Рекомендации

  1. ✅ Используйте === и !== вместо == и !=
  2. ✅ Используйте скобки для явного указания порядка операций
  3. ✅ Избегайте побочных эффектов в инкременте/декременте
  4. ✅ Используйте составные операторы для краткости
  5. ✅ Преобразуйте типы явно для ясности
  6. ✅ Помните о приоритете операций
  7. ❌ Не используйте == для сравнения
  8. ❌ Не используйте инкремент в сложных выражениях
