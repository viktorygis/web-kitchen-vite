Высшие функции (Higher-Order Functions)

Высшая функция — это функция, которая принимает другую функцию в аргументы или возвращает функцию. Это основной инструмент функционального стиля в JavaScript.

Определение и преимущества

  • Повышают абстракцию
  • Упрощают повторное использование
  • Позволяют писать чистые функции и композиции

map и filter

const nums = [1,2,3,4];
// map — преобразование
const doubled = nums.map(x => x * 2); // [2,4,6,8]
// filter — отбор
const evens = nums.filter(x => x % 2 === 0); // [2,4]

reduce

reduce аккумулирует значение по массиву — мощный инструмент для агрегации и трансформации.

const sum = [1,2,3,4].reduce((acc, x) => acc + x, 0); // 10

// reduce для группировки
const names = [{tag:'a'},{tag:'b'},{tag:'a'}];
const grouped = names.reduce((acc, item) => {
  acc[item.tag] = (acc[item.tag] || 0) + 1;
  return acc;
}, {});
console.log(grouped); // { a: 2, b: 1 }

Композиция и чистые функции

Чистые функции и композиция упрощают тестирование и читаемость кода.

const add1 = x => x + 1;
const mul2 = x => x * 2;
const compose = (f, g) => x => f(g(x));
const h = compose(add1, mul2);
console.log(h(3)); // add1(mul2(3)) = 7

Примеры

Пайплайн операций

const data = [1,2,3,4,5];
const result = data
  .filter(x => x % 2 === 1)
  .map(x => x * 10)
  .reduce((acc, x) => acc + x, 0);
console.log(result); // (1 + 3 + 5) * 10 = 90

Ловушки

  • map/filter создают новые массивы — учтите расход памяти
  • reduce может быть сложен для чтения если делает слишком много
  • Не мутируйте исходные данные — prefer immutability
