Что такое JSON

JSON (JavaScript Object Notation) — это текстовый формат для хранения и обмена данными. Он часто используется, когда сервер и браузер передают информацию друг другу.

❕ Важно помнить: JSON — это строка, а не объект JavaScript. По внешнему виду он похож на объект, но это не одно и то же.

JSON удобен тем, что его легко читать человеку и обрабатывать программно. Именно поэтому он стал одним из самых популярных форматов обмена данными в веб-разработке.

Пример JSON

{
  "name": "Иван",
  "age": 25,
  "isStudent": false
}

Отличия JSON от объекта JavaScript

Объект JavaScript используется внутри кода, а JSON — для передачи и хранения данных в текстовом виде. Несмотря на похожий синтаксис, между ними есть несколько важных отличий.

Объект JavaScript

let person = {
  name: "Иван",
  age: 25
};

Тот же объект в формате JSON

{
  "name": "Иван",
  "age": 25
}

Основные отличия

Объект JavaScript JSON
Используется внутри программы Используется для хранения и обмена данными
Не является строкой Всегда является строкой
Ключи могут быть без кавычек Ключи всегда записываются в двойных кавычках
Может содержать функции Не поддерживает функции
Может содержать undefined Не поддерживает undefined
Можно создавать методы объекта Содержит только данные

Запомните: JSON предназначен только для хранения и передачи данных. Если нужно отправить объект JavaScript на сервер, сначала его преобразуют в JSON с помощью JSON.stringify(), а после получения данных обратно превращают в объект с помощью JSON.parse().

JSON.stringify()

Если нужно отправить объект JavaScript на сервер, его сначала нужно преобразовать в строку JSON. Для этого используется метод JSON.stringify().

❕ Важно: JSON.stringify() не изменяет исходный объект. Он возвращает новую строку.

Пример преобразования объекта в JSON

let person = {
  name: "Иван",
  age: 25
};

let personJSON = JSON.stringify(person);

console.log(personJSON);
// {"name":"Иван","age":25}

Такой формат удобно отправлять на сервер, потому что данные в сети обычно передаются в текстовом виде.

JSON.parse()

Чтобы преобразовать строку JSON обратно в объект JavaScript, используется метод JSON.parse(). Это нужно, когда сервер присылает нам данные в JSON-формате, а работать с ними удобнее как с объектом.

Пример преобразования JSON в объект

let personJSON = '{"name": "Иван", "age": 25}';

let person = JSON.parse(personJSON);

console.log(person);
// { name: "Иван", age: 25 }

Если строка пустая или содержит невалидный JSON, JSON.parse() выдаст ошибку.

Пример ошибки

let emptyString = '';

let result = JSON.parse(emptyString);
// Ошибка: Unexpected end of JSON input

Работа с fetch()

Когда браузер получает данные с сервера, они часто приходят в формате JSON. Для этого используется fetch(), а затем метод response.json().

❕ Важно: response.json() возвращает Promise, поэтому результат нужно обрабатывать через .then() или async/await.

Пример с fetch()

fetch("https://jsonplaceholder.typicode.com/users/1")
  .then(response => {
    if (!response.ok) {
      throw new Error("HTTP error " + response.status);
    }

    return response.json();
  })
  .then(data => {
    console.log(`Имя: ${data.name}`);
    console.log(`Email: ${data.email}`);
  })
  .catch(error => {
    console.error("Ошибка:", error.message);
  });

Сначала мы проверяем, успешно ли выполнился запрос, а затем превращаем JSON в обычный объект JavaScript.

async/await + fetch()

Тот же запрос можно записать более удобно с помощью async/await. Такой вариант часто читается проще, особенно если логика запроса становится сложнее.

Пример с async/await

async function getUser() {
  try {
    const response = await fetch("https://jsonplaceholder.typicode.com/users/1");

    if (!response.ok) {
      throw new Error("HTTP error " + response.status);
    }

    const data = await response.json();

    console.log(`Имя: ${data.name}`);
    console.log(`Email: ${data.email}`);
  } catch (error) {
    console.error("Ошибка:", error.message);
  }
}

getUser();

В этом варианте код выглядит более последовательным: сначала запрос, затем проверка ответа, потом получение данных.

Итоги

JSON — это текстовый формат для обмена данными между сервером и клиентом. Он похож на объект JavaScript, но не является объектом.

Для преобразования объекта в JSON используется JSON.stringify(), а для обратного преобразования — JSON.parse(). При работе с сервером JSON часто получают через fetch(), а для удобной обработки можно использовать async/await.

Понимание этих инструментов — основа для работы с API и современными веб-приложениями.
