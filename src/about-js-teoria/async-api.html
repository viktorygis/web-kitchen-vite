Работа с API в JavaScript

API (Application Programming Interface) — интерфейс, который позволяет программам взаимодействовать друг с другом. Веб‑приложения используют API для получения и отправки данных.

С помощью встроенной функции fetch в JavaScript можно отправлять HTTP‑запросы и работать с ответами (обычно в формате JSON).

Коротко — зачем API

  • Интеграция с внешними сервисами (платежи, карты, соцсети).
  • Получение актуальных данных (погода, новости).
  • Отправка форм/данных на сервер.
  • Автоматизация и расширение функциональности без переписывания сервера.

Основные понятия

  • REST — распространённый архитектурный стиль для веб‑API.

  • Эндпоинт — URL, по которому доступна определённая функциональность API.

  • Request / Response — запрос к API и ответ от него.

  • JSON — текстовый формат обмена данными (обычно используется в API).

  • Токен авторизации — ключ доступа к защищённым методам API.

  • response.ok — свойство, показывающее успешность HTTP‑ответа (true при статусах 200–299).

fetch

fetch — встроенная функция для отправки HTTP‑запросов. Возвращает промис с объектом Response.

Базовый пример использования

fetch("https://api.example.com/data")
  .then(response => response.json())
  .then(data => {
    console.log(data);
  })
  .catch(error => {
    console.error("Произошла ошибка:", error);
  });

Важно: fetch не считает HTTP‑статусы (404, 500) ошибками промиса — проверяйте response.ok вручную.

JSON

JSON (JavaScript Object Notation) — текстовый формат для обмена данными, который легко преобразуется в объекты JavaScript.

Пример JSON

{
  "name": "Анна",
  "age": 25,
  "skills": ["HTML", "CSS", "JavaScript"]
}

Как работать с JSON и fetch

fetch("https://jsonplaceholder.typicode.com/users/1")
  .then(response => {
    if (!response.ok) throw new Error("HTTP error " + response.status);
    return response.json();
  })
  .then(data => {
    console.log(`Имя: ${data.name}, Email: ${data.email}`);
  })
  .catch(err => {
    console.error("Ошибка:", err.message);
  });

Обработка ошибок

Правильная обработка ошибок делает интеграции надёжными: проверяйте статус ответа, ловите исключения и показывайте пользователю понятные сообщения.

Пример: проверка response.ok

fetch("https://jsonplaceholder.typicode.com/invalid-url")
  .then(response => {
    if (!response.ok) {
      throw new Error("Ошибка: " + response.status);
    }
    return response.json();
  })
  .then(data => console.log(data))
  .catch(err => console.error("Произошла ошибка:", err.message));

Используйте try/catch с async/await для более прямолинейной обработки ошибок внутри async‑функций.

Отправка данных: POST

POST используется для отправки данных на сервер. Обычно данные отправляют в формате JSON и указывают заголовок Content-Type: application/json.

Пример POST

fetch("https://jsonplaceholder.typicode.com/posts", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    title: "Новый пост",
    body: "Это содержимое поста",
    userId: 1
  })
})
  .then(response => {
    if (!response.ok) throw new Error("Ошибка: " + response.status);
    return response.json();
  })
  .then(data => console.log("Успешно отправлено:", data))
  .catch(err => console.error("Ошибка:", err.message));

Не забывайте сериализовать объект в JSON с помощью JSON.stringify() перед отправкой в body.

Примеры

Ниже — компактные рабочие примеры с публичным тестовым API jsonplaceholder.typicode.com.

Пример: Получение списка товаров (GET)

fetch("https://api.example.com/products")
  .then(r => { if (!r.ok) throw new Error(r.status); return r.json(); })
  .then(data => console.log(data))
  .catch(err => console.error("Ошибка:", err.message));

Пример: Получение текущей погоды (GET)

fetch("https://api.weather.com/current?city=Moscow")
  .then(r => { if (!r.ok) throw new Error(r.status); return r.json(); })
  .then(data => console.log(`Температура: ${data.temp}°C`))
  .catch(err => console.error("Ошибка:", err.message));

Пример: Отправка отзыва (POST)

fetch("https://jsonplaceholder.typicode.com/feedback", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ message: "Отличный сервис!" })
})
  .then(r => r.json())
  .then(data => console.log("Ответ сервера:", data))
  .catch(err => console.error("Ошибка:", err.message));

Пример: Удаление пользователя (DELETE)

fetch("https://jsonplaceholder.typicode.com/user/123", { method: "DELETE" })
  .then(r => {
    if (r.ok) console.log("Пользователь удален");
    else throw new Error("Ошибка удаления");
  })
  .catch(err => console.error("Ошибка:", err.message));

Советы и типичные ошибки

fetch возвращает промис — обрабатывайте результат через .then или async/await.

Проверяйте response.ok — HTTP‑статус 404/500 не вызовет отклонение промиса сам по себе.

JSON — текст: используйте response.json() и ловите ошибки парсинга.

Для защищённых API храните токены в переменных окружения и никогда не коммитьте их в репозиторий.

Частая ошибка — неверный заголовок Content-Type или забытый токен, из‑за чего сервер возвращает 401/403.

Лучшие практики

Храните ключи и токены в безопасном месте (переменные окружения), логируйте ошибки и документируйте API для команды.

Используйте AbortController для отмены долгих запросов и таймаутов; применяйте Promise.allSettled, если нужны результаты всех задач.
