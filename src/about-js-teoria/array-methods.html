Методы массивов JavaScript

Методы массивов — это встроенные функции для работы с массивами. JavaScript предоставляет более 30 методов для разных задач: добавление/удаление элементов, поиск, преобразование данных, проверка условий и многое другое.

📚 Структура руководства

Все методы разделены на три основные категории в зависимости от того, как они влияют на исходный массив:

  • ⚠️ Мутирующие методы — изменяют исходный массив "на месте"
  • ✅ Немутирующие методы — возвращают новый массив без изменения исходного
  • 🔁 Итерационные методы — проходят по элементам и выполняют действия

💡 Совет: Выбирайте методы в зависимости от задачи. Для React/Vue предпочитайте немутирующие методы, чтобы избежать побочных эффектов.

Классификация методов

Группа Что делает Методы
⚠️ Мутирующие
(изменяют массив "на месте")		 Добавляют/удаляют/изменяют элементы напрямую в исходном массиве push(), pop(), unshift(), shift(), splice(), sort(), reverse(), fill()
✅ Немутирующие
(возвращают новый массив)		 Создают новый массив или значение без изменения исходного concat(), slice(), join(), indexOf(), includes(), find(), map(), filter(), flat(), at()
🔁 Итерационные
(проходят по элементам)		 Выполняют функцию для каждого элемента или аккумулируют значения forEach(), every(), some(), reduce(), reduceRight()

🎯 Как выбрать правильный метод?

  • Нужен новый массив? → используйте map(), filter(), concat(), slice()
  • Нужно изменить исходный массив? → используйте push(), pop(), splice(), sort()
  • Нужно выполнить действие для каждого элемента? → используйте forEach(), map()
  • Нужно проверить условие? → используйте includes(), find(), every(), some()
  • Нужно получить одно значение из массива? → используйте reduce(), join()

Полные разделы руководства

⚠️ Мутирующие методы

Методы, которые изменяют исходный массив "на месте". Полезны для производительности, но требуют осторожности при работе с React/Vue props и state.

Методы:

  • push() — добавление в конец
  • pop() — удаление с конца
  • unshift() — добавление в начало
  • shift() — удаление с начала
  • splice() — универсальное добавление/удаление
  • sort() — сортировка
  • reverse() — разворот
  • fill() — заполнение одинаковым значением

✅ Немутирующие методы

Методы, которые возвращают новый массив или значение без изменения исходного. Рекомендуются для React/Vue и функционального программирования.

Методы:

  • concat() — объединение массивов
  • slice() — копирование части
  • join() — объединение в строку
  • indexOf() — поиск индекса
  • includes() — проверка наличия
  • find() — поиск по условию
  • map() — преобразование
  • filter() — фильтрация
  • flat() — выравнивание
  • at() — доступ с отриц. индексами

🔁 Итерационные методы

Методы, которые проходят по элементам массива и выполняют действия. Полезны для обработки данных и проверки условий.

Методы:

  • forEach() — итерация без возврата
  • every() — проверка "все ли"
  • some() — проверка "есть ли"
  • reduce() — аккумуляция слева
  • reduceRight() — аккумуляция справа

Совет: Используйте map() и filter() из немутирующих методов вместо reduce() для лучшей читаемости, когда это возможно.

💡 Лучшие практики

1. Для React/Vue предпочитайте немутирующие методы

Неправильно:

const todos = [...state.todos];
todos.push(newTodo); // мутируем!
setState(todos);

Правильно:

setState(prev => [...prev.todos, newTodo]); // новый массив

2. Цепляйте методы для большей ясности

Вместо вложенных циклов:

const users = data
  .filter(item => item.active)
  .map(item => ({ id: item.id, name: item.name }))
  .sort((a, b) => a.name.localeCompare(b.name));

3. Используйте правильный метод для задачи

  • map() + filter() часто читабельнее чем один reduce()
  • find() лучше чем filter(...)[0] для одного элемента
  • includes() лучше чем indexOf() !== -1
  • some()/every() лучше чем циклы с break/continue

4. Избегайте мутации в callback функциях

Плохо:

users.forEach(user => {
  user.processed = true; // мутируем объект!
});

Хорошо:

const processed = users.map(user => ({
  ...user,
  processed: true
}));

5. Проверяйте результат поиска

Методы вроде find(), at(), indexOf() могут вернуть undefined или -1:

const user = users.find(u => u.id === id);
if (user) {
  console.log(user.name); // безопасно
}
