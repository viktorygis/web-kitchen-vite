Немутирующие методы массивов

Немутирующие методы — это методы, которые не изменяют исходный массив, а возвращают новый массив или новое значение. Они безопасны для использования с данными, которые нужно сохранить неизменёнными.

✅ Преимущества немутирующих методов

  • Безопасны в React/Vue (не нужны предостережения о мутации props/state)
  • Проще отследить изменения данных
  • Можно цеплять методы: arr.filter(...).map(...).join(...)
  • Исходный массив остаётся неизменным (functional programming)

💡 Производительность: Немутирующие методы создают новые массивы, что требует больше памяти. Для очень больших массивов и производительно-критичного кода это может быть важным фактором. Но в большинстве случаев удобство кода важнее.

concat() — Объединение массивов

concat() - метод объединяет два или более массивов и возвращает новый массив. Не изменяет исходные массивы.

Синтаксис:

arr.concat(array1, array2, ..., value1, value2, ...)

Параметры:

  • Массивы или значения для объединения

Возвращает:

  • Новый массив с объединёнными элементами

Примеры:

const arr1 = [1, 2];
const arr2 = [3, 4];
const arr3 = [5, 6];

const combined = arr1.concat(arr2, arr3);
console.log(combined); // [1, 2, 3, 4, 5, 6]
console.log(arr1); // [1, 2] — не изменился

// Объединение с отдельными значениями
const result = arr1.concat('a', 'b', arr2);
console.log(result); // [1, 2, 'a', 'b', 3, 4]

⚠️ Частые ошибки

1. Забывают присвоить результат

concat() не изменяет исходный массив:

const arr = [1, 2];
arr.concat([3, 4]); // результат не сохранён

console.log(arr); // [1, 2] — не изменился!

// Правильно:
const combined = arr.concat([3, 4]);

2. Путают с push() при объединении массивов

const arr = [1, 2];
arr.push([3, 4]); // добавляет МАССИВ как элемент
console.log(arr); // [1, 2, [3, 4]]

// concat() разворачивает массивы:
const combined = arr.concat([3, 4]); // [1, 2, 3, 4]

💡 Когда использовать

  • Объединение нескольких массивов в новый
  • Когда нужно сохранить исходные массивы неизменными
  • Альтернатива: spread оператор [...arr1, ...arr2, ...arr3]

slice() — Копирование части массива

slice() - метод копирует часть массива и возвращает новый массив. Не изменяет исходный массив.

Синтаксис:

arr.slice([start], [end])

Параметры:

  • start - индекс начала (включается), по умолчанию 0
  • end - индекс конца (НЕ включается), по умолчанию arr.length

Возвращает:

  • Новый массив с элементами из диапазона [start, end)

Примеры:

const arr = ['a', 'b', 'c', 'd', 'e'];

// Копия всего массива
const copy = arr.slice(); // ['a', 'b', 'c', 'd', 'e']

// С начала до индекса 3 (не включая 3)
const part1 = arr.slice(0, 3); // ['a', 'b', 'c']

// От индекса 2 до конца
const part2 = arr.slice(2); // ['c', 'd', 'e']

// Последние 2 элемента
const last2 = arr.slice(-2); // ['d', 'e']

// Без первого и последнего элемента
const middle = arr.slice(1, -1); // ['b', 'c', 'd']

⚠️ Частые ошибки

1. Путают slice() со splice()

slice() не мутирует, splice() мутирует:

const arr = [1, 2, 3, 4];
arr.slice(1, 3);  // [2, 3] — не изменяет arr
arr.splice(1, 2); // [2, 3] — изменяет arr на [1, 4]

2. Забывают, что end не включается

const arr = [0, 1, 2, 3, 4];
arr.slice(1, 3); // [1, 2] — индекс 3 не включен!

3. Используют отрицательные индексы неправильно

Отрицательный индекс считается с конца, но диапазон остаётся действительным:

const arr = ['a', 'b', 'c', 'd'];
arr.slice(-2); // ['c', 'd'] — последние 2
arr.slice(-3, -1); // ['b', 'c'] — с третьей с конца до второй с конца

💡 Когда использовать

  • Копирование массива: arr.slice()
  • Извлечение части массива без мутации
  • Удаление первых/последних элементов: arr.slice(1), arr.slice(0, -1)
  • Альтернатива: spread оператор для полной копии [...arr]

join() — Объединение в строку

join() - метод объединяет все элементы массива в строку с заданным разделителем.

Синтаксис:

arr.join([separator])

Параметры:

  • separator - строка для разделения элементов, по умолчанию запятая (,)

Возвращает:

  • Строка с объединёнными элементами

Примеры:

const arr = ['Hello', 'World', 'JavaScript'];

// По умолчанию через запятую
console.log(arr.join()); // 'Hello,World,JavaScript'

// Через пробел
console.log(arr.join(' ')); // 'Hello World JavaScript'

// Через дефис
console.log(arr.join('-')); // 'Hello-World-JavaScript'

// Без разделителя
console.log(arr.join('')); // 'HelloWorldJavaScript'

// Числовой массив
const nums = [1, 2, 3, 4, 5];
console.log(nums.join(', ')); // '1, 2, 3, 4, 5'

⚠️ Частые ошибки

1. Забывают преобразовать в строку

join() преобразует элементы в строки, но null и undefined становятся пустыми строками:

const arr = [1, null, 3, undefined, 5];
console.log(arr.join(', ')); // '1, , 3, , 5'

2. Путают с split()

join() — массив в строку, split() — строка в массив:

const arr = ['a', 'b', 'c'];
const str = arr.join('-'); // 'a-b-c'
const back = str.split('-'); // ['a', 'b', 'c']

💡 Когда использовать

  • Вывод списка пользователей: names.join(', ')
  • Создание CSV: row.join(',')
  • Путь к файлу: pathParts.join('/')
  • HTML список: items.join('</li><li>')

indexOf() и lastIndexOf() — Поиск индекса

indexOf() - метод возвращает индекс первого вхождения элемента в массив, или -1 если элемента нет.

Синтаксис:

arr.indexOf(searchElement, [fromIndex])
arr.lastIndexOf(searchElement, [fromIndex])

Параметры:

  • searchElement - элемент для поиска
  • fromIndex - индекс для начала поиска (необязательно)

Возвращает:

  • Индекс элемента или -1 если не найден

Примеры:

const arr = ['a', 'b', 'c', 'b', 'd'];

// Первое вхождение
console.log(arr.indexOf('b')); // 1

// Поиск с начала с позиции 2
console.log(arr.indexOf('b', 2)); // 3

// Элемента нет
console.log(arr.indexOf('z')); // -1

// Последнее вхождение
console.log(arr.lastIndexOf('b')); // 3

// Проверка наличия элемента
if (arr.indexOf('c') !== -1) {
  console.log('Элемент найден');
}

⚠️ Частые ошибки

1. Используют с объектами или массивами

indexOf() использует строгое равенство (===), поэтому с объектами не работает:

const arr = [{id: 1}, {id: 2}];
console.log(arr.indexOf({id: 1})); // -1 — разные объекты!

// Используйте find() для поиска по свойству:
const found = arr.find(item => item.id === 1);

2. Забывают проверить результат

Индекс 0 — валидный результат, нельзя использовать if (index):

const arr = ['a', 'b', 'c'];
const index = arr.indexOf('a'); // 0

// Неправильно:
if (index) { } // 0 — falsy!

// Правильно:
if (index !== -1) { } // Проверяем на -1

💡 Когда использовать

  • Проверка наличия примитива в массиве
  • Поиск индекса для splice: arr.splice(arr.indexOf(item), 1)
  • Для объектов/сложных условий используйте find() или findIndex()

includes() — Проверка наличия

includes() - метод проверяет, содержит ли массив элемент, и возвращает true или false.

Синтаксис:

arr.includes(searchElement, [fromIndex])

Параметры:

  • searchElement - элемент для поиска
  • fromIndex - индекс для начала поиска (необязательно)

Возвращает:

  • true если элемент найден, false если нет

Примеры:

const arr = ['apple', 'banana', 'orange'];

// Проверка наличия
console.log(arr.includes('banana')); // true
console.log(arr.includes('grape')); // false

// С начала с позиции 1
console.log(arr.includes('apple', 1)); // false

// С NaN
const withNaN = [1, 2, NaN, 4];
console.log(withNaN.includes(NaN)); // true — особенность includes!

⚠️ Частые ошибки

1. Путают includes() с indexOf()

includes() возвращает boolean, indexOf() — индекс:

const arr = ['a', 'b', 'c'];

arr.includes('b'); // true
arr.indexOf('b'); // 1

// includes() более читаемо для проверок:
if (arr.includes('b')) { } // лучше, чем
if (arr.indexOf('b') !== -1) { }

2. Используют с объектами

Как и indexOf(), работает только с примитивами:

const arr = [{id: 1}, {id: 2}];
arr.includes({id: 1}); // false — разные объекты

// Используйте some():
arr.some(item => item.id === 1); // true

💡 Когда использовать

  • Проверка наличия в условиях: if (allowedMethods.includes(method))
  • Валидация: validStatuses.includes(user.status)
  • Более читаемо чем indexOf() !== -1

find() и findIndex() — Поиск по условию

find() - метод возвращает первый элемент массива, который удовлетворяет функции-тесту, или undefined если ничего не найдено.

Синтаксис:

arr.find(callback(element, index, array))
arr.findIndex(callback(element, index, array))

Параметры:

  • callback - функция, возвращающая true/false для проверки условия

Возвращает:

  • find() — первый элемент, удовлетворяющий условию, или undefined
  • findIndex() — индекс первого элемента, или -1 если не найден

Примеры:

const users = [
  { id: 1, name: 'Аня', age: 25 },
  { id: 2, name: 'Петя', age: 30 },
  { id: 3, name: 'Виктор', age: 28 }
];

// Найти первого пользователя старше 26
const user = users.find(u => u.age > 26);
console.log(user); // { id: 2, name: 'Петя', age: 30 }

// Найти индекс
const index = users.findIndex(u => u.id === 2);
console.log(index); // 1

// Не найдено
const notFound = users.find(u => u.age > 50);
console.log(notFound); // undefined

⚠️ Частые ошибки

1. Путают find() с filter()

find() возвращает первый элемент, filter() — все подходящие:

const arr = [1, 2, 3, 4, 5];

arr.find(x => x > 2); // 3 — первый элемент
arr.filter(x => x > 2); // [3, 4, 5] — все элементы

2. Используют неправильное условие

Callback должен возвращать boolean:

// Неправильно — return пропущен:
const found = users.find(u => { u.age > 25 });

// Правильно:
const found = users.find(u => u.age > 25);

3. Не обрабатывают undefined

Если ничего не найдено, вернётся undefined:

const found = users.find(u => u.id === 999); // undefined
console.log(found.name); // TypeError!

// Правильно — проверяем:
if (found) {
  console.log(found.name);
}

💡 Когда использовать

  • Поиск объекта по свойству: users.find(u => u.id === id)
  • Найти первый элемент, удовлетворяющий условию
  • Если нужны ВСЕ элементы — используйте filter()

map() — Преобразование элементов

map() - метод преобразует каждый элемент массива с помощью функции и возвращает новый массив.

Синтаксис:

arr.map(callback(element, index, array))

Возвращает:

  • Новый массив с преобразованными элементами

Примеры:

const nums = [1, 2, 3, 4, 5];

// Умножить на 2
const doubled = nums.map(x => x * 2);
console.log(doubled); // [2, 4, 6, 8, 10]

// Преобразовать строки в числа
const strs = ['1', '2', '3'];
const numbers = strs.map(Number);
console.log(numbers); // [1, 2, 3]

// Извлечь свойства из объектов
const users = [
  { name: 'Аня', age: 25 },
  { name: 'Петя', age: 30 }
];
const names = users.map(u => u.name);
console.log(names); // ['Аня', 'Петя']

// С индексом
const indexed = nums.map((x, i) => `${i}: ${x}`);
console.log(indexed); // ['0: 1', '1: 2', ...]

⚠️ Частые ошибки

1. Забывают return в стрелочной функции

// Неправильно:
const result = nums.map(x => { x * 2 }); // [undefined, undefined, ...]

// Правильно:
const result = nums.map(x => x * 2); // [2, 4, 6, ...]

// Или с блоком:
const result = nums.map(x => { return x * 2 });

2. Путают map() с forEach()

map() возвращает новый массив, forEach() не возвращает ничего:

const arr = [1, 2, 3];

const mapped = arr.map(x => x * 2); // [2, 4, 6]
const iterated = arr.forEach(x => x * 2); // undefined

3. Не сохраняют результат

nums.map(x => x * 2); // результат выброшен!

// Правильно:
const doubled = nums.map(x => x * 2);

💡 Когда использовать

  • Преобразование одного массива в другой
  • Извлечение свойств: users.map(u => u.id)
  • Преобразование типов: strs.map(Number)
  • React render списков: items.map(item => <Item key={item.id} {...item} />)

filter() — Фильтрация элементов

filter() - метод фильтрует элементы массива на основе условия и возвращает новый массив с подходящими элементами.

Синтаксис:

arr.filter(callback(element, index, array))

Возвращает:

  • Новый массив с элементами, для которых callback вернул true

Примеры:

const nums = [1, 2, 3, 4, 5];

// Только чётные числа
const evens = nums.filter(x => x % 2 === 0);
console.log(evens); // [2, 4]

// Элементы больше 3
const greater = nums.filter(x => x > 3);
console.log(greater); // [4, 5]

// Удалить null и undefined
const arr = [1, null, 2, undefined, 3];
const clean = arr.filter(x => x != null);
console.log(clean); // [1, 2, 3]

// Фильтр объектов
const users = [
  { name: 'Аня', active: true },
  { name: 'Петя', active: false },
  { name: 'Виктор', active: true }
];
const activeUsers = users.filter(u => u.active);
console.log(activeUsers); // 2 пользователей

⚠️ Частые ошибки

1. Мутируют исходный массив

filter() не мутирует, но легко перепутать с методами вроде splice():

const arr = [1, 2, 3, 4];
const filtered = arr.filter(x => x > 2);

console.log(arr); // [1, 2, 3, 4] — не изменился
console.log(filtered); // [3, 4]

2. Забывают return в callback

// Неправильно:
const filtered = arr.filter(x => { x > 2 }); // пустой массив!

// Правильно:
const filtered = arr.filter(x => x > 2);

3. Используют в условии неправильный результат

const result = arr.filter(x => x > 2);

if (result) { } // всегда true, даже если пуст!

// Правильно:
if (result.length > 0) { }

💡 Когда использовать

  • Фильтрация по условию: items.filter(item => item.active)
  • Удаление элементов: arr.filter(x => x !== toRemove)
  • Поиск по типу: items.filter(item => item instanceof User)
  • Комбинировать с map: arr.filter(...).map(...)

flat() и flatMap() — Выравнивание массивов

flat() - метод выравнивает вложенные массивы на указанную глубину и возвращает новый массив.

Синтаксис:

arr.flat([depth])
arr.flatMap(callback(element, index, array))

Параметры:

  • depth - глубина выравнивания, по умолчанию 1, можно использовать Infinity

Примеры:

// flat() — выравнивание на 1 уровень
const arr1 = [1, [2, 3], [4, [5, 6]]];
console.log(arr1.flat()); // [1, 2, 3, 4, [5, 6]]

// На 2 уровня
console.log(arr1.flat(2)); // [1, 2, 3, 4, 5, 6]

// На любую глубину
console.log(arr1.flat(Infinity)); // [1, 2, 3, 4, 5, 6]

// flatMap() — map + flat(1)
const users = [
  { name: 'Аня', hobbies: ['чтение', 'спорт'] },
  { name: 'Петя', hobbies: ['игры', 'музыка'] }
];
const hobbies = users.flatMap(u => u.hobbies);
console.log(hobbies); // ['чтение', 'спорт', 'игры', 'музыка']

// Эквивалентно:
const hobbies2 = users.map(u => u.hobbies).flat();

⚠️ Частые ошибки

1. Забывают указать глубину

По умолчанию flat() выравнивает только на 1 уровень:

const arr = [1, [2, [3, [4]]]];
arr.flat(); // [1, 2, [3, [4]]] — выравнен 1 уровень!
arr.flat(2); // [1, 2, 3, [4]]
arr.flat(Infinity); // [1, 2, 3, 4]

2. Путают flatMap с map + flat

const arr = [1, 2, 3];

// flatMap использует flat(1):
const result1 = arr.flatMap(x => [x, x * 2]); // [1, 2, 2, 4, 3, 6]

// map().flat(2) и т.д.:
const result2 = arr.map(x => [x, x * 2]).flat(2); // то же самое

3. Используют на очень глубоких структурах

Для очень глубоких вложенных структур используйте рекурсию или специальные функции.

💡 Когда использовать

  • Объединение результатов поиска в подмассивах
  • Выравнивание API ответов с вложенными данными
  • flatMap для одновременного map и flat: arr.flatMap(x => [x.id, x.name])

at() — Доступ с отрицательными индексами

at() - метод возвращает элемент массива по индексу, поддерживая отрицательные индексы для доступа с конца. (ES2022)

Синтаксис:

arr.at(index)

Параметры:

  • index - индекс элемента, может быть отрицательным

Возвращает:

  • Элемент по индексу или undefined если индекс вне границ

Примеры:

const arr = ['a', 'b', 'c', 'd', 'e'];

// Обычный доступ
console.log(arr.at(0)); // 'a'
console.log(arr.at(2)); // 'c'

// Отрицательные индексы — с конца!
console.log(arr.at(-1)); // 'e' — последний элемент
console.log(arr.at(-2)); // 'd' — предпоследний
console.log(arr.at(-5)); // 'a' — первый

// Вне границ
console.log(arr.at(10)); // undefined
console.log(arr.at(-10)); // undefined

⚠️ Важные моменты

1. Это новый метод (ES2022)

Может не поддерживаться в старых браузерах. Проверьте совместимость:

// Альтернатива для старых браузеров:
arr[arr.length - 1]; // вместо arr.at(-1)

2. Удобнее чем старый синтаксис

Сравнение способов получить последний элемент:

// Старый способ:
arr[arr.length - 1];
arr.slice(-1)[0];

// Новый способ (более читаемо):
arr.at(-1);

💡 Когда использовать

  • Последний элемент: arr.at(-1) вместо arr[arr.length - 1]
  • Второй с конца: arr.at(-2)
  • Более читаемый код с отрицательными индексами
  • Поддержка браузеров: проверьте перед использованием
