Свойство overflow в CSS

overflow управляет тем, как содержимое блока ведет себя, если оно выходит за пределы его размеров.
Это важно для создания скроллируемых областей, обрезки лишнего текста, адаптивных карточек и таблиц.

Значения свойства overflow

  • visible — По умолчанию. Содержимое выходит за границы блока, ничего не обрезается.
  • hidden — Лишнее содержимое скрывается/обрезается, скролл не появляется.
  • scroll — Появляется скролл, даже если содержимое влезает полностью.
  • auto — Скролл появляется только когда содержимое не помещается.
  • clip — Обрезает содержимое без скролла (современная замена hidden, без создания scroll container).
  • overflow-x и overflow-y — Управляют переполнением по горизонтали и вертикали отдельно.
Для текста часто используют overflow: hidden + text-overflow: ellipsis + white-space: nowrap для появления троеточия.

Примеры использования overflow

visible
По умолчанию (visible)
Содержимое выходит за рамки блока и видно полностью.
Это длинный текст, который выйдет за границы контейнера и будет виден.

.block {
  overflow: visible; /* default */
}
hidden
Обрезка (hidden)
Всё, что не помещается, скрывается без скролла.
Это длинный текст, который не поместится и будет обрезан.

.block {
  overflow: hidden;
}
scroll
Принудительный скролл
Появится скролл, даже если содержимое влезает.
Это очень длинный текст, который обязательно вызовет появление полосы прокрутки даже если влезает.

.block {
  overflow: scroll;
}
auto
Автоматический скролл
Скролл появится только если не хватает места.
Это длинный текст, который вызовет скролл только если не помещается.

.block {
  overflow: auto;
}
overflow-x/y
Только по одной оси
Скролл или обрезка только горизонтально или вертикально.
Длинная строка для проверки горизонтального скролла или обрезки.

.block {
  overflow-x: auto;
  overflow-y: hidden;
}
clip
clip (обрезка без скролла)
Обрезает лишнее, не делает контейнер скроллируемым (CSS3+).
Это длинный текст, и он просто будет обрезан, не появится даже скролл.

.block {
  overflow: clip;
}
ellipsis
Троеточие для текста
Часто используют для однострочных заголовков и карточек.
Это очень длинный текст для демонстрации работы троеточия.

.block {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}

Советы и лайфхаки по overflow

  • Для скроллируемых областей (таблицы, галереи, меню) сочетайте overflow: auto/scroll и фиксированную высоту/ширину.
  • Для скрытия лишнего текста — overflow: hidden + text-overflow: ellipsis + white-space: nowrap .
  • overflow: clip не создаёт scroll container, и не позволяет прокручивать содержимое.
  • Свойства overflow-x и overflow-y независимы, удобно для горизонтальных/вертикальных списков.
  • Следите за доступностью: скрытый скролл может затруднить навигацию для пользователей.
Совет: Проверяйте overflow на мобильных устройствах — там часто появляются горизонтальные скроллы из-за длинных строк или таблиц!

Свойство visibility

Свойство visibility управляет видимостью элемента на странице, но не влияет на его место в потоке и размеры.
Это удобно, когда нужно временно скрыть элемент, не удаляя его из разметки и не меняя расположение соседей.

  • visible — (по умолчанию) элемент виден.
  • hidden — элемент невидим, но занимает то же пространство на странице.
  • collapse — для таблиц: ячейки исчезают и не занимают место (у display: table и подобных элементов).
.item1
Видимый
visibility: visible
Скрытый

Элемент скрыт visibility: hidden 


.item1 { visibility: visible; }
.item2 { visibility: hidden; }
Отличие от display: none :
visibility: hidden скрывает элемент, но сохраняет его место на странице.
display: none полностью убирает элемент и его пространство из потока.
