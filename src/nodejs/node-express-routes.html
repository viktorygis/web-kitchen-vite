Запросы и маршруты в Express.js

Маршруты — это пути обработки HTTP-запросов, которые определяют, какую функцию вызвать при обращении клиента к конкретному URL.
В Express маршруты являются основой приложения и связывают URL с действиями сервера.

Маршруты состоят из трёх компонентов: метода (GET, POST и т.д.), пути (/users, /products/:id) и обработчика (функция, которая выполняется).

Основы маршрутизации

Маршрут — определение пути и обработчика для этого пути. Синтаксис: app.method(path, handler).
Express сравнивает URL запроса с определенными маршрутами в порядке их объявления.

Простой маршрут:

const express = require('express');
const app = express();

// Основной маршрут
app.get('/', (req, res) => {
  res.send('Добро пожаловать на главную! 🏠');
});

// Маршрут подраздела
app.get('/about', (req, res) => {
  res.send('Это страница О нас 📋');
});

// Маршрут с параметром
app.get('/users/:id', (req, res) => {
  const userId = req.params.id;
  res.send(`Пользователь с ID: ${userId}`);
});

// Запуск сервера на порту 3000
app.listen(3000, () => {
  console.log('Сервер запущен на http://localhost:3000');
});

Как работает маршрут:

  • Метод — тип запроса (get, post, put, delete).
  • Путь — URL-строка или паттерн ('/', '/about', '/users/:id').
  • Обработчик — функция, которая получает объекты запроса req и ответа res.

Порядок имеет значение! Express проверяет маршруты в том порядке, в котором они определены. Если несколько маршрутов совпадают, выполняется первый.

HTTP методы

HTTP методы — это операции, которые клиент просит выполнить на сервере. Каждый метод используется для разных целей: чтение, создание, изменение или удаление данных.

Таблица HTTP методов:

Метод Синтаксис Назначение Пример
GET app.get() Получить данные Загрузить список товаров
POST app.post() Создать новые данные Отправить форму регистрации
PUT app.put() Обновить данные целиком Изменить профиль пользователя
PATCH app.patch() Обновить часть данных Изменить только имя пользователя
DELETE app.delete() Удалить данные Удалить товар из корзины

Примеры каждого метода:

GET — получение данных:
// Получить список всех товаров
app.get('/products', (req, res) => {
  const products = [
    { id: 1, name: 'Ноутбук', price: 50000 },
    { id: 2, name: 'Телефон', price: 20000 }
  ];
  res.json(products);
});
POST — создание новых данных:
// Создать новый товар
app.post('/products', (req, res) => {
  const newProduct = req.body; // Данные из тела запроса
  res.status(201).json({
    message: 'Товар создан успешно! ✅',
    product: newProduct
  });
});
PUT — обновление всего объекта:
// Обновить товар полностью
app.put('/products/:id', (req, res) => {
  const id = req.params.id;
  const updatedData = req.body;
  res.json({
    message: `Товар ${id} обновлен полностью`,
    data: updatedData
  });
});
PATCH — частичное обновление:
// Обновить только цену товара
app.patch('/products/:id', (req, res) => {
  const id = req.params.id;
  const partialUpdate = req.body; // Только поля для обновления
  res.json({
    message: `Товар ${id} частично обновлен`,
    update: partialUpdate
  });
});
DELETE — удаление данных:
// Удалить товар
app.delete('/products/:id', (req, res) => {
  const id = req.params.id;
  res.json({
    message: `Товар ${id} удален ❌`
  });
});

PUT vs PATCH: PUT заменяет весь объект, PATCH обновляет только указанные поля. В REST API PATCH предпочтительнее для экономии трафика.

Параметры запроса

Параметры запроса — это данные, которые клиент передает серверу. Express позволяет получать параметры из разных источников: URL, query-строки и тела запроса.

Типы параметров:

  • req.params — параметры из пути URL (например, /users/:id).
  • req.query — query-параметры из строки адреса (например, ?search=text&sort=asc).
  • req.body — данные из тела запроса (POST, PUT, PATCH).
  • req.headers — заголовки HTTP-запроса.

1. Параметры пути, маршрута (req.params):

Параметры пути — это переменные части URL, обозначаются двоеточием (:paramName).

// Маршрут с параметром id
app.get('/users/:id', (req, res) => {
  const userId = req.params.id;
  res.json({ message: `Пользователь ${userId}`, id: userId });
});

// Маршрут с несколькими параметрами
app.get('/users/:userId/posts/:postId', (req, res) => {
  const { userId, postId } = req.params;
  res.json({
    message: `Пост ${postId} пользователя ${userId}`,
    userId,
    postId
  });
});

Примеры URL:

  • /users/123 → req.params.id = '123'
  • /users/456/posts/789 → req.params.userId = '456', req.params.postId = '789'

2. Query-параметры (req.query):

Query-параметры (параметры строки запроса) — это дополнительные параметры после ? в URL, используются для фильтрации, сортировки.

Они обычно идут после знака ? и разделяются с помощью &. Параметры запроса часто используются для фильтрации данных или для передачи информации от клиента на сервер.

Пример URL с параметром запроса:
http://localhost:3000/search?query=express&page=2

Здесь query и page — это параметры запроса, которые мы можем извлечь на сервере.

Для получения параметров запроса в Express используется объект req.query. 

app.get('/search', (req, res) => {
  const query = req.query.query; // Получаем параметр 'query' из строки запроса
  const page = req.query.page; // Получаем параметр 'page'
  res.send(`Поиск по запросу: ${query}, Страница: ${page}`);
});

Примеры URL:

  • /products?search=phone → req.query.search = 'phone'
  • /products?category=electronics&sort=price → req.query = { category: 'electronics', sort: 'price' }

3. Тело запроса (req.body):

Тело запроса — это данные, передаваемые в POST, PUT, PATCH запросах. Нужно подключить middleware express.json().

middleware - это функции, которые выполняются во время обработки запроса. Они могут изменять объект запроса и ответа, завершать цикл обработки запроса или передавать управление следующему middleware.

const express = require('express');
const app = express();

// Middleware для парсинга JSON
app.use(express.json());

// POST запрос с данными в теле
app.post('/users', (req, res) => {
  const { name, email, age } = req.body;
  res.status(201).json({
    message: 'Пользователь создан',
    user: { name, email, age }
  });
});

app.listen(3000);

Пример запроса:

// Отправка данных через fetch
fetch('http://localhost:3000/users', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    name: 'Иван',
    email: 'ivan@mail.com',
    age: 25
  })
});

// На сервере req.body = { name: 'Иван', email: 'ivan@mail.com', age: 25 }

4. Комбинированный пример:

app.put('/api/users/:id', (req, res) => {
  const id = req.params.id;              // Из пути: /api/users/123
  const { search, sort } = req.query;    // Из URL: ?search=john&sort=asc
  const { name, email } = req.body;      // Из тела запроса

  res.json({
    id,
    search,
    sort,
    name,
    email,
    message: 'Получены все типы параметров'
  });
});

Совет: Используйте req.params для идентификаторов ресурсов, req.query для фильтрации и сортировки, req.body для основных данных.

Пример запроса

Параметры маршрута и запроса позволяют динамически изменять вывод данных в зависимости от того, что передаёт клиент в URL. 

app.get('/products/:category', (req, res) => {
  const category = req.params.category; // Получаем параметр 'category' из URL
  const page = req.query.page || 1; // Получаем параметр 'page' из строки запроса, если он есть
  res.send(`Товары категории: ${category}, Страница: ${page}`);
});

Здесь :category — это параметр маршрута, а page — параметр запроса. В зависимости от того, что передано в URL, сервер будет отправлять ответ с информацией о товаре и странице. Мы можем извлечь с помощью req.query

🚀 Что будет происходить на сервере?

Когда клиент запрашивает путь, например, /products/electronics?page=2, сервер получает electronics как значение параметра маршрута и page=2 как параметр запроса.

Затем сервер отправляет ответ: Товары категории: electronics, Страница: 2

Параметр по умолчанию

Иногда параметры запроса могут быть необязательными. Например, если параметр не передан, мы можем задать значение по умолчанию. 

http://localhost:3000/products?category=electronics&page=3

app.get('/products', (req, res) => {
  const category = req.query.category || 'all'; // Если параметр не передан, по умолчанию будет 'all'
  const page = req.query.page || 1; // Если параметр 'page' не передан, по умолчанию будет 1
  res.send(`Категория: ${category}, Страница: ${page}`);
});

Здесь параметр category обязателен, а page — это необязательный параметр. Если его нет, мы можем задать значение по умолчанию, например, 1.

Если параметр не передан, по умолчанию будет all, а вот если параметр page не передан, по умолчанию будет 1

Ошибка при отсутствии обязательного параметра

Иногда бывает, что необходимые параметры запроса не переданы. В таких случаях мы можем отправить ошибку или сообщение, что параметр обязательный. 

app.get('/products', (req, res) => {
  const category = req.query.category;
  const page = req.query.page || 1;

  if (!category) { // Проверяем, был ли передан параметр 'category'
    return res.status(400).send('Параметр "category" обязателен! 🚨');
  }

  res.send(`Товары категории: ${category}, Страница: ${page}`);
});

Здесь, если параметр category не передан, сервер возвращает ошибку с статусом 400 и сообщением Параметр "category" обязателен! 🚨

Middleware (промежуточное ПО)

Middleware — функции, которые выполняются во время обработки запроса, перед или после основного обработчика маршрута. Они могут изменять объект запроса, ответа или завершать обработку.

Структура middleware:

const middleware = (req, res, next) => {
  // Код выполняется перед маршрутом
  console.log(`Запрос: ${req.method} ${req.url}`);

  // next() передает управление следующему middleware или маршруту
  next();

  // Этот код выполняется после маршрута (если нет res.send())
};

Параметры middleware:

  • req — объект запроса с информацией о клиенте, параметрами и данными.
  • res — объект ответа для отправки данных клиенту.
  • next — функция для передачи управления следующему middleware. Если не вызвать, запрос зависнет.

1. Глобальный middleware (применяется ко всем маршрутам):

// Middleware для логирования ВСЕХ запросов
app.use((req, res, next) => {
  console.log(`⏰ ${new Date().toISOString()}`);
  console.log(`📍 ${req.method} ${req.url}`);
  next(); // Передаем управление дальше
});

// Встроенный middleware для парсинга JSON
app.use(express.json());

// Встроенный middleware для раздачи статических файлов
app.use(express.static('public'));

app.get('/test', (req, res) => {
  res.send('Hello!');
});

2. Middleware для конкретного маршрута:

// Middleware для проверки аутентификации
const checkAuth = (req, res, next) => {
  const token = req.headers.authorization;
  if (token) {
    console.log('✅ Пользователь авторизован');
    next();
  } else {
    res.status(401).json({ error: 'Требуется авторизация' });
  }
};

// Использование middleware для конкретного маршрута
app.get('/dashboard', checkAuth, (req, res) => {
  res.send('Добро пожаловать на панель управления!');
});

3. Цепочка middleware:

// Middleware 1: логирование
const logger = (req, res, next) => {
  console.log('🔍 Логирование:', req.url);
  next();
};

// Middleware 2: проверка токена
const validateToken = (req, res, next) => {
  console.log('🔐 Проверка токена');
  next();
};

// Middleware 3: установка пользователя
const setUser = (req, res, next) => {
  req.user = { id: 1, name: 'Иван' };
  console.log('👤 Пользователь установлен');
  next();
};

// Применить цепочку к маршруту
app.get('/profile', logger, validateToken, setUser, (req, res) => {
  res.json({ user: req.user });
});

4. Встроенные middleware Express:

Middleware Функция Пример
express.json() Парсинг JSON в теле запроса app.use(express.json())
express.static() Раздача статических файлов app.use(express.static('public'))
express.urlencoded() Парсинг данных формы app.use(express.urlencoded({ extended: true }))

Важно! Порядок подключения middleware критичен. Middleware выполняются в том порядке, в котором их подключить. Middleware должны быть подключены ДО маршрутов, которые их используют.

Методы ответа (res)

Методы ответа — это функции объекта res, которые позволяют отправить данные клиенту в разных форматах и с разными статусами.

Основные методы res:

Метод Описание Пример
res.send() Отправить текст или HTML res.send('Hello')
res.json() Отправить JSON res.json({ id: 1 })
res.status() Установить HTTP-статус res.status(404)
res.redirect() Перенаправить на другой URL res.redirect('/home')
res.sendFile() Отправить файл res.sendFile('file.pdf')

1. res.send() — отправка текста/HTML:

app.get('/text', (req, res) => {
  res.send('Это текстовый ответ! 📄');
});

app.get('/html', (req, res) => {
  res.send('<h1>Это HTML-ответ</h1><p>С разметкой!</p>');
});

2. res.json() — отправка JSON:

app.get('/user', (req, res) => {
  const user = {
    id: 1,
    name: 'Иван',
    email: 'ivan@mail.com'
  };
  res.json(user); // Автоматически устанавливает Content-Type: application/json
});

3. res.status() — установка статуса с ответом:

// ✅ Успешный запрос
app.get('/success', (req, res) => {
  res.status(200).json({ message: 'Успешно!' });
});

// ✅ Ресурс создан
app.post('/users', (req, res) => {
  res.status(201).json({ message: 'Пользователь создан', id: 1 });
});

// ❌ Ошибка: ресурс не найден
app.get('/notfound', (req, res) => {
  res.status(404).json({ error: 'Не найдено' });
});

// ❌ Ошибка: сервер
app.get('/error', (req, res) => {
  res.status(500).json({ error: 'Внутренняя ошибка сервера' });
});

4. res.redirect() — перенаправление:

// Перенаправить на другую страницу
app.get('/old-page', (req, res) => {
  res.redirect('/new-page');
});

app.get('/new-page', (req, res) => {
  res.send('Вы на новой странице!');
});

5. Цепирование методов:

// Методы можно цеплять
app.post('/create', (req, res) => {
  res.status(201).json({ message: 'Создано успешно' });
});

// Эквивалентно:
app.post('/create', (req, res) => {
  res.status(201);
  res.json({ message: 'Создано успешно' });
});

Таблица HTTP статусов:

Код Название Значение
200 OK Успешный запрос
201 Created Ресурс создан
204 No Content Успешно, но нет содержимого
400 Bad Request Неверный запрос
401 Unauthorized Требуется авторизация
403 Forbidden Доступ запрещен
404 Not Found Ресурс не найден
500 Server Error Ошибка на сервере

Продвинутая маршрутизация

Продвинутая маршрутизация включает использование паттернов, регулярных выраж��ний, группировки маршрутов и создание модульных роутеров для структурированного кода.

1. Маршруты с регулярными выражениями:

Regex маршруты позволяют более гибко определять пути, используя регулярные выражения.

// Маршрут, совпадающий с /users/123, но не с /users/abc
app.get(/^\/users\/\d+$/, (req, res) => {
  res.send('Это числовой ID пользователя');
});

// Маршрут, совпадающий с /files/ и расширениями .pdf, .doc
app.get(/\/files\/(.*\.(pdf|doc))$/, (req, res) => {
  const file = req.params[0];
  res.send(`Запрошен файл: ${file}`);
});

2. Параметры маршрута с ограничениями:

// Параметр только для цифр
app.get('/posts/:id(\\d+)', (req, res) => {
  res.json({ postId: req.params.id });
});

// Это совпадет: /posts/123
// Это НЕ совпадет: /posts/abc

3. Группировка маршрутов (Router):

Router — это мини-приложение для группировки связанных маршрутов. Позволяет организовать код в отдельные модули.

// users.js — модуль с маршрутами пользователей
const express = require('express');
const router = express.Router();

// Все маршруты в этом файле начинаются с /users
router.get('/', (req, res) => {
  res.json({ message: 'Список пользователей' });
});

router.get('/:id', (req, res) => {
  res.json({ message: `Пользователь ${req.params.id}` });
});

router.post('/', (req, res) => {
  res.status(201).json({ message: 'Пользователь создан' });
});

module.exports = router;
// app.js — главное приложение
const express = require('express');
const usersRouter = require('./routes/users');
const postsRouter = require('./routes/posts');

const app = express();

// Подключить маршруты
app.use('/api/users', usersRouter);    // /api/users, /api/users/:id
app.use('/api/posts', postsRouter);    // /api/posts, /api/posts/:id

app.listen(3000);

4. Обработка 404 (маршруты не найдены):

Обработчик 404 должен быть подключен ДО других маршрутов, чтобы ловить несоответствующие URL.

// Все маршруты приложения выше...

// Обработчик 404 — ловит все оставшиеся маршруты
app.use((req, res) => {
  res.status(404).json({
    error: 'Страница не найдена',
    url: req.url,
    method: req.method
  });
});

5. Обработка ошибок (error middleware):

Error middleware имеет 4 параметра и ловит ошибки из других middleware.

// Обработчик ошибок — ВСЕГДА 4 параметра
app.use((err, req, res, next) => {
  console.error('Ошибка:', err.message);
  res.status(err.status || 500).json({
    error: err.message || 'Внутренняя ошибка сервера',
    status: err.status || 500
  });
});

// Передача ошибки в error middleware
app.get('/danger', (req, res, next) => {
  const error = new Error('Что-то пошло не так!');
  error.status = 400;
  next(error); // Передаем ошибку в error middleware
});

Порядок подключения Router: Router должны быть подключены до обработчиков 404 и error, но после основных middleware.

Типичные ошибки

Типичные ошибки при работе с маршрутами и запросами — это распространённые проблемы, которые замораживают запрос, теряют данные или вызывают неопределённое поведение.

❌ Ошибка 1: забывчивый next():

// ❌ НЕПРАВИЛЬНО — запрос зависнет
app.use((req, res, next) => {
  console.log('Логирование');
  // Забыли вызвать next() — запрос никогда не дойдет до маршрута
});

app.get('/', (req, res) => {
  res.send('Hello'); // Это никогда не выполнится
});

// ✅ ПРАВИЛЬНО
app.use((req, res, next) => {
  console.log('Логирование');
  next(); // Передаем управление дальше
});

❌ Ошибка 2: отправка ответа дважды:

// ❌ НЕПРАВИЛЬНО — "Cannot set headers after they are sent"
app.get('/', (req, res) => {
  res.json({ message: 'Первый ответ' });
  res.json({ message: 'Второй ответ' }); // Ошибка!
});

// ✅ ПРАВИЛЬНО
app.get('/', (req, res) => {
  const response = { message: 'Один ответ' };
  res.json(response); // Отправить один раз
});

❌ Ошибка 3: порядок middleware имеет значение:

// ❌ НЕПРАВИЛЬНО — маршруты не найдут middleware
app.get('/', (req, res) => {
  res.json(req.body); // req.body еще не парсена!
});

app.use(express.json()); // Middleware подключен ТО ЖЕ

// ✅ ПРАВИЛЬНО
app.use(express.json()); // Middleware ПЕРВЫМ

app.get('/', (req, res) => {
  res.json(req.body); // Теперь req.body доступна
});

❌ Ошибка 4: неправильный порядок маршрутов:

// ❌ НЕПРАВИЛЬНО — специфичный маршрут не выполнится
app.get('/:id', (req, res) => {
  res.send('Это обработчик :id');
});

app.get('/profile', (req, res) => {
  res.send('Это профиль'); // Никогда не выполнится (совпал с /:id)
});

// ✅ ПРАВИЛЬНО — специфичные маршруты ПЕРЕД общими
app.get('/profile', (req, res) => {
  res.send('Это профиль');
});

app.get('/:id', (req, res) => {
  res.send('Это обработчик :id');
});

❌ Ошибка 5: забывчивый req.body парсинг:

// ❌ НЕПРАВИЛЬНО — req.body будет undefined
app.post('/user', (req, res) => {
  console.log(req.body); // undefined!
  res.json(req.body);
});

// ✅ ПРАВИЛЬНО
app.use(express.json()); // Подключить middleware для парсинга JSON

app.post('/user', (req, res) => {
  console.log(req.body); // Теперь есть данные
  res.json(req.body);
});

⚠️ Важные моменты:

  • Всегда вызывайте next() в middleware, если нужно передать управление дальше.
  • Не отправляйте ответ дважды — используйте if/else для разных случаев.
  • Подключайте middleware ДО маршрутов, которые их используют.
  • Специфичные маршруты пишите ДО общих параметрических маршрутов.

Лучшие практики

Лучшие практики — это проверенные подходы, которые делают код чище, безопаснее и легче для поддержки.

✅ 1. Организуйте маршруты в отдельные файлы:

// routes/users.js
const express = require('express');
const router = express.Router();

router.get('/', (req, res) => {
  res.json({ users: [] });
});

module.exports = router;

// app.js
app.use('/api/users', require('./routes/users'));

✅ 2. Используйте middleware для валидации:

// Middleware для проверки данных
const validateUser = (req, res, next) => {
  const { name, email } = req.body;
  if (!name || !email) {
    return res.status(400).json({ error: 'Name и email обязательны' });
  }
  next();
};

app.post('/users', validateUser, (req, res) => {
  res.status(201).json({ user: req.body });
});

✅ 3. Обрабатывайте ошибки корректно:

// Используйте try-catch для асинхронного кода
app.get('/data', async (req, res, next) => {
  try {
    const data = await fetchData();
    res.json(data);
  } catch (err) {
    next(err); // Передаем ошибку в error middleware
  }
});

✅ 4. Используйте правильные HTTP статусы:

// GET — успех
app.get('/users', (req, res) => {
  res.status(200).json(users); // 200 OK
});

// POST — создано
app.post('/users', (req, res) => {
  res.status(201).json(newUser); // 201 Created
});

// DELETE — успех без содержимого
app.delete('/users/:id', (req, res) => {
  res.status(204).send(); // 204 No Content
});

// Ошибка клиента
app.get('/notfound', (req, res) => {
  res.status(404).json({ error: 'Not found' });
});

// Ошибка сервера
app.get('/error', (req, res) => {
  res.status(500).json({ error: 'Server error' });
});

✅ 5. Логируйте запросы:

// Middleware для логирования
app.use((req, res, next) => {
  const start = Date.now();

  res.on('finish', () => {
    const duration = Date.now() - start;
    console.log(`${req.method} ${req.url} ${res.statusCode} (${duration}ms)`);
  });

  next();
});

✅ 6. Группируйте связанные маршруты:

// ❌ Плохо — все маршруты в одном файле
app.get('/users', ...);
app.post('/users', ...);
app.get('/posts', ...);
app.post('/posts', ...);

// ✅ Хорошо — отдельные роутеры
app.use('/api/users', userRouter);
app.use('/api/posts', postRouter);
app.use('/api/comments', commentRouter);

✅ 7. Используйте константы для путей:

// constants/routes.js
const ROUTES = {
  USERS: '/api/users',
  POSTS: '/api/posts',
  HOME: '/'
};

// app.js
app.get(ROUTES.HOME, (req, res) => {
  res.send('Home');
});

app.use(ROUTES.USERS, userRouter);

Итоги

  • Маршруты связывают URL с обработчиками через метод, путь и функцию.
  • HTTP методы: GET (читать), POST (создать), PUT (обновить), PATCH (частичное обновление), DELETE (удалить).
  • req.params — параметры из пути URL, req.query — query-параметры, req.body — данные из тела.
  • Middleware — функции, выполняющиеся перед маршрутом. Важен порядок и вызов next().
  • res методы: res.send(), res.json(), res.status(), res.redirect().
  • Router позволяет группировать связанные маршруты в отдельные модули.
  • Порядок имеет значение — middleware перед маршрутами, специфичные маршруты перед общими параметрическими.
  • Обработка ошибок — используйте error middleware с 4 параметрами и правильные HTTP статусы.
