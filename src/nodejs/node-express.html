ExpressJS - фреймворк для Node.js

ExpressJS — это минималистичный и гибкий фреймворк для создания веб-приложений на Node.js.
Он упрощает создание HTTP-серверов, обработку запросов и ответов, управление маршрутами и многое другое.
Фреймворк — это готовый набор инструментов и правил, которые помогают быстрее разрабатывать приложения.

ExpressJS — это фреймворк для создания серверов на Node.js. ,а вот что делает его таким крутым:
◽ Простой и быстрый — Express позволяет за несколько строк кода настроить сервер. ⏱️
◽ Гибкость — ты можешь настроить сервер, как тебе нужно, добавлять обработчики для различных типов запросов и многое другое. 🎨
◽ Множество готовых решений — у Express есть большая экосистема, где ты можешь найти решения для множества задач. 🔧

Что такое ExpressJS

ExpressJS — это популярный фреймворк для создания веб-приложений и API на Node.js. Он позволяет быстро создавать серверы, обрабатывать HTTP-запросы, управлять маршрутами и строить сложные приложения.

Основные возможности:

  • Маршрутизация — определение путей обработки запросов.
  • Middleware — функции для обработки запросов.
  • Шаблонизация — использование шаблонов для рендеринга HTML.
  • Статические файлы — раздача CSS, JS, изображений.
  • Обработка ошибок — централизованная обработка ошибок.

ExpressJS используется в тысячах приложений по всему миру — от стартапов до крупных компаний. Это один из самых популярных фреймворков для Node.js.

Установка ExpressJS

Для работы с ExpressJS вам нужно установить Node.js и npm (менеджер пакетов).

Шаг 1: Создайте папку проекта

mkdir my-express-app
cd my-express-app

Шаг 2: Инициализируйте проект

Открой командную строку и перейди в папку, где хочешь создать проект. Напиши команду:

npm init -y

Эта команда создаст файл package.json с базовой конфигурацией проекта.

Шаг 3: Установите ExpressJS

Теперь, когда проект готов, установим Express с помощью команды:

npm install express

Эта команда установит Express и добавит его в зависимости нашего проекта.

После установки ExpressJS будет доступен в папке node_modules и указан в файле package.json.

Шаг 4: Создание простого сервера

Шаг 5: Сохрани файл, запустите сервер командой

node app.js

Создание простого сервера

Давайте создадим простой HTTP-сервер с помощью ExpressJS. 

// Импортируем ExpressJS
const express = require('express');

// Создаём приложение
const app = express();

// Определяем порт
const PORT = 3000;

// Обработчик для главной страницы
app.get('/', (req, res) => {
  res.send('Добро пожаловать на главную страницу!');
});

// Запускаем сервер
app.listen(PORT, () => {
  console.log(`Сервер запущен на http://localhost:${PORT}`);
});
Как это работает?
  • require('express') — импортируем модуль ExpressJS.
  • express() — создаём приложение ExpressJS.
  • app.get() — определяем обработчик для GET-запроса.
  • res.send() — отправляем ответ клиенту.
  • app.listen() — запускаем сервер на указанном порту.

Сохраните этот код в файл app.js и запустите командой node app.js. Затем откройте браузер и перейдите на http://localhost:3000.

Маршруты

Маршруты — это пути, по которым приложение обрабатывает запросы. Каждый маршрут имеет путь (URL) и обработчик (функцию).
Маршруты в ExpressJS позволяют нам обрабатывать запросы, поступающие на сервер, и решать, что с ними делать.

Каждый маршрут обрабатывает определённый путь и тип запроса. Мы можем создавать маршруты для разных страниц или API.

Примеры маршрутов:

app.get('/', (req, res) => {
  res.send('Привет, мир! 🌍');
});

app.get('/') — это маршрут для обработки GET-запросов на главную страницу / (корень сайта). Когда кто-то посещает твой сайт, например, по адресу http://localhost:3000 (если сервер работает на порту 3000), Express будет "слушать" этот запрос и вызовет функцию, которая стоит в этом маршруте.
После мы передаём функцию, которая принимает два параметра: объект req (запрос) и объект res (ответ). В этой функции мы отправляем ответ через res.send()

Пример:

В нашем примере мы отправляем текст "Это наш проект на ExpressJS! 🚀" на страницы "О нас" /about. и возвращаем информацию о проекте. 

app.get('/about', (req, res) => {
  res.send('Это наш проект на ExpressJS! 🚀');
});

HTTP методы

ExpressJS поддерживает различные HTTP методы для обработки разных типов запросов.

GET - получение данных

app.get('/products', (req, res) => {
  res.json({ products: ['Товар 1', 'Товар 2', 'Товар 3'] });
});

POST - создание данных

app.post('/products', (req, res) => {
  // req.body содержит данные из тела запроса
  const newProduct = req.body;
  res.json({ message: 'Товар создан', product: newProduct });
});

PUT - обновление данных

app.put('/products/:id', (req, res) => {
  const productId = req.params.id;
  res.json({ message: `Товар ${productId} обновлен` });
});

DELETE - удаление данных

app.delete('/products/:id', (req, res) => {
  const productId = req.params.id;
  res.json({ message: `Товар ${productId} удален` });
});

Для обработки данных из тела запроса используйте middleware express.json(): app.use(express.json());

Middleware

Middleware(перевод: промежуточное ПО) — это функции, которые выполняются во время обработки запроса. Они могут изменять объект запроса req, объект ответа res или завершать обработку запроса.

Middleware может использоваться для логирования, проверки аутентификации, парсинга данных и многого другого.

Пример простого middleware:

const express = require('express');
const app = express();

// Middleware для логирования
app.use((req, res, next) => {
  console.log(`Запрос: ${req.method} ${req.url}`);
  next(); // Передаём управление дальше
});

// Встроенный middleware для парсинга JSON
app.use(express.json());

// Middleware для статических файлов
app.use(express.static('public'));

app.get('/', (req, res) => {
  res.send('Привет!');
});

app.listen(3000);
Параметры middleware:
  • req — объект запроса с информацией о клиенте.
    req (request - запрос) — это объект, который содержит информацию о запросе, который пришёл на сервер. Он включает в себя данные о пути запроса, параметрах, заголовках и теле запроса.
  • res — объект ответа для отправки данных клиенту.
    res (response - ответ) — это объект, который используется для отправки ответа клиенту. С его помощью можно отправлять текст, HTML, JSON и другие данные.
  • next — функция для передачи управления следующему middleware.
    next() (следующий) — это функция, которая вызывается внутри middleware, чтобы передать управление следующему middleware в цепочке. Если не вызвать next(), запрос "зависнет" и не будет обработан дальше.

Пример маршрута с параметром req:

Здесь req.params.name позволяет нам получить параметр name, который передан в URL. Например, если запрос был сделан на /greet/Дима, то req.params.name будет равно "Дима"

app.get('/greet/:name', (req, res) => {
  const name = req.params.name; // Получаем параметр из URL
  res.send(`Привет, ${name}! 👋`); // Отправляем ответ с использованием параметра
});

Пример маршрута с параметром res:

А вот объект res в Express используется для отправки ответа клиенту.
Мы можем отправить данные в различных форматах, например, текст, JSON, HTML и т. д. Объект res предоставляет различные методы для отправки этих данных, такие как res.send(), res.json(), res.status() и другие.

app.get('/user', (req, res) => {
  const user = {
    name: 'Дмитрий',
    age: 25
  };
  res.json(user); // Отправляем объект user в формате JSON
});

Тут мы отправлям JSON-данные с помощью метода res.json()

Методы объекта res:

Это методы объекта res, которые позволяют отправлять ответы клиенту в различных форматах и с различными статусами.

res.send() - отправляет текстовый ответ клиенту.
res.json() - отправляет JSON-ответ клиенту.
res.status() - устанавливает HTTP-статус ответа.

Порядок подключения middleware важен! Обработчики маршрутов выполняются в том порядке, в котором они определены.

Отправка данных

В ExpressJS мы можем отправлять данные клиенту в различных форматах, таких как текст, HTML, JSON и другие.

  • Текст — с помощью res.send()
  • JSON — с помощью res.json()
  • Статус — с помощью res.status()

Пример отправки текста:

app.get('/text', (req, res) => {
  res.send('Это текстовый ответ! 📄');
});

Пример отправки JSON:

app.get('/json', (req, res) => {
  const data = { message: 'Это JSON-ответ! 📦' };
  res.json(data);
});

Пример отправки статуса :

Например, если нам нужно вернуть ошибку, мы можем установить статус 404 и отправить текстовое сообщение

app.get('/notfound', (req, res) => {
  res.status(404).send('Страница не найдена! 😕');
});

Метод res.send() автоматически определяет тип данных и устанавливает соответствующий заголовок Content-Type. Метод res.json() устанавливает заголовок Content-Type: application/json.

Обработка запросов

Обработка запросов — это процесс получения данных от клиента и отправки ответа обратно. В ExpressJS мы можем обрабатывать различные типы запросов, такие как GET, POST, PUT, DELETE и другие. Каждый тип запроса имеет свои особенности и используется для разных целей.

params (перевод "параметры") - объект, содержащий параметры маршрута из URL. Например, если маршрут определен как /greet/:name, то req.params.name будет содержать значение параметра name.

Пример обработки GET-запроса:

app.get('/greet/:name', (req, res) => {
  const name = req.params.name; // Получаем параметр 'name' из URL
  res.send(`Привет, ${name}! 👋`); // Отправляем персонализированное сообщение
});

Здесь мы используем req.params.name для получения значения параметра из URL и отправляем персонализированное сообщение.

Пример обработки POST-запроса:

app.post('/data', (req, res) => {
  res.send('Это POST-запрос! 📤');
});

Пример обработки PUT-запроса:

app.put('/data', (req, res) => {
  res.send('Это PUT-запрос! 🔄');
});
