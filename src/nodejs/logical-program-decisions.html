Управление логикой приложения

Как программа принимает решения

Логика приложения — это набор правил, по которым программа принимает решения: что выполнить, когда выполнить и какой результат вернуть пользователю.

Любая программа состоит не только из хранения данных, но и из проверки условий:

  • Если пользователь авторизован — показать личный кабинет.
  • Если пароль неверный — вернуть ошибку.
  • Если товар есть в наличии — разрешить покупку.
  • Если данные некорректные — остановить выполнение.

Условия if / else

Условие if помогает программе делать выбор между несколькими вариантами, проверяя, верно ли какое-то утверждение. 

let age = 20;

if (age >= 18) {
  console.log("Ты взрослый!");
}
  • Оно выполняется, если условие внутри if истинно true
  • В круглых скобках ( ) пишем условие, которое должно быть проверено. Например, x > 5, age === 18 и т. д.
  • В фигурных скобках{ } (или без них, если код один) пишем то, что нужно выполнить, если условие выполнится.

const score = 75;

if(score >= 90){
 console.log("Отлично");
}
else if(score >= 60){
 console.log("Хорошо");
}
else{
 console.log("Нужно повторить");
}

Конструкция switch

Конструкция switch позволяет проверять значение переменной и выполнять разные действия в зависимости от этого значения.

const fruit = "apple";

switch (fruit) {
  case "apple":
    console.log("Яблоко");
    break;
  case "banana":
    console.log("Банан");
    break;
  default:
    console.log("Неизвестный фрукт");
}

Логические операторы

Логические операторы позволяют объединять несколько условий в одно. Они помогают программе принимать решения на основе нескольких факторов.

Оператор Описание Пример
&& Логическое И (AND) user.age >= 18 && user.active
|| Логическое ИЛИ (OR) if(user.age >= 18 || user.active){ console.log("Можно войти"); }
! Логическое НЕ (NOT) if(!isBlocked){ console.log("Пользователь активен"); }

Функции и логика

Функции помогают организовать код и сделать его более читаемым. Логику программы можно разделить на функции, каждая из которых выполняет определённую задачу.

function checkAge(age) {
  if (age >= 18) {
    return "Ты взрослый!";
  } else {
    return "Ты ещё ребёнок!";
  }
}

Проверка данных

Проверка данных — это процесс проверки входных данных на корректность перед их использованием в программе. Это помогает избежать ошибок и обеспечить безопасность приложения.

function validateEmail(email) {
  const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;

  return emailRegex.test(email);
}

Бизнес-логика

Бизнес-логика — это набор правил и процессов, которые определяют, как работает приложение в контексте конкретного бизнеса или задачи. Она включает в себя все условия, проверки и действия, которые необходимо выполнить для достижения целей приложения.

function calculateDiscount(price, userType) {
  if (userType === "premium") {
    return price * 0.8; // 20% скидка для премиум
  } else if (userType === "regular") {
    return price * 0.9; // 10% скидка для обычных пользователей
  } else {
    return price; // без скидки
  }
}

Логика в middleware

Middleware — это функции, которые выполняются между запросом и ответом в веб-приложении. Они могут использоваться для проверки условий, аутентификации, логирования и других задач.

function authMiddleware(req, res, next) {
  if (req.user) {
    next(); // пользователь авторизован, продолжаем
  } else {
    return res.status(401).send("Не авторизован");
  }
}

Обработка ошибок

Обработка ошибок — это процесс выявления и управления ошибками, которые могут возникнуть во время выполнения программы. Это помогает предотвратить сбои и улучшить пользовательский опыт.

const jsonString = '{"name":"Иван"}';

try {
  const data = JSON.parse(jsonString);
  console.log(data);
} catch (error) {
  console.error("Ошибка при разборе JSON:", error);
}

Практические примеры

Примеры показывают, как применять условия, логические операторы и функции в реальных сценариях.

function canAccessContent(user) {
  if (user.age >= 18 && user.subscription === "active") {
    return true; // доступ разрешен
  } else {
    return false; // доступ запрещен
  }
}
