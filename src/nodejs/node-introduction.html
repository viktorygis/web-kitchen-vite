Введение в Node.js

Node.js — это среда выполнения JavaScript, которая позволяет запускать код вне браузера.

Возможности Node.js

  • Создание серверов.
  • Разработка REST API.
  • Работа с базами данных.
  • Создание CLI-приложений.
  • Автоматизация задач.
  • Backend-разработка.

Для чего это нужно🤷‍♂️

Когда ты хочешь создать сервер, который будет работать 24/7, тебе нужен инструмент, который может обрабатывать запросы, управлять файлами, работать с пользователями — вот для этого и используется Node.js! 🎯

В отличие от обычного JavaScript, который работает в браузере, Node.js позволяет запускать твой код на сервере, чтобы ты мог создавать приложения, которые взаимодействуют с базами данных, обрабатывают запросы и многое другое

Как работает Node.js

Node.js использует движок V8 для выполнения JavaScript и асинхронную модель обработки событий.
V8 - это движок, разработанный Google для выполнения JavaScript в браузере Chrome. Node.js использует его для выполнения кода на сервере.
Асинхронная модель позволяет Node.js обрабатывать множество запросов одновременно, не блокируя выполнение кода.

Архитектура Node.js

  • V8 — движок выполнения JavaScript.
  • Event Loop — механизм обработки событий.
  • Libuv — библиотека для асинхронных операций.
  • Node API — встроенные возможности платформы.

Установка Node.js

Установка Node.js - Перейди на сайт Node.js и скачай последнюю стабильную версию (LTS). ◽ Установи Node.js, следуя инструкциям на экране.

Проверка установки

После установки проверьте версии Node.js и NPM. Открой терминал и введи команду:

node -v — проверка версии Node.js.
npm -v — проверка версии NPM.

Если всё установлено правильно, ты увидишь версию Node.js (например, v14.17.6).

Запуск JavaScript в Node.js

Запуск JavaScript в Node.js осуществляется через командную строку. Создайте файл app.js с простым кодом:

// app.js
console.log("Hello Node.js");

Запуск выполняется командой:

node app.js

Модули Node.js

Модули — это отдельная часть кода, которая выполняет определённую задачу. Модули помогают разделить код на логические блоки, каждый из которых можно использовать по отдельности.

Модули - это файлы JavaScript, которые можно импортировать и использовать в других файлах. Node.js поддерживает два типа модулей: CommonJS и ES Modules.

CommonJS

CommonJS - это стандарт модулей, используемый в Node.js. Модули экспортируются с помощью module.exports и импортируются с помощью команды require().

const http = require('http');

//где const http - это переменная, в которую мы сохраняем модуль http,
//который мы импортируем с помощью require('http').
//require - это функция, которая позволяет импортировать модули в Node.js.
//Она принимает имя модуля в качестве аргумента и возвращает объект,
//который представляет этот модуль.
//'http' - это имя модуля, который мы хотим импортировать.
//В данном случае мы импортируем встроенный модуль http,
//который позволяет создавать HTTP-серверы и обрабатывать HTTP-запросы и ответы.

Ты говоришь Node.js, что хочешь подключить модуль http, который уже есть в стандартной библиотеке Node.
После этого ты получаешь доступ ко всем функциям, которые этот модуль предоставляет. Это как если бы ты открыл ящик с инструментами и достал нужный тебе инструмент для работы с HTTP.

// math.js
function sum(a, b) {
  return a + b;
}

module.exports = sum;

ES Modules

ES Modules - это современный стандарт модулей, который поддерживается в Node.js. Модули экспортируются с помощью export и импортируются с помощью import.

export function sum(a, b) {
  return a + b;
}

Использование ES Modules

Для использования ES Modules в Node.js необходимо указать тип модуля в файле package.json:

{
  "type": "module"
}

Пример 

//main.js
import { sum } from './math.js';
console.log(sum(2, 3)); // 5

//math.js
export function sum(a, b) {
  return a + b;
}

NPM — менеджер пакетов

NPM (Node Package Manager) — это менеджер пакетов для Node.js, который позволяет устанавливать и управлять библиотеками и зависимостями.
Библиотеки - это готовые решения, которые можно использовать в своих проектах.
Зависимости - это библиотеки, от которых зависит ваш проект. То есть, если ваш проект использует какую-либо библиотеку, она становится зависимостью вашего проекта.

Пример использования NPM

Для начала работы с NPM необходимо инициализировать проект и установить необходимые пакеты. Например, для установки фреймворка Express:

//Инициализация проекта
npm init

//Установка Express
npm install express

Создание HTTP-сервера

Для того чтобы запустить сервер в Node.js, мы используем встроенный модуль http. Этот модуль помогает создавать серверы, которые могут принимать запросы от пользователей и отправлять ответы.

Мы создаём сервер с помощью метода createServer() и говорим ему, чтобы он слушал на определённом порту (например, на порту 3000).

Вот пример минимального сервера, который отвечает "Hello, World!" на все запросы:

const http = require('http');  // Подключаем модуль http

// Создаём сервер, который будет отвечать на запросы
http.createServer((req, res) => {
  res.writeHead(200, {'Content-Type': 'text/plain'});  // Устанавливаем статус ответа и тип данных
  res.end('Hello, World!\n');  // Отправляем ответ
}).listen(3000, 'localhost');  // Сервер слушает на порту 3000

Итоги

  • Node.js позволяет запускать JavaScript вне браузера.
  • Основой платформы является движок V8.
  • Модули помогают делить код на части.
  • NPM используется для управления зависимостями.
  • Node.js подходит для серверов, API и автоматизации.
