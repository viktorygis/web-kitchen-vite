Синхронность и асинхронность в JavaScript

Синхронный код выполняется последовательно — каждая строка ждёт завершения предыдущей.
Асинхронный код позволяет продолжить выполнение, не ожидая результата долгой операции.

Пример синхронного кода

console.log('Первая задача');
console.log('Вторая задача');
console.log('Третья задача');
//Вывод будет: Первая задача, Вторая задача, Третья задача

Пример асинхронного кода

console.log('Первая задача');
setTimeout(() => {
  console.log('Вторая задача (асинхронно)');
}, 1000);
console.log('Третья задача');
//Вывод будет: Первая задача, Третья задача, Вторая задача (через 1 секунду)

В JavaScript асинхронность критична для сетевых запросов, работы с файлами и задержек. Без неё пользовательский интерфейс замерзал бы на каждый HTTP-запрос.

Где используется асинхронность

  • Сетевые запросы (fetch, axios).
  • Работа с файловой системой (Node.js).
  • Таймеры и задержки (setTimeout, setInterval).
  • Операции с базами данных.
  • Параллельные вычисления.

Основные понятия

Синхронность и асинхронность — два способа организации выполнения кода в зависимости от времени ожидания результатов операций.

  • Синхронный код — выполняется блокирующим образом, строки выполняются одна за другой.
  • Асинхронный код — не блокирует выполнение, результат приходит позже.
  • Callback (колбэк) — функция, вызываемая после завершения асинхронной операции.
  • Promise (промис) — объект, представляющий будущее значение или ошибку.
  • Async/Await (асинхронные функции) — современный синтаксис для работы с асинхронным кодом.
  • Event Loop (событийный цикл) — механизм, который управляет выполнением асинхронного кода.

Синхронный код

Синхронный код выполняется последовательно. Каждая строка ждёт завершения предыдущей.

Пример синхронного кода

console.log("Начало");

function add(a, b) {
  return a + b;
}

const result = add(2, 3); // Выполняется сразу
console.log(result); // 5

console.log("Конец");

Вывод:

Начало
5
Конец

Проблема: блокирующие операции

Проблема: если операция занимает время (сетевой запрос, чтение файла), весь код замерзает в ожидании. 

console.log("Запрос к серверу...");

// Эта функция имитирует долгую операцию
function slowOperation() {
  let sum = 0;
  for (let i = 0; i < 1000000000; i++) {
    sum += i; // Много вычислений, занимает время
  }
  return sum;
}

const data = slowOperation(); // Код здесь ждёт!
console.log("Данные получены:", data);

// Эта строка выполнится только после slowOperation
console.log("Продолжение работы");

Проблема: пока выполняется slowOperation, ничего больше не может выполниться. Интерфейс замерзнет.

Асинхронный код

Асинхронный код выполняется не по очереди, а продолжает работать, не ожидая завершения других операций.

Как работает асинхронность

Допустим, одна из задач занимает время, но программа не будет ждать её завершения. Тут у нас у второй задачи стоит таймер на 2 секунды.

console.log('Задача 1: Начинаем');
setTimeout(() => {
  console.log('Задача 2: Асинхронная операция');
}, 2000);
console.log('Задача 3: Всё идёт параллельно');
Вывод будет такой:
Задача 1: Начинаем
Задача 3: Всё идёт параллельно
Задача 2: Асинхронная операция
Пояснение:
  • Мы сразу выводим "Задача 1".
  • Потом ставим операцию с задержкой в 2 секунды.
  • Параллельно выполняется "Задача 3".

Преимущества асинхронности

  • Код не блокируется, интерфейс остаётся отзывчивым.
  • Можно выполнять несколько операций одновременно.
  • Лучше использование ресурсов сервера.

setTimeout()

setTimeout() — встроенная функция JavaScript, которая позволяет выполнить код через определённое время (в миллисекундах).

Например, если ты хочешь выполнить какое-то действие через 3 секунды, ты используешь setTimeout().

setTimeout(() => {
  console.log('Прошло 3 секунды');
}, 3000);

В этом примере сообщение появится через 3 секунды. Таймаут не блокирует остальной код.

setTimeout() внутри цикла

setTimeout() откладывает выполнение функции, и если запускать его внутри цикла, все функции сработают почти одновременно после одинаковой задержки.

for (let i = 0; i < 3; i++) {
  setTimeout(() => {
    console.log(`Задача № ${i}`);
  }, 1000);
}

clearTimeout()

clearTimeout() — встроенная функция JavaScript, которая отменяет выполнение кода, запланированного с помощью setTimeout().

Для этого нужно передать идентификатор таймаута в clearTimeout().

const timeoutId = setTimeout(() => {
  console.log('Прошло 3 секунды');
}, 3000);

clearTimeout(timeoutId);

Что произойдёт, если использовать clearTimeout() до завершения таймаута?
Запланированная функция не будет выполнена.

setInterval()

setInterval() — встроенная функция JavaScript, которая выполняет код через определённые интервалы времени и повторяет его до отмены. Она отличается от setTimeout(), который выполняет код один раз после задержки [web:17][web:20].

Остановка через setTimeout

const intervalId = setInterval(() => {
  console.log('Прошло 1 секунда');
}, 1000);

setTimeout(() => {
  clearInterval(intervalId);
  console.log('Интервал остановлен');
}, 5000);

Остановка через clearInterval()

const intervalId = setInterval(() => {
  console.log('Каждую секунду!');
}, 1000);

clearInterval(intervalId);

Если нужен повторяющийся запуск, используют setInterval(), а для остановки — clearInterval() [web:2][web:20].

Сравнение методов

Сравнение setTimeout, clearTimeout и setInterval — функций для работы с таймерами и интервалами.

Функция Назначение Повторяется Отмена Когда использовать
setTimeout Выполнить код через время ❌ Нет, один раз ✅ clearTimeout(id) Задержки, отложенные действия, запросы после паузы
clearTimeout Отменить setTimeout — — Отмена таймера до его выполнения
setInterval Выполнить код повторно ✅ Да, каждый интервал ✅ clearInterval(id) Часы, счётчики, периодические обновления, опросы сервера

Практическое сравнение

setTimeout — один раз:

setTimeout(() => console.log("Выполнится один раз"), 1000);

setInterval — повторно:

setInterval(() => console.log("Выполнится каждую секунду"), 1000);

setTimeout с рекурсией (альтернатива setInterval):

function repeatEverySecond() {
  console.log("Выполнится каждую секунду");

  // Запланировать следующее выполнение
  setTimeout(repeatEverySecond, 1000);
}

repeatEverySecond();

// Это более гибко, чем setInterval (можно менять интервал)

Лучшие практики

Лучшие практики при работе с таймерами и интервалами в JavaScript.

  • Всегда сохраняйте ID: чтобы иметь возможность отменить с clearTimeout/clearInterval.
  • Очищайте интервалы: забытые setInterval могут привести к утечкам памяти.
  • Проверяйте условия выхода: используйте if/break для остановки интервалов.
  • Избегайте вложенных таймеров: это усложняет отладку и может привести к ошибкам.
  • Используйте requestAnimationFrame: для анимаций вместо setTimeout/setInterval.
  • Передавайте функции: не вызывайте их сразу (неправильно: callback(), правильно: callback).
  • Очищайте при размонтировании: в React/Vue удаляйте интервалы в useEffect cleanup.

Пример: правильное использование в React

import { useEffect, useState } from 'react';

function Timer() {
  const [seconds, setSeconds] = useState(0);

  useEffect(() => {
    const intervalId = setInterval(() => {
      setSeconds(prev => prev + 1);
    }, 1000);

    // Cleanup: очищаем интервал при размонтировании
    return () => clearInterval(intervalId);
  }, []);

  return 
Прошло {seconds} секунд
;
}

Пример: отмена всех таймеров

const timers = [];

function addTimer(callback, delay) {
  const id = setTimeout(callback, delay);
  timers.push(id);
  return id;
}

function clearAllTimers() {
  timers.forEach(id => clearTimeout(id));
  timers.length = 0;
  console.log("Все таймеры отменены");
}

// Использование
addTimer(() => console.log("Таймер 1"), 1000);
addTimer(() => console.log("Таймер 2"), 2000);
addTimer(() => console.log("Таймер 3"), 3000);

// Отменить все
clearAllTimers();

Типичные ошибки

Ошибка 1: Вызов функции вместо передачи

// ❌ Неправильно: функция выполняется сразу
setTimeout(console.log("Привет"), 1000);

// ✅ Правильно: передаём функцию
setTimeout(() => console.log("Привет"), 1000);
setTimeout(console.log, 1000, "Привет");

Ошибка 2: Забыли сохранить ID

// ❌ Неправильно: не сможем отменить
setTimeout(() => console.log("Таймер"), 1000);

// ✅ Правильно: сохраняем ID
const timerId = setTimeout(() => console.log("Таймер"), 1000);
// Теперь можем отменить: clearTimeout(timerId);

Ошибка 3: Утечка памяти от setInterval

// ❌ Неправильно: интервал никогда не останавливается
setInterval(() => {
  console.log("Выполняется бесконечно");
}, 1000);

// ✅ Правильно: останавливаем интервал
const intervalId = setInterval(() => {
  console.log("Выполняется пока не отменим");
}, 1000);

// Когда нужно остановить:
clearInterval(intervalId);

Ошибка 4: Неправильная задержка

// ❌ Неправильно: задержка в секундах (но нужны миллисекунды)
setTimeout(() => console.log("Через 5 секунд"), 5); // Это 5ms!

// ✅ Правильно: 5000 миллисекунд = 5 секунд
setTimeout(() => console.log("Через 5 секунд"), 5000);

Частая ошибка: забыть очистить setInterval. Это может привести к утечкам памяти и множественным одновременным выполнениям.
Решение: всегда сохраняйте ID и вызывайте clearInterval при необходимости.

Итоги

  • Синхронный код выполняется блокирующим образом, строка за строкой.
  • Асинхронный код не блокирует выполнение, результат приходит позже.
  • Event Loop управляет выполнением синхронного и асинхронного кода.
  • setTimeout выполняет код один раз через определённую задержку.
  • clearTimeout отменяет ранее запланированный setTimeout.
  • setInterval выполняет код повторно через равные интервалы.
  • Всегда сохраняйте ID таймеров для их отмены.
  • Задержки указываются в миллисекундах, не секундах.
  • Очищайте интервалы, чтобы избежать утечек памяти.
  • setTimeout с задержкой 0 не значит "сразу" — это значит "когда Call Stack пуст".
