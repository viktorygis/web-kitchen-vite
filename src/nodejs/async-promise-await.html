async/await и Promise в Node.js

Promise — объект JavaScript, который представляет результат асинхронной операции.
async/await — удобный синтаксис для работы с Promise, позволяющий писать асинхронный код так, будто он выполняется последовательно.

В Node.js многие операции выполняются асинхронно: чтение файлов, запросы к базе данных, сетевые обращения и работа с API. Promise и async/await помогают управлять такими операциями без большого количества вложенных callback-функций.

Promise в JavaScript

Promise — специальный объект, который хранит состояние будущего результата асинхронной операции.

Promise — это как «обещание» выполнения какой-то асинхронной операции, которое в будущем либо выполнится успешно, либо с ошибкой. Когда операция завершена, Promise либо resolve (успех), либо reject (ошибка).

Пример создания Promise:

// Создание Promise
let promise = new Promise((resolve, reject) => {
  let success = true; // Сделаем успех
  if (success) {
    resolve("Операция успешна!");
  } else {
    reject("Произошла ошибка");
  }
});

Здесь мы создаём Promise с помощью конструктора Promise и проверяем переменную success. Если она true, Promise завершается успешно resolve. Если нет — с ошибкой reject.

Состояния Promise:

  • pending — операция выполняется.
  • fulfilled — операция успешно завершена.
  • rejected — произошла ошибка.

Создание Promise

Конструктор Promise — способ создать собственную асинхронную операцию. 

const userPromise = new Promise((resolve, reject) => {
  const success = true;

  if (success) {
    resolve("Пользователь найден");
  } else {
    reject("Ошибка загрузки");
  }
});

userPromise
  .then(result => console.log(result))
  .catch(error => console.log(error));

Методы Promise

  • then() — выполняется после успешного результата.
  • catch() — обрабатывает ошибки.
  • finally() — выполняется всегда после завершения.

Цепочка Promise:

fetchUser()
  .then(user => fetchPosts(user.id))
  .then(posts => console.log(posts))
  .catch(error => console.error(error))
  .finally(() => console.log("Готово"));

async/await

async/await — синтаксический способ работы с Promise, который делает асинхронный код более читаемым.
Это синтаксический сахар, который упрощает работу с асинхронным кодом.
async используется перед функцией, чтобы сделать её асинхронной. А await заставляет функцию "подождать", пока Promise не выполнится. 

async function example() {
  let result = await promise;
  console.log(result);
}

Здесь await ждёт, пока Promise не выполнится, и только потом продолжает выполнение кода.

  • async — делает функцию асинхронной и возвращает Promise.
  • await — ожидает завершение Promise внутри async-функции.

Базовый пример async/await

async function testAsync() {
  let result = await new Promise((resolve) => {
    setTimeout(() => resolve("Операция завершена!"), 1000); // Заполните время ожидания
  });
  console.log(result);
}

Последовательное выполнение

Последовательное выполнение — следующая операция начинается после завершения предыдущей. 

async function loadData() {
  const user = await getUser();
  const posts = await getPosts(user.id);

  console.log(posts);
}

loadData();

Параллельное выполнение Promise

Promise.all() — запускает несколько Promise одновременно и ждёт завершения всех операций. 

async function loadAll() {
  const [users, posts] = await Promise.all([
    getUsers(),
    getPosts()
  ]);

  console.log(users, posts);
}

loadAll();

async/await в Node.js

Асинхронные API Node.js часто возвращают Promise, поэтому async/await является основным способом работы с ними.

Чтение файла через fs/promises

const fs = require("fs/promises");

async function readFile() {
  const text = await fs.readFile(
    "data.txt",
    "utf8"
  );

  console.log(text);
}

readFile();

Обработка ошибок

try...catch — способ перехватывать ошибки внутри async-функций.

  • try — в этом блоке пишешь код, который может вызвать ошибку.
  • catch — в этом блоке обрабатываешь ошибку, если она возникла в блоке try
async function example() {
  try {
    let result = await someAsyncFunction();
    console.log(result);
  } catch (error) {
    console.log("Произошла ошибка:", error);
  }
}

Здесь try пытается выполнить код, а catch перехватывает ошибку, если она возникла.

Частая ошибка: Не обрабатывать ошибки Promise.
Решение: всегда использовать catch() или try...catch вместе с async/await.

async/await и Promise

async/await и Promise работают вместе, чтобы упрощать асинхронные операции.

  • Promise выполняется первым и возвращает результат.
  • await останавливает выполнение функции до завершения Promise
async function example() {
  let result = await new Promise((resolve) => {
    setTimeout(() => resolve("Задача выполнена!"), 1000);
  });
  console.log(result);
}

В этом примере await ждёт результат от Promise, а выполнение функции продолжается только после его получения.

Что происходит:
  • Создаётся Promise, который выполняется через 1 секунду.
  • await останавливает выполнение функции до завершения Promise.
  • После завершения Promise результат выводится в консоль.
Если не использовать await

Если не использовать await, Promise будет создан, но выполнение функции не будет ждать его завершения. В результате консоль может вывести Promise вместо его результата.

Если не использовать Promise

Если не использовать Promise, await будет ожидать обычное значение, и асинхронность теряется. В результате код может работать синхронно, что не всегда соответствует ожиданиям.

Если не использовать await и Promise

Если не использовать await и Promise, код будет выполняться синхронно, что может привести к блокировке выполнения и потере асинхронности. В результате приложение может стать медленным и неэффективным.

Promise.all()

Promise.all() — метод, который позволяет запускать несколько Promise одновременно и ждать их завершения. 

async function parallelTasks() {
let result = await Promise.all([
new Promise ((resolve) => setTimeout(resolve, 1000)),
new Promise ((resolve) => setTimeout(resolve, 2000)),
]);
  console.log("Все задачи завершены!");
}

В этом примере Promise.all() запускает две задачи параллельно. Первая задача завершится через 1 секунду, а вторая — через 2 секунды. После завершения обеих задач выводится сообщение..

Лучшие практики async/await

  • Используйте async/await вместо длинных цепочек then().
  • Всегда обрабатывайте возможные ошибки.
  • Используйте Promise.all() для независимых операций.
  • Не добавляйте лишний await без необходимости.

Итоги

  • Promise управляет результатом асинхронной операции.
  • async/await упрощает работу с Promise.
  • await можно использовать только внутри async-функции.
  • try...catch помогает безопасно обрабатывать ошибки.
