HTTP-запросы и ответы

HTTP-запросы и ответы — механизм обмена данными между клиентом (браузером или приложением) и сервером по протоколу HTTP.
Каждый запрос инициируется клиентом и завершается ответом сервера с данными или кодом ошибки.

HTTP лежит в основе работы большинства сайтов и веб-приложений: страницы, изображения, JSON-данные, формы и авторизация — всё это передаётся через HTTP.

Браузер отправляет запрос на сервер, чтобы получить нужные данные. Запросы бывают разных типов: GET — получить данные, POST — отправить данные на сервер.

Пример: простой запрос через fetch

fetch() — встроенный браузерный метод для выполнения HTTP-запросов.
Возвращает Promise, который разрешается в объект Response. 

fetch('https://jsonplaceholder.typicode.com/todos/1') // Запрос к серверу
  .then(response => response.json()) // Преобразуем ответ в JSON
  .then(data => console.log(data));  // Выводим данные в консоль

Что такое HTTP

HTTP (HyperText Transfer Protocol) — протокол прикладного уровня, описывающий правила обмена сообщениями между клиентом и сервером в сети.
Клиент всегда инициирует соединение; сервер отвечает данными или сообщением об ошибке.

Когда пользователь открывает страницу, браузер отправляет HTTP-запрос, а сервер возвращает HTTP-ответ: HTML, JSON, изображение, файл или код ошибки.

Где используется HTTP

  • Загрузка страниц сайта.
  • Получение данных из API.
  • Отправка форм и комментариев.
  • Авторизация и работа с сессиями.

Запрос и ответ

HTTP-обмен — процесс, состоящий из двух частей: запроса (request) от клиента и ответа (response) от сервера.
Каждая часть содержит метаданные (заголовки) и при необходимости тело с данными.

Структура HTTP-запроса

  • Метод — действие: получить, создать, изменить или удалить данные (GET, POST, PUT, DELETE).
  • URL — адрес ресурса на сервере.
  • Заголовки — дополнительная информация: тип контента, авторизация, кэш.
  • Тело запроса — данные, отправляемые на сервер (только для POST, PUT и др.).

Структура HTTP-ответа

  • Статус — числовой код результата: 200 (ОК), 404 (не найдено), 500 (ошибка сервера).
  • Заголовки — тип контента, кэширование, CORS и другая служебная информация.
  • Тело ответа — данные, которые сервер возвращает клиенту.

Чаще всего данные возвращаются в формате JSON — компактного текстового формата, нативно поддерживаемого JavaScript.

Код статуса Значение
200 Успех — данные возвращены
201 Ресурс создан
400 Некорректный запрос
401 Не авторизован
404 Ресурс не найден
500 Внутренняя ошибка сервера

Fetch API

Fetch API — встроенный браузерный интерфейс для выполнения HTTP-запросов без сторонних библиотек.
Метод fetch() принимает URL и параметры запроса, возвращает Promise с объектом Response. 

fetch('https://api.example.com/data')
  .then(response => {
    if (!response.ok) {
      throw new Error('Ошибка сети: ' + response.status);
    }
    return response.json(); // Преобразуем тело ответа в объект
  })
  .then(data => {
    console.log(data);
  })
  .catch(error => console.error('Ошибка:', error));

Сначала мы получаем объект Response, проверяем статус, затем преобразуем тело в JSON и работаем с данными.

fetch() поддерживает любой HTTP-метод, произвольные заголовки и тело запроса.

Важно: fetch() не выбрасывает ошибку при статусах 4xx/5xx. Всегда проверяйте response.ok вручную.

Как обрабатывать ответ сервера

Ответ сервера — объект Response, который возвращает fetch() после завершения запроса.
Чтобы получить данные из ответа, его нужно преобразовать — чаще всего методом .json().

Когда ты отправляешь HTTP-запрос с помощью fetch(), сервер отвечает на него. Ответ чаще всего приходит в формате JSON — его удобно читать и использовать в JavaScript. Важно научиться правильно обрабатывать этот ответ. 

// Отправляем запрос на сервер
fetch('https://jsonplaceholder.typicode.com/todos/1')
  .then(response => response.json()) // Преобразуем ответ в формат JSON
  .then(data => console.log(data));  // Выводим данные на экран

Что происходит в этом коде — шаг за шагом:

  • Шаг 1. fetch() отправляет запрос на сервер по указанному URL.
  • Шаг 2. Когда сервер вернёт ответ, первый .then() получает объект Response и вызывает .json() — это превращает ответ сервера в JavaScript-объект.
  • Шаг 3. Второй .then(data => console.log(data)) получает готовый объект и выводит его в консоль.

Разбор ответа на практическом примере

Практический разбор — пошаговый анализ реального JSON-ответа от сервера на конкретном примере.
Помогает понять, что именно возвращает сервер и как с этим работать в коде.

Возьмём реальный запрос к публичному API jsonplaceholder.typicode.com. Когда мы отправляем запрос на todos/1, сервер возвращает JSON-объект с данными о задании. 

fetch("https://jsonplaceholder.typicode.com/todos/1") // Отправляем запрос
  .then(response => response.json()) // Преобразуем ответ в JSON
  .then(data => console.log(data));  // Выводим результат

После выполнения этого кода в консоли появится следующий объект:

{
  userId: 1,
  id: 1,
  title: "Упражнение 5",
  completed: false
}

Это и есть данные, которые сервер вернул в ответ на наш запрос. Теперь с ними можно работать как с обычным JavaScript-объектом.

Свойства объекта ответа

Свойства объекта — именованные поля внутри JSON-ответа, каждое из которых хранит конкретное значение.
После преобразования ответа через .json() к ним можно обращаться через точку: data.title.

В консоли будет объект с четырьмя свойствами. Разберём каждое:

Свойство Тип Описание
userId number ID пользователя, которому принадлежит задание.
id number Уникальный ID самого задания.
title string Название задания — текстовое описание.
completed boolean Статус выполнения: true — выполнено, false — нет.

Необязательно выводить весь объект целиком. Можно обратиться к нужному свойству через точку и вывести только его: 

fetch('https://jsonplaceholder.typicode.com/todos/1')
  .then(response => response.json())
  .then(data => console.log(data.title)); // Выведет только название задания

В этом примере мы обращаемся только к свойству title и выводим его — всё остальное содержимое объекта игнорируется.

Обработка ошибок с .catch()

Обработка ошибок — механизм перехвата сбоев при выполнении запроса: недоступность сервера, проблемы с сетью или неверный URL.
Метод .catch() ловит любую ошибку, возникшую в цепочке .then().

Когда мы отправляем запрос на сервер, не всегда всё может пройти гладко. Сервер может не ответить или вернуть ошибку. Важно обрабатывать такие ситуации, чтобы приложение не «падало». 

fetch('https://jsonplaceholder.typicode.com/todos/1')
  .then(response => response.json())       // Преобразуем ответ в JSON
  .then(data => console.log(data))         // Если всё прошло хорошо — выводим данные
  .catch(error => console.log('Ошибка при запросе:', error)); // Ловим ошибку

Параметр error в методе .catch() — это объект с информацией о том, что пошло не так. Всегда указывайте его, чтобы вывести понятное сообщение пользователю или записать в лог.

Важно: .catch() перехватывает только сетевые ошибки (нет соединения, неверный URL). Ответы с кодами 4xx и 5xx ошибкой не считаются — для них нужно проверять response.ok вручную. 
// Правильно: проверяем статус + ловим сетевую ошибку
fetch('https://jsonplaceholder.typicode.com/todos/1')
  .then(response => {
    if (!response.ok) {
      throw new Error('Сервер вернул ошибку: ' + response.status);
    }
    return response.json();
  })
  .then(data => console.log(data))
  .catch(error => console.log('Ошибка:', error));

Параметры запроса (Query String)

Параметры запроса — дополнительные данные, которые передаются серверу прямо в URL после знака ?.
Позволяют фильтровать, сортировать или постранично загружать данные без изменения самого маршрута.

Иногда нам нужно отправить данные вместе с запросом, чтобы сервер выполнил нужную операцию. Параметры выглядят как ?ключ=значение, а если их несколько — соединяются через &.

Формат параметров в URL

  • Один параметр: /todos?userId=1
  • Несколько параметров: /todos?userId=1&completed=false
  • Удобный способ строить URL — через URLSearchParams.
// Запрос только заданий пользователя с userId=1
fetch('https://jsonplaceholder.typicode.com/todos?userId=1')
  .then(response => response.json())
  .then(data => console.log(data)); // Массив заданий этого пользователя

Для автоматического и безопасного формирования параметров используйте URLSearchParams — он корректно экранирует спецсимволы. 

// Строим URL с параметрами через URLSearchParams
const params = new URLSearchParams({ userId: 1, completed: false });
const url = 'https://jsonplaceholder.typicode.com/todos?' + params;

fetch(url)
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.log('Ошибка:', error));

Async/Await

async/await — синтаксический сахар над Promise, делающий асинхронный код линейным и читаемым.
async объявляет функцию асинхронной, await приостанавливает её выполнение до разрешения Promise. 

async function loadData() {
  try {
    const response = await fetch('https://api.example.com/data');

    if (!response.ok) {
      throw new Error('Ошибка: ' + response.status);
    }

    const data = await response.json();
    console.log(data);
  } catch (error) {
    // Перехватываем ошибки сети и статуса
    console.error('Ошибка:', error);
  }
}

loadData();

Такой подход удобен для последовательных запросов, сложной логики и обработки ошибок через стандартный try/catch.

Когда использовать async/await вместо .then()

  • Когда запросов несколько и они зависят друг от друга.
  • Когда нужно обрабатывать ошибки в одном блоке catch.
  • Когда код должен быть легко читаем и поддерживаем.

GET-запросы

GET — HTTP-метод для получения данных с сервера без их изменения.
Параметры передаются в URL, тело запроса отсутствует. Это самый распространённый метод.

GET применяют для загрузки списков, карточек, новостей, товаров и любых других ресурсов, которые нужно только прочитать. 

async function loadUsers() {
  const response = await fetch('https://api.example.com/users'); // метод GET по умолчанию

  if (!response.ok) {
    throw new Error('Не удалось загрузить пользователей');
  }

  const users = await response.json();
  console.log(users);
}

loadUsers();

GET-запросы идемпотентны — повторный запрос к тому же URL возвращает те же данные и не изменяет состояние сервера.

POST-запросы

POST — HTTP-метод для отправки данных на сервер с целью создания или обновления ресурса.
Данные передаются в теле запроса (body), а не в URL.

POST подходит для отправки форм, регистрации, создания комментариев, заявок и любых операций, меняющих состояние сервера. 

async function sendForm() {
  const response = await fetch('https://api.example.com/forms', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json' // Указываем формат тела
    },
    body: JSON.stringify({               // Преобразуем объект в JSON-строку
      name: 'Ivan',
      message: 'Hello HTTP'
    })
  });

  if (!response.ok) {
    throw new Error('Ошибка при отправке формы');
  }

  const result = await response.json();
  console.log(result);
}

sendForm();

Перед отправкой данные преобразуются в строку через JSON.stringify(). Сервер разбирает тело по заголовку Content-Type: application/json.

Метод Назначение Тело запроса
GET Получить данные Нет
POST Создать ресурс Да
PUT Заменить ресурс Да
PATCH Частично обновить Да
DELETE Удалить ресурс Нет / Да

Итоги

Итоги раздела — краткое резюме ключевых понятий и инструментов для работы с HTTP в JavaScript.
Используйте как шпаргалку при реализации запросов в проекте.

  • HTTP — протокол обмена данными между клиентом и сервером.
  • HTTP-запрос содержит метод, URL, заголовки и иногда тело.
  • HTTP-ответ содержит статус-код, заголовки и тело с данными.
  • fetch() — браузерный метод для отправки HTTP-запросов, возвращает Promise.
  • Ответ обрабатывается через .then(response => response.json()) — результат это JavaScript-объект.
  • К отдельным свойствам ответа обращаются через точку: data.title, data.id.
  • .catch() перехватывает сетевые ошибки; статусы 4xx/5xx проверяйте через response.ok.
  • Параметры запроса передаются в URL: ?userId=1&completed=false. Для построения — URLSearchParams.
  • Всегда проверяйте response.ok — fetch() не выбрасывает ошибку при 4xx/5xx.
  • async/await делает асинхронный код линейным и удобным для поддержки.
  • GET — читать данные, параметры в URL. POST — отправить данные, тело в body.
