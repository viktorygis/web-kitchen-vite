Взаимодействие и работа с API

Клиент-серверная архитектура — это способ, с помощью которого приложение делит обязанности между клиентом (пользовательским интерфейсом) и сервером (серверной частью)
Клиент — это устройство или программа (например, браузер), которая отправляет запросы на сервер.
Сервер — это компьютер, который обрабатывает запросы от клиента и отправляет ответы.

API (Application Programming Interface) — это интерфейс для взаимодействия между клиентом и сервером.
Он позволяет клиенту отправлять запросы на сервер и получать ответы, обычно в формате JSON.

Как работают запросы

Клиент-серверная архитектура работает следующим образом:

  • Клиент отправляет HTTP-запрос серверу. Это может быть запрос на страницу, данные или ресурсы.
  • Сервер обрабатывает этот запрос и посылает HTTP-ответ обратно клиенту, обычно в виде веб-страницы или данных (например, JSON).

Пример:

Предположим, вы хотите получить страницу с новостями. Браузер (клиент) отправляет запрос на сервер:

// Простой HTTP-запрос с использованием fetch (клиентский запрос)
fetch('https://example.com/news')
  .then(response => response.json())
  .then(data => console.log(data)) // Сервер вернёт данные о новостях
  .catch(error => console.error('Ошибка:', error));
Что здесь происходит?
  • fetch отправляет запрос на сервер по указанному URL.
  • Сервер обрабатывает запрос и отправляет ответ в виде данных (например, новостей в формате JSON).
  • data — это ответ от сервера, который будет отображён в консоли.

Что такое API

API Application Programming Interface — это набор инструментов, который позволяет одному приложению взаимодействовать с другим. Когда клиент (например, браузер) хочет получить данные или выполнить операцию на сервере, он может отправить запрос через API.

Пример простого запроса:

Когда ты заходишь на сайт, браузер отправляет запрос на сервер через API, и сервер возвращает данные (например, новости). Все это происходит в несколько секунд! 

// Запрос к API для получения данных
fetch('https://api.example.com/news')
  .then(response => response.json())
  .then(data => {
    console.log('Новости:', data); // сервер вернёт список новостей
  })
  .catch(error => console.error('Ошибка:', error));
Как это работает?
  • Мы используем метод fetch, чтобы отправить запрос на сервер.
  • Сервер, получив запрос, возвращает данные о новостях в формате JSON.
  • Ответ от сервера обрабатывается и выводится в консоль.

API позволяет приложениям обмениваться данными и функциональностью. Например, когда ты используешь карту на сайте, она может получать данные о местоположении через API картографического сервиса.

Как работают запросы

Обычно работа с API строится по схеме: клиент отправляет запрос, сервер его обрабатывает и возвращает ответ.

Основные части запроса

  • URL — адрес, куда отправляется запрос.
  • Метод — что именно нужно сделать: получить, создать, изменить или удалить данные.
  • Заголовки — дополнительная информация о запросе.
  • Тело запроса — данные, которые отправляются на сервер.

Самый частый формат ответа от API — JSON. Его удобно читать в JavaScript и использовать в интерфейсе.

Fetch API

Для работы с API в браузере часто используют встроенный метод fetch().

Он отправляет запрос по указанному адресу и возвращает Promise. 

fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => {
    console.log(data);
  });

Сначала мы получаем ответ, затем преобразуем его в JSON и только после этого работаем с данными.

Async/Await

Async/await делает работу с асинхронным кодом более удобной и читаемой.

Вместо цепочек .then() можно писать код почти как обычный последовательный скрипт. 

async function loadData() {
  const response = await fetch('https://api.example.com/data');
  const data = await response.json();

  console.log(data);
}

Такой подход особенно удобен, когда нужно выполнять несколько запросов, обрабатывать ошибки или строить более сложную логику.

GET-запросы

GET используют, когда нужно получить данные с сервера.

Это самый распространённый метод для загрузки списков, карточек, новостей, товаров и других данных. 

async function loadUsers() {
  const response = await fetch('https://api.example.com/users', {
    method: 'GET'
  });

  const users = await response.json();
  console.log(users);
}

GET-запросы обычно не передают данные в теле запроса, а используют параметры в URL.

POST-запросы

POST используют, когда нужно отправить данные на сервер.

Это подходит для форм, комментариев, заявок, регистрации и создания новых записей. 

async function sendForm() {
  const response = await fetch('https://api.example.com/forms', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      name: 'Ivan',
      message: 'Hello API'
    })
  });

  const result = await response.json();
  console.log(result);
}

При POST-запросе данные обычно передаются в body, а перед отправкой преобразуются в JSON с помощью JSON.stringify().

Итоги

  • API позволяет приложениям обмениваться данными.
  • Запросы обычно отправляются с помощью fetch().
  • Ответ от сервера часто приходит в формате JSON.
  • async/await делает асинхронный код удобнее.
  • GET используют для получения данных, POST — для отправки данных.
