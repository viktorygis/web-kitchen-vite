HTTP-запросы и ответы

HTTP — это протокол, по которому браузер, приложение или другой клиент запрашивает данные у сервера и получает ответ.
Каждый запрос начинается с клиента и завершается ответом сервера.

HTTP лежит в основе работы большинства сайтов и веб-приложений: страницы, изображения, JSON-данные, формы и авторизация — всё это обычно передаётся через HTTP.

Рис. 1. Клиент-серверная архитектура

Когда мы работаем в интернете, наш браузер (или любое другое приложение) отправляет запросы на сервер, чтобы получить нужную информацию. Эти запросы называются HTTP-запросами. Они могут быть разного типа, например GET (для получения данных) или POST (для отправки данных на сервер).

Пример использования fetch

fetch('https://jsonplaceholder.typicode.com/todos/1') // Это запрос, который отправляется на сервер
  .then(response => response.json()) // Сервер возвращает ответ в формате JSON
  .then(data => console.log(data)); // Мы выводим данные на экран

Что такое HTTP

HTTP расшифровывается как HyperText Transfer Protocol. Он описывает, как клиент и сервер обмениваются сообщениями в сети.

Когда пользователь открывает страницу, браузер отправляет HTTP-запрос, а сервер возвращает HTTP-ответ: HTML, JSON, изображение, файл или сообщение об ошибке.

Где используется HTTP

  • Загрузка страниц сайта.
  • Получение данных из API.
  • Отправка форм и комментариев.
  • Авторизация и работа с сессиями.

Запрос и ответ

HTTP-обмен состоит из двух основных частей: request и response.

HTTP-запрос

  • Метод — что нужно сделать: получить, создать, изменить или удалить данные.
  • URL — адрес ресурса.
  • Заголовки — дополнительная информация о запросе.
  • Тело запроса — данные, которые отправляются на сервер.

HTTP-ответ

  • Статус — результат обработки запроса, например 200, 404 или 500.
  • Заголовки — тип контента, кэширование и другая служебная информация.
  • Тело ответа — данные, которые сервер возвращает клиенту.

Чаще всего данные возвращаются в формате JSON, потому что его удобно обрабатывать в JavaScript.

Fetch API

В браузере для отправки HTTP-запросов часто используют встроенный метод fetch(). Он принимает адрес ресурса и возвращает Promise, который разрешается в объект Response. 

fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => {
    console.log(data);
  })
  .catch(error => console.error('Ошибка:', error));

Сначала мы получаем ответ от сервера, затем преобразуем его в JSON и только после этого работаем с данными.

`fetch()` полезен не только для получения данных, но и для отправки запросов с разными методами и заголовками.

Async/Await

async/await делает асинхронный код более читаемым и похожим на обычный последовательный код. 

async function loadData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();

    console.log(data);
  } catch (error) {
    console.error('Ошибка:', error);
  }
}

Такой подход удобно использовать, когда нужно обрабатывать ошибки, выполнять несколько запросов или строить сложную логику загрузки данных.

GET-запросы

GET используют, когда нужно получить данные с сервера.

Это самый распространённый метод для загрузки списков, карточек, новостей, товаров и других ресурсов. 

async function loadUsers() {
  const response = await fetch('https://api.example.com/users', {
    method: 'GET'
  });

  const users = await response.json();
  console.log(users);
}

GET-запросы обычно не содержат тело запроса, а параметры передаются в URL.

POST-запросы

POST используют, когда нужно отправить данные на сервер.

Этот метод подходит для форм, комментариев, заявок, регистрации и создания новых записей. 

async function sendForm() {
  const response = await fetch('https://api.example.com/forms', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      name: 'Ivan',
      message: 'Hello HTTP'
    })
  });

  const result = await response.json();
  console.log(result);
}

При POST-запросе данные обычно передаются в body, а перед отправкой преобразуются в JSON с помощью JSON.stringify().

Итоги

  • HTTP — это протокол обмена данными между клиентом и сервером.
  • HTTP-запрос содержит метод, URL, заголовки и иногда тело.
  • HTTP-ответ содержит статус, заголовки и тело ответа.
  • fetch() используется для отправки HTTP-запросов в браузере.
  • async/await делает работу с запросами удобнее и понятнее.
  • GET применяют для получения данных, POST — для отправки данных.
