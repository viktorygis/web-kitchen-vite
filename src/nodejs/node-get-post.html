GET и POST запросы в JavaScript

HTTP-методы — способы указать серверу, что именно нужно сделать с ресурсом: получить, создать, обновить или удалить.
Самые используемые из них — GET (получить данные) и POST (отправить данные).

В браузере HTTP-запросы отправляются через встроенный метод fetch(). Он возвращает Promise, который разрешается в объект Response с данными от сервера.

Зачем нужны GET и POST

  • GET — загрузить список товаров, получить профиль пользователя, запросить данные о погоде.
  • POST — отправить форму регистрации, создать комментарий, добавить заказ в базу.
  • Правильный выбор метода делает API понятным и предсказуемым.

Основные понятия

Ключевые термины — без понимания этих понятий сложно разобраться в том, как работают запросы.
Каждый термин — отдельная часть механизма HTTP-обмена.

  • fetch() — встроенная браузерная функция для отправки HTTP-запросов. Возвращает Promise.
  • Request (запрос) — сообщение от клиента серверу. Содержит метод, URL, заголовки и иногда тело.
  • Response (ответ) — сообщение от сервера клиенту. Содержит статус-код, заголовки и тело с данными.
  • response.ok — булево свойство, которое равно true при статусах 200–299 и false при 4xx/5xx.
  • response.json() — метод, который преобразует тело ответа из JSON-строки в JavaScript-объект.
  • JSON.stringify() — обратная операция: превращает JavaScript-объект в JSON-строку для отправки в теле запроса.
  • URLSearchParams — встроенный класс для безопасного формирования параметров в URL.

GET-запросы

GET — HTTP-метод для получения данных с сервера без их изменения.
Это метод по умолчанию в fetch() — его не нужно указывать явно.

GET применяют для загрузки любых данных: списков пользователей, карточек товаров, статей, погоды. Параметры фильтрации и поиска передаются прямо в URL.

Простейший GET-запрос:

// GET — метод по умолчанию, указывать необязательно
fetch('https://jsonplaceholder.typicode.com/users/1')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('Ошибка:', error));

GET с явным указанием метода:

fetch('https://jsonplaceholder.typicode.com/users/1', {
  method: 'GET' // явно, но необязательно
})
  .then(response => response.json())
  .then(data => console.log(data));

Параметры запроса в URL:

Параметры передаются после ? в формате ключ=значение. Несколько параметров соединяются через &. Используются для фильтрации, сортировки и постраничной загрузки. 

// GET с параметром userId=1
fetch('https://jsonplaceholder.typicode.com/todos?userId=1')
  .then(response => response.json())
  .then(data => console.log(data));

GET-запросы идемпотентны — повторный запрос к тому же URL даёт тот же результат и не меняет состояние сервера. Идемпотентны — значит, что многократное выполнение одного и того же запроса не изменяет состояние сервера.

Параметры запроса (Query String)

Query String — часть URL после знака ?, содержащая пары ключ=значение.
Несколько параметров соединяются через &. Используются для фильтрации, сортировки и постраничной загрузки.

Параметры вручную в URL:

// Получить задания только пользователя с userId=1
fetch('https://jsonplaceholder.typicode.com/todos?userId=1')
  .then(response => response.json())
  .then(data => console.log(data)); // массив заданий

Несколько параметров:

// Фильтр: userId=1 и только выполненные задания
fetch('https://jsonplaceholder.typicode.com/todos?userId=1&completed=true')
  .then(response => response.json())
  .then(data => console.log(data));

Для автоматического и безопасного построения параметров используйте URLSearchParams — он правильно экранирует спецсимволы.

URLSearchParams — рекомендуемый способ:

// Безопасное формирование параметров
const params = new URLSearchParams({
  userId: 1,
  completed: false
});

const url = 'https://jsonplaceholder.typicode.com/todos?' + params;
// Результат: .../todos?userId=1&completed=false

fetch(url)
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('Ошибка:', error));
⚠️ Осторожно: не вставляйте данные от пользователя прямо в URL строкой — это может сломать запрос или создать уязвимость. Всегда используйте URLSearchParams.

Обработка ответа сервера

Response — объект, который возвращает fetch() после завершения запроса.
Чтобы получить данные, ответ нужно преобразовать методом .json() — он возвращает ещё один Promise.

Получение всего объекта:

fetch('https://jsonplaceholder.typicode.com/todos/1')
  .then(response => response.json()) // Promise → объект
  .then(data => {
    console.log(data);
    // { userId: 1, id: 1, title: "...", completed: false }
  });

Обращение к отдельному свойству:

fetch('https://jsonplaceholder.typicode.com/todos/1')
  .then(response => response.json())
  .then(data => {
    console.log(data.title);     // название задания
    console.log(data.completed); // статус: true/false
    console.log(data.userId);    // ID пользователя
  });

Вывод нескольких данных из полученного ответа от сервера:

В этом примере мы получаем задание по ID и выводим несколько его свойств в консоль. 

//Пример ответа от сервера (JSON):
{
  "userId": 4,
  "id": 12,
  "title": "Упражнение 11",
  "completed": false
}
//Отправляем GET-запрос на сервер
fetch('https://jsonplaceholder.typicode.com/todos/1') // Отправляем запрос
  .then(response => response.json()) // Преобразуем ответ в JSON
  .then(data => {console.log('ID пользователя:', data.userId); // Выводим ID пользователя
    console.log('ID задания:', data.id); // Выводим ID задания
    console.log('Завершено ли задание?', data.completed); // Выводим статус выполнения
  });

Пояснение к коду:

  • Мы отправляем GET-запрос на сервер по URL https://jsonplaceholder.typicode.com/todos/1.
  • Метод response.json() преобразует ответ в объект JavaScript.
  • Далее мы обращаемся к свойствам объекта data и выводим их в консоль.

Типичные свойства JSON-ответа

Свойство Тип Описание
id number Уникальный идентификатор ресурса
userId number ID пользователя-владельца
title string Название или заголовок
completed boolean Статус выполнения задания

POST-запросы

POST — HTTP-метод для отправки данных на сервер с целью создания нового ресурса.
Данные передаются в теле запроса (body), а не в URL. Метод нужно указывать явно.

Тело запроса — это данные, которые мы отправляем на сервер. Обычно это JSON-объект, но может быть и FormData, XML или другой формат. Например: { "title": "Новая запись", "body": "Содержимое записи", "userId": 1 }

POST используют при регистрации, отправке форм, создании комментариев, заказов и любых данных, которые нужно сохранить на сервере.

Базовый POST-запрос:

fetch('https://jsonplaceholder.typicode.com/posts', {
  method: 'POST',                          // указываем метод явно
  headers: {
    'Content-Type': 'application/json'     // говорим серверу формат данных
  },
  body: JSON.stringify({                   // преобразуем объект в JSON-строку
    title: 'Новая запись',
    body: 'Содержимое записи',
    userId: 1
  })
})
  .then(response => response.json())
  .then(data => console.log('Создано:', data))
  .catch(error => console.error('Ошибка:', error));
Объяснение:
  • Метод method Чтобы указать, что это POST-запрос, нужно прописать method: 'POST'. Это говорит серверу, что мы отправляем данные.
  • Заголовки headers указываем Content-Type: 'application/json', чтобы сообщить серверу, что мы отправляем данные в формате JSON.
  • В поле body мы передаем данные, которые хотим отправить. Мы используем метод JSON.stringify(), чтобы преобразовать объект JavaScript в строку формата JSON, потому что сервер ожидает строку в формате JSON.

Ответ сервера

Когда мы отправляем POST-запрос, сервер может вернуть подтверждение с данными, которые мы отправили. Это обычно происходит через объект JSON, который содержит ID созданного ресурса или сообщение о статусе.

fetch('https://jsonplaceholder.typicode.com/todos', {
  method: 'POST', // Указываем метод запроса
  headers: {
    'Content-Type': 'application/json', // Тип данных
  },
  body: JSON.stringify({
    userId: 1,
    title: 'Новая задача',
    completed: false
  })
})
  .then(response = > response.json()) // Преобразуем ответ в JSON
  .then(data = > {
    console.log('Создано новое задание: ', data); // Выводим данные, которые вернул сервер
  })
  .catch(error = > console.log('Ошибка при отправке запроса:', error)); // Обрабатываем ошибку

💡 Запомни! После отправки POST-запроса, всегда нужно проверять ответ от сервера. Используй .then(response = > response.json()), чтобы преобразовать его в формат, с которым можно работать.

Обработка ошибок

Обработка ошибок — необходимая часть любого запроса, защищающая приложение от неожиданных сбоев.
Ошибки бывают двух видов: сетевые (нет соединения) и HTTP (сервер вернул 4xx/5xx).

⚠️ Важно: fetch() выбрасывает ошибку только при сетевых проблемах. Ответы со статусами 404 и 500 ошибкой не считаются — Promise выполняется успешно. Всегда проверяйте response.ok.

Правильная обработка обоих видов ошибок:

fetch('https://jsonplaceholder.typicode.com/todos/1')
  .then(response => {
    if (!response.ok) {
      // HTTP-ошибка (404, 500 и т.д.)
      throw new Error('Ошибка HTTP: ' + response.status);
    }
    return response.json();
  })
  .then(data => console.log(data))
  .catch(error => {
    // Ловит и сетевые ошибки, и throw из .then()
    console.error('Что-то пошло не так:', error.message);
  });

То же самое с async/await:

async function loadData() {
  try {
    const response = await fetch('https://jsonplaceholder.typicode.com/todos/1');

    if (!response.ok) {
      throw new Error('Ошибка HTTP: ' + response.status);
    }

    const data = await response.json();
    console.log(data);
  } catch (error) {
    // try/catch ловит и сетевые, и HTTP-ошибки
    console.error('Ошибка:', error.message);
  }
}

loadData();

Коды статусов HTTP

Код Значение Что делать
200 Успех Обработать данные из response.json()
201 Ресурс создан Обычно возвращается после POST
400 Некорректный запрос Проверить данные, которые отправляете
401 Не авторизован Проверить токен / заголовок Authorization

GET vs POST — сравнение

Выбор метода — не произвольный: GET и POST имеют чёткие области применения, которые соответствуют семантике HTTP.
Использование правильного метода делает API понятным и предсказуемым.

Критерий GET POST
Назначение Получить данные Создать / отправить данные
Данные Только в URL (?key=value) В теле запроса (body)
Тело запроса Нет Да — JSON, FormData и др.
Идемпотентность ✅ Да — повтор безопасен ❌ Нет — повтор создаст дубль
Кэширование ✅ Браузер может кэшировать ❌ Не кэшируется
Безопасность данных Данные видны в URL и логах Данные скрыты в теле
Пример использования Список товаров, профиль, поиск Форма, регистрация, комментарий

Наглядное сравнение в коде:

// ✅ GET — получить пользователя
fetch('https://jsonplaceholder.typicode.com/users/1')
  .then(r => r.json())
  .then(data => console.log(data));

// ✅ POST — создать пост
fetch('https://jsonplaceholder.typicode.com/posts', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ title: 'Заголовок', userId: 1 })
})
  .then(r => r.json())
  .then(data => console.log(data));

Типичные ошибки

Типичные ошибки — проблемы, с которыми сталкиваются при работе с GET и POST почти все.
Знание этих ловушек сэкономит часы отладки.

❌ Не проверяют response.ok

// ❌ Неправильно: 404 не вызовет ошибку промиса
fetch('https://jsonplaceholder.typicode.com/todos/99999')
  .then(response => response.json()) // выполнится даже при 404!
  .then(data => console.log(data));

// ✅ Правильно: проверяем статус
fetch('https://jsonplaceholder.typicode.com/todos/99999')
  .then(response => {
    if (!response.ok) throw new Error('Статус: ' + response.status);
    return response.json();
  })
  .then(data => console.log(data))
  .catch(err => console.error(err.message));

❌ Забывают JSON.stringify() в POST

// ❌ Неправильно: отправляем объект напрямую
fetch('/api/posts', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: { title: 'Заголовок' } // объект, не строка — ошибка!
});

// ✅ Правильно: преобразуем в строку
fetch('/api/posts', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ title: 'Заголовок' })
});

❌ Забывают заголовок Content-Type

// ❌ Неправильно: сервер не знает формат данных
fetch('/api/posts', {
  method: 'POST',
  body: JSON.stringify({ title: 'Заголовок' }) // нет headers!
});

// ✅ Правильно: указываем тип содержимого
fetch('/api/posts', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ title: 'Заголовок' })
});

❌ Передают приватные данные через GET

// ❌ Неправильно: пароль виден в URL, логах, истории браузера
fetch('/login?password=secret123');

// ✅ Правильно: приватные данные — только в body POST
fetch('/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ login: 'user', password: 'secret123' })
});

Лучшие практики

Лучшие практики — подходы, которые делают код с запросами надёжным, читаемым и безопасным.
Следуйте им с первых проектов, чтобы не переучиваться позже.

✅ Всегда проверяйте response.ok

async function safeFetch(url, options) {
  const response = await fetch(url, options);
  if (!response.ok) {
    throw new Error('HTTP ошибка: ' + response.status);
  }
  return response.json();
}

Оберните повторяющуюся проверку в вспомогательную функцию.

✅ Используйте URLSearchParams для параметров

// ✅ Безопасно и чисто
const params = new URLSearchParams({ userId: 1, page: 2 });
fetch('/api/todos?' + params);

// ❌ Хрупко и опасно при спецсимволах
fetch('/api/todos?userId=' + userId + '&page=' + page);

✅ Предпочитайте async/await для сложной логики

// ✅ Читаемо, удобно отлаживать
async function loadProfile(id) {
  const response = await fetch('/api/users/' + id);
  if (!response.ok) throw new Error('Не найден');
  return response.json();
}

// ❌ Глубокая вложенность — труднее читать
fetch('/api/users/' + id)
  .then(r => r.ok ? r.json() : Promise.reject(r.status))
  .then(data => { /* ... */ })
  .catch(err => { /* ... */ });

✅ Выбирайте метод по смыслу операции

  • GET — только читаем данные, ничего не меняем на сервере.
  • POST — создаём новый ресурс или отправляем данные.
  • PUT / PATCH — обновляем существующий ресурс.
  • DELETE — удаляем ресурс.

Итоги

Итоги — краткое резюме всего, что разобрали на странице.
Используйте как шпаргалку при работе с запросами.

Что мы изучили

  • GET — получить данные, параметры передаются в URL через ?key=value.
  • POST — отправить данные, тело запроса передаётся в body как JSON-строка.
  • fetch() по умолчанию выполняет GET, для POST нужно явно указать method: 'POST'.
  • response.json() преобразует тело ответа в JavaScript-объект.
  • К отдельным полям обращаемся через точку: data.title, data.id.
  • fetch() не выбрасывает ошибку при 4xx/5xx — всегда проверяйте response.ok.
  • Для формирования параметров используйте URLSearchParams.
  • Приватные данные (пароли, токены) — только через POST в body, никогда в URL.

Ключевые правила

  1. Проверяйте response.ok в каждом запросе.
  2. Для POST всегда используйте JSON.stringify() и заголовок Content-Type: application/json.
  3. Используйте URLSearchParams для безопасного построения URL с параметрами.
  4. Выбирайте метод по смыслу: GET — читать, POST — создавать.
  5. Оборачивайте запросы в try/catch при использовании async/await.
