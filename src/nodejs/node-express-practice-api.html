📝 Практика по модулю

Задача: создать API для работы с уроками

План работы

  1. Создай новый проект Node.js.
  2. Установи Express.
  3. Сделай server.js или app.js.
  4. Подключи express.json(), чтобы сервер понимал JSON из POST, PUT, PATCH.
  5. Создай массив с данными в памяти, например lessons = [].
  6. Определи базовый роут /lessons.

Маршруты

  1. Сделай GET /lessons — вернуть все уроки.
  2. Сделай GET /lessons/:id — вернуть один урок по id.
  3. Сделай POST /lessons — добавить новый урок.
  4. Сделай PUT /lessons/:id — заменить урок по id.
  5. Сделай PATCH /lessons/:id — обновить урок по id.
  6. Сделай DELETE /lessons/:id — удалить урок по id.

Структура кода

  1. Вынеси массив с данными в отдельный файл data/lessons.js.
  2. Вынеси обработчики в отдельный файл controllers/lessons.js.
  3. Вынеси маршруты в отдельный файл routes/lessons.js.
  4. Вынеси подключение роутера в routes/index.js или сразу в server.js.
  5. Если модуль большой, раздели логику на routes, controllers, data.

Проверка логики

  1. В каждом обработчике сначала находи урок по id.
  2. Если урок не найден, возвращай 404.
  3. Если данные пришли невалидные, возвращай 400.
  4. Для POST добавляй новый объект с новым id.
  5. Для PUT заменяй все поля объекта.
  6. Для PATCH обновляй только переданные поля.
  7. Для DELETE удаляй объект из массива и возвращай подтверждение.

Тестирование

  1. Проверь все запросы в браузере или Postman.
  2. Сначала протестируй GET /lessons.
  3. Потом GET /lessons/1.
  4. Потом POST /lessons с JSON body.
  5. Потом PUT, PATCH, DELETE.
  6. Убедись, что ответы приходят в нужном формате JSON.

Что сдать

  1. Исходный код server.js.
  2. Код роутов.
  3. Примеры запросов и ответов.
  4. Краткое описание, что делает каждый метод.

Как удобнее делать по шагам

  1. Сначала реализуй только GET /lessons.
  2. Потом добавь GET /lessons/:id.
  3. Потом POST.
  4. Потом PUT и PATCH.
  5. В конце DELETE и обработку ошибок.

Решение

Решение задачи будет представлено в виде кода, который можно использовать в своем проекте. Ниже приведен пример реализации API для работы с уроками.

1.Создай новый проект Node.js.

Создай папку для проекта с названием nodejs-express-practice-api и инициализируй его с помощью команды npm init -y.

После этого в проектной папке появится файл package.json.

2.Установи Express.

Установи Express с помощью команды npm install express.

После этого в проектной папке появится папка node_modules и файл package-lock.json.

3.Сделай server.js или app.js.

Создай файл app.js в корне проекта и добавь следующий код:

// Импортируем ExpressJS
const express = require('express');

// Создаём приложение
const app = express();

// Определяем порт
const PORT = 3000;

// Обработчик для главной страницы
app.get('/', (req, res) => {
  res.send('Добро пожаловать на главную страницу!');
});

// Запускаем сервер
app.listen(PORT, () => {
  console.log(`Сервер запущен на http://localhost:${PORT}`);
});

Теперь можно запустить сервер с помощью команды node app.js и открыть в браузере http://localhost:3000, чтобы увидеть сообщение "Добро пожаловать на главную страницу!".

4.Подключи express.json(), чтобы сервер понимал JSON из POST, PUT, PATCH.

Добавь в app.js следующую строку после создания приложения:

app.use(express.json());

Теперь сервер сможет обрабатывать JSON данные, отправленные в теле запросов POST, PUT и PATCH.

5.Создай массив с данными в памяти, например lessons = [].

Добавь в app.js после создания приложения (строка const app = express();) следующий код для создания массива уроков:

let lessons = [
  { id: 1, title: 'Урок 1', description: 'Описание урока 1' },
  { id: 2, title: 'Урок 2', description: 'Описание урока 2' },
  { id: 3, title: 'Урок 3', description: 'Описание урока 3' }
];
Промежуточный файл app.js
// Импортируем ExpressJS
const express = require('express');

// Создаём приложение
const app = express();
// Подключаем middleware для обработки JSON
app.use(express.json());

// Массив с данными уроков
let lessons = [
  { id: 1, title: 'Урок 1', description: 'Описание урока 1' },
  { id: 2, title: 'Урок 2', description: 'Описание урока 2' },
  { id: 3, title: 'Урок 3', description: 'Описание  урока 3' }
];

// Определяем порт
const PORT = 3000;

// Обработчик для главной страницы
app.get('/', (req, res) => {
  res.send('Добро пожаловать на главную страницу!');

// Запускаем сервер
app.listen(PORT, () => {
  console.log(`Сервер запущен на http://localhost:${PORT}`);

6.Определи базовый роут /lessons.

Добавь в app.js обработчик для маршрута /lessons:

app.get('/lessons', (req, res) => {
  res.json(lessons);
});

Теперь при обращении к http://localhost:3000/lessons в браузере или Postman, сервер вернет JSON массив с уроками.

7.Реализуй остальные маршруты: GET /lessons/:id, POST /lessons, PUT /lessons/:id, PATCH /lessons/:id, DELETE /lessons/:id.

Добавь в app.js следующие обработчики:

// GET /lessons/:id - вернуть один урок по id
app.get('/lessons/:id', (req, res) => {
  const lessonId = parseInt(req.params.id);
  const lesson = lessons.find(l => l.id === lessonId);
  if (!lesson) {
    return res.status(404).json({ error: 'Урок не найден' });
  }
  res.json(lesson);
});

// POST /lessons - добавить новый урок
app.post('/lessons', (req, res) => {
  const { title, description } = req.body || {};

  if (!title) {
    return res.status(400).json({ error: 'Поле title обязательно' });
  }

  const newLesson = {
    id: lessons.length + 1,
    title,
    description: description || ''
  };

  lessons.push(newLesson);
  res.status(201).json(newLesson);
});

// PUT /lessons/:id - заменить урок по id
app.put('/lessons/:id', (req, res) => {
  const lessonId = parseInt(req.params.id);
  const lessonIndex = lessons.findIndex(l => l.id === lessonId);
  if (lessonIndex === -1) {
    return res.status(404).json({ error: 'Урок не найден' });
  }
  lessons[lessonIndex] = {
    id: lessonId,
    title: req.body.title,
    description: req.body.description
  };
  res.json(lessons[lessonIndex]);
});

// PATCH /lessons/:id - обновить урок по id
app.patch('/lessons/:id', (req, res) => {
  const lessonId = parseInt(req.params.id);
  const lesson = lessons.find(l => l.id === lessonId);
  if (!lesson) {
    return res.status(404).json({ error: 'Урок не найден' });
  }
  if (req.body.title) lesson.title = req.body.title;
  if (req.body.description) lesson.description = req.body.description;
  res.json(lesson);
});

// DELETE /lessons/:id - удалить урок по id
app.delete('/lessons/:id', (req, res) => {
  const lessonId = parseInt(req.params.id);
  const lessonIndex = lessons.findIndex(l => l.id === lessonId);
  if (lessonIndex === -1) {
    return res.status(404).json({ error: 'Урок не найден' });
  }
  lessons.splice(lessonIndex, 1);

  res.json({ message: 'Урок удален' });
});

8.Запусти сервер и проверь все маршруты в Postman или браузере.

Запусти сервер с помощью команды node app.js

Проверь все маршруты в Postman или браузере. Например, открой http://localhost:3000/lessons для получения всех уроков, http://localhost:3000/lessons/1 для получения урока с id 1

Используй POST, PUT, PATCH и DELETE для добавления, обновления и удаления уроков. Чтобы проверить POST, PUT, PATCH и DELETE, используй Postman или curl с соответствующими методами и JSON телом запроса.

Как проверить POST-запрос

Postman - это удобный инструмент для тестирования API. Ты можешь скачать его с официального сайта https://www.postman.com/downloads/ и использовать для отправки запросов к твоему серверу.

raw - это формат тела запроса, который позволяет отправлять данные в виде текста. В Postman можно выбрать "raw" и указать тип данных, например JSON, чтобы отправлять JSON объекты в теле запроса.

Открой Postman, выбери метод POST, введи URL http://localhost:3000/lessons, перейди на вкладку "Body", выбери "raw" и "JSON", вставь JSON тело запроса, например: { "title": "Новый урок", "description": "Описание нового урока" } и нажми "Send".

Мы получим ответ от сервера с созданным уроком, например: { "id": 4, "title": "Новый урок", "description": "Описание нового урока" }.

Как проверить PUT-запрос

Используй Postman для отправки PUT-запроса на http://localhost:3000/lessons/1 с JSON телом запроса, например: { "title": "Обновленный урок", "description": "Обновленное описание урока" }. Сервер вернет обновленный урок с id 1.

Как проверить PATCH-запрос

Используй Postman или curl для отправки PATCH-запроса на http://localhost:3000/lessons/1 с JSON телом запроса, например: { "title": "Частично обновленный урок" }. Сервер вернет обновленный урок с id 1.

9. Структура кода

Когда приложение становится больше, не стоит хранить все данные, маршруты и обработчики в одном файле app.js. Лучше разделить проект на отдельные части: данные, контроллеры и маршруты. Такой подход делает код понятнее, удобнее для поддержки и позволяет легко расширять приложение.

Структура проекта

Создай следующую структуру папок: 

project
│
├── app.js                         - Главный файл приложения
│
├── data                            - Данные приложения
│   └── lessons.js                  - Массив уроков
│
├── controllers                     - Логика обработки запросов
│   ├── getAllLessons.js            - Получить все уроки
│   ├── getLessonById.js            - Получить урок по id
│   ├── createLesson.js             - Создать урок
│   ├── updateLesson.js             - Полностью обновить урок
│   ├── patchLesson.js              - Частично обновить урок
│   └── deleteLesson.js             - Удалить урок
│
├── routes                          - Маршруты приложения
│   │
│   └── v1                          - Версия API v1
│       ├── index.js                - Главный файл маршрутов

1. Выносим данные в data/lessons.js

Файл data/lessons.js содержит данные уроков. Сейчас используется обычный массив, но в будущем его можно заменить на базу данных.

2. Контроллеры (controllers)

Контроллеры содержат основную логику приложения. Каждый контроллер отвечает за отдельное действие: получение данных, создание, изменение или удаление уроков.

Например, файл:

// controllers/getAllLessons.js
const lessons = require('../data/lessons');

const getAllLessons = (req, res) => {
  res.json(lessons);
};

module.exports = getAllLessons;

Такой подход позволяет разделить код. Вместо одного большого файла:

controllers/lessons.js

Создаются отдельные файлы:

controllers
│
├── getAllLessons.js
├── getLessonById.js
├── createLesson.js
├── updateLesson.js
├── patchLesson.js
└── deleteLesson.js
3. Маршруты API: routes/v1/lessons.js

Файл маршрутов отвечает только за URL. Он получает запрос и передает его нужному контроллеру. 

// routes/v1/lessons.js
const express = require('express');
const router = express.Router();

const getAllLessons = require('../../controllers/getAllLessons');
const getLessonById = require('../../controllers/getLessonById');
const createLesson = require('../../controllers/createLesson');
const updateLesson = require('../../controllers/updateLesson');
const patchLesson = require('../../controllers/patchLesson');
const deleteLesson = require('../../controllers/deleteLesson');

// Получить все уроки
router.get('/', getAllLessons);

// Получить урок по id
router.get('/:id', getLessonById);

// Создать новый урок
router.post('/', createLesson);

// Обновить урок полностью
router.put('/:id', updateLesson);

// Частично обновить урок
router.patch('/:id', patchLesson);

// Удалить урок
router.delete('/:id', deleteLesson);

module.exports = router;

express.Router() создает отдельный маршрутизатор. Благодаря этому маршруты можно разделять по разным файлам.

4. Объединяем маршруты в routes/v1/index.js

Файл routes/v1/index.js объединяет все маршруты версии 1. Он подключает маршруты уроков и может подключать другие маршруты в будущем.

// routes/v1/index.js
const express = require('express');
const router = express.Router();
const lessonsRouter = require('./lessons');

// Подключаем маршруты уроков
router.use('/lessons', lessonsRouter);

module.exports = router;

Теперь все маршруты уроков начинаются с: /lessons

5. Подключаем API в app.js

В app.js подключаем маршруты версии 1. Все запросы к /api/v1 будут обрабатываться соответствующими маршрутами.

// app.js
const express = require('express');
const routes = require('./routes/v1');

const app = express();

const PORT = 3000;

// Обработка JSON
app.use(express.json());

// Главная страница
app.get('/', (req,res)=>{
  res.send(
    'Добро пожаловать на главную страницу!'
  );
});

// Подключение API версии 1
app.use('/api/v1', routes);

app.listen(PORT,()=>{
 console.log(
  `Сервер запущен на http://localhost:${PORT}`
 );
});
Почему используется /api/v1?

Префикс /api показывает, что это API-запросы. Версия v1 позволяет в будущем создать новую версию API и сохранить совместимость со старым кодом.

После подключения маршрутов получаются следующие адреса: 

GET
http://localhost:3000/api/v1/lessons

GET один урок
http://localhost:3000/api/v1/lessons/1

POST создать урок
http://localhost:3000/api/v1/lessons
Версии API

В больших проектах API может развиваться. Например:

/api/v1/lessons

/api/v2/lessons

Первая версия может продолжать работать для старых клиентов, а новая версия будет использовать новый формат данных. 

/api/v1/lessons // старая версия

/api/v2/lessons // новая версия

В учебном проекте используется только версия /api/v1. В реальных приложениях версионирование API помогает безопасно развивать сервер.

10. Проверка логики контроллеров

Каждый контроллер должен правильно обрабатывать входные данные, проверять ошибки и возвращать соответствующий HTTP-статус.

Получение урока по id (GET /lessons/:id)

Сначала получаем id из параметров запроса и проверяем его корректность. Если урок с таким id не найден, возвращаем ошибку 404.

Создание урока (POST /lessons)

Проверяем данные, которые пришли в теле запроса. Если обязательные поля отсутствуют или имеют неверный формат, возвращаем ошибку 400. После проверки создаём новый объект урока с новым уникальным id и добавляем его в массив.

Полное обновление урока (PUT /lessons/:id)

Сначала проверяем id и ищем существующий урок. Если урок не найден, возвращаем 404. При успешной проверке полностью заменяем данные урока, сохраняя только его id.

Частичное обновление урока (PATCH /lessons/:id)

Проверяем id и наличие урока. Затем изменяем только те поля, которые были переданы в запросе. Остальные данные объекта остаются без изменений.

Удаление урока (DELETE /lessons/:id)

Проверяем id и ищем урок в массиве. Если урок не найден, возвращаем 404. Если урок найден, удаляем его из массива и возвращаем подтверждение удаления.

Используемые HTTP-статусы

200 OK
Запрос успешно выполнен.

201 Created
Объект успешно создан (используется при POST).

400 Bad Request
Ошибка в данных запроса:
- неправильный id;
- отсутствуют обязательные поля;
- неверный формат данных.

404 Not Found
Объект с указанным id не найден.

Таким образом каждый контроллер отвечает только за свою задачу: получение, создание, обновление или удаление данных.
