Алгоритмы сортировки

Сортировка — это упорядочивание элементов набора данных по определённому признаку: числа по возрастанию, слова по алфавиту, товары по цене.
Это одна из самых частых операций в программировании: без неё сложно построить поиск, аналитику или красивый вывод данных пользователю.

Алгоритмов сортировки много, и они отличаются не результатом (отсортированный массив), а тем, как быстро и какой ценой они его получают. Разберёмся, какие бывают подходы и когда какой выбирать.

Сравнение алгоритмов сортировки

Сложность алгоритма — примерное число операций, которое нужно выполнить в зависимости от размера массива n.

Таблица сравнения — время работы и память для основных алгоритмов сортировки.

Алгоритм Среднее время Память Когда использовать
Сортировка пузырьком — сравнивает соседние элементы. O(n²) O(1) Для обучения, не для реальных задач.
Сортировка выбором — ищет минимум на каждом шаге. O(n²) O(1) Маленькие массивы, простота важнее скорости.
Сортировка вставками — вставляет элемент в отсортированную часть. O(n²) O(1) Почти отсортированные или небольшие массивы.
Быстрая сортировка — делит массив вокруг опорного элемента. O(n log n) O(log n) Общий случай, самый быстрый на практике.
Сортировка слиянием — делит массив пополам и сливает отсортированные части. O(n log n) O(n) Когда важна стабильность и предсказуемая скорость.

💡 O(n log n) — это ощутимо быстрее, чем O(n²), на больших массивах: для 1000 элементов разница может быть в десятки раз.

Простые алгоритмы сортировки

Простые алгоритмы — сортировки со сложностью O(n²), которые легко понять и реализовать, но которые медленно работают на больших данных.

Сортировка пузырьком

Сортировка пузырьком (bubble sort) — на каждом проходе сравнивает пары соседних элементов и меняет их местами, если порядок неверный.

  • После каждого прохода самый большой ещё не отсортированный элемент "всплывает" в конец.
  • Проходов нужно столько же, сколько элементов в массиве.

Реализация:

function bubbleSort(arr) {
  for (let i = 0; i < arr.length - 1; i++) {
    for (let j = 0; j < arr.length - 1 - i; j++) {
      if (arr[j] > arr[j + 1]) {
        [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]];
      }
    }
  }
  return arr;
}

bubbleSort([5, 2, 9, 1, 5]); // [1, 2, 5, 5, 9]

Сортировка выбором

Сортировка выбором (selection sort) — на каждом шаге ищет минимальный элемент в неотсортированной части и ставит его в начало.

Реализация:

function selectionSort(arr) {
  for (let i = 0; i < arr.length - 1; i++) {
    let minIndex = i;
    for (let j = i + 1; j < arr.length; j++) {
      if (arr[j] < arr[minIndex]) {
        minIndex = j;
      }
    }
    [arr[i], arr[minIndex]] = [arr[minIndex], arr[i]];
  }
  return arr;
}

selectionSort([5, 2, 9, 1, 5]); // [1, 2, 5, 5, 9]

Сортировка вставками

Сортировка вставками (insertion sort) — берёт очередной элемент и вставляет его на нужное место в уже отсортированной части массива.

Реализация:

function insertionSort(arr) {
  for (let i = 1; i < arr.length; i++) {
    const current = arr[i];
    let j = i - 1;
    while (j >= 0 && arr[j] > current) {
      arr[j + 1] = arr[j];
      j--;
    }
    arr[j + 1] = current;
  }
  return arr;
}

insertionSort([5, 2, 9, 1, 5]); // [1, 2, 5, 5, 9]

Быстрая сортировка и сортировка слиянием

Эффективные алгоритмы — сортировки со сложностью O(n log n), которые устроены сложнее, но заметно быстрее на больших массивах.

Быстрая сортировка

Быстрая сортировка (quick sort) — выбирает опорный элемент, делит массив на меньшие и большие относительно него части, а затем рекурсивно сортирует каждую часть.

  • Использует рекурсию: подробнее — в уроке про рекурсию.
  • На практике часто быстрее сортировки слиянием за счёт меньшего расхода памяти.

Реализация:

function quickSort(arr) {
  if (arr.length <= 1) return arr;

  const pivot = arr[arr.length - 1];
  const left = arr.filter((x, i) => x < pivot && i !== arr.length - 1);
  const right = arr.filter((x, i) => x >= pivot && i !== arr.length - 1);

  return [...quickSort(left), pivot, ...quickSort(right)];
}

quickSort([5, 2, 9, 1, 5]); // [1, 2, 5, 5, 9]

Сортировка слиянием

Сортировка слиянием (merge sort) — делит массив пополам до одиночных элементов, а затем последовательно сливает отсортированные части в один массив.

Реализация:

function mergeSort(arr) {
  if (arr.length <= 1) return arr;

  const mid = Math.floor(arr.length / 2);
  const left = mergeSort(arr.slice(0, mid));
  const right = mergeSort(arr.slice(mid));

  const result = [];
  while (left.length && right.length) {
    result.push(left[0] <= right[0] ? left.shift() : right.shift());
  }
  return [...result, ...left, ...right];
}

mergeSort([5, 2, 9, 1, 5]); // [1, 2, 5, 5, 9]
// результат

Типичные ошибки

Сортировка чисел как строк — стандартный Array.prototype.sort() в JavaScript по умолчанию сравнивает элементы как строки.

Решение: передавайте функцию сравнения — arr.sort((a, b) => a - b) для чисел по возрастанию.

Использование O(n²) на больших данных — пузырёк, выбор и вставки резко замедляются, когда элементов становится много.

Решение: для массивов от нескольких тысяч элементов используйте быструю сортировку или слияние.

Забытая стабильность сортировки — не все алгоритмы сохраняют порядок равных элементов.

Решение: если порядок равных элементов важен (например, при многоступенчатой сортировке таблицы), выбирайте стабильный алгоритм, например сортировку слиянием.

Советы по выбору алгоритма

  • На практике почти всегда можно использовать встроенную сортировку языка — она уже оптимизирована.
  • Реализуйте сортировки вручную в первую очередь для обучения и для собеседований.
  • Для почти отсортированных данных сортировка вставками может быть быстрее быстрой сортировки.
  • Если важна стабильность порядка равных элементов — выбирайте сортировку слиянием.

Итоги

  • ✅ Сортировка — упорядочивание данных по заданному признаку.
  • ✅ Простые алгоритмы (пузырёк, выбор, вставки) понятны, но работают за O(n²).
  • ✅ Быстрая сортировка и сортировка слиянием работают за O(n log n) и подходят для больших данных.
  • ✅ На практике чаще всего используют встроенную сортировку языка, а не реализуют алгоритм вручную.

Часто задаваемые вопросы

В общем случае быстрая сортировка показывает лучшее среднее время на практике, хотя в худшем случае её сложность совпадает с простыми алгоритмами — O(n²). Для гарантированного O(n log n) в любом случае подходит сортировка слиянием.

Обычно нет — языки программирования уже предоставляют оптимизированную встроенную сортировку. Ручная реализация нужна для обучения, понимания сложности алгоритмов и на технических собеседованиях.

Стабильная сортировка сохраняет исходный порядок элементов, которые считаются равными по ключу сортировки. Это важно, когда данные уже отсортированы по одному признаку, а вы досортировываете их по другому.
