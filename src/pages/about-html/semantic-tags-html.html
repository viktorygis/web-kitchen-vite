HTML: Семантические теги

Семантические теги — это специальные элементы HTML, которые описывают смысловую структуру контента на странице. Используя такие теги, вы помогаете браузерам, поисковым системам и вспомогательным технологиям (например, скринридерам) лучше понимать содержимое сайта.

Зачем нужны семантические теги?

  • Доступность: Теги позволяют вспомогательным технологиям (скринридерам и др.) правильно озвучивать и структурировать страницу.
  • SEO: Поисковые системы лучше индексируют и ранжируют сайты с корректной семантикой.
  • Поддержка: Код становится понятнее для других разработчиков, легче проверять и расширять.

Примеры популярных семантических тегов:

<header> , <nav> , <main> , <footer> , <article> , <section> , <aside> .

Структурные теги

Структурные семантические теги задают основную организацию страницы и разделов. Они определяют «каркас» сайта: шапка, подвал, главное содержимое, навигация.

  • <header> — "шапка" страницы или раздела, обычно содержит заголовок, лого, навигацию.
  • <nav> — блок для основного навигационного меню или списка ссылок.
  • <main> — уникальное основное содержимое страницы (только один на странице).
  • <footer> — "подвал" страницы или раздела: контакты, копирайты, доп. ссылки.

  • <header>

    <header> — Шапка страницы или раздела, содержит вводную информацию, меню, логотип, заголовки.

    Где использовать: В начале страницы, статьи, секции. Можно использовать несколько <header> на разных уровнях.

    • class — CSS-класс для стилизации.
    • id — уникальный идентификатор.

    Примечание: Внутри <header> нельзя размещать <footer> , <address> .

    Пример:

    <header>
  <h2>Заголовок сайта</h2>
  <nav>...</nav>
</header>

  • <nav>

    <nav> — Навигационный блок, хранит основные ссылки для перемещения по сайту или разделу.

    Обычно содержит: Списки ссылок ( <ul> , <a> ).

    • class — CSS-класс для стилизации.
    • id — уникальный идентификатор.

    Примечание: Может быть несколько <nav> (главное меню, футер, боковое меню).

    Пример:

    <nav>
  <a href="#home">Главная</a>
  <a href="#about">О нас</a>
</nav>

  • <main>

    <main> — Единственное на странице место для основного уникального содержимого.

    Внутри: Ключевой контент: статьи, разделы, изображения, видео.

    • class — CSS-класс для стилизации.
    • id — уникальный идентификатор.

    Примечание: Только один <main> на странице. Не размещайте внутри <header> , <nav> , <footer> , <aside> .

    Пример:

    <main>
  <section>Основной контент</section>
</main>

  • <footer>

    <footer> — Подвал страницы или секции, содержит копирайт, контакты, ссылки на политику и т.д.

    Можно использовать: Для всей страницы, для статей или разделов.

    • class — CSS-класс для стилизации.
    • id — уникальный идентификатор.

    Примечание: Внутри <footer> нельзя размещать <header> , <nav> .

    Пример:

    <footer>
  <p>Контактная информация</p>
</footer>

Контентные теги

Контентные семантические теги служат для выделения самостоятельных или логически связанных блоков информации.

  • <section> — логический раздел документа (может содержать свой заголовок).
  • <article> — самостоятельный блок контента (новость, пост, статья).
  • <aside> — побочный контент (боковая колонка, реклама, доп. ссылки).

  • <section>

    <section> — Логический раздел, группирует схожий по смыслу контент (например, услуги, новости, отзывы).

    Рекомендуется: Начинайте <section> с заголовка ( <h2> - <h6> ).

    • class — CSS-класс для стилизации.
    • id — якорь для навигации.

    Примечание: Не используйте <section> только для стилизации — он нужен для логической группировки!

    Пример:

    <section>
  <h2>Заголовок раздела</h2>
  <p>Содержимое раздела</p>
</section>

  • <article>

    <article> — Самостоятельная публикация: статья, новость, пост, комментарий.

    Структура: Может содержать заголовки, текст, списки, <section> и другие элементы.

    • class — CSS-класс для стилизации.
    • id — уникальный идентификатор.

    Примечание: <article> должен быть самостоятельным и может быть распространён вне контекста сайта.

    Пример:

    <article>
  <h2>Заголовок статьи</h2>
  <p>Содержимое статьи</p>
</article>

  • <aside>

    <aside> — Боковое или дополнительное содержание, не связанное напрямую с основным (например, боковая колонка, реклама, ссылки).

    Использование: Для информации, которая дополняет основной контент, но не является его частью.

    • class — CSS-класс для стилизации.
    • id — уникальный идентификатор.

    Пример:

    <aside>
  <h3>Дополнительная информация</h3>
  <p>Боковой контент или реклама</p>
</aside>

Медиа и детали

Теги для описания иллюстраций, дат, времени и других деталей, которые помогают структурировать и описывать контент более точно.

  • <figure> — иллюстрация, диаграмма, медиа; <figcaption> — подпись к иллюстрации.
  • <time> — дата или время для распознавания машинами (например, для событий, публикаций).

  • <figure> и <figcaption>

    <figure> — содержит иллюстрацию, фото, схему, диаграмму.
    <figcaption> — подпись к иллюстрации, размещается внутри <figure> .

    Использование: Для любых медиа, которым требуется подпись или пояснение.

    • class — CSS-класс для стилизации.
    • id — уникальный идентификатор.

    Пример:

    <figure>
  <img src="photo.jpg" alt="Фото">
  <figcaption>Подпись к фото</figcaption>
</figure>

  • <time>

    <time> — тег для даты или времени, используемый для событий, публикаций, расписаний.

    Атрибут datetime: Позволяет задать дату/время для машинного анализа.

    • datetime — формат даты/времени (например, 2025-08-18 ).

    Пример:

    <time datetime="2025-08-18">18 августа 2025</time>

Примеры использования семантических тегов

Семантические теги делают код HTML осмысленным, улучшают доступность и визуальную структуру страницы для людей и поисковых систем.

Визуальная структура

<header> — шапка сайта
(логотип, меню)
<nav> — навигационное меню
<main> — основное содержимое
<section> — раздел
<article> — статья/новость
<figure> — иллюстрация
<figcaption> — подпись к картинке
<aside> — боковая колонка
<footer> — подвал страницы
(контакты, копирайт, дата)

HTML-код


<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <title>Пример семантической страницы</title>
  </head>
  <body>
    <header>
      <h1>Заголовок сайта</h1>
      <nav>
        <ul>
          <li><a href="#">Главная</a></li>
          <li><a href="#">О нас</a></li>
        </ul>
      </nav>
    </header>

    <main>
      <section>
        <article>
          <h2>Статья</h2>
          <p>Текст статьи.</p>
          <figure>
            <img src="photo.jpg" alt="Фото">
            <figcaption>Подпись к фото</figcaption>
          </figure>
        </article>
      </section>
      <aside>
        <h3>Боковая колонка</h3>
        <p>Дополнительная информация.</p>
      </aside>
    </main>

    <footer>
      <p>Копирайт © 2025</p>
      <time datetime="2025-08-18">18 августа 2025</time>
    </footer>
  </body>
</html>

Такая структура помогает поисковым системам, скринридерам и разработчикам быстро понять смысловую организацию страницы.

Как правильно использовать семантические теги

Семантические теги делают ваш HTML осмысленным и структурированным, облегчая работу поисковых систем и вспомогательных технологий.

  • Используйте <header> , <main> , <footer> для базовой структуры страницы.
  • <nav> — только для основного меню или навигационных блоков, чтобы обозначить область с ссылками по сайту.
  • <article> — для самостоятельных материалов, например, статьи, новости, посты.
    <section> — для логических разделов внутри <main> или <article> .
  • <aside> — для дополнительной информации, рекламы, вспомогательных ссылок, боковой колонки.
  • Не используйте семантические теги ради их наличия — выбирайте тот, что отражает смысл контента.
  • Проверяйте разметку в валидаторе HTML для обнаружения ошибок.

Советы и best practices

Применяйте семантику осознанно — она упрощает поддержку сайта, делает код читабельным и доступным.

  • Пишите код так, чтобы его было удобно читать без CSS — структура должна быть ясна даже без оформления.
  • Для повышения доступности используйте атрибуты, например aria-label для <nav> или других важных областей.
  • Избегайте избыточного вложения тегов ("дивовый ад"). Используйте семантику для логичной структуры, а <div> и <span> — только для группировки и стилизации.
  • Структурируйте контент так, чтобы пользователю было легко ориентироваться на странице.
  • Семантические теги помогают поисковым системам и вспомогательным технологиям — используйте их вместо несемантических контейнеров там, где это возможно.
