Руководство по свойствам display: block,inline и inline-block

Свойство display определяет, как элемент отображается в потоке документа.
В этом руководстве мы рассмотрим ключевые режимы отображения — block , inline и inline-block — и разницу между ними.

display: block

Элемент занимает всю доступную ширину, начиная с новой строки. Блоки могут иметь отступы, padding, ширину и высоту.


.block {
  display: block;
  background: #e3f2fd;
  padding: 10px;
  margin-bottom: 10px;
}

display: inline

Элемент располагается в одну строку с другими элементами.
Нельзя задавать ширину и высоту, но можно использовать padding и margin по горизонтали. 


.inline {
  display: inline;
  background: #ffe0b2;
  padding: 4px 8px;
  margin-right: 8px;
}

display: inline-block

Элемент отображается в строке, как inline , но при этом можно задавать width , height , padding и margin по всем сторонам. 


.inline-block {
  display: inline-block;
  background: #dcedc8;
  padding: 10px;
  width: 120px;
  height: 60px;
  margin-right: 10px;
  text-align: center;
}

Какие элементы по умолчанию имеют display: inline, block, inline-block

Значения по умолчанию задаются таблицами стилей браузера (UA styles) и могут немного отличаться между движками и платформами. Ниже — наиболее распространённые категории.

Чаще всего display: block

  • div, p, pre, hr
  • header, footer, main, nav
  • section, article, aside, address
  • h1–h6
  • ul, ol, menu, dl
  • form, fieldset, legend
  • figure, figcaption
  • details

display: inline (фразовый контент)

  • span, a, br, wbr
  • em, strong, b, i, u, s, small
  • sub, sup, mark, q, cite
  • abbr, time, data, var, kbd, code, samp
  • label, bdi, bdo
  • img, video, audio, canvas, iframe*

Чаще всего display: inline-block

  • button
  • input
  • select
  • textarea
  • progress, meter
  • output
Примечания:
  • Многие «медийные» и встроенные элементы (например, img , video , canvas , iframe ) являются встроенными по умолчанию ( inline ), но как заменяемые элементы допускают размеры и ведут себя «как inline-block » с точки зрения коробочной модели.
  • li по умолчанию имеет display: list-item , а таблицы — свои типы отображения ( table , table-row , table-cell и т. д.).
  • У элемента summary по умолчанию обычно display: list-item , а у details — block .
  • CSS‑фреймворки и нормалайзеры могут переопределять эти значения. Проверяйте «Computed» в DevTools для актуального проекта.
