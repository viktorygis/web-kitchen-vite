Отступы margin collapsing-margin padding Примеры

Что такое отступы в CSS?

В CSS есть два вида отступов — margin и padding . Они помогают управлять пространством вокруг и внутри элементов, делая макет аккуратным и удобным для восприятия.

margin — создаёт пространство вокруг элемента.

padding — создаёт пространство внутри элемента между содержимым и границей.

Используя margin и padding , ты словно работаешь дизайнером интерьера: всё становится гармонично и удобно!

margin — внешний отступ

margin — это личное пространство элемента.
Оно создаёт внешние отступы, то есть расстояние между элементом и его соседями.

С помощью margin можно:

  • Отделять элементы друг от друга
  • Центрировать элементы: margin: 0 auto;
  • Настраивать отступы отдельно для каждой стороны: margin-top , margin-right , margin-bottom , margin-left
  • Задавать отрицательные значения для сдвига элементов
  • Использовать сокращённую запись: margin: 10px 20px 30px 40px;

Пример внешнего отступа (margin):

Каждому блоку задан margin: 10px; . Между ними появляется пространство. 


<div style="margin: 10px; background-color: lightblue">Я люблю воздух вокруг!</div>
<div style="margin: 10px; background-color: lightgreen">И я тоже!</div>
Я люблю воздух вокруг!
И я тоже!

Margin collapsing (схлопывание margin)

Когда два вертикальных margin соприкасаются, они объединяются в один, равный наибольшему (или сумме с учётом знаков, если есть отрицательные). Явление происходит у блочных элементов в нормальном потоке (block formatting context).

Когда марджины схлопываются

  • Смежные вертикальные отступы соседних блоков: нижний margin предыдущего и верхний следующего.
  • Между родителем и его первым ребёнком: верхний margin первого ребёнка со «верхом» родителя.
  • Между родителем и его последним ребёнком: нижний margin последнего ребёнка с «низом» родителя.
  • В «пустых» блоках без border/padding/внутреннего контента — верхний и нижний margin самого блока схлопываются.

Если среди участвующих есть отрицательные margin, результат — алгебраическая сумма: max(положительные) + (самый «большой по модулю» отрицательный).

Пример: соседние блоки

Фактическое расстояние между блоками — 30px (из 30px и 20px берётся наибольший).

<div style="margin-bottom: 30px; background: #e3f2fd; padding: 10px">Элемент 1 (mb: 30px)</div>
<div style="margin-top: 20px; background: #bbdefb; padding: 10px">Элемент 2 (mt: 20px)</div>
Элемент 1 (margin-bottom: 30px)
Элемент 2 (margin-top: 20px)

Пример: родитель и первый ребёнок

У родителя нет padding / border / overflow , поэтому margin-top ребёнка «выпрыгивает» наверх родителя .

Ребёнок 1 (margin-top: 24px)
Ребёнок 2 (margin-top: 12px)
Переключить предотвращение схлопывания:

Текущий режим: без предотвращения (коллапс включён)

Как надёжно избежать схлопывания?

  • Рекомендуется: display: flow-root; у родителя — создаёт новый BFC, не меняя поток. 
    .parent { display: flow-root; }
  • overflow: hidden|auto|clip; — тоже создаёт BFC, но:
    • hidden/clip могут обрезать содержание;
    • auto может показать полосы прокрутки.
  • padding-top или border-top у родителя — простые, но добавляют визуальное пространство.
  • display: flex или grid у родителя — предотвращают коллапс, но меняют модель раскладки детей.

Где коллапса нет по определению

  • У flex‑ и grid‑элементов (дети flex/grid и их соседи не схлопывают марджины вертикально).
  • У абсолютно позиционированных элементов ( position: absolute/fixed ).
  • У плавающих элементов ( float ) и их потомков снаружи — margins не схлопываются через float.
  • По горизонтали марджины никогда не схлопываются, только по вертикали в нормальном блоковом контексте.

Типичные заблуждения

  • position: relative , outline , background , transform , box-shadow — не влияют на схлопывание.
  • «Сброшу на margin-top: 0 у первого ребёнка» — это работает локально, но не объясняет и не решает саму причину коллапса.

padding — внутренний отступ

padding — это как уютная подушка внутри твоего элемента.
Оно создаёт внутренние отступы между содержимым элемента и его границами.

С помощью padding можно:

  • Добавлять пространство между текстом и краями блока
  • Создавать "воздух" внутри кнопок и карточек
  • Настраивать внутренние отступы отдельно для каждой стороны: padding-top , padding-right , padding-bottom , padding-left
  • Использовать сокращённую запись: padding: 15px 30px;

Пример внутреннего отступа (padding):


<div style="padding: 15px; border: 2px solid black; background: #f5f5f5; display: inline-block">
  Текст внутри блока
</div>
Текст внутри блока

Примеры применения margin и padding

Настройка отступов для каждой стороны


div {
  margin-top: 20px;    /* Отступ сверху */
  margin-bottom: 10px; /* Отступ снизу */
  padding-right: 10px; /* Внутренний отступ справа */
}
Пример блока с индивидуальными отступами (margin и padding)

Сокращённая запись отступов


div {
  margin: 10px 20px 30px 40px; /* верх, право, низ, лево */
  padding: 15px;               /* все стороны = 15px */
}
Пример блока с сокращённой записью
(margin: 10px 20px 30px 40px; padding: 15px)

margin: 10px 20px; — верх/низ = 10px, право/лево = 20px

margin: 10px; — все стороны = 10px

CSS читает значения по часовой стрелке, начиная с верхней стороны.

Памятка

  • margin — для пространства вокруг элемента. 🏞️
  • padding — для пространства внутри элемента. 🛏️
  • ✂️ Сокращённая запись — твой лучший друг, если нужно писать меньше, а делать больше.
