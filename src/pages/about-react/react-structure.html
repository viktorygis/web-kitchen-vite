Структура компонентов в React

Структура компонентов — важная часть архитектуры React-приложения.
Организация файлов и разделение ответственности позволяют легко поддерживать и расширять проект.

Структура файлов в проекте


src/
  components/
    Button.js
    Card.js
    List.js
  pages/
    Home.js
    About.js
  App.js
  index.js
Компоненты обычно хранятся в папке components , страницы — в pages .

Именование компонентов

  • Компоненты — с большой буквы: Button , Header .
  • Файл компонента — Button.js , Card.js .
  • Папки группируются по смыслу: components , features , pages .

Архитектура и разделение ответственности

  • UI-компоненты: только отображают данные (Button, Card).
  • Контейнеры/страницы: управляют состоянием, бизнес-логикой.
  • Композиция: крупные компоненты строятся из мелких.
  • Переиспользуемые блоки — отдельные компоненты.
Разделяйте UI и логику для масштабируемости!

Практика: файл компонента


/* src/components/Button.js */
function Button({children, onClick}) {
  return <button onClick={onClick}>{children}</button>;
}
export default Button;

Советы и типичные ошибки

  • Не смешивайте бизнес-логику и отображение.
  • Не делайте компоненты слишком большими.
  • Переиспользуйте компоненты — не копируйте код.
  • Соблюдайте единый стиль именования.

Лучшие практики

  • Группируйте компоненты по смыслу и назначению.
  • Документируйте ответственность каждого компонента.
  • Используйте PropTypes или TypeScript для типизации.
  • Структурируйте проект для легкой навигации.

Примеры структуры компонентов

Компонент страницы


/* src/pages/Home.js */
function Home() {
  return <div>Главная страница</div>;
}
export default Home;

Композиция компонентов


/* src/pages/Profile.js */
import Card from '../components/Card';
function Profile() {
  return (
    <Card>
      <h2>Профиль пользователя</h2>
    </Card>
  );
}
export default Profile;
