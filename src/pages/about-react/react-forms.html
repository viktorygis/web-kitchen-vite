Формы в React

Формы — способ получать данные от пользователя.
В React формы обычно реализуются как управляемые компоненты.

Управляемые компоненты


function App() {
  const [value, setValue] = React.useState("");
  return (
    <input value={value} onChange={e => setValue(e.target.value)} />
  );
}
Управляемый компонент — значение хранится в state, меняется через onChange.

Обработка событий формы


function App() {
  const [value, setValue] = React.useState("");
  function handleSubmit(e) {
    e.preventDefault();
    alert("Отправлено: " + value);
  }
  return (
    <form onSubmit={handleSubmit}>
      <input value={value} onChange={e => setValue(e.target.value)} />
      <button>OK</button>
    </form>
  );
}
onSubmit — обработчик отправки формы.

Множественные элементы формы


function App() {
  const [form, setForm] = React.useState({name:"", email:""});
  function handleChange(e) {
    setForm(f => ({...f, [e.target.name]: e.target.value}));
  }
  return (
    <form>
      <input name="name" value={form.name} onChange={handleChange} />
      <input name="email" value={form.email} onChange={handleChange} />
    </form>
  );
}
Для сложных форм удобно хранить все значения в одном объекте.

Практика: форма обратной связи


<div id="root"></div>
<script type="text/babel">
function App() {
  const [form, setForm] = React.useState({name:"", feedback:""});
  function handleChange(e) {
    setForm(f => ({...f, [e.target.name]: e.target.value}));
  }
  function handleSubmit(e) {
    e.preventDefault();
    alert(`Спасибо, ${form.name}! Ваш отзыв: ${form.feedback}`);
    setForm({name:"", feedback:""});
  }
  return (
    <form onSubmit={handleSubmit}>
      <input name="name" value={form.name} onChange={handleChange} placeholder="Имя" />
      <input name="feedback" value={form.feedback} onChange={handleChange} placeholder="Отзыв" />
      <button>Отправить</button>
    </form>
  );
}
const root = ReactDOM.createRoot(document.getElementById('root'));
root.render(<App />);
</script>

Советы и типичные ошибки

  • Держите значения формы в state.
  • Очищайте форму после отправки.
  • Проверяйте данные до отправки (валидация).

Лучшие практики

  • Выносите логику формы в отдельные компоненты и хуки.
  • Документируйте структуру формы для команды.
  • Для сложных форм используйте useReducer.
