Вложенные маршруты в React Router

Вложенные маршруты позволяют строить иерархию страниц — разделы с собственными подстраницами.
Это удобно для личных кабинетов, магазинов, профилей и других сложных интерфейсов.

Структура вложенных маршрутов


<Routes>
  <Route path="/profile" element={<Profile />}>
    <Route path="settings" element={<Settings />} />
    <Route path="orders" element={<Orders />} />
  </Route>
</Routes>
Вложенные маршруты описываются внутри родительского Route !

Outlet — место для вложенных компонентов


import { Outlet, Link } from 'react-router-dom';

function Profile() {
  return (
    <div>
      <nav>
        <Link to="settings">Настройки</Link>
        <Link to="orders">История</Link>
      </nav>
      <Outlet /> {/* Здесь будут подстраницы! */}
    </div>
  );
}
Outlet — специальный компонент, куда React Router вставляет дочерние маршруты.

Страница по умолчанию для вложенного маршрута


<Route path="" element={<Settings />} />
Пустой путь "" означает, что при переходе на /profile загрузится Settings .

Практика: личный кабинет с вложенными страницами


import { BrowserRouter, Routes, Route, Link, Outlet } from 'react-router-dom';

function Settings() { return <div>Настройки</div>; }
function Orders() { return <div>История заказов</div>; }

function Profile() {
  return (
    <div>
      <h2>Личный кабинет</h2>
      <nav>
        <Link to="settings">Настройки</Link>
        <Link to="orders">История</Link>
      </nav>
      <Outlet />
    </div>
  );
}

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/profile" element={<Profile />}>
          <Route path="" element={<Settings />} />
          <Route path="settings" element={<Settings />} />
          <Route path="orders" element={<Orders />} />
        </Route>
      </Routes>
    </BrowserRouter>
  );
}

Советы и типичные ошибки

  • Не забывайте про Outlet в родительском компоненте!
  • Вложенные маршруты — это не просто слэш в пути, а иерархия в Routes !
  • Используйте относительные ссылки to="settings" внутри вложенного маршрута.

Лучшие практики

  • Общий UI (заголовок, меню) — в родителе, контент — в дочерних маршрутах.
  • Делайте страницу по умолчанию с пустым path для удобства.
  • Документируйте вложенную структуру маршрутов для команды.
