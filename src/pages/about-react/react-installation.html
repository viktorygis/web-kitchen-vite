Установка React

React — библиотека для создания пользовательских интерфейсов. Использует синтаксис JSX, работает на основе виртуального DOM и компонентного подхода. Каждый компонент — это функция, возвращающая JSX-разметку.

Основные понятия

  • React — библиотека для создания пользовательских интерфейсов.
  • Пользовательский интерфейс — всё, что видит и с чем взаимодействует пользователь.
  • JSX (JavaScript XML) — синтаксис для разметки в JavaScript.
  • Виртуальное дерево DOM — абстракция для эффективного обновления интерфейса.
  • Дерево DOM — структура HTML-документа в виде узлов.
  • Компонентный подход — интерфейс строится из небольших переиспользуемых компонентов.
  • Компонент — функция, возвращающая JSX. Может принимать входные данные (props) и управлять своим состоянием.

Способы установки

  • CDN — быстрое подключение через теги <script> в HTML (для изучения, прототипов).
  • Vite — современный быстрый инструмент сборки, стандарт для новых проектов React.
  • Create React App — классический способ для сложных или старых проектов.

CDN-способ подходит для учебных или маленьких проектов.
Для современных и крупных приложений используйте Vite.
Create React App — для поддержки старых проектов.

Установка React через CDN

CDN — для прототипов, тестирования и маленьких учебных проектов.

Подключение

<script src="https://unpkg.com/react@18/umd/react.development.js"></script>
<script src="https://unpkg.com/react-dom@18/umd/react-dom.development.js"></script>
<script src="https://unpkg.com/babel-standalone@6/babel.min.js"></script>

Пример HTML-страницы с React через CDN

<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>React CDN</title>
    <script src="https://unpkg.com/react@18/umd/react.development.js"></script>
    <script src="https://unpkg.com/react-dom@18/umd/react-dom.development.js"></script>
    <script src="https://unpkg.com/babel-standalone@6/babel.min.js"></script>
  </head>
  <body>
    <div id="root"></div>
    <script type="text/babel">
      function App() {
        const [message, setMessage] = React.useState("Привет, React!");
        return (
          <div>
            <h1>{message}</h1>
            <button onClick={() => setMessage("Вы только что нажали кнопку!")}>Изменить сообщение</button>
          </div>
        );
      }
      const root = ReactDOM.createRoot(document.getElementById('root'));
      root.render(<App />);
    </script>
  </body>
</html>

Установка React через Vite (рекомендуется для новых проектов)

Vite — современный стандарт для быстрой и удобной разработки на React.
Поддерживает горячую перезагрузку, минимальную структуру, мгновенный старт.
Рекомендуется для всех новых проектов!

Шаги установки (2025, актуально):

  1. Убедитесь, что установлены Node.js и npm (node -v и npm -v).

  2. Откройте терминал и выполните команду:

    npm create vite@latest my-app

  3. Выберите стрелками React и JavaScript (или TypeScript по желанию).

  4. Выберите

  5. Для Yarn: yarn create vite my-app

  6. Перейдите в папку проекта:

    cd my-app

  7. Установите зависимости:

    npm install

  8. Запустите проект в режиме разработки:

    npm run dev

Проект откроется по адресу http://localhost:5173

Пример структуры проекта Vite + React


my-app/
├─ src/
│  ├─ App.jsx
│  ├─ main.jsx
│  └─ ...
├─ index.html
├─ package.json
└─ vite.config.js
Объяснение:
  • src/ — папка для исходного кода, содержит компоненты и точки входа.
  • index.html — шаблон HTML, в который Vite вставляет скомпилированные скрипты.
  • package.json — файл с зависимостями и скриптами проекта.
  • vite.config.js — конфигурация Vite (необязательно, только для кастомизации).

Пример простого компонента

import React from "react";

function App() {
  return (
    <h1>Привет, React!</h1>
  );
}

export default App;
Преимущества Vite: мгновенный запуск, быстрая горячая перезагрузка, гибкая настройка, современная архитектура.
Документация: vitejs.dev/guide/

Установка React через Create React App

Create React App — классический способ для старых/поддерживаемых проектов.
Сейчас для новых проектов рекомендуется Vite.
Create React App был признан устаревшим. Используйте его только для поддержки старого кода или если нужен максимально быстрый старт без сложной настройки.

Шаги установки

  1. Создание нового проекта:

    npx create-react-app my-react-app

  2. Перейдите в папку проекта:

    cd my-react-app

  3. Запустите сервер разработки:

    npm start

Проект откроется по адресу http://localhost:3000

Пример структуры проекта Create React App

my-react-app/
├─ node_modules/
├─ public/
├─ src/
│  ├─ App.jsx
│  ├─ index.js
│  └─ ...
├─ package.json
├─ package.json-lock.json
├─ README.md
└─ ...
Объяснение:
  • node_modules/ — папка с установленными зависимостями.
  • public/ — статические файлы, доступные по URL.
  • src/ — папка для исходного кода, содержит компоненты и точки входа.
  • package-lock.json — фиксирует версии зависимостей (необязательно для понимания).
  • package.json — файл с зависимостями и скриптами проекта.
  • README.md — документация проекта.

Пример простого компонента

import React from "react";
function App() {
  return (
    <h1>Привет, React!</h1>
  );
}
export default App;
Преимущества Create React App: быстрый старт, много готовых настроек, поддержка старого кода.
Недостатки: медленнее Vite, сложная настройка, устаревший инструмент.
Документация: create-react-app.dev/docs

Сравнение Vite и Create React App

Критерий Vite Create React App (CRA)
Команда установки npm create vite@latest my-app npx create-react-app my-app
Время установки Очень быстро (секунды) Дольше (минуты)
Старт разработки cd my-app
npm install
npm run dev 		cd my-app
npm start
Горячая перезагрузка Мгновенная, без перезагрузки страницы Медленнее, часто перезагружает страницу
Базовая структура проекта Минимум файлов, только самое необходимое Много файлов и настроек «из коробки»
Структура проекта Гибкая, можно организовать по своему усмотрению Стандартная, рекомендуется придерживаться
Гибкость конфигурации Легко настраивается (vite.config.js) Сложно, требуется «eject» для кастомизации
Скорость сборки Очень высокая (ESBuild/Vite) Медленнее (Webpack)
Совместимость с modern JS Отличная (ESM, современный синтаксис) Требуется настройка для новых возможностей
Поддержка Современный стандарт, быстро развивается Официально поддерживается, но развивается медленнее
Документация vitejs.dev/guide create-react-app.dev/docs
Вывод:
Vite — лучший выбор для современных проектов.
CRA — только если нужен максимально быстрый старт без сложной настройки или для поддержки старого кода.

Фреймворки на базе React

На базе React построены популярные фреймворки для создания сложных приложений:

  • Next.js — для серверного рендеринга и статической генерации сайтов.
  • React Router (v7) — для маршрутизации в одностраничных приложениях.
  • Expo (для нативных приложений) — для разработки мобильных приложений на React Native.
  • TanStack Start (Beta) — для управления данными и кэшированием в React-приложениях.
  • RedwoodJS — для создания full-stack приложений на React с интеграцией бэкенда.

Эти фреймворки нужно устанавливать после создания базового React-приложения через Vite или CRA, в зависимости от требований проекта.

То есть сначала устанавливается Vite, а затем уже фреймворки на их основе.

Эти фреймворки предоставляют готовые решения для маршрутизации, управления данными, оптимизации производительности и других задач, упрощая разработку сложных приложений на React.

Советы и типичные ошибки

  • Для production-проектов используйте Vite.
  • Проверяйте, чтобы Node.js и npm были актуальными (Node.js 18+, npm 9+).
  • Не храните весь код в одном файле — используйте структуру папок.
  • Для экспериментов и обучения CDN идеально подойдёт.
Частая ошибка: использовать CDN для серьёзных проектов. Это может привести к проблемам с производительностью, безопасностью и поддержкой. Для реальных приложений всегда выбирайте Vite или CRA.

Лучшие практики

  • Делите интерфейс на небольшие компоненты.
  • Используйте props и state для управления данными.
  • Соблюдайте структуру проекта: src/ для компонентов и страниц.
  • Следите за обновлениями React и экосистемы.
  • Используйте Vite для современных проектов.
Вывод: Начинайте знакомство с React через CDN, но для серьёзной разработки переходите на Vite — это быстро, удобно и современно!
