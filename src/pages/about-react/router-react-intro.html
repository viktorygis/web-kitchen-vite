Введение в React Router

React Router — стандартная библиотека для организации навигации в React-приложениях.
Позволяет строить SPA, где разные URL отображают разные компоненты.

Установка React Router


npm install react-router-dom
Используйте react-router-dom для браузерных приложений.

Основы маршрутизации


import { BrowserRouter, Routes, Route, Link } from 'react-router-dom';

function App() {
  return (
    <BrowserRouter>
      <nav>
        <Link to="/">Главная</Link>
        <Link to="/about">О сайте</Link>
      </nav>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </BrowserRouter>
  );
}
BrowserRouter — корневой компонент для работы с URL.
Routes — контейнер маршрутов.
Route — отдельный маршрут.

Маршруты и ссылки


<Link to="/profile">Профиль</Link>
<Route path="/profile" element={<Profile />} />
Link — компонент для перехода по маршрутам без перезагрузки страницы!

Практика: простая навигация


/* App.js */
import { BrowserRouter, Routes, Route, Link } from 'react-router-dom';

function Home() { return <h2>Главная</h2>; }
function About() { return <h2>О сайте</h2>; }

function App() {
  return (
    <BrowserRouter>
      <nav>
        <Link to="/">Главная</Link>
        <Link to="/about">О сайте</Link>
      </nav>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </BrowserRouter>
  );
}

Советы и типичные ошибки

  • Все маршруты размещайте внутри BrowserRouter .
  • Используйте только Link для перехода между страницами.
  • Проверяйте, что путь маршрута не перекрывает другие страницы.

Лучшие практики

  • Группируйте маршруты по разделам.
  • Используйте отдельные компоненты для каждой страницы.
  • Документируйте структуру маршрутов для команды.
