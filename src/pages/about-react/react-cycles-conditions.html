Циклы и условия в React

JSX не поддерживает циклы и условия напрямую!
Всё делается через JavaScript: map() для списков, if / ? / && для условий.
Это позволяет делать интерфейс динамическим и адаптивным.

Циклы: map для рендеринга списков

Используйте map для создания элементов из массива. Циклы for и forEach использовать нельзя! 

const users = ["Аня", "Макс", "Иван"];
<ul>
  {users.map(name => <li key={name}>{name}</li>)}
</ul>

Условия: if, ? и &&

  • Вне JSX используйте обычный if .
  • Внутри JSX — только ? (тернарный оператор) или && . 

{isLoggedIn ? <p>Добро пожаловать!</p> : <p>Войдите</p>}
{count > 0 && <p>Есть элементы</p>}

Практика: фильтр списка


function FilterDemo() {
  const [filter, setFilter] = React.useState("");
  const users = ["Аня", "Макс", "Иван"];
  const filtered = users.filter(u => u.toLowerCase().includes(filter.toLowerCase()));
  return (
    <div>
      <input placeholder="Фильтр" value={filter} onChange={e => setFilter(e.target.value)} />
      <ul>
        {filtered.length ? filtered.map(u => <li key={u}>{u}</li>) : <li>Нет совпадений</li>}
      </ul>
    </div>
  );
}

Советы и типичные ошибки

  • Для списков используйте только map , не for !
  • Для условий внутри JSX — только ? или && .
  • Старайтесь не писать длинные вложенные условия.

Лучшие практики

  • Готовьте данные (фильтрация/сортировка) вне JSX.
  • Избегайте лишнего дублирования условий.
  • Для больших вложенных условий используйте отдельные функции.

Примеры циклов и условий

Список товаров

Пояснение: 🛒 Используется map для рендеринга массива объектов, а также условие для пустого списка. 

function Products({items}) {
  return (
    <ul>
      {items.length
        ? items.map(i => <li key={i.id}>{i.name}</li>)
        : <li>Нет товаров</li>}
    </ul>
  );
}
