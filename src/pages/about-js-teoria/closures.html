Замыкания и лексическое окружение в JavaScript

Это фундаментальная часть JavaScript, которая помогает понять, как работают функции, как они "помнят" свои данные и почему код ведёт себя так, а не иначе.

Замыкание — это механизм, который позволяет функции "запоминать" переменные из области, где она была создана, даже если эта область уже недоступна.

Лексическое окружение

Лексическое окружение - Внутренний механизм JavaScript, который определяет, какие переменные доступны функции при её вызове.
Каждая функция "запоминает", где она была объявлена, и сохраняет доступ к переменным из этой области.

💬 Как это работает:
Когда JavaScript выполняет код, он "смотрит", где была объявлена функция, и запоминает доступные ей переменные. 
function sayHello() {
    const message = "Привет, мир!";
    console.log(message);
}

sayHello(); // "Привет, мир!"

Что происходит?

  • Функция sayHello "видит" переменную message , потому что она объявлена внутри неё.
  • Когда мы вызываем sayHello , JavaScript "подглядывает" в лексическое окружение функции и находит message .

Что такое замыкание?

Замыкание — это когда функция "запоминает" переменные из своего окружения, даже если внешняя функция уже завершила работу.

💡 Замыкания — это как личный помощник для каждой функции, который помогает ей помнить важные вещи.
  • Позволяет создавать приватные переменные и функции, недоступные из глобального контекста.
  • Удобно для хранения состояния между вызовами функции.
function outer() {
    const message = "Я замыкание!";
    return function inner() {
        console.log(message);
    };
}

const myFunction = outer();
myFunction(); // "Я замыкание!"
Пошаговый разбор:
  • outer — внешняя функция, внутри которой создаётся переменная message .
  • Внутри outer возвращается функция inner , использующая message .
  • Когда вызывается outer() , она возвращает функцию inner , а myFunction хранит её.
  • Даже после завершения outer , переменная message остаётся доступной внутри inner благодаря замыканию.
  • Вызов myFunction() выведет message — потому что inner "помнит" своё окружение. 
function createGreeting(name) {
    return function () {
        console.log("Hello, " + name);
    };
}

const greet = createGreeting("Alice");
greet(); // "Hello, Alice"
Пошаговый разбор:
  • createGreeting — функция с параметром name .
  • Она возвращает новую функцию, использующую этот name .
  • Вызов createGreeting("Alice") создаёт замыкание, где name равен "Alice".
  • greet хранит внутреннюю функцию, у которой есть доступ к name из внешней функции.
  • При вызове greet() будет использовано значение name , даже если createGreeting уже завершила выполнение.

Примеры использования замыканий

Пример 1: Счётчик с замыканием

Этот пример демонстрирует, как замыкания позволяют создавать функции, которые могут сохранять своё состояние между вызовами.

function counter() {
    let count = 0;
    return function increment() {
        count++;
        console.log("Текущий счётчик:", count);
    };
}
const increment = counter();
increment(); // 1
increment(); // 2
increment(); // 3
Пошаговый разбор:
  • counter — внешняя функция, внутри которой объявляется count .
  • Возвращается функция increment , которая увеличивает count и выводит его.
  • Вызов counter() возвращает increment , замыкающую переменную count .
  • Каждый вызов increment() работает с тем же значением count .
  • Значение count недоступно снаружи, только внутри замыкания.

Пример 2: Счётчик с начальным значением

Этот пример показывает, как замыкания позволяют создавать функции с приватным состоянием, которое сохраняется между вызовами.

function createCounter(initialValue) {
    let count = initialValue;
    return function counter() {
        count++;
        console.log("Текущий счётчик:", count);
    };
}
const myCounter = createCounter(10);
myCounter(); // 11
myCounter(); // 12
Пошаговый разбор:
  • createCounter — функция с параметром initialValue , задающим начальное значение счётчика.
  • Внутри создаётся переменная count равная initialValue .
  • Возвращается функция counter , работающая с переменной count .
  • Вызов createCounter(10) создаёт замыкание, где count стартует с 10.
  • Каждый вызов myCounter() увеличивает и выводит count , сохраняя состояние между вызовами.
🎯 Замыкания позволяют создавать функции, которые могут использовать переменные из внешнего контекста — даже если этот контекст уже не существует.

Пример 3: Счётчик с замыканием

Этот пример демонстрирует, как замыкания позволяют создавать функции, которые могут сохранять своё состояние между вызовами.

function createCounter() {
    let count = 0;
    return function () {
        count++;
        console.log(`Счётчик: ${count}`);
    };
}

const counter = createCounter();
counter(); // 1
counter(); // 2
counter(); // 3
Пошаговый разбор:
  • createCounter — функция, внутри которой объявляется переменная count .
  • Внутренняя функция увеличивает count и выводит результат в шаблонной строке.
  • counter хранит внутреннюю функцию, замыкающую count .
  • Каждый вызов counter() увеличивает значение count и выводит его.
  • count недоступна вне замыкания, её значение хранится только внутри замкнутой функции.
  • Переменная count создаётся внутри функции createCounter .
  • Она остаётся доступной для возвращаемой функции благодаря замыканию.

Пример 4 из реального проекта: обработка событий и приватное состояние

Этот пример демонстрирует, как замыкания позволяют создавать функции, которые могут сохранять своё состояние между вызовами.


// Модуль для управления всплывающими окнами (popup) на сайте

function createPopupManager() {
    let isOpen = false; // приватная переменная состояния

    function open() {
        if (!isOpen) {
            // Логика открытия окна
            document.getElementById('popup').style.display = 'block';
            isOpen = true;
        }
    }

    function close() {
        if (isOpen) {
            // Логика закрытия окна
            document.getElementById('popup').style.display = 'none';
            isOpen = false;
        }
    }

    function toggle() {
        if (isOpen) {
            close();
        } else {
            open();
        }
    }

    // Возвращаем публичный интерфейс, а isOpen остаётся приватным
    return {
        open,
        close,
        toggle
    };
}

// Использование в проекте:
const popupManager = createPopupManager();

document.getElementById('open-btn').onclick = popupManager.open;
document.getElementById('close-btn').onclick = popupManager.close;
document.getElementById('toggle-btn').onclick = popupManager.toggle;
Объяснение:
Переменная isOpen инкапсулирована внутри замыкания createPopupManager и недоступна извне. Только методы open , close и toggle могут менять её значение, что делает логику управления окном надёжной и безопасной для внешнего кода.

Практика: Замыкания в действии

Попробуй создать и использовать замыкание-счётчик

Значение счётчика: 0

Пример кода:

// Создаём замыкание-счётчик
function createCounter() {
    let count = 0;
    return {
        increment: function() {
            count++;
            return count;
        },
        reset: function() {
            count = 0;
            return count;
        }
    };
}

const counter = createCounter();
counter.increment(); // 1
counter.increment(); // 2
counter.reset();     // 0

Пошаговый разбор:

  • Вызов createCounter() создаёт замыкание, внутри которого переменная count доступна только методам increment и reset .
  • Каждый вызов increment() увеличивает count и возвращает новое значение.
  • Вызов reset() сбрасывает count в 0.
  • Значение count не доступно снаружи напрямую — оно инкапсулировано в замыкании.
Практика: Замыкания удобно использовать для создания приватных данных и управления внутренним состоянием объектов и функций, недоступных извне.

Советы по работе с замыканиями

  • Используйте замыкания для инкапсуляции данных и создания приватных переменных.
  • Замыкания полезны для реализации шаблонов, таких как фабрики функций и декораторы.
  • Будьте осторожны с памятью: замыкания могут удерживать переменные, которые больше не нужны, если неаккуратно обращаться с ссылками.
Иногда замыкания могут приводить к утечкам памяти, если они ссылаются на большие объекты, которые больше не используются.

Лучшие практики

  • Явно обозначайте, где используются замыкания, чтобы код был понятнее для других разработчиков.
  • Не создавайте лишние замыкания, если в них нет необходимости — это может затруднить отладку и понимание кода.
  • Используйте замыкания для создания приватных методов и переменных в модулях и компонентах.
  • Очищайте ссылки на переменные, если замыкание больше не требуется.
Замыкания — мощный инструмент, но используйте их осознанно. Понимание их работы поможет писать более надёжный и чистый код!
