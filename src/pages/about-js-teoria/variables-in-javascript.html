Основы JavaScript: Переменные

Переменные в JavaScript используются для хранения данных.
Они могут содержать информацию о товарах, сообщениях, пользователях и любых других данных.

Основные понятия:

  • Именованное хранилище данных
  • Могут изменяться (кроме констант)
  • Имеют область видимости

Где используются:

  • Хранение промежуточных результатов
  • Работа с пользовательскими данными
  • Управление состоянием приложений

Объявление переменных

Объявление переменной — создание именованного хранилища для данных.
В JavaScript используются три ключевых слова: let , const и var .

Простое объявление:

let message = 'Hello!'; // объявление с инициализацией
let age; // объявление без значения

Множественное объявление:

let user = 'John',
    age = 25,
    isAdmin = false;

(хотя лучше объявлять по одной переменной на строку)

Особенности объявления:

Ключевое слово Область видимости Можно изменять Поднятие (hoisting)
let Блочная Да Без инициализации
const Блочная Нет Без инициализации
var Функциональная Да С инициализацией (undefined)

Типы переменных

Изменяемые переменные (let)

let counter = 0;
counter = 1; // значение можно изменить
let price;
price = 100; // можно присвоить позже

Константы (const)

const PI = 3.14;
// PI = 3.1415; // Ошибка!

const user = { name: 'John' };
user.name = 'Pete'; // Можно, т.к. изменяется свойство объекта
// user = {}; // Ошибка!

Когда что использовать:

  • const — по умолчанию, если значение не меняется
  • let — если значение будет изменяться
  • var — только для поддержки старого кода

Именование переменных

Имя переменной должно четко отражать её назначение.
В JavaScript используют camelCase для имен переменных и UPPER_CASE для констант.

Правила именования:

  • Может содержать буквы, цифры, $, _
  • Не может начинаться с цифры
  • Регистр имеет значение (myVar ≠ myvar)
  • Нельзя использовать зарезервированные слова

Примеры имен:

let userName; // хорошее
let shoppingCart; // хорошее
let 123abc; // плохое (начинается с цифры)
let let; // плохое (ключевое слово)

Рекомендации по именованию

  1. Используйте понятные, описательные имена
  2. Избегайте сокращений (data → userData)
  3. Для булевых переменных используйте is/has/can (isActive, hasPermission)
  4. Для констант — UPPER_CASE с подчеркиваниями (MAX_SIZE)
