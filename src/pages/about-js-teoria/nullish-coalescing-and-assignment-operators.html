Операторы нулевого слияния и присваивания

Операторы ?? и ??= позволяют удобно работать с null и undefined .
Они часто используются для задания значений по умолчанию и безопасного присваивания.

  • Нулевое слияние: ?? — возвращает первое определённое значение ( ≠ null/undefined ).
  • Нулевое присваивание: ??= — присваивает, если переменная null или undefined .

?? — оператор нулевого слияния

Возвращает первое значение, которое не null и не undefined .

Примеры:

let user;
let defaultName = "Аноним";
alert(user ?? defaultName); // "Аноним"
//возвращает первое определённое значение

let a = null, b = undefined, c = "Текст";
alert(a ?? b ?? c ?? "Пусто"); // "Текст"
//возвращает первое определённое значение

Применение для значений по умолчанию:

function getName(name) {
  return name ?? "Гость";
}
console.log(getName());         // "Гость"
console.log(getName("Вася"));  // "Вася"

Сравнение ?? и ||

|| возвращает первое "истинное" значение,
?? — первое определённое ( ≠ null/undefined ).

|| использует ToBoolean для проверки истинности, а ?? — для проверки на null и undefined . То есть ?? более строгий, чем || , так как || считает 0 , "" , false , NaN "ложными", а ?? только null и undefined "неопределёнными".

Примеры:

let height = 0;>alert(height || 100); // 100

( 0 считается ложным) 

alert(height ?? 100); // 0

( 0 — определённое значение)

??= — оператор нулевого присваивания

Присваивает новое значение только если переменная null или undefined .

Примеры:

let userAge = null;
userAge ??= 18;
alert(userAge); // 18

let userAge = 25;
userAge ??= 30;
alert(userAge); // 25

Приоритет оператора ??

У ?? низкий приоритет, используйте скобки для корректных вычислений. 

let area = (height ?? 100) * (width ?? 50);

Внимание!
?? нельзя смешивать с || и && без скобок.
let x = (a && b) ?? c; — правильно
let x = a && b ?? c; — ошибка

Ловушки и рекомендации

  • Используйте ?? для значений по умолчанию, если важны 0 или "" .
  • Не смешивайте ?? с || / && без скобок — получите синтаксическую ошибку.
  • ??= — удобно для инициализации переменных только если они "пустые".
