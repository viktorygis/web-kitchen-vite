Логические операторы в JavaScript

Логические операторы позволяют комбинировать условия, возвращать значения выражений и управлять потоком исполнения. В JavaScript операторы || и && возвращают один из операндов (а не просто true/false ) и используют механизм «сокращённого вычисления».

Основные логические операторы:

  • || — логическое ИЛИ
  • && — логическое И
  • ! — логическое НЕ
  • ||= , &&= — логические операторы присваивания
  • !! — идиома приведения к булевому типу
Отдельно: оператор нулевого слияния ?? и логическое присваивание ??= разбираются на другой странице.

|| — логическое ИЛИ

Возвращает первый «истинный» (truthy) операнд или последний, если все оказались «ложными» (falsy). Истинность определяется приведением к логическому типу: Boolean(операнд) .

Алгоритм:

  1. Читает операнды слева направо
  2. Преобразует текущий операнд к логическому значению
  3. Если он truthy — немедленно возвращает исходное значение
  4. Если все falsy — возвращает последний операнд

Простые примеры:

console.log(true  || true);   // true
console.log(false || true);   // true
console.log(false || false);  // false

console.log(1 || 0);          // 1 (первый truthy)
console.log(null || 0 || "A"); // "A"
console.log(null || undefined || 0); // 0 (все falsy)

Поиск первого содержательного значения:

const firstName = "";
const lastName  = "";
const nickName  = "Суперкодер";

const userLabel = firstName || lastName || nickName || "Аноним";
console.log(userLabel); // "Суперкодер"

||= — логическое присваивание ИЛИ

Сокращённая запись: a ||= b эквивалентна if (!a) a = b; (но с тонкостью: проверяется truthy/falsy).

Примеры:

let configTitle = "";
configTitle ||= "Заголовок по умолчанию";
console.log(configTitle); // "Заголовок по умолчанию"

let port = 0;
port ||= 3000;
console.log(port); // 3000 (потому что 0 - falsy)
Осторожно: Значение 0 или пустая строка "" могут быть значимыми. Оператор ||= их перезапишет. Для «только null/undefined» используйте ??= .

&& — логическое И

Возвращает первый «ложный» (falsy) операнд или последний, если все были truthy. Используется для условного выполнения/доступа.

Алгоритм:

  1. Идёт слева направо
  2. Преобразует операнд к логическому
  3. Если операнд falsy — возвращает его исходное значение
  4. Если все truthy — возвращает последний

Примеры:

console.log(true  && true);   // true
console.log(false && true);   // false
console.log(1 && 0);          // 0 (первый falsy)
console.log(1 && 2 && 3);     // 3 (все truthy)
console.log(null && "текст"); // null

Условное выполнение:

isLoggedIn && showDashboard();
// Аналогично:
if (isLoggedIn) showDashboard();

&&= — логическое присваивание И

a &&= b эквивалентно if (a) a = b; (проверяется truthy).

Пример:

let message = "Привет";
message &&= message + ", пользователь!";
console.log(message); // "Привет, пользователь!"

let counter = 0;
counter &&= 10;
console.log(counter); // 0 (не изменился, т.к. 0 falsy)

! — логическое НЕ

Инвертирует булево значение. Сначала приводит операнд к логическому, затем возвращает противоположное.

Примеры:

console.log(!true);     // false
console.log(!0);        // true
console.log(!"");       // true
console.log(!"abc");    // false
console.log(!null);     // true
Приоритет: У оператора ! приоритет выше, чем у && и || , поэтому он выполняется раньше.

!! — двойное НЕ (приведение к Boolean)

Идиома для явного получения булева значения: !!value равнозначно Boolean(value) .

Примеры:

console.log(!!"текст");  // true
console.log(!!"");       // false
console.log(!!0);        // false
console.log(!!1);        // true
console.log(!!null);     // false
console.log(Boolean("текст")); // true

Приоритет и порядок вычисления

Приоритет (от более высокого к низкому): ! → && → || . Скобки всегда можно (и нужно для читаемости) использовать для явного порядка.

Пример:

console.log(!true || false); // false, т.к. !true -> false, false || false -> false
console.log(true && false || true); // (true && false) -> false; false || true -> true
console.log(true && (false || true)); // (false || true) -> true; true && true -> true

Практические шаблоны использования

1. Значение по умолчанию

function greet(name) {
  name = name || "Гость"; // Если name falsy
  console.log("Привет,", name);
}

2. Безопасный доступ (до появления optional chaining)

user && user.profile && user.profile.avatar && show(user.profile.avatar);

3. Короткое выполнение вместо if

isAdmin && grantAccess(); // if (isAdmin) grantAccess();

4. Условное логирование

DEBUG && console.log("STATE =", state);

5. Инициализация (но аккуратно с 0 и "")

let timeout = passedTimeout || 500; // 0 потеряется
Для «только null/undefined» используйте ?? и ??= , чтобы не терять допустимые значения 0 , "" .

Ловушки и рекомендации

  • Falsy значения: 0 , "" , null , undefined , NaN , false .
  • a ||= b перезапишет 0 и "" , даже если это желаемые значения.
  • Смешивать побочные эффекты в условиях ( a() && b() ) — снижает читаемость.
  • Скобки повышают ясность: не бойтесь писать (cond1 && cond2) || fallback .
  • Для доступа к свойствам лучше использовать ?. (optional chaining), чем длинные цепочки && .
  • Двойное отрицание !! — идиома, но не злоупотребляйте; иногда Boolean(value) читабельнее.
