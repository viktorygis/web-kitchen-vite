Получение элементов

Вся веб-страница — это дерево элементов, с которым можно работать через JavaScript.
Зачем это нужно?

  • Найти нужный блок — например, заголовок, кнопку, список.
  • Изменить содержимое — заменить текст, спрятать или показать элемент.
  • Добавить интерактив — реагировать на действия пользователя (например, по клику).

Как это сделать?
В JavaScript есть специальные методы поиска — они позволяют быстро найти нужный элемент по id , классу , тегу или CSS-селектору .
Выберите подходящий способ — и управляйте любым элементом на странице!

Основные методы поиска элементов в DOM

🔎 document.getElementById

getElementById — быстрый способ найти элемент по уникальному атрибуту id . 

<h1 id="main-title">Добро пожаловать!</h1>

Как использовать:

  1. document — доступ к DOM.
  2. Метод .getElementById .
  3. В скобках — строка с id элемента.
const title = document.getElementById("main-title");
console.log(title); // <h1 id="main-title">Добро пожаловать!</h1>
Обратите внимание:
• id должен быть уникальным на странице
• Если id повторяется, будет найден только первый элемент
• Если элемента нет, результат — null

🌟 document.querySelector

querySelector — универсальный способ поиска по CSS‑селектору . 

<p class="text">Привет, DOM!</p>
const paragraph = document.querySelector(".text");
console.log(paragraph); // <p class="text">Привет, DOM!</p>

Как использовать:

  1. document — доступ к DOM.
  2. Метод .querySelector .
  3. В скобках — CSS‑селектор искомого элемента.
  • .text — класс (точка перед именем)
  • #header — id (решётка перед идентификатором)
  • div > p — первый <p> внутри <div>
Особенности:
• Возвращает только первый найденный элемент
• Для всех совпадений — используйте querySelectorAll
• Если элемента нет — результат null

🤔 Когда использовать какой метод?

  • getElementById — для поиска по уникальному id (быстро и надёжно)
  • querySelector — для гибкого поиска по классу, тегу, вложенности, сложному селектору
Резюме: Для уникальных элементов — id, для остального — CSS‑селекторы.

Получить элементы по классу: getElementsByClassName

Этот метод возвращает коллекцию всех элементов с указанным классом.
Получается HTMLCollection — похожий на массив объект, с которым удобно работать в цикле. 

<p class="main-text">Первый абзац</p>
<p class="main-text">Второй абзац</p>
<p>Обычный абзац</p>

const textElements = document.getElementsByClassName("main-text");
console.log(textElements);

for (let i = 0; i < textElements.length; i++) {
  console.log(textElements[i].textContent);
}
• Коллекция обновляется при изменении DOM.
• Нет Array-методов, но можно работать через индексы.
• Если элементов нет, коллекция пустая.

Получить элементы по тегу: getElementsByTagName

Позволяет получить HTMLCollection всех элементов с заданным тегом, например, все <p> или <div> на странице. 

<p>Первый абзац</p>
<p>Второй абзац</p>
<div>Какой-то блок</div>

const allParagraphs = document.getElementsByTagName("p");
console.log(allParagraphs);

for (let i = 0; i < allParagraphs.length; i++) {
  console.log(allParagraphs[i].textContent);
}
• Работает для любого тега.
• Коллекция обновляется при изменении DOM.
• Если элементов нет, коллекция пустая.

Получить все элементы по селектору: querySelectorAll

Возвращает NodeList всех элементов, подходящих под CSS-селектор. Можно использовать любые сложные селекторы. 

<p class="main-text">Первый абзац</p>
<p class="main-text">Второй абзац</p>
<div class="main-text">Какой-то блок</div>

const allMainTexts = document.querySelectorAll(".main-text");
console.log(allMainTexts);

allMainTexts.forEach(el => {
  console.log(el.textContent);
});
• NodeList поддерживает forEach .
• Список статический (не обновляется при изменении DOM).
• Если элементов нет, NodeList пустой.
