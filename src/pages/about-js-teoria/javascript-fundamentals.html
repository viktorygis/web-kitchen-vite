Основы JavaScript

JavaScript — это язык программирования, используемый для создания интерактивных веб-страниц.
Он позволяет браузеру выполнять код, который мы пишем.

Основные характеристики:

  • Интерпретируемый язык
  • Динамическая типизация
  • Поддержка объектно-ориентированного программирования
  • Кроссплатформенность

Где выполняется:

  • В браузере пользователя
  • На сервере (Node.js)
  • В мобильных приложениях

Тег «script»

Тег <script> — используется для вставки JavaScript кода в HTML документ.
Может содержать код непосредственно или ссылаться на внешний файл.

Встроенный скрипт:

Скрипт выполняется автоматически при загрузке страницы.

<script>
  alert('Привет, мир!');
</script>

Внешний скрипт:

Для улучшения структуры кода рекомендуется использовать внешние файлы.

<script src="/path/to/script.js"></script>

Атрибуты тега script

Атрибут Назначение Современное использование
type Определял тип скрипта Не требуется (по умолчанию JavaScript)
language Указывал язык скрипта Устарел, не используется
async/defer Управление загрузкой скриптов Актуально для оптимизации

Структура кода JavaScript

Инструкции

Инструкции — это команды, которые выполняют действия. Каждая инструкция обычно заканчивается точкой с запятой. 

alert('Привет');
alert('Мир');

Точка с запятой

Хотя JavaScript часто сам добавляет точки с запятой, их явное использование предотвращает ошибки:

// Без точки с запятой может вызвать проблемы
let a = 1
let b = 2
[a, b] = [b, a]

Рекомендации:

  • Всегда используйте точки с запятой для явного завершения инструкций
  • Разделяйте логические блоки пустыми строками
  • Соблюдайте единый стиль кодирования

Комментарии в JavaScript

Комментарии — это части кода, которые игнорируются интерпретатором.
Они служат для объяснения работы кода.

Типы комментариев:

// Однострочный комментарий

/*
  Многострочный
  комментарий
*/

JSDoc комментарии:

/**
 * Возвращает сумму двух чисел
 * @param {number} a Первое число
 * @param {number} b Второе число
 * @returns {number} Сумма a и b
 */
function sum(a, b) {
  return a + b;
}

Правила хороших комментариев:

  1. Объясняйте "почему", а не "что" (код должен говорить сам за себя)
  2. Избегайте избыточных комментариев
  3. Используйте JSDoc для документирования функций
  4. Обновляйте комментарии при изменении кода
