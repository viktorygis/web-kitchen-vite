Операторы сравнения в JavaScript

Операторы сравнения возвращают логическое значение true или false .
В нестрогих сравнениях происходит автоматическое приведение типов.
Понимание правил приведения помогает избегать скрытых ошибок.

Общее:

  • Строгие: === , !== — без приведения типов.
  • Нестрогие: == , != — с приведением типов.
  • Реляционные: > < >= <= — числовое сравнение (если обе строки — лексикографическое).

== — нестрогое равенство

Сравнивает значения с приведением типов.

Основные трансформации:

  • '5' == 5 → true (строка → число)
  • true == 1 , false == 0 → true
  • '' == 0 , ' ' == 0 → true
  • null == undefined → true (уникальное правило)
  • '0' == false → true
  • [] == '' → true , [] == 0 → true
  • NaN == NaN → false

Примеры:

// Лучше избегать без явной причины
console.log('5' == 5);        // true
console.log(false == 0);      // true
console.log([] == '');        // true
console.log([] == 0);         // true
console.log([0] == 0);        // true

=== — строгое равенство

Не приводит типы. Истинно, только если тип и значение совпадают.

Ключевые моменты:

  • '5' === 5 → false
  • 0 === false → false
  • NaN === NaN → false (используйте Number.isNaN() )

Примеры:

console.log('5' === 5);  // false
console.log(5 === 5);    // true
console.log(Number.isNaN(NaN)); // true

!= — нестрогое неравенство

Логическое отрицание результата == с теми же преобразованиями типов.

Типичные результаты:

  • 5 != '5' → false
  • null != undefined → false
  • '' != 0 → false

Примеры:

console.log(5 != '5');          // false
console.log(null != undefined); // false
console.log('' != 0);           // false

!== — строгое неравенство

Истинно, если типы различны или значения различны.

Особенности:

  • 5 !== '5' → true
  • 5 !== 5 → false
  • NaN !== NaN → true

Примеры:

console.log(5 !== '5'); // true
console.log(5 !== 5);  // false
console.log(NaN !== NaN); // true

Реляционные: > , < , >= , <=

Если обе строки — лексикографическое сравнение. Иначе — числовое приведение.

Примеры поведения:

  • '10' > 2 → true
  • '2' > '15' → true (лексикографически)
  • null >= 0 → true (null → 0)
  • null == 0 → false (особое правило)
  • undefined становится NaN → сравнения false

Примеры:

console.log('2' > '15'); // true (строки)
console.log('2' > 15);   // false (числа)
console.log(null >= 0);  // true
console.log(null == 0);  // false

Сравнение строк

Посимвольно по Unicode. Первое различие символов решает результат.

Примеры:

  • 'abc' < 'abd' → true
  • '2' > '15' → true
  • 'apple' < 'apricot' → true

Код:

console.log('2' > '15');       // true
console.log('apple' < 'apricot'); // true

null и undefined

Имеют специальное правило равенства между собой и ведут себя по-разному при приведении к числам.

Правила:

  • null == undefined → true
  • null === undefined → false
  • null в числовом сравнении → 0
  • undefined → NaN ⇒ сравнения >, <, >=, <= → false

Примеры:

console.log(null == undefined); // true
console.log(null >= 0);         // true (null -> 0)
console.log(null == 0);         // false
console.log(undefined > 0);     // false

NaN

NaN не равно ничему, даже самому себе.

Проверки:

  • NaN === NaN → false
  • Number.isNaN(NaN) → true
  • isNaN('foo') → true (нестрогое приведение)

Примеры:

console.log(NaN === NaN);        // false
console.log(Number.isNaN(NaN));  // true
console.log(isNaN('foo'));       // true

Мини-таблица приведения (избранные случаи)

Выражение Результат Причина
'5' == 5 true Строка → число
false == 0 true false → 0
'' == 0 true '' → 0
[] == '' true [] → ''
[] == 0 true [] → '' → 0
[0] == 0 true [0] → '0' → 0
null == undefined true Спец. правило
null == 0 false Правило не действует

Ловушки и рекомендации

  • Предпочитайте === / !== вместо == / != .
  • null и undefined — равны только друг другу.
  • Используйте Number.isNaN() для проверки NaN.
  • Строки сравниваются лексикографически: '2' > '15' → true .
  • Пустые массивы могут вводить в заблуждение: [] == '' , [] == 0 .
