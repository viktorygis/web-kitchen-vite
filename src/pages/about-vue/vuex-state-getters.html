Основы работы с состоянием

Основы работы с состоянием - часть большого курса по Vuex. Рекомендуется пройти сначала введение в Vuex.

Механизмы Vuex:

  • state – где хранятся все данные 💾
  • getters – позволяют получить вычисляемые данные 🎯
  • mutations – изменяют state, но только синхронно ✏️
  • actions – выполняют асинхронные задачи и вызывают mutations 🔄

Различия между ними:

  • state – просто хранит данные, не содержит логики
  • getters – вычисляют и возвращают данные на основе state
  • mutations – единственный способ изменить state, работают синхронно
  • actions – могут быть асинхронными, вызывают mutations для изменения state

В этом уроке мы подробно рассмотрим state, getters и mutations, чтобы вы могли эффективно управлять данными в вашем Vue-приложении с помощью Vuex! 🚀

📌 State

State – это централизованное хранилище данных вашего приложения. Вместо того чтобы передавать данные через props и события между компонентами, вы можете хранить их в одном месте – в Vuex state! 💾

Как создать state

import { createStore } from "vuex";

const store = createStore({
  state: {
    count: 0, // 💾 Глобальное хранилище для счётчика
    user: { name: "Алексей", role: "admin" } // 👤 Данные пользователя
  }
});

export default store;

⚠️ На что обратить внимание?

  • ✔ state – основное место хранения всех данных приложения
  • ✔ Нельзя изменять state напрямую – для этого используются mutations
  • ✔ В state можно хранить числа, строки, объекты, массивы

Теперь у нас есть единое хранилище данных – все компоненты могут обращаться к нему без props и emit! 🚀

📌 Getters

getters – это вычисляемые свойства на основе state, которые легко использовать в компонентах.

Для чего нужны getters?

  • 🎯 Позволяют получать обработанные данные из state
  • 🧠 Хранят логику вычислений в одном месте
  • ⚡️ Упрощают доступ к часто используемым значениям

Как создать getters

const store = createStore({
  state: {
    count: 5,
    user: { name: "Алексей", role: "admin" }
  },
  getters: {
    doubleCount: state => state.count * 2, // 📊 Удваиваем значение
    isAdmin: state => state.user.role === "admin" // 🔑 Проверяем роль пользователя
  }
});
  • state передаётся первым аргументом в getters
  • getters не изменяют данные, а только вычисляют
  • Используются как computed-свойства в компонентах

Computed-свойства - это свойства, которые автоматически пересчитываются при изменении зависимостей. В Vuex getters работают аналогично, предоставляя удобный способ получения вычисленных данных из state.

Пример простого getter

doubleCount: state => state.count * 2,

🔍 Разбираем:

doubleCount – это имя getter, которое мы будем использовать в компонентах. Функция принимает state и возвращает удвоенное значение счётчика.

Использование getters в компоненте


<script>
export default {
  computed: {
    doubleCount() {
      return this.$store.getters.doubleCount; // 📊 Получаем обработанные данные
    }
  }
};
</script>

🔍 Как это работает?

  1. this – ссылка на текущий компонент Vue
  2. $store – доступ к глобальному хранилищу Vuex
  3. getters – объект с вычисляемыми значениями
  4. doubleCount – имя getter, который мы хотим получить

🔥 Теперь можно использовать doubleCount в шаблоне: 

<p>Удвоенное значение: {{ doubleCount }}</p>
getters позволяют хранить логику в одном месте и не засорять компоненты вычислениями! 🎯

📌 Mutations

Mutations – это единственный способ изменить данные в state. Они работают синхронно и вызываются через метод commit().

❌ Нельзя менять state напрямую: this.$store.state.count++ — это ошибка! Будьте аккуратны!

✅ Используйте mutations: this.$store.commit('имяMutation')

Как работают mutations?

  • mutations — специальные функции внутри Vuex, которые изменяют state
  • работают только синхронно
  • вызываются через commit()

Пример использования mutations

Пример: счётчик, который можно увеличить или уменьшить кнопками в компоненте.

1. Создаём mutations в store.js

const store = createStore({
  state: {
    count: 0 // 💾 Глобальная переменная
  },
  mutations: {
    increment(state) {
      state.count++; // ✏️ Увеличиваем значение на 1
    },
    decrement(state) {
      state.count--; // ✏️ Уменьшаем значение на 1
    }
  }
});
Что здесь происходит?
  • increment и decrement – имена mutations
  • каждая mutation принимает state как первый аргумент
  • внутри mutation мы изменяем state.count

2. Вызываем mutations в компоненте:

<script>
export default {
  methods: {
    increase() {
      this.$store.commit("increment"); // 🔄 Увеличиваем счётчик
    },
    decrease() {
      this.$store.commit("decrement"); // 🔽 Уменьшаем счётчик
    }
  }
};
</script>
Что здесь происходит?
  • в методах компонента вызываем this.$store.commit("имяMutation")
  • это запускает соответствующую mutation в Vuex
  • mutation изменяет state, и компоненты автоматически обновляются

3. Шаблон с кнопками:

<template>
  <p>Счётчик: {{ $store.state.count }}</p>
  <p>Удвоенное значение: {{ $store.getters.doubleCount }}</p>
  <button @click="increase">➕ Увеличить</button>
  <button @click="decrease">➖ Уменьшить</button>
</template>
Что здесь происходит?
  • отображаем текущее значение счётчика из state
  • отображаем удвоенное значение через getter
  • кнопки вызывают методы для изменения счётчика через mutations

Важно помнить:

  • Изменять state можно только через mutations и commit
  • В компоненте вызываются commit("mutationName") для обновления данных в Vuex
  • Mutations должны быть синхронными, для асинхронных действий используются actions (обсудим далее)

📌 Actions

Actions - это механизм Vuex для выполнения асинхронных операций перед изменением состояния через mutations.

Зачем нужны actions?

  • 🔄 Позволяют выполнять асинхронные задачи (таймауты, запросы к серверу)
  • ⚙️ Вызывают mutations для изменения state после завершения асинхронной операции
  • 🧩 Помогают разделить логику асинхронных операций и синхронных изменений состояния

Как добавить actions в store.js

Чтобы добавить action, нужно расширить объект store новым свойством actions:

const store = createStore({
  state: {
    count: 0
  },
  mutations: {
    increment(state) {
      state.count++;
    }
  },
  actions: {
    // Асинхронное увеличение счетчика
    asyncIncrement({ commit }) {
      setTimeout(() => {
        commit("increment"); // 🔄 Вызываем мутацию через 2 секунды
      }, 2000);
    }
  }
});
Разбираем код:
  • actions – объект, где объявляются все actions
  • asyncIncrement – имя action для асинхронного увеличения счётчика
  • первый аргумент функции – объект контекста, из которого мы берём commit
  • внутри action используем setTimeout для имитации асинхронной операции
  • после задержки вызываем commit("increment"), чтобы изменить state через mutation

Как вызвать action из компонента?

Чтобы вызвать action из компонента, используйте метод dispatch() .
dispatch() - это способ запустить action в Vuex.

❌ Ошибка: нельзя вызвать action через commit():
this.$store.commit("asyncIncrement");
mutations не могут быть асинхронными!

✅ Правильно: используйте dispatch() для вызова action:
this.$store.dispatch("asyncIncrement");

<script>
export default {
  methods: {
    asyncIncrease() {
      this.$store.dispatch("asyncIncrement"); // 🔄 Запускаем асинхронное действие
    }
  }
};
</script>
Разбираем код:
  • в методе компонента вызываем this.$store.dispatch("asyncIncrement")
  • это запускает action asyncIncrement в Vuex
  • после 2 секунд action вызывает mutation increment, которая изменяет state
<template>
  <button @click="asyncIncrease">Асинхронно увеличить</button>
</template>
Разбираем код:
  • кнопка в шаблоне вызывает метод asyncIncrease
  • метод запускает action через dispatch()
  • после задержки state обновляется через mutation
  • ✔ dispatch() — только для actions
  • ✔ commit() — только для mutations
  • ✔ Actions идеально подходят для таймаутов, серверных запросов, сложных операций

Обобщим:

// Мутация (для синхронных изменений)
this.$store.commit("increment");

// Action (для асинхронных изменений)
this.$store.dispatch("asyncIncrement");

FAQ / Ошибки

  • В: Почему нельзя менять state напрямую?
    О: Любые изменения должны контролироваться — это делает поведение приложения предсказуемым!
  • В: Можно ли использовать несколько getters и mutations?
    О: Да! Объявляйте столько, сколько нужно для логики приложения.
  • В: Как отлаживать getters и mutations?
    О: Используйте консольные логи внутри функций getters/mutations, чтобы видеть, когда и с какими аргументами они вызываются.
  • В: Почему mutations только синхронные?
    О: Это правило Vuex — чтобы не возникало путаницы при изменении данных. Для асинхронных операций используйте actions.
