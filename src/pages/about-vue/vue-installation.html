Установка Vue через CDN и CLI

Vue.js — современный фреймворк для создания интерактивных веб-приложений. Он удобен, гибок и отлично подходит как для маленьких, так и для больших проектов.
В этой инструкции рассмотрим два способа установки: через CDN для быстрого старта и CLI для профессиональной разработки.

Основные понятия

  • Vue.js — фреймворк для создания пользовательских интерфейсов.
  • Работает по принципу реактивности : изменения данных автоматически отображаются на странице.
  • Компонентный подход : проект строится из переиспользуемых частей.
  • Два способа подключения:
    • CDN — быстрое подключение через тег <script> в HTML.
    • CLI — инструмент для создания крупных проектов с удобной структурой и сборкой.
  • Vue подходит для любого уровня: от новичков до профессионалов.

Примеры установки Vue

Установка Vue через CDN

<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>
  • Добавьте строку перед закрывающим </body> в вашем HTML.
  • Vue будет доступен во всём окне браузера.
  • Удобно для тестирования, прототипов и маленьких проектов.
<!DOCTYPE html>
  <html lang="ru">
    <head>
      <meta charset="UTF-8">
      <meta name="viewport" content="width=device-width, initial-scale=1.0">
      <title>Vue CDN</title>
      <script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>
    </head>
    <body>
      <div id="app">
        <h1>{{ message }}</h1>
      </div>
      <script>
        const app = Vue.createApp({
          data() {
            return {
               message: "Привет, Vue!"
            };
          }
        });
        app.mount("#app");
      </script>
    </body>
  </html>
Объяснение кода
  • {{ message }} — это интерполяция, Vue подставит значение переменной.
  • Vue.createApp() создает приложение, app.mount("#app") — привязывает его к HTML.
  • Весь JS-код можно размещать прямо внутри HTML-файла.

Установка Vue через CLI

Для серьёзных проектов рекомендуется использовать Vue CLI — это инструмент командной строки, который создаёт структуру проекта и настраивает сборку.

1. Установка Vue CLI

//Установка Vue CLI (если не установлен)
npm install -g @vue/cli

2. Создание нового проекта

//Создание нового проекта
vue create my-vue-app
Объяснение кода
  • Первой командой устанавливается глобально инструмент Vue CLI.
  • Второй командой создаётся новый проект со структурой и конфигом.
  • CLI предложит выбрать настройки — новичкам можно выбрать Default.

3. Запуск проекта

//Переход в папку проекта
cd my-vue-app
//запуск сервера
npm run serve
Объяснение кода
  • Запускает локальный сервер для разработки.
  • После запуска проект откроется по адресу http://localhost:8080 . 
<template>
   <h1>Привет, Vue!</h1>
</template>

<script>
export default {
   name: "App"
};
</script>

<style>
h1 {
   color: blue;
}
</style>
Объяснение кода
  • Это основной компонент src/App.vue в CLI-проекте.
  • Всё, что в теге <template> , отображается на странице.
  • Стили, JS и шаблон в одном файле — это удобно!

🚀 Вывод: CDN-подключение быстрое и простое, но для больших проектов лучше использовать CLI — так вы получите мощный инструмент для разработки с поддержкой компонентов, маршрутизации и сборки.

Практика: минимальное Vue-приложение через CDN

Описание: создадим простое Vue-приложение, которое отображает сообщение и меняет его при нажатии кнопки.

Пример:

<div id="app">
  <h1>{{ message }}</h1>
  <button @click="message = 'Вы только что нажали кнопку!'">Изменить сообщение</button>
</div>

<script>
const app = Vue.createApp({
  data() {
    return {
      message: "Привет, Vue!"
    }
  }
});
app.mount('#app');
</script>
Что нужно сделать:
  1. Создайте HTML-файл и подключите Vue через CDN.
  2. Добавьте HTML-разметку с заголовком и кнопкой.
  3. Создайте Vue-приложение с реактивной переменной message.
  4. Привяжите обработчик события к кнопке для изменения сообщения.
  5. Запустите приложение с помощью mount('#app').
  6. Попробуйте изменить сообщение, нажав кнопку ниже.

{{ message }}

Объяснение кода
  • Интерполяция {{ message }} отображает текущее значение переменной message .
  • Директива @click привязывает обработчик события к кнопке — при нажатии изменяется значение message .
  • Метод Vue.createApp() создает Vue-приложение с реактивными данными.
Этот пример показывает реактивность Vue — при нажатии на кнопку значение message изменится и сразу отобразится в заголовке.

Советы и типичные ошибки

  • CDN-способ подходит только для учебных или маленьких проектов.
  • Для крупных приложений используйте CLI — получите управление зависимостями и сборкой.
  • Проверяйте, чтобы тег <script> с Vue был последним перед </body> .
  • В CLI-проектах не изменяйте public/index.html напрямую — работайте с компонентами в src/ .
Частая ошибка: забыть про mount("#app") — без этого приложение не привяжется к HTML.

Лучшие практики

  • Всегда отделяйте данные, логику и шаблон в компонентах.
  • Используйте компоненты для повторяющихся частей интерфейса.
  • В больших проектах ведите структуру папок и файлов — не храните весь код в одном файле.
  • Следите за обновлениями Vue — новые версии приносят улучшения и новые возможности.
Начинайте с CDN, чтобы попробовать, но переходите на CLI для серьёзных задач — это откроет все возможности Vue!
