API во Vue: краткий обзор

В Vue под “API” подразумевается не серверное взаимодействие, а способы работы с возможностями самого фреймворка.
Основные подходы — Options API (классический) и Composition API (современный). Они позволяют создавать реактивные компоненты любого уровня сложности.

Options API (Классический подход)

Options API — “традиционный” синтаксис Vue 2 и совместим с Vue 3.
Описывается с помощью объектных опций: data, methods, computed, watch, props и др. 

<template>
  <div>
    Привет, {{ name }}!
    <button @click="sayHi">Поздороваться</button>
  </div>
</template>

<script>
export default {
  data() { return { name: "Vue" } },
  methods: {
    sayHi() { alert(`Привет, ${this.name}!`) }
  }
}
</script>

Composition API (Современный подход)

Composition API — синтаксис Vue 3 для создания логики через функции и реактивные переменные.
Используются примитивы ref, reactive, computed, watch и hooks. 

<script setup>
import { ref } from 'vue'
const name = ref('Vue 3')
function sayHi() {
  alert(`Привет, ${name.value}!`)
}
</script>

<template>
  <div>Привет, {{ name }}! <button @click="sayHi">Поздороваться</button></div>
</template>
  • Повторное использование логики через composables (пользовательские функции).
  • Явный импорт реактивных инструментов из vue.
  • Совместим с TypeScript и идеально для сложных компонентов.

Реактивность во Vue

Основной принцип Vue — реактивное отслеживание данных.
Любые изменения реактивных переменных немедленно отражаются в шаблоне.

  • ref() — реактивное значение-примитив
  • reactive() — реактивный объект
  • computed() — реактивное вычисляемое свойство на основе других реактивных значений
  • watch() — слушатель изменений реактивных данных
<script setup>
import { ref, computed, watch } from 'vue'
const count = ref(0)
const double = computed(() => count.value * 2)
watch(count, (newVal) => console.log('Новое значение:', newVal))
</script>

Хуки жизненного цикла компонента

У каждого Vue-компонента есть цикл жизни: создание, монтирование, обновление и уничтожение.
Вы можете выполнять действия в определённые моменты с помощью хуков: 

<script setup>
import { onMounted, onUnmounted } from 'vue'
onMounted(() => {
  console.log('Компонент смонтирован!')
})
onUnmounted(() => {
  console.log('Компонент уничтожен!')
})
</script>
  • onBeforeMount — до монтирования
  • onMounted — после монтирования
  • onUpdated — после обновления
  • onUnmounted — после удаления компонента

Работа с DOM через ref

Для получения доступа к DOM-элементу используйте директиву ref внутри шаблона и функцию ref из API. 

<template>
  <input ref="inputEl" />
  <button @click="focusInput">Фокус</button>
</template>
<script setup>
import { ref, onMounted } from 'vue'
const inputEl = ref(null)
function focusInput() {
  inputEl.value && inputEl.value.focus()
}
</script>

Best Practices для работы с API Vue

  • Выбирайте Composition API для новых проектов и сложных компонентов.
  • Options API хорош для небольших и учебных примеров.
  • Логику, которую можно переиспользовать, выносите в composables (пользовательские функции).
  • Используйте хуки жизненного цикла для асинхронных запросов, подписок, очистки ресурсов.
  • Для сложной реактивности предпочитайте computed и watch, избегайте сложных вложенных watch без необходимости.
  • Всегда используйте ref и reactive для реактивности внутри Composition API.
🚀 Главное: Организуйте логику понятно, используйте сильные стороны двух API, не бойтесь совмещать варианты там, где это реально ускоряет разработку.
