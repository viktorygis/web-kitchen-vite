Введение во Vue JS

Vue.js — прогрессивный JavaScript-фреймворк для создания пользовательских интерфейсов. Он прост в освоении, гибок и мощен.
Vue позволяет создавать реактивные веб-приложения с минимальными усилиями.

История фреймворка

Vue был создан Эваном Ю в 2014 году как легковесная альтернатива более крупным фреймворкам, таким как Angular и React. С тех пор Vue быстро набрал популярность благодаря своей простоте и эффективности.

Что нужно знать о Vue

Vue ориентирован на декларативное программирование и компонентный подход. Это позволяет разработчикам создавать сложные интерфейсы, разбивая их на небольшие, переиспользуемые части.

Vue сочетает в себе лучшие идеи из других фреймворков, предлагая декларативный подход к созданию интерфейсов и мощную систему компонентов.
Сегодня Vue используется во множестве проектов, от небольших сайтов до крупных корпоративных приложений.

Установка Vue

Установка Vue проста и может быть выполнена несколькими способами в зависимости от ваших потребностей.
Вот два основных метода:

Быстрый старт через CDN

Самый простой способ начать использовать Vue — подключить его через CDN (Content Delivery Network).

<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>
Объяснение кода:
  • Этот скрипт подключает последнюю версию Vue 3 к вашему проекту.
  • Подходит для небольших проектов и быстрого прототипирования.

Профессиональная разработка через CLI

Для более сложных проектов рекомендуется использовать Vue CLI (Command Line Interface) для создания и управления проектами Vue.

npm install -g @vue/cli
vue create my-vue-app
Объяснение кода:
  • Первая команда устанавливает Vue CLI глобально на вашем компьютере.
  • Вторая команда создаёт новый проект Vue с именем my-vue-app.
  • Этот метод подходит для профессиональной разработки с использованием современных инструментов сборки и управления зависимостями.

🚀 Теперь вы готовы начать работу с Vue!
Выберите подходящий способ установки и приступайте к созданию своих первых Vue-приложений.

Шаблоны Vue

Шаблоны Vue — это HTML-разметка, которая может содержать динамические данные и директивы Vue для управления поведением элементов.

Основные элементы шаблона Vue

  • Шаблоны должны быть обернуты в один корневой элемент (например, <div>).
  • Используйте двойные фигурные скобки {{ }} для интерполяции данных.
  • Директивы Vue начинаются с префикса v- и управляют поведением элементов.
  • Шаблоны могут содержать условные конструкции и циклы с помощью директив.

Пример шаблона Vue

<div id="app">
  <h1>Привет, {{ name }}!</h1>
  <a v-bind:href="website">Мой сайт</a>
  <ul>
    <li v-for="user in users" :key="user">{{ user }}</li>
  </ul>
</div>
<script>
  Vue.createApp({
    data() {
      return {
        name: "Алексей",
        website: "https://example.com",
        users: ["Аня", "Макс", "Иван"]
      }
    }
  }).mount("#app");
</script>
Объяснение кода:
  • {{ name }} — подставляет значение переменной name из data().
  • v-bind:href="website" — — дт связывает-с атрибут href с переменной website.
  • v-for="user in users" :key="user" — перебирает массив users и создаёт элемент для каждого значения.

Интерполяция

Интерполяция - это способ внедрения динамических данных в HTML-шаблоны Vue с помощью двойных фигурных скобок {{ }}.
Позволяет отображать значения переменных и выражений прямо в разметке.

Пример интерполяции:

В этом примере заголовок отображает значение переменной name из data().

<h1>Имя: {{ name }}</h1>
Объяснение:
  • {{ name }} — синтаксис интерполяции для вывода значения переменной name.
  • При изменении значения name в data(), заголовок автоматически обновится на странице.

Пример полного приложения с интерполяцией:

<div id="app">
  <h1>Привет, {{ name }}!</h1>
</div>

<script>
  Vue.createApp({
    data() {
      return {
        name: "Алексей"
      }
    }
  }).mount("#app");
</script>
Объяснение кода:
  • Шаблон содержит заголовок с интерполяцией {{ name }}.
  • В data() определена переменная name со значением "Алексей".
  • При запуске приложения Vue автоматически подставит значение name в заголовок.

🚀 Интерполяция — простой и мощный способ отображения динамических данных в шаблонах Vue, обеспечивающий реактивность и удобство разработки.

Директивы Vue

Директивы Vue — специальные атрибуты в шаблоне, начинающиеся с префикса v-, которые указывают Vue, как обрабатывать элементы.
Они позволяют добавлять динамическое поведение, условный рендеринг, циклы и многое другое.

Основные директивы Vue:

  • v-bind — связывает атрибуты HTML с данными из data().
  • v-if — условно рендерит элемент в DOM.
  • v-show — управляет видимостью элемента через CSS.
  • v-for — перебирает массив и отображает список элементов.
  • v-model — двустороннее связывание данных с элементами формы(текстовое поле, чекбокс, радио-кнопки, выпадающее меню). Она автоматически обновляет значение переменной в data(), когда пользователь изменяет данные в поле ввода.

Для чего нужны:

  • Управление отображением элементов (например, условный вывод с помощью v-if).
  • Перебор массивов и создание списков с помощью v-for.
  • Связывание атрибутов HTML с данными из data() с помощью v-bind.
  • Двустороннее связывание данных с элементами формы через v-model.

Альтернативы:

  • Использование JavaScript для манипуляции DOM напрямую (менее эффективно и реактивно).
  • Другие фреймворки, такие как React или Angular, которые имеют свои собственные подходы к управлению состоянием и рендерингом.

🚀 Директивы — мощный инструмент для управления поведением элементов в шаблонах Vue, обеспечивающий реактивность и удобство разработки.
✅ Без директив работает vue? Нет, директивы являются неотъемлемой частью Vue.js и обеспечивают его реактивность и динамическое поведение.

Работа с данными через data()

В Vue все реактивные данные приложения объявляются внутри функции data(), которая возвращает объект с переменными.
Эти переменные автоматически становятся реактивными и могут быть использованы в шаблоне.

Для чего нужна data():

  • 🗄️ Хранить данные приложения
  • 🎛️ Управлять интерфейсом
  • ⚙️ Делать страницу динамической

Объявление переменных в data():

Объявление переменных в data() - это создание свойств объекта, которые Vue будет отслеживать и обновлять в шаблоне при изменении.
Это делается внутри функции data(), которая возвращает объект с нужными переменными.

<script>
Vue.createApp({
  data() {
    return {
      message: "Привет, Vue!",
      count: 0
    }
  }
});
</script>
Объяснение кода:
  • Функция data() возвращает объект с переменными message и count.
  • Эти переменные становятся реактивными и могут быть использованы в шаблоне.

Использование переменных в шаблоне:

Использование переменных в шаблоне - это процесс внедрения реактивных данных, объявленных в data(), в HTML-разметку Vue с помощью интерполяции или директив.
Это позволяет динамически отображать и обновлять содержимое страницы на основе состояния приложения.

<div id="app">
  <h1>{{ message }}</h1>
  <p>Счётчик: {{ count }}</p>
</div>
Объяснение кода:
  • В шаблоне используется интерполяция {{ message }} для отображения значения переменной message.
  • Переменная count также отображается в параграфе с помощью интерполяции.
  • При изменении значений в data(), шаблон автоматически обновится.

Динамическое изменение данных

Динамическое изменение данных в Vue происходит через методы, которые вызываются при событиях или других условиях. Эти изменения автоматически обновляют шаблон, благодаря реактивности Vue. 

<div id="app">
  <button @click="count++">+1</button>
  <p>Счётчик: {{ count }}</p>
</div>
Объяснение кода:
  • При клике на кнопку значение переменной count увеличивается на 1.
  • Шаблон автоматически обновляет отображение счётчика благодаря реактивности Vue.

🚀 data() — основа для хранения и управления реактивными данными в Vue.

✅ Объявление переменных в data() делает их доступными для использования в шаблоне и обеспечивает автоматическое обновление интерфейса при изменении данных.

✅ Использование переменных в шаблоне позволяет создавать динамические и интерактивные веб-приложения с минимальными усилиями.

✅ Динамическое изменение данных через методы обеспечивает реактивность и удобство взаимодействия с пользователем.

События во Vue

События в Vue — это способ реагировать на действия пользователя: 👆 клики, 👆 наведение мыши, ⌨️ нажатия клавиш и 🖱️ другие взаимодействия.
С помощью событий можно создавать 🖥️интерактивные интерфейсы, улучшать 🌟пользовательский опыт и ⚙️делать приложения более динамичными.

Обработка событий в Vue

v-on — директива Vue для обработки событий.
Она позволяет привязывать события к элементам и вызывать методы при их срабатывании.
Существует короткая запись для v-on: — символ @.

Пример использования:

В этом примере кнопка увеличивает счётчик при каждом клике.

<div id="demo-events">
  <button @click="count++">+1</button>
  <p>Счётчик: {{ count }}</p>
</div>
<script>
Vue.createApp({
  data() { return { count: 0 } }
}).mount("#demo-events");
</script>
Объяснение кода:
  • @click — это сокращённая запись для v-on:click, которая привязывает событие клика к кнопке.
  • count++ — это выражение, которое увеличивает значение переменной count на 1 при каждом клике.
  • @click="count++" — это присваение обработчика события клика, который изменяет значение count.
  • {{ count }} — интерполяция, которая отображает текущее значение переменной count в параграфе.
  • { count: 0 } — инициализация (инициализация это установка начального значения) переменной count в data() с начальным значением 0.

Модификаторы событий во Vue

Модификаторы событий во Vue — специальные суффиксы, добавляемые к директиве v-on (или её сокращённой форме @), которые изменяют поведение обработчиков событий.
Они позволяют легко управлять распространением событий, предотвращать действия по умолчанию и выполнять другие полезные операции без необходимости писать дополнительный код.

Основные модификаторы событий:

  • .stop — останавливает распространение события (event propagation).
  • .prevent — предотвращает действие по умолчанию (например, отправку формы).
  • .capture — использует фазу захвата для события.
  • .self — срабатывает только если событие произошло на самом элементе, а не на его потомках.
  • .once — обработчик сработает только один раз и затем будет удалён.

Пример использования модификаторов:

В этом примере кнопка показывает сообщение при клике, но останавливает распространение события. Без модификатора событие распространялось бы дальше по DOM дереву, что может привести к нежелательным последствиям, таким как повторное срабатывание обработчиков событий на родительских элементах.

<div id="app">
  <button @click.stop="handleClick">Кликни меня</button>
</div>
<script>
Vue.createApp({
  methods: {
    handleClick() {
      alert("Кнопка нажата!");
    }
  }
}).mount("#app");
</script>
Объяснение кода:
  • @click.stop="handleClick" — привязывает событие клика к кнопке с модификатором .stop, который останавливает распространение события.
  • Метод handleClick вызывается при клике на кнопку и отображает сообщение.

Основные события во Vue

v-model

v-model - директива Vue для двустороннего связывания данных между элементами формы и переменными в data().
Позволяет автоматически обновлять значение переменной при изменении ввода пользователя и наоборот.

Для чего используется:

  • Автоматическое обновление переменных при вводе в текстовые поля, то есть <input type="text">
  • Связывание состояния чекбоксов то есть <input type="checkbox">
  • Привязка выбранной опции в группе радио-кнопок то есть <input type="radio">
  • Привязка выбранных значаний в выпадающих списках к данным select то есть <select>

Пример использования v-model:

<div id="demo-vmodel">
  <input v-model="message" placeholder="Введите сообщение">
  <p>Вы ввели: {{ message }}</p>
</div>
<script>
Vue.createApp({
  data() {
    return { message: "" }
  }
}).mount("#demo-vmodel");
</script>
Объяснение кода:
  • v-model="message" связывает значение текстового поля с переменной message в data().
  • При вводе текста в поле, значение message автоматически обновляется.
  • Интерполяция {{ message }} отображает текущее значение переменной на странице.
🚀 v-model — мощный инструмент для создания интерактивных форм и управления состоянием ввода пользователя в Vue.js.
✅ Обеспечивает двустороннее связывание данных между элементами формы и переменными, упрощая разработку динамических интерфейсов.

Методы в Vue

methods — это функции, определённые внутри объекта methods в компоненте Vue. Они обычно используются для обработки событий, таких как клики по кнопкам, изменения ввода и другие взаимодействия пользователя.

Где используются методы:

  • Обработчики событий (например, @click, @input)
  • Логика, связанная с действиями пользователя
  • Манипуляции с данными компонента

Как использовать методы:

Методы определяются в объекте methods компонента Vue. Они могут быть вызваны из шаблона с помощью директивы @click, @input и других.

Пример 1:

Создадим простой счётчик с кнопкой для увеличения значения.

Счётчик: {{ count }}

<div id="counter-demo">
  <p>Счётчик: {{ count }}</p>
  <button @click="increaseCount">Увеличить</button>
</div>

<script>
Vue.createApp({
  data() {
    return { count: 0 }
  },
  methods: {
    increaseCount() {
      this.count++;
    }
  }
}).mount("#counter-demo");
</script>
Объяснение кода:

computed: вычисляемые свойства

computed определяются в объекте computed. Пересчитываются только при изменении зависимостей (например, data()) и кешируются.

Что нужно знать о computed:

  • Используются для вычислений на основе данных компонента.
  • Пересчитываются только при изменении зависимостей.
  • Кешируются для оптимизации производительности.

В отличие от методов, которые выполняются при каждом вызове, вычисляемые свойства кешируются до тех пор, пока не изменятся их зависимости. Это делает их идеальными для сложных вычислений, которые не нужно выполнять каждый раз при рендеринге.

Пример:

Создадим вычисляемое свойство, которое возвращает удвоенное значение счётчика.

Исходное значение: {{ computedCount }}

Удвоенное (computed): {{ doubleCount }}

<div id="computed-demo">
  <h4>Исходное значение: {{ computedCount }}</h4>
  <h4>Удвоенное (computed): {{ doubleCount }}</h4>
  <button @click="computedCount++">Увеличить</button>
</div>

<script>
Vue.createApp({
  data() { return { computedCount: 5 } },
  computed: {
    doubleCount() { return this.computedCount * 2 }
  }
}).mount("#computed-demo");
</script>
Объяснение кода:
  • В объекте data() определена переменная computedCount, инициализированная значением 5.
  • В объекте computed определено вычисляемое свойство doubleCount, которое возвращает удвоенное значение computedCount.
  • В шаблоне отображаются оба значения, и кнопка увеличивает computedCount на 1 при каждом клике.

🚀 computed идеально подходят для производительных вычислений, которые зависят от реактивных данных.

watch: наблюдатель

watch определяются в объекте watch. Следят за изменением переменной и запускают функцию, например, при изменении data().

Что нужно знать о watch:

  • Используются для асинхронных действий и побочных эффектов.
  • Следят за изменением реактивных данных.
  • Не кешируются.

В отличие от computed , которые предназначены для вычислений на основе данных, наблюдатели watch используются для выполнения действий в ответ на изменения данных, таких как запросы к API или обновление других частей состояния.

Пример:

Создадим наблюдатель, который будет логировать изменения счётчика в консоль.

Счётчик: {{ scoring }}

<div id="watch-demo">
  <h4>Счётчик: {{ scoring }}</h4>
  <button @click="scoring++">Увеличить</button>
</div>
<script>
Vue.createApp({
  data() { return { scoring: 0 } },
  watch: {
    scoring(newValue, oldValue) {
      console.log(`Счётчик изменился с ${oldValue} на ${newValue}`);
    }
  }
}).mount("#watch-demo");
</script>
Объяснение кода:
  • В объекте data() определена переменная scoring, инициализированная значением 0.
  • В объекте watch определён наблюдатель для переменной scoring, который логирует в консоль старое и новое значение при каждом изменении.
  • В шаблоне отображается текущее значение счётчика, и кнопка увеличивает его на 1 при каждом клике.

🚀 watch подходит для асинхронных действий и побочных эффектов.
Например, запрос к API при изменении значения.

Циклы и условия во Vue

Циклы и условные конструкции во Vue — ключевые инструменты для динамического отображения данных и управления видимостью элементов на странице.

Циклы и условные конструкции нужны для:

  • Динамического создания списков и элементов на основе данных
  • Управления видимостью контента в зависимости от состояния приложения
  • Создания интерактивных интерфейсов с адаптивным отображением информации

Основные директивы:

  • v-for — директива для перебора массивов и отображения элементов списка. Она позволяет динамически создавать элементы на основе данных, что делает интерфейсы более гибкими и интерактивными.
  • v-if — директива для условного отображения элементов на основе логических выражений. Если условие истинно, элемент отображается; если ложно — скрывается.
  • v-else — директива для отображения альтернативного контента, если условие v-if не выполнено. Используется вместе с v-if для создания ветвлений в отображении.
  • v-else-if — директива для добавления дополнительных условий к цепочке v-if/v-else. Позволяет проверять несколько условий последовательно. Используется вместе с v-if и v-else.
  • v-show — директива для условного отображения элементов с помощью CSS-свойства display. Элемент остаётся в DOM, но его видимость управляется стилями.

Пример использования циклов и условий:

Создадим список пользователей с условием отображения сообщения, если список пуст.

<div id="demo-cycles">
  <ul>
    <li v-for="user in users" :key="user">{{ user }}</li>
  </ul>
  <p v-if="users.length === 0">Нет пользователей</p>
</div>
<script>
Vue.createApp({
  data() { return { users: ["Аня", "Макс", "Иван"] } }
}).mount("#demo-cycles");
</script>
Объяснение кода:
  • Директива v-for="user in users" перебирает массив users и создаёт элемент списка для каждого пользователя.
  • Атрибут :key="user" помогает Vue эффективно отслеживать элементы при обновлении списка.
  • Директива v-if="users.length === 0" отображает сообщение "Нет пользователей", если массив пуст.
🚀 Циклы и условия во Vue позволяют создавать динамические и интерактивные интерфейсы, адаптирующиеся к состоянию данных.
Чаще всего используется вместе директива v-for и условные конструкции v-if, v-else.
Реже используется v-show для управления видимостью через CSS.

Практика: мини-приложение на Vue

Давайте создадим простое мини-приложение на Vue, которое позволяет добавлять пользователей в список и отображать приветствие.

В этом примере мы используем v-model для двустороннего связывания данных, v-for для отображения списка пользователей, v-if для условного отображения приветствия и методы Vue для добавления пользователей и переключения видимости приветствия.

Код мини-приложения:

<div id="vue-demo">
  <input v-model="newUser" placeholder="Добавить пользователя">
  <button @click="addUser">Добавить</button>
  <ul>
    <li v-for="user in users" :key="user">{{ user }}</li>
  </ul>
  <button @click="toggleHello">Показать/скрыть приветствие</button>
  <p v-if="showHello">Привет, {{ users[0] || 'гость' }}!</p>
</div>

<script>
Vue.createApp({
  data() {
    return {
      users: ["Аня", "Макс", "Иван"],
      newUser: "",
      showHello: true
    }
  },
  methods: {
    addUser() {
      if (this.newUser) {
        this.users.push(this.newUser);
        this.newUser = "";
      }
    },
    toggleHello() {
      this.showHello = !this.showHello;
    }
  }
}).mount('#vue-demo');
</script>
Объяснение кода:
  • Поле ввода с v-model="newUser" связывает значение с переменной newUser.
  • Кнопка "Добавить" вызывает метод addUser, который добавляет нового пользователя в массив users.
  • Список пользователей отображается с помощью v-for, перебирая массив users.
  • Кнопка "Показать/скрыть приветствие" вызывает метод toggleHello, который переключает значение showHello.
  • Приветствие отображается условно с помощью v-if="showHello", показывая имя первого пользователя или "гость", если список пуст.

Советы и частые ошибки

Вот несколько советов, которые помогут вам эффективно использовать Vue и избежать распространённых ошибок:

  • Всегда используйте директивы Vue (v-if, v-for, v-model) для управления отображением и взаимодействием с данными.
  • Не изменяйте напрямую свойства объекта или массива в data(); используйте методы Vue для реактивности.
  • Используйте уникальные ключи с директивой v-for для оптимальной производительности при рендеринге списков.
  • Избегайте сложной логики в шаблонах; лучше выносите её в методы или вычисляемые свойства.
  • Проверяйте консоль браузера на наличие ошибок Vue, которые могут помочь в отладке.
Следуя этим советам, вы сможете создавать более надёжные и эффективные приложения на Vue!

Вот несколько лучших практик для работы с Vue, которые помогут вам создавать чистый и поддерживаемый код:

  • Разделяйте логику и представление, используя методы и вычисляемые свойства вместо сложных выражений в шаблонах.
  • Используйте компоненты для повторно используемых частей интерфейса, чтобы улучшить структуру и читаемость кода.
