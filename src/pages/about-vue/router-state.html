Хранение состояния маршрутов: query и hash в Vue Router

Состояние маршрута — это параметры, которые сохраняются в адресной строке (URL) и позволяют не терять фильтры, сортировку и позицию на странице даже после обновления!
Это важно для поиска, фильтрации товаров в магазине, сортировки, перехода к нужному блоку и удобства пользователя.

Что изучаем?

  • Как использовать query для передачи параметров в URL
  • Что такое hash и зачем он нужен
  • Как программно менять query и hash
  • Можно ли их комбинировать?

Передача параметров через query

Query-параметры идут после ? в URL и позволяют фильтровать или искать данные.
Они сохраняются даже после перезагрузки страницы!

Пример:

<router-link :to="{ path: '/products', query: { category: 'laptops' } }">📱 Фильтр: Ноутбуки</router-link>

Итоговый URL: /products?category=laptops

Как получить query в компоненте?

this.$route.query.category // "laptops"

Можно передавать сразу несколько параметров: /search?keyword=Vue&sort=popular

Программно изменить query:

this.$router.push({ path: "/products", query: { category: "phones" } });

Query сохраняется в истории и после обновления страницы.
Для удаления query передай пустой объект: 

this.$router.push({ path: "/products", query: {} });

Якорь ( hash ) в маршруте

Hash — всё, что после # в URL.
Используется для прокрутки к нужному элементу на странице (например, #features или #reviews ).

Как добавить hash в ссылку:

<router-link :to="{ path: '/docs', hash: '#installation' }">Перейти к установке</router-link>

Программно перейти к якорю:

this.$router.push({ path: "/about", hash: "#features" });

Как узнать hash в компоненте?

this.$route.hash // Например: "#setup"

Hash работает только внутри одной страницы и не переключает маршрут.
Если нет нужного id в разметке, hash не сработает.

Пример кнопки для перехода к блоку с id="features": 

<button @click="goToFeatures">🔗 Перейти к функциям</button>

<script>
export default {
  methods: {
    goToFeatures() {
      this.$router.push({ path: "/about", hash: "#features" });
    }
  }
}
</script>

Комбинируем query и hash

Можно передать и фильтр, и якорь одновременно:

this.$router.push({ path: "/products", query: { category: "phones" }, hash: "#filters" });
// URL: /products?category=phones#filters
  • Query сохраняется даже после перезагрузки 🔄
  • Hash просто прокручивает страницу к нужному элементу

Что произойдет при перезагрузке /products?category=phones#reviews ?

Параметры и якорь останутся в адресе.
После загрузки страницы фильтр будет активен, а страница прокрутится к блоку с id="reviews".

Порядок обработки:

  • Vue Router сначала разбирает path
  • Затем применяет query
  • В конце переходит к hash (якорю), если он есть

Практика: работа с query и hash

  • Сделай ссылку, которая фильтрует товары по категории ноутбуки
  • Сделай кнопку, которая по клику прокручивает страницу к блоку с id="features"
  • Сделай ссылку, которая фильтрует по цене и сразу прокручивает к блоку с фильтрами

App.vue

<!-- App.vue -->
<template>
  <div>
    <router-link :to="{ path: '/products', query: { category: 'laptops' } }">Ноутбуки</router-link>
    <button @click="goToFeatures">К функциям</button>
    <router-link :to="{ path: '/products', query: { sort: 'price' }, hash: '#filters' }">Сортировать и к фильтрам</router-link>
    <router-view/>
  </div>
</template>

<script>
export default {
  methods: {
    goToFeatures() {
      this.$router.push({ path: "/about", hash: "#features" });
    }
  }
}
</script>

Частые ошибки и вопросы

  • Ошибка: this.router.push(...) — забыли $ перед router!
  • Ошибка: не передали query или hash как объект!
  • Вопрос: Как одновременно передать query и hash?
    this.$router.push({ path: "/products", query: { sort: "price" }, hash: "#filters" })
  • Вопрос: Почему параметры и hash не исчезают после обновления?
    — Они хранятся в адресе и восстанавливаются при загрузке страницы.
Query и hash — твой инструмент для удобной и "живой" навигации!
