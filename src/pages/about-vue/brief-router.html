Маршрутизация во Vue

Vue Router — официальная библиотека для маршрутизации в приложениях на Vue.js. Она позволяет создавать одностраничные приложения (SPA) с навигацией между "страницами" без перезагрузки.
Маршрутизация во Vue.js - это процесс управления навигацией между различными компонентами и представлениями в одностраничном приложении (SPA) с использованием библиотеки Vue Router.
Навигация осуществляется путем определения маршрутов, которые связывают URL-пути с соответствующими Vue-компонентами, позволяя пользователям переходить между различными частями приложения без полной перезагрузки страницы.
маршруты — это правила, которые определяют, какой компонент должен отображаться при переходе на определенный URL-путь.
Виды маршрутов: статические (фиксированные пути), динамические (с параметрами), вложенные (children), редиректы и обработка ошибок 404.

Основные возможности Vue Router:

  • 📍 Определение маршрутов: связывание URL-путей с компонентами Vue.
  • 🔄 Навигация: переход между страницами без перезагрузки страницы.
  • 🧩 Динамические маршруты: использование параметров в URL для отображения различных данных.
  • 📂 Вложенные маршруты: создание иерархии страниц и компонентов.
  • ➡️ Программная навигация: переход по маршрутам через код (например, при клике на кнопку).
  • 📋 Состояние маршрутов: управление историей браузера и состоянием навигации.
  • ↪️ Редиректы: перенаправление пользователей с одного пути на другой.
  • ❓ Обработка ошибок 404: отображение страницы ошибки при неправильном URL.
  • 🔐 Защита маршрутов: реализация аутентификации и авторизации для доступа к определенным страницам.

Структура роутера в проекте

Структура роутера в проекте - это организация файлов и компонентов, связанных с маршрутизацией, для обеспечения удобной навигации и управления состоянием приложения.
Обычно маршруты настраиваются в отдельном файле (например, router.js), а компоненты-страницы хранятся в папке views/. Главный компонент App.vue содержит <router-view> для отображения текущего маршрута.

Типичная структура проекта с Vue Router:

📂 src/
 ├── 📂 components/
 ├── 📂 views/
 │    ├── Home.vue
 │    ├── About.vue
 │    ├── Profile.vue
 │    ├── NotFound.vue
 │    ├── Dashboard.vue
 ├── router.js
 ├── App.vue
 ├── main.js
Описание ключевых файлов и папок:
  • views/ — папка для компонентов-страниц.
  • router.js — файл для настройки маршрутов.
  • App.vue — главный компонент с <router-view>.
  • main.js — точка входа, где создается Vue-приложение и подключается роутер.

Виды маршрутов во Vue Router

Виды маршрутов во Vue Router включают статические маршруты с фиксированными путями, динамические маршруты с параметрами, вложенные маршруты (children), редиректы и обработку ошибок 404.
Каждый вид маршрута служит для различных целей и позволяет создавать гибкую и удобную навигацию в одностраничных приложениях (SPA).

Тип маршрута Описание Пример
Статический маршрут Фиксированный путь без параметров. /home
Динамический маршрут Путь с параметрами, например, ID пользователя. /user/:id
Вложенный маршрут Маршруты внутри других маршрутов. /admin/users/:id/profile
Редирект Перенаправление с одного пути на другой. /old-page → /new-page

Навигация между страницами

Навигация между страницами во Vue.js осуществляется с помощью Vue Router, который позволяет пользователям переходить между различными компонентами и представлениями без перезагрузки страницы.
Навигация может быть реализована с помощью компонентов <router-link> для создания ссылок и <router-view> для отображения текущего маршрута.

Основные способы навигации:

Ссылки в шаблоне:

<template>
  <div>
    <nav>
      <router-link to="/">Главная</router-link>
      <router-link to="/about">О нас</router-link>
      <router-link :to="'/profile/1'">Профиль 1</router-link>
    </nav>
    <router-view></router-view>
  </div>
</template>

<script>
export default {
  name: 'App'
}
</script>
Объяснение кода:
  • <router-link> — компонент для создания ссылок на маршруты.
  • Атрибут to указывает путь маршрута.
  • <router-view> — место для отображения текущего компонента маршрута.

Программная навигация в методах

Используйте this.$router.push() для перехода по маршрутам из методов Vue-компонентов:

methods: {
    goToProfile(userId) {
        this.$router.push(`/profile/${userId}`);
    }
}
Объяснение кода:
  • this.$router.push() — метод для программного перехода на указанный маршрут.
  • Можно использовать динамические параметры в пути.

🚀 Вывод: Навигация во Vue.js с помощью Vue Router обеспечивает плавный и удобный переход между страницами, улучшая пользовательский опыт в одностраничных приложениях.

Статические маршруты

Статические маршруты — это маршруты, которые не содержат параметров и всегда ведут к одному и тому же компоненту. Они используются для отображения фиксированных страниц. Например, главная страница или страница "О нас". 

{
  path: '/about',
  component: About
}

Динамические маршруты с параметрами

Динамические маршруты - это маршруты, которые содержат параметры, позволяющие отображать различные данные в зависимости от значения этих параметров. Например , профиль пользователя с уникальным ID.
В Vue Router динамические параметры обозначаются двоеточием (:) в пути маршрута.

Пример динамического маршрута:

Маршрут для профиля пользователя с параметром id:

{ path: '/profile/:id', name: 'profile', component: Profile }

Получение параметров в компоненте:

В компоненте можно получить доступ к параметрам маршрута через this.$route.params:

export default {
  computed: {
    userId() {
      return this.$route.params.id;
    }
  }
}

Создание ссылок с параметрами:

Используйте <router-link> с динамическим значением для параметра:

<router-link :to="`/profile/${user.id}`">Профиль</router-link>

Несколько параметров в маршруте:

Маршрут с несколькими динамическими параметрами:

{ path: '/category/:categoryId/product/:productId', component: Product }

Вложенные маршруты

Вложенные маршруты - это маршруты, которые находятся внутри других маршрутов, позволяя создавать иерархическую структуру страниц и компонентов.
В Vue Router вложенные маршруты определяются с помощью свойства children в объекте маршрута.

Как использовать вложенные маршруты:

  • Определите родительский маршрут с дочерними маршрутами в свойстве children.
  • Используйте <router-view> внутри родительского компонента для отображения дочерних компонентов.

Пример вложенных маршрутов:

Маршрут Dashboard с двумя вложенными маршрутами:

{
  path: "/dashboard",
  component: Dashboard,
  children: [
    { path: "analytics", component: Analytics },
    { path: "settings", component: Settings }
  ]
}

Навигация по вложенным маршрутам:

Используйте <router-link> для перехода к вложенным маршрутам:

<router-link to="/dashboard/analytics">Аналитика</router-link>
<router-link to="/dashboard/settings">Настройки</router-link>

Отображение вложенных маршрутов:

Для отображения вложенной страницы — в Dashboard.vue используйте <router-view> (будет внутри родителя).

<template>
  <div>
    <h2>Панель управления</h2>
    <router-view></router-view>
  </div>
</template>

Программная навигация

Программная навигация - это способ перехода между маршрутами в Vue.js с помощью методов Vue Router, таких как this.$router.push() и this.$router.replace().
Этот подход позволяет выполнять навигацию в ответ на события, такие как клики на кнопки или завершение асинхронных операций.

Метлоды программной навигации:

  • this.$router.push(location) — добавляет новый маршрут в историю браузера.
  • this.$router.replace(location) — заменяет текущий маршрут без добавления в историю.
  • this.$router.go(n) — перемещается вперед или назад в истории на n шагов.

Как использовать программную навигацию:

  • Используйте методы this.$router.push(), когда нужно перейти на новый маршрут, например, после успешной отправки формы.
  • Используйте this.$router.replace(), чтобы заменить текущий маршрут,например, после логина пользователя.
  • Вызывайте эти методы внутри методов Vue-компонентов.
  • Можно использовать this.$router.push(), this.$router.replace() и другие для перехода по маршрутам.
  • Можно передавать параметры и query-параметры при навигации.

Пример программной навигации:

Переход на страницу профиля пользователя при клике на кнопку:

<template>
  <div>
    <button @click="goToProfile(1)">Перейти к профилю 1</button>
  </div>
</template>
<script>
export default {
  methods: {
    goToProfile(userId) {
      this.$router.push(`/profile/${userId}`);
    }
  }
}
</script>
Объяснение кода:
  • При клике на кнопку вызывается метод goToProfile.
  • Метод использует this.$router.push() для перехода на маршрут профиля с указанным ID.

🚀 Вывод: Программная навигация во Vue.js с помощью Vue Router позволяет гибко управлять переходами между страницами, улучшая взаимодействие пользователя с приложением.

Состояние маршрутов и история навигации

Состояние маршрутов - это информация о текущем маршруте, включая параметры, query-параметры и метаданные.
Vue Router позволяет управлять историей навигации с помощью методов, таких как this.$router.go(n), для перемещения вперед или назад в истории браузера.

Основные аспекты состояния маршрутов:

  • this.$route.params — доступ к динамическим параметрам маршрута.
  • this.$route.query — доступ к query-параметрам URL.
  • this.$route.name — имя текущего маршрута.
  • this.$route.fullPath — полный путь текущего маршрута.

Управление историей навигации:

  • this.$router.go(n) — перемещение на n шагов вперед или назад в истории.
  • this.$router.back() — переход на предыдущий маршрут.
  • this.$router.forward() — переход на следующий маршрут.

Передача параметров через query

Query-параметры позволяют передавать дополнительные данные в URL в формате ключ-значение. Они добавляются после знака вопроса (?) в URL.

Пример передачи query-параметров при навигации:

this.$router.push({ path: '/search', query: { q: 'vue' } });
Объяснение кода:
  • Переход на маршрут /search с query-параметром q, равным 'vue'.

Получение query-параметров в компоненте:

В компоненте можно получить доступ к query-параметрам через this.$route.query:

export default {
  computed: {
    searchQuery() {
      return this.$route.query.q;
    }
  }
}
Объяснение кода:
  • Используется вычисляемое свойство searchQuery для получения значения query-параметра q.

Якорь ( hash ) в маршрутах

Якорь ( hash ) позволяет ссылаться на определенные части страницы, используя символ # в URL. Это полезно для навигации к определенным секциям внутри одной страницы.

Пример использования якоря в маршруте:

this.$router.push({ path: '/about', hash: '#team' });
Объяснение кода:
  • Переход на маршрут /about с якорем #team, который указывает на секцию "team" на странице "about".

Комбинирование query-параметров и якоря

Можно одновременно использовать query-параметры и якорь при навигации:

this.$router.push({ path: '/search', query: { q: 'vue' }, hash: '#results' });
Объяснение кода:
  • path: '/search' - маршрут для перехода, где search — это путь к странице поиска.
  • query: { q: 'vue' } - передача query-параметра q со значением 'vue'.
  • hash: '#results' - указание якоря #results для навигации к определенной секции на странице.

Редиректы и обработка ошибок 404

Редиректы — это перенаправления пользователей с одного маршрута на другой. Они полезны для управления устаревшими ссылками или упрощения навигации.
Обработка ошибок 404 позволяет отображать специальную страницу, когда пользователь пытается перейти на несуществующий маршрут.

Как настроить редиректы и 404:

  • Используйте свойство redirect для создания редиректов.
  • Создайте маршрут с шаблоном /:pathMatch(.*)* для обработки всех неизвестных путей.

Примеры кода:

Редирект с главной страницы на /dashboard и маршрут 404:

{
  path: "/",
  redirect: "/dashboard" // Редирект с главной страницы
},
{
  path: "/:pathMatch(.*)*", // Маршрут 404
  component: NotFound
}
Объяснение кода:
  • Первый маршрут перенаправляет пользователей с / на /dashboard.
  • Второй маршрут ловит все неизвестные пути и отображает компонент NotFound.

Создание компонента 404:

Пример простого компонента для страницы 404:

<template>
  <div>
    <h2>404 - Страница не найдена</h2>
    <router-link to="/dashboard">Вернуться на главную</router-link>
  </div>
</template>

Защита маршрутов

Защита маршрутов — это механизм, позволяющий контролировать доступ к определенным маршрутам в приложении на Vue.js.
С помощью навигационных охранников можно реализовать аутентификацию и авторизацию пользователей, предотвращая доступ к защищенным страницам без соответствующих прав.

Типы навигационных охранников:

  • Глобальные охранники: применяются ко всем маршрутам. Пример: router.beforeEach().
  • Охранники маршрута: определяются в объекте маршрута с помощью свойства beforeEnter.
  • Охранники компонента: определяются внутри компонентов с помощью хуков жизненного цикла:
    • beforeRouteEnter - вызывается перед входом на маршрут для компонента.
    • beforeRouteUpdate - вызывается при обновлении маршрута для компонента.
    • beforeRouteLeave - вызывается перед выходом с маршрута.

Пример глобального охранника:

Проверка аутентификации перед переходом на защищенный маршрут:

router.beforeEach((to, from, next) => {
  if (to.meta.requiresAuth && !isAuthenticated()) {
    next('/login'); // Перенаправление на страницу логина
  } else {
    next(); // Разрешение перехода
  }
});
Объяснение кода:
  • Проверяется, требует ли маршрут аутентификации через to.meta.requiresAuth.
  • Если пользователь не аутентифицирован, происходит перенаправление на страницу логина.
  • Если все условия выполнены, вызывается next() для разрешения перехода.

🚀 навигационные охранники — позволяют контролировать доступ к маршрутам, обеспечивая безопасность и управление пользовательским опытом.

Практика: настрой маршруты!

Попрактикуйтесь в создании маршрутов во Vue Router, реализовав следующие задачи:

  • Создайте статические маршруты для главной страницы и страницы "О нас".
  • Добавьте динамический маршрут для профиля пользователя с параметром ID.
  • Настройте вложенные маршруты внутри панели управления (Dashboard).
  • Реализуйте редирект с главной страницы на панель управления.
  • Создайте маршрут для обработки ошибок 404.

Пример решения:

// Пример настройки маршрутов в router.js
import { createRouter, createWebHistory } from 'vue-router';
import Dashboard from './views/Dashboard.vue';
import About from './views/About.vue';
import Profile from './views/Profile.vue';
import NotFound from './views/NotFound.vue';

const routes = [
  { path: '/', redirect: '/dashboard' },
  { path: '/dashboard', component: Dashboard },
  { path: '/about', component: About },
  { path: '/profile/:id', name: 'profile', component: Profile },
  { path: '/:pathMatch(.*)*', component: NotFound }
];
const router = createRouter({
  history: createWebHistory(),
  routes,
});
export default router;
Объяснение кода:
  • Настроены статические маршруты для /dashboard и /about.
  • Добавлен динамический маршрут для профиля пользователя с параметром id.
  • Реализован редирект с главной страницы на панель управления.
  • Создан маршрут для обработки ошибок 404.
