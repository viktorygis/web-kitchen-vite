Введение в emits

$emit - это способ, с помощью которого дочерний компонент может отправлять события родительскому компоненту во Vue.js.
Это позволяет дочерним компонентам сообщать родителям о действиях или изменениях, чтобы те могли реагировать соответствующим образом.
Название "$emit" происходит от английского слова "emit", что означает "излучать" или "передавать".

Для чего используется $emit:

  • Сообщение родительскому компоненту о действиях пользователя в дочернем компоненте.
  • Передача данных или изменений состояния от дочернего компонента к родителю.
  • Создание взаимодействия между компонентами без прямого изменения данных родителя из дочернего компонента.

Базовая передача событий

Базовая передача событий через $emit: - это когда дочерний компонент отправляет простое событие родительскому компоненту без дополнительных данных.
Например, когда пользователь нажимает кнопку в дочернем компоненте, и дочерний компонент сообщает об этом родителю, чтобы тот мог выполнить какое-то действие, например, увеличить счётчик.

Пример: увеличение счётчика:

Родительский компонент 

<template>
  <div>
    <h2>Счётчик: {{ count }}</h2>
    <CounterButton @increase="incrementCount" />
  </div>
</template>

<script>
import CounterButton from './CounterButton.vue';

export default {
  components: { CounterButton },
  data() {
    return { count: 0 };
  },
  methods: {
    incrementCount() {
      this.count++;
    }
  }
};
</script>
Объяснение кода:
  • В родительском компоненте есть переменная count , которая хранит текущее значение счётчика.
  • Родительский компонент включает дочерний компонент CounterButton и слушает событие increase с помощью @increase="incrementCount" .
  • Когда дочерний компонент отправляет событие increase , вызывается метод incrementCount , который увеличивает значение count на 1.

Дочерний компонент: CounterButton.vue

<template>
  <button @click="$emit('increase')">+1</button>
</template>
Объяснение кода:
  • В дочернем компоненте есть кнопка, которая при нажатии вызывает $emit('increase') .
  • Это отправляет событие increase родительскому компоненту.
Кратко:
  • Дочерний компонент не меняет данные родителя напрямую!
  • Он только сообщает о действии — родитель реагирует и меняет данные.

Передача объектов и форм

$emit может передавать сложные данные, такие как объекты или формы.
Это полезно, когда дочерний компонент собирает данные от пользователя и отправляет их родителю.

Пример: форма пользователя

Родительский компонент получает объект с именем и возрастом пользователя от дочернего компонента.


<div id="app">
  <h2>Пользователь: {{ user.name }}, {{ user.age }} лет</h2>
  <child-component @send-user="updateUser"></child-component>
</div>

<script>
Vue.createApp({
  data() {
    return {
      user: { name: "Неизвестно", age: 0 }
    };
  },
  methods: {
    updateUser(newUser) {
      this.user = newUser;
    }
  }
}).component("child-component", {
  template: `
    <div>
      <input v-model="name" placeholder="Имя">
      <input v-model="age" type="number" placeholder="Возраст">
      <button @click="$emit('send-user', { name, age })">Отправить</button>
    </div>
  `,
  data() {
    return {
      name: "",
      age: null
    };
  }
}).mount("#app");
</script>
Объяснение кода:
  • В родительском компоненте есть объект user , который хранит имя и возраст пользователя.
  • Родительский компонент включает дочерний компонент child-component и слушает событие send-user с помощью @send-user="updateUser" .
  • Когда пользователь заполняет форму в дочернем компоненте и нажимает кнопку, $emit('send-user', { name, age }) отправляет объект с именем и возрастом родителю.
  • Родительский компонент получает этот объект в методе updateUser и обновляет свой объект user .
Так часто отправляются формы:
  • Дочерний компонент собирает данные
  • Через $emit передаёт их родителю
  • Родитель обновляет свои данные

Передача нескольких аргументов

$emit может передавать сразу несколько значений.
Это удобно для передачи ID и статуса задачи, например. 



<div id="app">
  <h2>Статус задачи:</h2>
  <p>ID: {{ taskInfo.id }}, Статус: {{ taskInfo.status }}</p>
  <child-component @update-task="handleTaskUpdate"></child-component>
</div>

<script>
Vue.createApp({
  data() {
    return {
      taskInfo: { id: null, status: 'Не задан' }
    };
  },
  methods: {
    handleTaskUpdate(taskId, newStatus) {
      this.taskInfo.id = taskId;
      this.taskInfo.status = newStatus;
    }
  }
}).component("child-component", {
  template: `<button @click="$emit('update-task', 101, 'выполнено')">Обновить задачу</button>`
}).mount("#app");
</script>
Передавать можно сколько угодно аргументов!
Главное — соблюдать порядок.

Несколько дочерних компонентов: сообщения

Родитель может принимать сообщения от нескольких дочерних компонентов.


<div id="app">
  <h2>Сообщения от детей:</h2>
  <child-component @child-event="handleEvent('Компонент 1', $event)"></child-component>
  <child-component @child-event="handleEvent('Компонент 2', $event)"></child-component>
  <p>{{ message }}</p>
</div>

<script>
Vue.createApp({
  data() {
    return { message: "" }
  },
  methods: {
    handleEvent(name, msg) {
      this.message = `${name} говорит: ${msg}`;
    }
  }
})
.component("child-component", {
  template: `
    <button @click="$emit('child-event', 'Привет, родитель!')">
      Нажми меня! 👍
    </button>
  `
})
.mount("#app");
</script>
Родитель слушает события от любых детей.
Можно узнать, кто отправил сообщение, и обработать его по-своему.
