Реактивность в Vuex: асинхронные действия и API

Реактивность в Vuex — это способность автоматически отслеживать изменения состояния и обновлять интерфейс пользователя.

Vuex идеально хранит и меняет состояние через state и mutations, но настоящий веб живёт асинхронными запросами! Тут вступают в игру actions — они работают с async/await, делают запросы к API и обновляют состояние без багов.

Что мы изучим?

  • 📡 Получить пользователей из API
  • 🛒 Загрузить товары
  • ⏳ Отправить данные и дождаться ответа сервера

Если использовать только mutations, мы скоро столкнёмся с тем, что они не умеют ждать ответа от API!

Почему нужны actions во Vuex

  • Mutations — синхронные, не могут ждать API
  • Actions — асинхронные, работают с промисами и async/await
  • Actions вызывают mutations после получения данных

Таким образом, Vuex остаётся реактивным и управляемым, даже при работе с асинхронными операциями!

🔥 Зачем нужны actions?

В Vuex есть два основных способа изменять состояние: mutations и actions. Но почему нам нужны именно actions для асинхронных операций?

Mutations vs Actions: в чём разница?

Mutations

  • Mutations отвечают только за изменение состояния (state), но они обязаны быть синхронными. Если ты попробуешь встроить в мутацию API-запрос или задержку через setTimeout, получишь ошибку или сломанное состояние.
  • API-запросы по природе асинхронные — они требуют времени, ведь приходят из внешнего мира. Простая мутация не сможет дождаться ответа!

Actions

Actions — это “оркестратор” асинхронных операций во Vuex.
Они могут:

  • Выполнять асинхронный код (Promise, async/await, setTimeout, fetch и др).
  • Контролировать последовательность: сначала загрузить что-то с сервера, после этого вызвать мутацию, чтобы обновить state. Это важно для правильного состояния приложения.
  • Комбинировать несколько действий — например, последовательно запросить товары и пользователей, обновить состояние, обработать ошибки.
  • Делать дополнительные проверки до обновления state (например, валидацию, фильтрацию, сортировку данных).
  • Выполнять бизнес-логику и “склеивать” цепочки событий приложения (например, авторизация пользователя, затем загрузка корзины).
  • Логировать, отправлять события аналитике, реагировать на ошибки сервера — всё это место actions, а не mutations.

🔥 Зачем нужны actions?

Actions позволяют централизованно управлять асинхронной логикой, обеспечивая чистоту и предсказуемость состояния приложения.

Коротко: как работает схема?

  1. Компонент вызывает dispatch action.
  2. Action делает асинхронный запрос или ждет события.
  3. Когда данные готовы — вызывает commit mutation.
  4. Mutation обновляет state, что сразу отражается во всех реактивных компонентах.

Без actions сложно:

  • Нельзя гарантировать “чистое” состояние — мутации могут завершаться до получения данных.
  • Появляется дублирование кода в компонентах (fetch, then, commit в каждом), сложнее поддерживать.

С actions легко:

  • Всё асинхронное — в одном месте!
  • Приложение становится масштабируемым и управляемым.
  • Чистый и читаемый код: компоненты заботятся только о запуске action, не думают о подробностях API.

Итог: actions — центр асинхронной логики Vuex, делают приложение гибким и надежным!

📌 Как actions загружают данные из API

Рассмотрим пример: мы хотим загрузить список пользователей с внешнего API и сохранить их в состоянии Vuex.

Шаг 1: Определяем state и mutations

const store = createStore({
  state: {
    users: [] // 👥 Здесь храним список пользователей
  },
  mutations: {
    setUsers(state, users) {
      state.users = users; // 🔄 Обновляем state
    }
  },
  actions: {
    async fetchUsers({ commit }) {
      const response = await fetch("https://jsonplaceholder.typicode.com/users");
      const data = await response.json();
      commit("setUsers", data); // 📥 Передаём данные в mutations
    }
  }
});
Пояснение:
  • state.users: массив для хранения загруженных пользователей.
  • mutation setUsers: обновляет состояние с новыми данными.
  • action fetchUsers: асинхронно запрашивает пользователей с API и вызывает мутацию для обновления состояния.

Шаг 2: Вызов actions из компонента

methods: {
  loadUsers() {
    this.$store.dispatch("fetchUsers"); // 🚀 Запускаем action
  }
}
Пояснение:
  • Метод loadUsers вызывает dispatch для запуска action fetchUsers.
  • Action выполняет асинхронный запрос и обновляет состояние через мутацию.

Шаг 3: Использование данных в компоненте

computed: {
  users() {
    return this.$store.state.users; // 👥 Получаем пользователей из state
  }
}
Пояснение:
  • Вычисляемое свойство users возвращает список пользователей из состояния Vuex.
  • Компонент автоматически обновится, когда состояние изменится после загрузки данных.

Шаг 4: Шаблон для отображения пользователей

<template>
  <div>
    <button @click="loadUsers">Загрузить пользователей</button>
    <ul>
      <li v-for="user in users" :key="user.id">{{ user.name }}</li>
    </ul>
  </div>
</template>
Пояснение:
  • Кнопка вызывает метод loadUsers для загрузки пользователей.
  • Список пользователей отображается с помощью v-for, автоматически обновляется при изменении состояния.

Вывод: Vuex позволяет удобно и эффективно управлять асинхронными данными в приложении.

🎯 Как запускать actions из компонентов

Есть два основных способа вызвать actions из Vue-компонентов:

Способ 1: стандартный dispatch()

стандартный dispatch() - это метод Vuex для вызова actions напрямую из компонента.


methods: {
  loadUsers() {
    this.$store.dispatch("fetchUsers"); // Запускаем actions
  }
}

Неудобно — слишком многословно!

Способ 2: mapActions — быстро и красиво

mapActions — это хелпер Vuex, который позволяет быстро “маппить” actions в методы компонента.

<script>
import { mapActions } from "vuex";
export default {
  methods: {
    ...mapActions(["fetchUsers"])
  }
};
 </script>
  • Можно просто вызывать this.fetchUsers() — Vuex сам отправит запрос
  • Нет лишнего кода, легко читать и поддерживать

📌 Как actions ловят ошибки

При работе с API всегда есть риск ошибок: сервер может не ответить, данные могут быть некорректными. Важно правильно обрабатывать эти ошибки в actions, чтобы приложение оставалось стабильным.

Пример: обработка ошибок в actions

actions: {
  async fetchUsers({ commit }) {
    try {
      const response = await fetch("https://jsonplaceholder.typicode.com/users");
      if (!response.ok) {
        throw new Error("Ошибка загрузки данных!");
      }
      const data = await response.json();
      commit("setUsers", data);
    } catch (error) {
      console.error("🚨 Что-то пошло не так:", error.message);
    }
  }
}
Пояснение:
  • Весь код — внутри try: безопасно отправляем запрос
  • Ошибки ловятся catch — функция не сломает приложение
  • Можно показать пользователю ошибку или сохранить её в state

Пример: как запускать actions из UI

Допустим, у нас есть кнопка, которая загружает пользователей при клике. Вот как это сделать с помощью mapActions:

<template>
  <button @click="fetchUsers">Загрузить пользователей</button>
</template>
<script>
import { mapActions } from "vuex";
export default {
  methods: {
    ...mapActions(["fetchUsers"])
  }
};
</script>
Пояснение:
  • При клике на кнопку вызывается actions.fetchUsers()
  • Vuex сам делает запрос и обновляет state
  • UI остаётся простым и чистым

Вывод: использование mapActions упрощает вызов actions из компонентов и делает код чище.
