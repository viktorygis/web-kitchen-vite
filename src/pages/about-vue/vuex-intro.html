Что такое Vuex?

Vuex — это централизованное хранилище данных для Vue-приложений.
Он позволяет управлять состоянием всего приложения из одного места.
Vuex называют "официальным менеджером состояния для Vue.js".
Vuex называется так потому что он вдохновлён концепцией Flux от Facebook и Redux от сообщества React.

Для чего нужен Vuex?

  • 📦 Централизованное хранилище данных
  • 🔄 Синхронизация состояния между компонентами
  • 🛠️ Упрощённое управление сложным состоянием
  • 🔍 Легкая отладка и отслеживание изменений состояния
  • ⚙️ Чёткая структура для масштабируемых приложений

Как установить Vuex?

Vuex можно установить несколькими способами: через CDN, NPM или Yarn.
📌 Важно: Убедитесь, что используете Vuex версии 4 для Vue 3 и Vuex версии 3 для Vue 2

CDN-подключение (Vue 3 + Vuex 4):

Если вы не используете систему сборки, можно подключить Vuex напрямую через CDN. 

<script src="https://unpkg.com/vue@3"></script>
<script src="https://unpkg.com/vuex@4"></script>

👉 Сначала подключите Vue, затем Vuex. После этого можно использовать Vue.createApp и Vuex.createStore (см. далее).

NPM-установка

Установите Vuex через NPM, если используете систему сборки.

npm install vuex@4 --save

Yarn-установка

Установите Vuex через Yarn, если используете систему сборки.

yarn add vuex@4

Импорт и настройка (для Vue 3, системы сборки)

Пример настройки Vuex в проекте с использованием системы сборки.
Обычно это делается в файле main.js. 

import { createApp } from 'vue';
import { createStore } from 'vuex';
import App from './App.vue';

const store = createStore({
  // ваша конфигурация store
});

const app = createApp(App);
app.use(store); // Подключаем Vuex
app.mount('#app');
Что здесь происходит?
  • ✅ Импортируем createStore из Vuex для создания хранилища.
  • ✅ Создаём экземпляр хранилища с помощью createStore.
  • ✅ Регистрируем Vuex в приложении через app.use(store).

Важно: Для Vue 2 используется Vue.use(Vuex) и другие CDN:
<script src="https://unpkg.com/vue@2"></script>
<script src="https://unpkg.com/vuex@3"></script>
Для Vue 3 используйте только app.use(store) и Vuex 4!

Создаём файл store.js и настраиваем Vuex

После установки пора настроить хранилище для приложения.

store.js — файл, в котором находится конфигурация хранилища данных для всего приложения.

Пример store.js

import { createStore } from "vuex";

export default createStore({
  state: {
    count: 0 // 💾 Здесь хранятся данные
  },
  mutations: {
    increment(state) {
      state.count++; // ✏️ Функция для изменения state
    }
  }
});

Обозначения:

  • createStore — функция для создания хранилища Vuex
  • state — объект с данными приложения
  • mutations — функции для изменения данных в state
🔥 Теперь у нас есть полноценное хранилище данных! 🚀

Подключение Vuex к приложению

Окей, мы создали store.js, но Vue пока о нём ничего не знает. Нужно подключить Vuex к нашему приложению в main.js.

main.js — точка входа приложения, здесь мы связываем Vuex и Vue.

Добавляем Vuex в приложение:

import { createApp } from "vue";
import App from "./App.vue";
import store from "./store"; // 📥 Импортируем store

const app = createApp(App);
app.use(store); // 🔗 Подключаем Vuex
app.mount("#app");

🔍 Что здесь происходит?

  • ✅ import store from "./store" — импортируем хранилище
  • ✅ app.use(store) — регистрируем Vuex в приложении
  • ✅ Теперь все компоненты могут использовать Vuex! 🎉
Store готов к работе! Теперь можно получать и изменять данные глобально. 🚀

Как проверить, что Vuex работает?

После подключения store нам нужно убедиться, что он действительно работает!

📌 Проверяем Vuex в консоли браузера:
  1. Открываем DevTools (F12) → Console
  2. Вбиваем команду:
console.log(window.store.state);

✅ Если Vuex подключён правильно, ты увидишь объект с данными: 

{ count: 0 }
Vuex готов к работе — можно пользоваться глобальными данными и изменять их из любого компонента!

Зачем нужен Vuex?

В небольших приложениях можно обойтись локальным состоянием компонентов. Но когда приложение растёт, управление состоянием становится сложнее.

Проблемы без Vuex:

  • 🔄 Дублирование данных в разных компонентах
  • ❌ Сложность отслеживания изменений состояния
  • ⚠️ Ошибки из-за несогласованности данных

Преимущества использования Vuex:

  • ✅ Централизованное управление состоянием
  • ✅ Легкий доступ к данным из любого компонента
  • ✅ Чёткая структура для изменений состояния через mutations
  • ✅ Упрощённое тестирование и отладка
Vuex помогает создавать масштабируемые и поддерживаемые приложения!

Как устроен Vuex?

Vuex состоит из нескольких ключевых концепций, которые помогают управлять состоянием приложения.

Основные части Vuex:

  • State — объект, хранящий данные приложения
  • Mutations — функции для синхронного изменения state
  • Actions — функции для асинхронных операций и вызова mutations
  • Getters — вычисляемые свойства для получения данных из state
Каждая часть играет важную роль в управлении состоянием и обеспечивает предсказуемость изменений!

Как получить доступ к данным Vuex?

После настройки Vuex, компоненты могут легко получать доступ к данным из хранилища.

Пример доступа к state в компоненте:

<template>
  <div>
    <h1>Счётчик: {{ count }}</h1>
    <button @click="increment">+1</button>
  </div>
</template>
<script>
export default {
  computed: {
    count() {
      return this.$store.state.count; // Получаем count из Vuex
    }
  },
  methods: {
    increment() {
      this.$store.commit("increment"); // Вызываем mutation для изменения count
    }
}
</script>

Объяснение:

  • this.$store.state.count — доступ к данным из state
  • this.$store.commit("increment") — вызов mutation для изменения данных
Таким образом, компоненты могут легко читать и изменять глобальное состояние приложения!

Использование mapState для упрощения доступа к данным

Vuex предоставляет удобный хелпер mapState, который упрощает получение данных из state в компонентах.

Пример использования mapState:

import { mapState } from "vuex";
<template>
  <div>

    <h1>Счётчик: {{ count }}</h1>
    <button @click="increment">+1</button>
  </div>
</template>
<script>
export default {
  computed: {
    ...mapState(["count"]) // Используем mapState для получения count
  },
  methods: {
    increment() {
      this.$store.commit("increment"); // Вызываем mutation для изменения count
    }
  }
</script>

Пояснение:

  • mapState автоматически создаёт вычисляемые свойства для указанных полей state
  • Это сокращает код и делает его чище!
Использование mapState делает работу с Vuex ещё проще и удобнее!

Практика: используем Vuex в компонентах

  • Подключи хранилище Vuex в своём приложении
  • Получай данные из state через computed или mapState
  • Продумай, как бы организовал информацию о пользователе, корзину и тему оформления

store.js (простая версия)


// store.js
const store = Vuex.createStore({
  state() {
    return {
      count: 0,
      user: { name: "Аня", loggedIn: false },
      cart: []
    };
  },
  mutations: {
    increment(state) {
      state.count++;
    },
    login(state, name) {
      state.user = { name, loggedIn: true };
    },
    addToCart(state, item) {
      state.cart.push(item);
    }
  }
});

App.vue

<!-- App.vue -->
<template>
  <div>
    <h2>Счётчик: {{ count }}</h2>
    <button @click="increment">+1</button>
    <h4>Пользователь: {{ user.name }} ({{ user.loggedIn ? "в системе" : "гость" }})</h4>
    <button @click="login">Войти как Катя</button>
    <h4>Корзина: {{ cart.length }} товаров</h4>
    <button @click="addToCart">Добавить товар</button>
  </div>
</template>

<script>
import { mapState } from "vuex";
export default {
  computed: {
    ...mapState(["count", "user", "cart"])
  },
  methods: {
    increment() { this.$store.commit("increment"); },
    login() { this.$store.commit("login", "Катя"); },
    addToCart() { this.$store.commit("addToCart", "Товар №1"); }
  }
}
</script>

Вопросы и распространённые ошибки при работе с Vuex

1. Почему Vuex не работает в моём приложении?

Убедитесь, что вы правильно подключили Vuex к вашему приложению через app.use(store) в main.js.

2. Как отлаживать состояние Vuex?

Используйте Vue Devtools для просмотра состояния Vuex и отслеживания изменений в реальном времени.

3. Можно ли использовать Vuex с Vue 2?

Да, но нужно использовать Vuex версии 3 и подключать его через Vue.use(Vuex).

Если у вас есть другие вопросы или проблемы, не стесняйтесь обращаться к официальной документации Vuex или сообществу разработчиков!
