Введение во Vue.js

Vue.js - это прогрессивный JavaScript-фреймворк для создания пользовательских интерфейсов. Он разработан для того, чтобы быть адаптивным и легко интегрируемым в существующие проекты. Vue.js позволяет создавать интерактивные веб-приложения с использованием декларативного подхода и компонентной архитектуры.

Основные концепции Vue.js

Установка Vue.js

Для начала работы с Vue.js, вы можете подключить его через CDN. Добавьте следующий тег <script> в ваш HTML-файл перед закрывающим тегом </body> но перед скиптом с vue:

<script src="https://unpkg.com/vue@3/dist/vue.global.prod.js"></script>

Создание простого приложения Vue.js

С использованием шаблона и интерполяции данных, где шаблон это HTML-код, который определяет структуру и внешний вид вашего приложения (начинается с <div>), а интерполяция ( {{ }})данных позволяет динамически отображать данные из вашего приложения в шаблоне. (Интерполяция - это способ вставки значений переменных или выражений в текстовый контент HTML.)

Привет, {{ name }}!

Код приложения:

<div id="app">
  <h2>Привет, {{ name }}!</h2> //Шаблон с привязкой данных
</div>
<script>
  const {createApp} = Vue;
  createApp({
    data() {
      return {
        name: 'Мир' //Данные приложения
      }
    }
  }).mount('#app');
</script>

Директивы

Директивы Vue.js - это специальные атрибуты, которые начинаются с префикса v- и предоставляют дополнительные функциональные возможности для элементов HTML. Они позволяют вам связывать данные, обрабатывать события и управлять отображением элементов в вашем приложении.

v-bind

Директива v-bind используется для связывания атрибутов HTML с данными из вашего приложения Vue.js. В этом примере мы связываем атрибут href ссылки с переменной url из данных приложения.

Перейти на Vue.js

Код приложения:

<div id="app-bind">
  <a v-bind:href="url" target="_blank">Перейти на Vue.js</a> //Директива v-bind для связывания атрибута href
</div>
<script>
  const {createApp} = Vue;
  createApp({
    data() {
      return {
        url: 'https://vuejs.org/' //Данные приложения
      }
    }
  }).mount('#app-bind');
</script>

v-if

Директива v-if используется для условного отображения элементов в Vue.js. В этом примере мы используем v-if для отображения сообщения только тогда, когда переменная isVisible установлена в true.

Этот заголовок отображается, если значение show — true.

Заголовок скрыт, потому что show = false.

Код приложения:

<div id="app-if">
  <h4 v-if="show">Этот заголовок отображается, если значение show — true.</h4> //Директива v-if для условного отображения
  <p v-if="!show">Заголовок скрыт, потому что show = false.</p>
</div>
<script>
  const {createApp} = Vue;
  createApp({
    data() {
      return {
        show: true //Данные приложения
      }
    }
  }).mount('#app-if');
</script>

v-show

Директива v-show управляет видимостью элемента через CSS-свойство display. Элемент всегда присутствует в DOM, но скрывается, если условие isVisible установлено в false.

Этот заголовок видим, если isVisible — true.

Заголовок скрыт, потому что isVisible = false.

Код приложения:

<div id="app-show">
  <h4 v-show="isVisible">Этот заголовок видим, если isVisible — true.</h4>
  <p v-show="!isVisible">Заголовок скрыт, потому что isVisible = false.</p>
</div>
<script>
  const {createApp} = Vue;
  createApp({
    data() {
      return {
        isVisible: true
      }
    }
  }).mount('#app-show');
</script>

v-model

Директива v-model обеспечивает двухстороннюю привязку данных между элементом формы и состоянием приложения. В этом примере при изменении текста в поле ввода меняется и отображаемое приветствие.

Привет, {{ username }}!

Код приложения:

<div id="app-model">
  <input v-model="username" placeholder="Введите имя">
  <p>Привет, {{ username }}!</p>
</div>
<script>
  const {createApp} = Vue;
  createApp({
    data() {
      return {
        username: ""
      }
    }
  }).mount('#app-model');
</script>

v-on

Директива v-on используется для обработки событий в Vue.js. В этом примере мы обрабатываем событие клика на кнопку и вызываем метод increment, который увеличивает значение счетчика.

Количество кликов: {{ count }}

Код приложения:

<div id="app-on">
  <button v-on:click="increment">Кликни меня</button> //Директива v-on для обработки события клика
  <p>Количество кликов: {{ count }}</p>
</div>
<script>

  const {createApp} = Vue;
  createApp({
    data() {
      return {
        count: 0 //Данные приложения
      }
    },
    methods: {
      increment() { //Метод для увеличения счетчика
        this.count++;
      }
    }
  }).mount('#app-on');
</script>

Работа с данными через data()

В Vue.js, функция data() используется для определения реактивных данных вашего компонента. Она возвращает объект, содержащий все переменные и их начальные значения, которые будут использоваться в вашем приложении.

Данные, определенные в функции data(), становятся реактивными, что означает, что любые изменения этих данных автоматически обновляют соответствующие части пользовательского интерфейса.

{{ message }}

Код приложения:

<div id="app-data">
  <h2>{{ message }}</h2>
  <input v-model="message" placeholder="Измени меня">
</div>
<script>
  const {createApp} = Vue;
  createApp({
    data() {
      return {
        message: "Привет, Vue.js!"
      }
    }
  }).mount('#app-data');
</script>

Методы в Vue.js

Методы в Vue.js - это функции, которые определены внутри объекта methods вашего компонента. Они используются для обработки событий, выполнения вычислений и изменения данных вашего приложения.

Методы могут быть вызваны из шаблона с помощью директивы v-on или напрямую из других методов внутри компонента.

{{ greeting }}

Код приложения:

<div id="app-methods">
  <button v-on:click="greet">Поприветствовать</button>
  <p>{{ greeting }}</p>
</div>
<script>
  const {createApp} = Vue;
  createApp({
    //Данные приложения
    data() {
      return {
        greeting: ""
      }
    },
    //Методы приложения
    methods: {
      greet() {
        this.greeting = "Привет от Vue.js!";
      }
    }
  }).mount('#app-methods');
</script>

Передача параметров и работа с this в методах Vue.js

В Vue.js методы могут принимать параметры, что позволяет передавать данные при вызове метода. Внутри методов, ключевое слово this ссылается на текущий экземпляр Vue, что позволяет получать доступ к данным и другим методам компонента.

Использование параметров в методах позволяет создавать более гибкие и переиспользуемые функции внутри вашего Vue-приложения.

{{ greeting }}

Код приложения:

<div id="app-params">
  <input v-model="name" placeholder="Введите ваше имя">
  <button v-on:click="greet(name)">Поприветствовать</button>
  <p>{{ greeting }}</p>
</div>
<script>
  const {createApp} = Vue;
  createApp({
    data() {
      return {
        name: "",
        greeting: ""
      }
    },
    methods: {
      greet(name) {
        this.greeting = `Привет, ${name}! Добро пожаловать в Vue.js!`;
      }
    }
  }).mount('#app-params');
</script>

Вычисляемые свойства (computed) в Vue.js

Вычисляемые свойства в Vue.js - это свойства, которые зависят от других реактивных данных и автоматически обновляются при изменении этих данных. Они определяются внутри объекта computed вашего компонента и используются для выполнения вычислений на основе текущего состояния данных.

Вычисляемые свойства полезны для оптимизации производительности, так как они кэшируются и пересчитываются только тогда, когда их зависимости изменяются.

Полное имя: {{ fullName }}

Код приложения:

<div id="app-computed">
  <input v-model="firstName" placeholder="Имя">
  <input v-model="lastName" placeholder="Фамилия">
  <p>Полное имя: {{ fullName }}</p>
</div>
<script>
  const {createApp} = Vue;
  createApp({
    //Данные приложения
    data() {
      return {
        firstName: "",
        lastName: ""
      }
    },
    //Вычисляемые свойства
    computed: {
      fullName() {
        return `${this.firstName} ${this.lastName}`;
      }
    }
  }).mount('#app-computed');
</script>

Наблюдатели (watch) в Vue.js

Наблюдатели в Vue.js - это специальные функции, которые следят за изменениями реактивных данных и выполняют определенные действия при их изменении. Они определяются внутри объекта watch вашего компонента и полезны для выполнения асинхронных операций или сложной логики, когда данные изменяются.

Наблюдатели позволяют реагировать на изменения данных без необходимости использовать методы или вычисляемые свойства, что делает их удобными для определенных сценариев.

Ответ: {{ answer }}

Код приложения:

<div id="app-watch">
  <input v-model="question" placeholder="Задайте вопрос">
  <p>Ответ: {{ answer }}</p>
</div>
<script>
  const {createApp} = Vue;
  createApp({
    //Данные приложения
    data() {
      return {
        question: "",
        answer: "Пока нет вопроса."
      }
    },
    //Наблюдатели
    watch: {
      question(newQuestion) {
        this.answer = "Думаю...";
        setTimeout(() => {
          this.answer = "Это интересный вопрос!";
        }, 1000);
      }
    }
  }).mount('#app-watch');
</script>

Пример Vue.js с использованием data, methods, computed и watch

В этом примере мы создадим простое приложение Vue.js, которое демонстрирует использование всех основных свойств: data, methods, computed и watch. Приложение позволит пользователю вводить свое имя и фамилию, а также задавать вопрос, на который будет дан ответ.

Полное имя: {{ fullName }}

Ответ: {{ answer }}

Код приложения:

<div id="app-full">
  <input v-model="firstName" placeholder="Имя">
  <input v-model="lastName" placeholder="Фамилия">
  <p>Полное имя: {{ fullName }}</p>

  <input v-model="question" placeholder="Задайте вопрос">
  <button v-on:click="askQuestion">Спросить</button>
  <p>Ответ: {{ answer }}</p>
</div>
<script>
  const {createApp} = Vue;
  createApp({
    //Данные приложения
    data() {
      return {
        firstName: "",
        lastName: "",
        question: "",
        answer: "Пока нет вопроса."
      }
    },
    //Вычисляемые свойства
    computed: {
      fullName() {
        return `${this.firstName} ${this.lastName}`;
      }
    },
    //Методы
    methods: {
      askQuestion() {
        this.answer = "Думаю...";
        setTimeout(() => {
          this.answer = "Это интересный вопрос!";
        }, 1000);
      }
    },
    //Наблюдатели
    watch: {
      question(newQuestion) {
        this.answer = "Пока нет вопроса.";
      }
    }
  }).mount('#app-full');
</script>

Код приложения:


<script>
  const { createApp } = Vue;
  createApp({
    data() {
      return {
        title: 'Органайзер задач',
        newTask: '',
        placeholderText: 'Введите новую задачу',
        filter: 'all',
        lastAction: '',
        tasks: [
          { id: 1, text: 'Изучить Vue', done: false, urgent: true },
          { id: 2, text: 'Выполнить задание', done: true, urgent: false }
        ]
      }
    },
    computed: {
      filteredTasks() {
        if (this.filter === 'done') return this.tasks.filter(t => t.done);
        if (this.filter === 'active') return this.tasks.filter(t => !t.done);
        return this.tasks;
      }
    },
    watch: {
      newTask(newValue) {
        if (newValue && newValue.toLowerCase().includes('важно')) {
          this.lastAction = 'Ключевое слово "важно" введено!';
        }
      },
      filter(newF) {
        this.lastAction = 'Изменён фильтр: ' + newF;
      }
    },
    methods: {
      addTask() {
        if (!this.newTask.trim()) return;
        this.tasks.push({
          id: Date.now(),
          text: this.newTask,
          done: false,
          urgent: /важн|🔥/i.test(this.newTask)
        });
        this.lastAction = 'Добавлена задача: ' + this.newTask;
        this.newTask = '';
      },
      deleteTask(idx) {
        this.lastAction = 'Удалена задача: ' + this.tasks[idx].text;
        this.tasks.splice(idx, 1);
      },
      toggle(idx) {
        this.tasks[idx].done = !this.tasks[idx].done;
        this.lastAction = `Статус "${this.tasks[idx].text}" изменён`;
      }
    }
  }).mount('#app-fullest');
</script>

<!-- Список задач -->
  <ul>
    <!--li - элемент списка задач
    v-for="(task, idx) in filteredTasks" - перебор массива filteredTasks с доступом к элементу task и его индексу idx,
    :key="task.id" - уникальный ключ для каждого элемента списка, важен для производительности,
    :class="{ done: task.done }" - динамическое присвоение класса done если задача выполнена,
    :style="{ opacity: task.done ? 0.6 : 1 }" - динамическое присвоение стиля прозрачности в зависимости от статуса задачи -->
    <li
      v-for="(task, idx) in filteredTasks"
      :key="task.id"
      :class="{ done: task.done }"
      :style="{ opacity: task.done ? 0.6 : 1 }"
    >
      <!-- Индикатор срочности задачи,
      v-if="task.urgent" - условное отображение если задача срочная,
      v-show="filter!=='done'" - скрытие индикатора если выбран фильтр 'done' -->
      <span v-if="task.urgent" style="color:red" v-show="filter!=='done'">🔥</span>
      <!--  Номер задачи, {{ idx+1 }} - интерполяция индекса задачи +1 -->
      <span>{{ idx+1 }}.</span>
      <!-- Текст задачи с возможностью переключения статуса
      @click="toggle(idx)" - обработка события клика для вызова метода toggle с передачей индекса задачи idx,
      style="cursor:pointer;" - изменение курсора при наведении на текст задачи -->
      <span @click="toggle(idx)" style="cursor:pointer;">
        <!-- Индикатор статуса задачи,  v-if="task.done" - условное отображение если задача выполнена -->
        <template v-if="task.done">(✓)</template>
        <!--  Индикатор статуса задачи,  v-else - отображение если задача не выполнена -->
        <template v-else>(…)</template>
        <!--  Текст задачи, {{ task.text }} - интерполяция текста задачи -->
        {{ task.text }}
      </span>
      <!-- Кнопка удаления задачи
      @click="deleteTask(idx)" - обработка события клика для вызова метода deleteTask с передачей индекса задачи idx,
      style="margin-left:10px;" - отступ слева для кнопки -->
      <button @click="deleteTask(idx)" style="margin-left:10px;">Удалить</button>
    </li>
  </ul>
  • v-for="(task, idx) in filteredTasks" - перебор массива задач filteredTasks для отображения каждой задачи в списке.
  • filteredTasks - вычисляемое свойство, которое возвращает отфильтрованный список задач в зависимости от выбранного фильтра (все, выполненные, активные).
  • :key="task.id" - уникальный ключ для каждого элемента списка, важен для производительности при обновлении списка.
  • :class="{ done: task.done }" - динамическое присвоение класса done элементу списка, если задача выполнена.
  • :style="{ opacity: task.done ? 0.6 : 1 }" - динамическое изменение прозрачности элемента списка в зависимости от статуса задачи.
  • v-if="task.urgent" - условное отображение индикатора срочности задачи, если задача помечена как срочная.
  • @click="toggle(idx)" - обработка события клика по тексту задачи для вызова метода toggle(), который переключает статус задачи между выполненной и активной.
  • @click="deleteTask(idx)" - обработка события клика по кнопке "Удалить" для вызова метода deleteTask(), который удаляет задачу из списка.
