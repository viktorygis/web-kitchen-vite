Vuex-хелперы: mapState, mapGetters, mapMutations, mapActions

Vuex-хелперы — это специальные функции (helpers) для сокращения кода в компонентах Vue при работе с хранилищем состояния (Vuex). Они помогают автоматически подключать и использовать state, getters, mutations и actions без многократного обращения к this.$store.

Без хелперов Vuex:

computed: {
  count() { return this.$store.state.count; },
  username() { return this.$store.state.user.name; }
}

❌ Многословно и неудобно!

С Vuex-хелперами:

import { mapState } from "vuex";
export default {
  computed: {
    ...mapState(["count"]),
    ...mapState({ username: state => state.user.name })
  }
}

✅ Очень коротко и наглядно!

Что мы изучим?

  • mapState — лёгкий доступ к state 💾
  • mapGetters — подключение обработанных вычислений (getters) 🎯
  • mapMutations — компактный вызов mutations ✏️
  • mapActions — лаконичные вызовы actions 🔄

💾 mapState

mapState — хелпер, который позволяет автоматически связывать свойства state с computed вашего компонента. Используется только внутри computed!

Для чего нужен mapState?

  • Упрощает доступ к состоянию Vuex
  • Сокращает количество кода в компонентах
  • Позволяет легко переименовывать свойства state при подключении

Пример использования mapState:

Было так: 
computed: {
  count() { return this.$store.state.count; },
  username() { return this.$store.state.user.name; }
}
Становится так: 
import { mapState } from "vuex";
export default {
  computed: {
    ...mapState(["count"]),
    ...mapState({ username: state => state.user.name })
  }
}

Что здесь происходит?

  • Импортируем хелпер mapState из Vuex
  • Внутри computed используем оператор spread ..., чтобы "развернуть" результат работы mapState
  • mapState(['count']) - подключает свойство count из state напрямую
  • mapState({ username: state => state.user.name }) - подключает свойство user.name из state и переименовывает его в username
  • Теперь в шаблоне можно использовать {{ count }} и {{ username }} без обращения к this.$store

Что нужно знать:

  • Можно использовать как массив строк, так и объект для переименования
  • Можно комбинировать несколько вызовов mapState внутри одного computed
  • Можно использовать функции для более сложной логики подключения state

🎯 mapGetters

mapGetters — хелпер для быстрого подключения геттеров из Vuex-store в computed вашего компонента. Используется только внутри computed!

Для чего нужен mapGetters?

  • Упрощает доступ к геттерам Vuex
  • Сокращает количество кода в компонентах
  • Позволяет легко переименовывать геттеры при подключении

Пример:

До: 
computed: {
  doubleCount() {
    return this.$store.getters.doubleCount;
  }
}
После: 
import { mapGetters } from "vuex";
export default {
  computed: {
    ...mapGetters(["doubleCount"])
  }
}

Что здесь происходит?

  • Импортируем хелпер mapGetters из Vuex
  • Внутри computed используем оператор spread ..., чтобы "развернуть" результат работы mapGetters
  • mapGetters(['doubleCount']) - подключает геттер doubleCount из Vuex-store напрямую
  • Теперь в шаблоне можно использовать {{ doubleCount }} без обращения к this.$store.getters

Что нужно знать:

  • Можно использовать как массив строк, так и объект для переименования
  • Можно комбинировать несколько вызовов mapGetters внутри одного computed
  • Геттеры должны быть заранее определены в вашем Vuex-store

✏️ mapMutations

mapMutations — хелпер для быстрого связывания методов компонента с мутациями Vuex. Используется только внутри methods!

Зачем нужен mapMutations?

  • Упрощает вызов мутаций Vuex
  • Сокращает количество кода в компонентах
  • Позволяет легко переименовывать методы при подключении

Пример использования mapMutations:

До (без mapMutations): 
methods: {
  increment() {
    this.$store.commit("increment");
  }
}
После (с mapMutations): 
import { mapMutations } from "vuex";
export default {
  methods: {
    ...mapMutations(["increment"])
  }
}

Что здесь происходит?

  • Импортируем хелпер mapMutations из Vuex
  • Внутри methods используем оператор spread ..., чтобы "развернуть" результат работы mapMutations
  • mapMutations(['increment']) - подключает метод increment, который вызывает мутацию Vuex с тем же именем
  • Теперь в шаблоне можно вызывать метод increment напрямую через @click="increment"

Что нужно знать:

  • Можно использовать как массив строк, так и объект для переименования
  • Можно комбинировать несколько вызовов mapMutations внутри одного methods
  • Мутации должны быть заранее определены в вашем Vuex-store

🔄 mapActions

mapActions — хелпер для быстрого связывания методов компонента с действиями (actions) Vuex. Используется только внутри methods!

Зачем нужен mapActions?

  • Упрощает вызов действий Vuex
  • Сокращает количество кода в компонентах
  • Позволяет легко переименовывать методы при подключении

Пример использования mapActions:

Без mapActions: 
methods: {
  fetchData() {
    this.$store.dispatch("fetchData");
  },
  sendData() {
    this.$store.dispatch("sendData");
  },
  updateUser() {
    this.$store.dispatch("updateUser");
  }
}
С mapActions: 
import { mapActions } from "vuex";
export default {
  methods: {
    ...mapActions(["fetchData", "sendData", "updateUser"])
  }
}

Что здесь происходит?

  • Импортируем хелпер mapActions из Vuex
  • Внутри methods используем оператор spread ..., чтобы "развернуть" результат работы mapActions
  • mapActions(['fetchData', 'sendData', 'updateUser']) - подключает методы, которые вызывают соответствующие действия Vuex
  • Теперь в шаблоне можно вызывать методы напрямую через @click="fetchData", @click="sendData" и т.д.

Что нужно знать:

  • Можно использовать как массив строк, так и объект для переименования
  • Можно комбинировать несколько вызовов mapActions внутри одного methods
  • Действия должны быть заранее определены в вашем Vuex-store

Таблица сравнения Vuex-хелперов

Хелпер Для чего Где использовать Что подключает
mapState Удобное подключение state computed Переменные из state
mapGetters Удобное подключение getters computed Геттеры из store
mapMutations Связка методов с mutations methods Mutation-функции
mapActions Связка методов с actions methods Action-функции

computed - для вычисляемых свойств,
methods - для методов

FAQ / Ошибки

  • В: Где можно использовать mapState и mapGetters?
    О: Только внутри computed.
  • В: А mapMutations и mapActions?
    О: Только внутри methods.
  • В: Можно ли объединять несколько хелперов?
    О: Да, используйте ... (spread) для каждого.
  • В: Можно ли переименовывать?
    О: Да, передавайте объект ({ myVal: 'getterName' })
  • В: Можно комбинировать свои методы и хелперы?
    О: Конечно, хоть вместе, хоть по отдельности!
