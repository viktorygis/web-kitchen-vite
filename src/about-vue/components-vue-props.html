Props во Vue

Props - это механизм передачи данных от родительского компонента к дочернему в Vue.js.
Они позволяют компонентам быть более гибкими и переиспользуемыми, принимая входные данные в виде свойств.
Название "props" происходит от слова "properties" (свойства).

Основные моменты о props:

  • Передают данные сверху вниз (от родителя к ребенку)
  • Реактивны: при изменении в родителе, обновляются и в дочернем компоненте
  • Нельзя менять props внутри дочернего компонента
  • Можно валидировать типы и задавать значения по умолчанию

Объявление props

Объявление props - это способ указать, какие свойства компонент ожидает получить от родителя.

Способы объявления props:

  • Через массив - простой синтаксис без валидации типов;
  • Через объект - с указанием типов и дополнительных параметров.
  • Можно комбинировать оба способа в одном компоненте.

Объявление через массив

Объявление через массив - это простой способ указать имена props без дополнительной информации.

Пример простого объявления props через массив:

props: ['name', 'age']
Объяснение кода:
  • props: - ключевое слово для объявления props в компоненте;
  • ['name', 'age'] - массив строк, где каждая строка представляет имя prop, который компонент ожидает получить от родителя.

Объявление через объект:

Объявление через объект - позволяет указать типы и дополнительные параметры для каждого prop.

Пример объявления props через объект:

props: {
   name: String,
   age: Number
}
Объяснение кода:
  • props: - ключевое слово для объявления props в компоненте;
  • {...} - объект, где каждый ключ представляет имя prop, а значение указывает ожидаемый тип данных.
  • name: String - ожидается строка для prop name;
  • age: Number - ожидается число для prop age;

Пример использования props в компоненте:

Создадим простой компонент UserCard, который принимает два props: name и age, и отображает их.

Передача props

Передача props - это процесс, при котором родительский компонент передает данные дочернему компоненту через атрибуты.
В Vue это делается с помощью синтаксиса двоеточия (:) или ключевого слова v-bind.

Пример передачи props от родителя к дочернему компоненту:

<user-card :name="userName" :age="userAge"></user-card>
Объяснение кода:
  • <user-card> - использование дочернего компонента UserCard;
  • :name="userName" - передача значения переменной userName из родительского компонента в prop name дочернего компонента;
  • :age="userAge" - передача значения переменной userAge из родительского компонента в prop age дочернего компонента;

Валидация props

Валидация props - это способ убедиться, что переданные данные имеют правильный тип и соответствуют ожиданиям компонента.

Можно указать:

  • Тип данных (String, Number, Boolean, Array, Object и т.д.)
  • Обязательность (required: true/false)
  • Значение по умолчанию (default: значение) - если проп не передан

Пример валидации props и задания значений по умолчанию:

props: {
  name: {
    type: String, // Ожидаем строку
    required: true // Обязательный параметр
  },
  age: {
    type: Number, // Ожидаем число
    default: 18   // Значение по умолчанию
  }
}
Объяснение кода:
  • name: - prop, который ожидает строку и является обязательным;
  • age: - prop, который ожидает число и имеет значение по умолчанию 18, если не передан.

Массивы, объекты и кастомная валидация

Props могут принимать сложные типы данных, такие как массивы и объекты.
Также можно создавать собственные функции валидации для более сложных требований.

Пример props с массивами:

props: {
  items: {
    type: Array,
    default: () => []
  }
}
Объяснение кода:
  • items: - prop, который ожидает массив и имеет значение по умолчанию пустой массив;

Пример props с объектами:

props: {
  user: {
    type: Object,
    default: () => ({})
  }
}
Объяснение кода:
  • user: - prop, который ожидает объект и имеет значение по умолчанию пустой объект;

Пример props с кастомной валидацией:

props: {
  age: {
    type: Number,
    validator: (value) => {
      return value >= 0 && value <= 120;
    }
  }
}
Объяснение кода:
  • age: - prop, который ожидает число;
  • validator: - функция, которая проверяет, что значение age находится в диапазоне от 0 до 120.

Пример props с массивами, объектами и кастомной валидацией:

props: {
  items: {
    type: Array,
    default: () => []
  },
  user: {
    type: Object,
    default: () => ({}),
    validator: (value) => {
      return value.name && value.age;
    }
  }
}
Объяснение кода:
  • items: - prop, который ожидает массив и имеет значение по умолчанию пустой массив;
  • user: - prop, который ожидает объект и имеет значение по умолчанию пустой объект;
  • validator: - функция, которая проверяет, что объект user содержит свойства name и age.
