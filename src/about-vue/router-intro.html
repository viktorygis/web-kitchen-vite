Маршрутизация во Vue

Маршрутизация во Vue.js — это процесс управления навигацией между различными страницами или компонентами в приложении на Vue.js с использованием библиотеки Vue Router.
Vue Router — стандартный инструмент для создания одностраничных и многостраничных приложений без перезагрузки.

Зачем нужен Vue Router?

  • Создание SPA (Single Page Application).
  • Управление переходами между страницами без перезагрузки.
  • Гибкая настройка маршрутов, редиректов и обработки ошибок.
  • Работа с динамическими маршрутами и параметрами.
  • Поддержка вложенных маршрутов для сложных интерфейсов.
  • Интеграция с Vuex и другими библиотеками для управления состоянием.
  • Обеспечение плавного пользовательского опыта с помощью переходов и анимаций.

Основные возможности Vue Router:

  • Определение маршрутов и их компонентов.
  • Навигация с помощью <router-link> и программно через $router.
  • Динамические маршруты с параметрами.
  • Вложенные маршруты для создания сложных интерфейсов.
  • Редиректы и обработка 404 ошибок.
  • Навигационные охранники (guards) для контроля доступа к маршрутам.
  • Поддержка истории браузера и хеш-маршрутизации.

Структура роутера в проекте

Структура роутера в проекте на Vue.js с использованием Vue Router обычно включает в себя несколько ключевых файлов и папок, которые организуют маршруты и компоненты приложения.

  • Маршруты выносите в отдельный файл src/router.js
  • Страницы размещайте в views/ — каждый маршрут связан с отдельным компонентом.
  • Используйте осмысленные и человекочитаемые пути: /about , /profile , /contacts и т.д.

Пример структуры проекта:

📂 src/
 ├── 📂 components/
 ├── 📂 views/
 │    ├── Home.vue
 │    ├── About.vue
 │    ├── Contacts.vue
 │    ├── NotFound.vue
 │    ├── Dashboard.vue
 ├── router.js
 ├── App.vue
 ├── main.js
Основные файлы роутера:

router.js — здесь описываются все маршруты и их компоненты.
views/ — содержат компоненты-страницы для переходов.

Навигация между страницами

Навигация между страницами - ключевая функция Vue Router, позволяющая пользователям перемещаться по различным частям вашего приложения без перезагрузки страницы. Это обеспечивает плавный и быстрый пользовательский опыт.
Навигация осуществляется с помощью компонента <router-link> и программно через методы роутера.

Пример: использование <router-link> для навигации

<router-link to="/">Главная</router-link>
<router-link to="/about">О нас</router-link>
<router-link to="/contacts">Контакты</router-link>
Что происходит в коде:

Каждый <router-link> создаёт ссылку на указанный маршрут. При клике на ссылку Vue Router обновляет URL и отображает соответствующий компонент в <router-view> без перезагрузки страницы.

<router-view> — это контейнер, в котором отображается компонент, соответствующий текущему маршруту.
<router-link> — это компонент для создания ссылок на маршруты.

Пример использования в App.vue:

<template>
  <div>
    <nav>
      <router-link to="/">Главная</router-link>
      <router-link to="/about">О нас</router-link>
      <router-link to="/contacts">Контакты</router-link>
    </nav>
    <router-view></router-view>
  </div>
</template>
Что происходит в коде:

В этом примере <router-link> создаёт навигационные ссылки, а <router-view> отображает соответствующий компонент в зависимости от текущего маршрута.

Использование <router-link> и <router-view> обеспечивает плавную навигацию в вашем приложении Vue.js без перезагрузки страницы.

Редиректы и обработка 404

Редиректы и обработка несуществующих маршрутов — важные аспекты навигации в веб-приложениях. Они помогают направлять пользователей к нужному контенту и обеспечивают хороший пользовательский опыт.

Для чего нужно:

  • Перенаправление с устаревших URL на новые.
  • Обработка ошибок при вводе неправильного адреса.
  • Улучшение навигации и пользовательского опыта.

Редиректы:

Редирект автоматически перенаправляет пользователя с одного URL на другой. Это полезно, если вы хотите изменить структуру URL или направить пользователей на главную страницу.

Виды редиректов:

  • Прямой редирект: перенаправляет с одного пути на другой.
    { path: '/', redirect: '/dashboard' }
  • Редирект с параметрами: сохраняет параметры при перенаправлении.
    { path: '/user/:id', redirect: to => `/profile/${to.params.id}` }

Обработка 404:

Обработка 404 позволяет показывать пользователю специальную страницу, если он пытается перейти на несуществующий маршрут. Это улучшает пользовательский опыт и помогает избежать путаницы.

Как настроить:

Добавьте маршрут с path, который ловит все неизвестные адреса:

{ path: '/:pathMatch(.*)*', component: NotFound }

Здесь NotFound — это компонент, отображающий сообщение об ошибке 404.

Пример настройки редиректов и 404:

//NotFound — компонент для страницы 404
import NotFound from '@/views/NotFound.vue';

const routes = [
  //Dashboard, About, Contacts — ваши компоненты-страницы
  { path: '/', redirect: '/dashboard' },
  { path: '/dashboard', component: Dashboard },
  { path: '/about', component: About },
  { path: '/contacts', component: Contacts },

  //Обработка 404 — ловит все неизвестные пути
  { path: '/:pathMatch(.*)*', component: NotFound }
];
Что происходит в коде:
  • Редирект с главной страницы на /dashboard.
  • Определены маршруты для Dashboard, About и Contacts.
  • Добавлен маршрут для обработки 404, который отображает компонент NotFound для всех неизвестных путей.

Важно: Всегда ставьте маршрут 404 в конец списка маршрутов, чтобы он не перехватывал другие пути.

Переходы между страницами: основные сценарии с <router-link>

Основные сценарии навигации — это ключевые способы использования компонента <router-link> для навигации в приложениях на Vue.js с Vue Router. Они охватывают различные ситуации, с которыми могут столкнуться разработчики при создании маршрутов и ссылок в своих приложениях.

Основные сценарии навигации:

  • Создавать навигацию с <router-link>
  • Работать с динамическими ссылками
  • Открывать страницы в новой вкладке
  • Превращать кнопки в навигационные элементы

Создавать навигацию с <router-link>

Используйте компонент <router-link> для создания ссылок на маршруты в вашем приложении Vue. Это позволяет пользователям переходить между страницами без перезагрузки. 

<router-link to="/">Главная</router-link>
<router-link to="/about">О нас</router-link>
<router-link to="/contacts">Контакты</router-link>

<router-link> автоматически обрабатывает навигацию и обновляет URL без перезагрузки страницы. Это обеспечивает плавный пользовательский опыт в одностраничных приложениях (SPA).

Работать с динамическими ссылками

Иногда необходимо передавать параметры в маршруты. Например, для отображения профиля пользователя по его ID:

<router-link :to="profilePath">Профиль</router-link>
...
<script>
export default {
  data() {
    return {
      profilePath: "/profile"
    }
  }
}
</script>

Используйте :to с переменной или вычисляемым свойством. Ссылка будет автоматически обновляться, если profilePath изменится.
Это удобно для ссылок с параметрами, например "/user/" + userId .

Открывать страницы в новой вкладке

Чтобы открыть ссылку в новой вкладке, используйте атрибуты target="_blank" и rel="noopener" . 

<router-link to="/about" target="_blank" rel="noopener">О нас</router-link>

target="_blank" открывает ссылку в новой вкладке, а rel="noopener" предотвращает потенциальные уязвимости безопасности, связанные с открытием новых вкладок.

Превращать кнопки в навигационные элементы

Иногда нужно использовать кнопки для навигации. Вместо <router-link> можно использовать метод $router.push() внутри обработчика событий.

<button @click="goToContacts">Контакты</button>
...<script>
export default {
  methods: {
    goToContacts() {
      this.$router.push('/contacts');
    }
  }
}
</script>

В методе goToContacts используется this.$router.push('/contacts') для навигации к маршруту /contacts без перезагрузки страницы.

Стилизация в навигации router-link-active

Vue Router автоматически добавляет класс router-link-active к активной ссылке. Это позволяет легко стилизовать текущий маршрут. 

<style>
.router-link-active {
    font-weight: bold;
    color: #42b983;
    text-decoration: underline;
}
</style>
<nav>
    <router-link to="/">Главная</router-link>
    <router-link to="/about">О нас</router-link>
    <router-link to="/contacts">Контакты</router-link>
</nav>

Теперь активная ссылка будет выделяться жирным шрифтом, зелёным цветом и подчёркиванием.

Лучшие практики роутинга

Чистый и понятный роутинг улучшает поддержку и масштабируемость вашего приложения. Следуйте этим рекомендациям, чтобы создать удобную навигацию для пользователей и разработчиков.

  • Используйте осмысленные и человекочитаемые пути (например, /about , /profile , /contacts ).
  • Группируйте маршруты по функциональности (например, все маршруты для админки в отдельном блоке).
  • Используйте вложенные маршруты для сложных страниц с несколькими уровнями навигации.
  • Всегда добавляйте обработку 404 для несуществующих маршрутов.
  • Используйте редиректы для устаревших URL или главной страницы.
  • Документируйте структуру маршрутов в проекте для удобства команды.
  • Избегайте слишком длинных и сложных путей, чтобы не запутать пользователей.
  • Регулярно пересматривайте и оптимизируйте маршруты по мере роста приложения.
Чистый роутинг — залог понятного и поддерживаемого проекта!

Пример: рабочий router.js с 404 и редиректом

router.js

import { createRouter, createWebHistory } from 'vue-router';
import Home from '@/views/Home.vue';
import About from '@/views/About.vue';
import Contacts from '@/views/Contacts.vue';
import Dashboard from '@/views/Dashboard.vue';
import NotFound from '@/views/NotFound.vue';

const routes = [
  { path: '/', redirect: '/dashboard' }, // Главная ведёт на Dashboard
  { path: '/dashboard', component: Dashboard },
  { path: '/about', component: About },
  { path: '/contacts', component: Contacts },
  //Обработка 404 — ловит все неизвестные пути
  { path: '/:pathMatch(.*)*', component: NotFound } // Все неизвестные URL — страница 404
];

const router = createRouter({
  history: createWebHistory(),
  routes
});

export default router;

router.js - основной файл маршрутизации, где определяются все маршруты приложения. Здесь происходит настройка роутера и его маршрутов.

App.js

<template>
  <router-view></router-view>
</template>

<script>
export default {
  name: 'App'
};
</script>

App.js - основной файл приложения, где происходит настройка корневого компонента и его маршрутов.

main.js

import { createApp } from 'vue';
import App from './App.vue';
import router from './router';

createApp(App)
  .use(router)
  .mount('#app');

main.js - точка входа в приложение, где создаётся экземпляр Vue и подключается роутер.

Пример компонента страницы (Home.vue)

<template>
  <div>
    <h2>Главная страница</h2>
    <p>Добро пожаловать!</p>
  </div>
</template>
<script>
export default {
  name: 'Home'
};
</script>

Home.vue - компонент главной страницы, отображает приветственное сообщение.

Пример компонента страницы (Dashboard.vue)

<template>
  <div>
    <h2>404 - Страница не найдена</h2>
    <p>Извините, такой страницы не существует.</p>
    <router-link to="/dashboard">Вернуться на главную</router-link>
  </div>
</template>
<script>
export default {
  name: 'NotFound'
};
</script>

Dashboard.vue - компонент страницы 404, информирует пользователя о несуществующем маршруте и предлагает вернуться на главную страницу.

Этот пример можно использовать как шаблон для настройки роутинга в вашем проекте.

  • Создайте файл src/router.js и добавьте туда код из примера.
  • Создайте компоненты-страницы в папке src/views/ : Home.vue , About.vue , Contacts.vue , Dashboard.vue , NotFound.vue .
  • Импортируйте и используйте роутер в вашем основном файле приложения (обычно main.js или main.ts ):

Что происходит в коде подробно:

  • Импортируются необходимые функции из vue-router и компоненты-страницы.
  • Определяется массив маршрутов с путями, компонентами, редиректами и обработкой 404.
  • Создаётся экземпляр роутера с использованием createRouter и createWebHistory для истории браузера.
  • Экспортируется роутер для использования в основном файле приложения.

Практика: настройте свои маршруты

Создайте свои страницы и настройте переходы, 404 и редирект.

Минимальный пример — две страницы и роутинг

<!-- router.js -->
import { createRouter, createWebHistory } from "vue-router";
import Home from "./views/Home.vue";
import About from "./views/About.vue";
const routes = [
  { path: "/", component: Home },
  { path: "/about", component: About }
];
const router = createRouter({
  history: createWebHistory(),
  routes
});
export default router;

<!-- -------------------------------- -->
<!-- Home.vue -->
<template>
  <div>
    <h2>Главная страница</h2>
    <p>Добро пожаловать!</p>
  </div>
</template>
<script>
export default {
  name: "Home"
};
</script>

<!-- -------------------------------- -->
<!-- About.vue -->
<template>
  <div>
    <h2>О нас</h2>
    <p>Мы изучаем Vue Router!</p>
  </div>
</template>
<script>
export default {
  name: "About"
};
</script>

<!-- -------------------------------- -->
<!-- App.vue -->
<template>
  <div>
    <router-link to="/">Главная</router-link>
    <router-link to="/about">О нас</router-link>
    <router-view></router-view>
  </div>
</template>

Задание для самостоятельной работы:

  • Добавьте страницу Contacts.vue и маршрут /contacts
  • Добавьте обработку 404 через NotFound.vue и маршрут /:pathMatch(.*)*
  • Сделайте редирект с / на /dashboard
  • Добавьте вложенные маршруты для /dashboard (опционально)
