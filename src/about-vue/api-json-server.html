🛠 JSON Server – быстрый фейковый API для практики и тестирования

JSON Server — это npm-пакет, который позволяет быстро создать фейковое REST API на основе JSON-файла. Это отличный инструмент для фронтенд-разработчиков, которые хотят тестировать и отлаживать свои приложения без необходимости разворачивать полноценный бэкенд.
REST API - это способ взаимодействия между клиентом и сервером через стандартные HTTP-запросы (GET, POST, PUT, DELETE и т.д.).
JSON-файл - это текстовый файл, в котором данные хранятся в формате JSON (JavaScript Object Notation) — удобном для обмена данными между клиентом и сервером.
npm-пакет - это библиотека или инструмент, который можно установить через менеджер пакетов npm (Node Package Manager) для использования в своих проектах на JavaScript.

Что JSON Server представляет собой и зачем он нужен?

  • Это инструмент для быстрого создания фейкового REST API на основе JSON-файла.
  • Позволяет фронтенд-разработчикам тестировать и отлаживать приложения без необходимости разворачивать полноценный бэкенд.
  • Поддерживает стандартные HTTP-методы: GET, POST, PUT, DELETE и другие.
  • Идеален для прототипирования, обучения и разработки фронтенд-приложений.

🚀 С JSON Server ты сможешь быстро настроить тестовую среду для своих фронтенд-проектов и сосредоточиться на разработке без лишних сложностей!

⚡ Как работает JSON Server?

Принцип работы JSON Server очень прост: ты создаёшь JSON-файл с данными, запускаешь сервер, и он автоматически предоставляет REST API для этих данных.

Основные шаги работы с JSON Server:

  • Создание JSON-файла: Ты создаёшь файл db.json, в котором определяешь свои данные в формате JSON. Например, можно создать коллекции пользователей, постов и комментариев.
  • Запуск сервера: С помощью команды json-server --watch db.json ты запускаешь сервер, который начинает слушать HTTP-запросы на определённом порту (по умолчанию это http://localhost:3000).
  • Взаимодействие с API: Теперь ты можешь отправлять HTTP-запросы (GET, POST, PUT, DELETE и т.д.) к серверу, чтобы получать, добавлять, обновлять или удалять данные из твоего JSON-файла.

Пример:

Если в твоём db.json есть коллекция пользователей, ты можешь получить список всех пользователей с помощью GET-запроса к http://localhost:3000/users.

🔥 JSON Server автоматически обрабатывает запросы и обновляет твой JSON-файл, что делает его идеальным инструментом для быстрого прототипирования и тестирования фронтенд-приложений!

📌 Как установить и настроить JSON Server?

Чтобы начать использовать JSON Server, следуй этим простым шагам:

1. Установи JSON Server глобально через npm:

Открой терминал и выполни команду:

npm install -g json-server

2. Создай файл db.json с нужными данными:

Вот пример простого db.json:

{
  "posts": [
    { "id": 1, "title": "Hello World", "author": "John" }
  ],
  "users": [
    { "id": 1, "name": "Alice" }
  ]
}

3. Запусти JSON Server с помощью команды:

В терминале, в той же папке, где находится db.json, выполни:

json-server --watch db.json

4. Сервер по умолчанию работает на http://localhost:3000.

Теперь ты можешь отправлять HTTP-запросы к своему фейковому API!

📝 Пример db.json и запуск сервера

Давай рассмотрим простой пример создания файла db.json и запуска JSON Server.

Создай файл db.json с таким содержимым:

{
  "users": [
    { "id": 1, "name": "Alice", "age": 25 },
    { "id": 2, "name": "Bob", "age": 30 }
  ]
}

Запусти JSON Server с помощью следующей команды:

npx json-server --watch db.json --port 5000

Теперь твой сервер работает по адресу http://localhost:5000!

Объяснение кода:
  • В файле db.json мы создали коллекцию users с двумя пользователями, каждый из которых имеет id, name и age.
  • Команда npx json-server --watch db.json --port 5000 запускает JSON Server, который следит за изменениями в файле db.json и обслуживает запросы на порту 5000.

🔥 GET-запрос: получение данных от JSON Server

Получение данных от JSON Server — для этого мы используем метод fetch в JavaScript и отправляем GET-запрос к нужному эндпоинту (например, /users).

Пример GET-запроса:

Проверим работу сервера простым GET-запросом. Запросим список пользователей:

fetch('http://localhost:5000/users', {
  method: "GET"
})
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('Ошибка:', error));

db.json:

{
  "users": [
    { "id": 1, "name": "Alice", "age": 25 },
    { "id": 2, "name": "Bob", "age": 30 }
  ]
}
Объяснение кода:
  • С помощью fetch отправляем запрос к /users (смотри на адрес!)
  • Первый then переводит ответ из JSON-строки в объект
  • Второй then выводит результат в консоль
  • catch ловит и выводит ошибку, если что-то пошло не так

🚀Можно так же обращаться к /posts или любым другим коллекциям, что указаны в db.json.
