Обработка ошибок маршрутизации в Vue Router

Ошибки маршрутизации — это ситуации, когда пользователь пытается перейти по маршруту, который не существует, не имеет доступа, или происходит сбой при переходе.
Грамотная обработка таких ситуаций важна для удобства пользователя и предотвращения "битых" страниц.

Что изучаем?

  • Как настроить страницу 404 для несуществующих маршрутов
  • Как перехватывать ошибки перехода с помощью navigation guards
  • Как программно обрабатывать ошибки перехода (try/catch)

Страница 404 (Не найдено)

404 — это специальная страница, которая показывается, если пользователь перешёл по несуществующему маршруту.

Как настроить?


// В массиве маршрутов добавьте "ловушку":
const routes = [
  // ваши маршруты...
  { path: '/:pathMatch(.*)*', name: 'NotFound', component: NotFound }
];

Теперь при переходе на любой неизвестный адрес пользователь увидит компонент NotFound.

Пример компонента:


<template>
  <div class="not-found">
    <h2>404 — Страница не найдена</h2>
    <router-link to="/">На главную</router-link>
  </div>
</template>

Перехват ошибок перехода (Navigation Guards)

Иногда нужно запретить переход или проверить доступ. Для этого используются guards — специальные хуки в маршрутизаторе.

Пример глобального guard:


router.beforeEach((to, from, next) => {
  if (to.meta.requiresAuth && !isLoggedIn()) {
    next('/login'); // Перенаправить на логин
  } else {
    next(); // Разрешить переход
  }
});

Можно использовать next(false), чтобы отменить переход, или next(error) — если произошла ошибка.

Ловим ошибку перехода:


router.onError((error) => {
  console.error('Ошибка маршрутизации:', error.message)
  // Можно показать уведомление или перекинуть на страницу ошибки
});

Программная обработка ошибок перехода

Методы router.push и router.replace возвращают Promise — можно ловить ошибки через catch: 


this.$router.push('/secret-page')
  .catch(err => {
    if (err.name === 'NavigationDuplicated') return; // игнорировать повторный переход
    alert('Ошибка перехода: ' + err.message);
  });

Важно: Не забывайте обрабатывать ошибки при программной навигации — это предотвращает появление неожиданных багов!

Практика: обработка ошибок маршрутизации

  • Добавь маршрут 404 и проверь переход на несуществующую страницу
  • Реализуй guard, запрещающий переход на страницу /secret без авторизации
  • Сделай кнопку, которая программно пытается перейти на /secret и обрабатывает ошибку

App.vue

<!-- App.vue -->
<template>
  <div>
    <router-link to="/">Главная</router-link>
    <router-link to="/secret">Секретная страница</router-link>
    <button @click="goToSecret">Перейти к секрету (catch)</button>
    <router-link to="/no-page">Несуществующая</router-link>
    <router-view/>
  </div>
</template>

<script>
export default {
  methods: {
    goToSecret() {
      this.$router.push('/secret').catch(err => {
        alert('Ошибка перехода: ' + err.message);
      });
    }
  }
}
</script>

NotFound.vue

<!-- NotFound.vue -->
<template>
  <div class="not-found">
    <h2>404 — Страница не найдена</h2>
    <router-link to="/">На главную</router-link>
  </div>
</template>

Частые ошибки и вопросы

  • Ошибка: Не добавили маршрут /:pathMatch(.*)* — 404 не срабатывает!
  • Ошибка: В guard забыли вызвать next() или вернуть результат — переход "зависает".
  • Вопрос: Как обработать ошибку перехода программно?
    this.$router.push('/something').catch(err => { ... })
  • Вопрос: Как настроить редирект на логин?
    В guard: next('/login')
