Вложенные маршруты во Vue Router

Вложенные маршруты помогают строить иерархию страниц, когда внутри одного раздела есть подстраницы: настройки, история заказов, подписки. Это как этажи и комнаты в большом торговом центре!

  • Группировка связанных страниц под одним "родителем" (например, /profile/settings).
  • Единое оформление и логика для раздела (меню, заголовок, боковая панель).
  • Чистый, поддерживаемый router.js без каши из десятков отдельных маршрутов.

Структура вложенных маршрутов

Вместо множества отдельных путей, объединяем их внутри одного родителя с помощью children : 

const routes = [
  {
    path: "/profile",
    component: Profile,
    children: [
      { path: "settings", component: Settings },
      { path: "orders", component: Orders }
    ]
  }
];
  • /profile – главная страница раздела.
  • /profile/settings – страница настроек.
  • /profile/orders – история заказов.

Важно! Не начинай вложенные пути с / — иначе они станут отдельными маршрутами.

Использование router-view для вложенных маршрутов

Чтобы вложенные компоненты действительно отображались внутри родителя, добавь в шаблон <router-view> : 

<template>
  <div>
    <h1>👤 Личный кабинет</h1>
    <nav>
      <router-link to="/profile/settings">⚙ Настройки</router-link>
      <router-link to="/profile/orders">🛍 История заказов</router-link>
    </nav>

    <router-view></router-view> <!-- Тут рендерятся дочерние компоненты! -->
  </div>
</template>
  • Вложенные страницы отображаются внутри родителя.
  • Меню и заголовок сохраняются при навигации.

Страница по умолчанию для вложенного маршрута

Если перейти на /profile , может появиться пустая страница. Чтобы показывать контент по умолчанию, добавь дочерний маршрут с пустым path : 

const routes = [
  {
    path: "/profile",
    component: Profile,
    children: [
      { path: "", component: Settings }, // по умолчанию
      { path: "orders", component: Orders }
    ]
  }
];
  • При переходе на /profile загружается Settings .
Без маршрута с пустым path родительская страница будет пустой!

Практика: настрой вложенные маршруты

  • Создай раздел /shop с дочерними страницами products и cart .
  • Добавь router-view в Shop.vue .
  • Сделай /shop/products страницей по умолчанию при переходе на /shop .

Минимальный пример вложенных маршрутов

router.js

// router.js
import { createRouter, createWebHistory } from "vue-router";
import Shop from "./views/Shop.vue";
import Products from "./views/Products.vue";
import Cart from "./views/Cart.vue";

const routes = [
  {
    path: "/shop",
    component: Shop,
    children: [
      { path: "", component: Products }, // по умолчанию
      { path: "products", component: Products },
      { path: "cart", component: Cart }
    ]
  }
];
const router = createRouter({
  history: createWebHistory(),
  routes
});
export default router;

Shop.vue

<!-- Shop.vue -->
<template>
  <div>
    <h2>Магазин</h2>
    <nav>
      <router-link to="/shop/products">Товары</router-link>
      <router-link to="/shop/cart">Корзина</router-link>
    </nav>
    <router-view></router-view>
  </div>
</template>

<script>
export default {
  name: "Shop"
};
</script>

Products.vue

<!-- Products.vue -->
<template>
  <div>
    <h3>Товары</h3>
    <ul>
      <li>Ноутбук</li>
      <li>Смартфон</li>
      <li>Наушники</li>
    </ul>
  </div>
</template>

Cart.vue

<!-- Cart.vue -->
<template>
  <div>
    <h3>Корзина</h3>
    <p>Ваша корзина пуста.</p>
  </div>
</template>

App.vue

<!-- App.vue -->
<template>
  <div>
    <router-link to="/shop">Магазин</router-link>
    <router-view></router-view>
  </div>
</template>

Что происходит?

  • В роутере раздел /shop содержит дочерние страницы.
  • В Shop.vue есть меню и router-view для дочерних компонентов.
  • По умолчанию отображается Products , если перейти на /shop .
Попрактикуйся, чтобы закрепить знание вложенных маршрутов!

Лучшие практики работы с вложенными маршрутами

  • Используй children для связанных страниц, не плодя отдельные маршруты.
  • Добавляй router-view в родительский компонент!
  • Делай страницу по умолчанию с пустым path в children .
  • Переиспользуй общий UI (заголовок, меню) в родителе — меньше дублирования.
  • Документируй маршруты и иерархию для команды.
  • Добавляй обработку ошибок для несуществующих вложенных страниц.
Вложенная структура маршрутов — залог чистой архитектуры и удобства поддержки!
