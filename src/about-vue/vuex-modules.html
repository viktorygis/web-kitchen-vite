Vuex-модули: структура, использование и преимущества

Vuex-модули — это способ структурировать хранилище состояния в крупном приложении Vue, разбивая store на независимые секции: user, cart, settings и др. Так ты избегашь хаоса и обеспечиваешь лёгкую поддержку кода!

Что мы изучим?

  • Как разделять state, getters, mutations и actions по модулям
  • Как импортировать и подключать модули в store.js
  • Как правильно получать и обновлять данные из разных модулей в компонентах

✅ Модули — ключ к устранению беспорядка в store!

📦 Проблема большого store и решение через модули

Когда всё лежит в одном store.js, появляются сложности в поддержке и развитии приложения.

❌ Всё в одном месте (данные пользователя, корзина покупок, настройки темы.) → хаос и ошибки!

Например:

const store = createStore({
  state: {
    user: { name: "Алексей", isAdmin: true },
    cart: [{ id: 1, name: "Ноутбук", price: 1000 }],
    theme: "dark"
  },
  mutations: {
    setUser(state, user) { state.user = user; },
    addToCart(state, item) { state.cart.push(item); },
    changeTheme(state, theme) { state.theme = theme; }
  }
});

✅ Решение: разбиваем store на модули.

📂 store
 ├── 📂 modules
 │    ├── user.js
 │    ├── cart.js
 │    ├── settings.js
 ├── store.js

Теперь каждый раздел — отдельный модуль!

🚀 Как создать модуль Vuex?

Чтобы создать модуль, нужно определить его state, mutations и actions в отдельном файле. Создаем файл store/modules/user.js:

Пример: user.js.

// store/modules/user.js
export default {
  state: { user: { name: "Алексей" } },
  mutations: {
    setUserName(state, newName) {
      state.user.name = newName;
    }
  },
  actions: {
    asyncChangeUserName({ commit }, newName) {
      setTimeout(() => { commit("setUserName", newName); }, 2000);
    }
  }
}

Пояснение:

  • state: хранит данные пользователя.
  • mutations: синхронно изменяют имя пользователя.
  • actions: асинхронно меняют имя с задержкой.

📥 Как импортировать и подключить модуль в store.js?

  1. Импортируем модуль: 
    import userModule from "./modules/user";
  2. Добавляем его в объект modules: 
    const store = createStore({
  modules: {
    user: userModule // Теперь доступен как store.state.user
  }
});

Теперь 👤-данные доступны через store.state.user.

🔎 Получение данных из модулей в компонентах

В обычном store: 

computed: {
  count() {
    return this.$store.state.count;
  }
}

В модульном store: 

computed: {
  userName() {
    return this.$store.state.user.user.name; // 📦 теперь данные из модуля
  }
}
  • В state модулей обращаемся через название модуля (user, cart)
  • Для хелперов типа mapState первым параметром передаём строку — имя модуля: 
    import { mapState } from "vuex";
computed: {
  ...mapState("cart", ["items"])
}
  • Без указания модуля данные ищутся в основном state!

Пример:

// Получить массив товаров из модуля cart
this.$store.state.cart.items

🔄 Обновление данных: Mutations и Actions в модулях

✏️ Mutations (синхронно)

  • Меняют state мгновенно
  • Вызываются commit()
const userModule = {
  state: { user: { name: "Алексей" } },
  mutations: {
    setUserName(state, newName) {
      state.user.name = newName;
    }
  }
}
// В компоненте:
this.$store.commit("setUserName", "Иван")

⏳ Actions (асинхронные задачи)

  • Для задержек, API-запросов и пр.
  • Вызываются dispatch() и в итоге commit()
const userModule = {
  state: { user: { name: "Алексей" } },
  mutations: {
    setUserName(state, newName) {
      state.user.name = newName;
    }
  },
  actions: {
    asyncChangeUserName({ commit }, newName) {
      setTimeout(() => {
        commit("setUserName", newName);
      }, 2000);
    }
  }
}
// В компоненте:
this.$store.dispatch("asyncChangeUserName", "Иван")

Сравнение mutations и actions

Способ Как вызываем? Когда использовать? Меняет state?
mutations (Мгновенно) commit("mutationName") При моментальном изменении Да, сразу
actions (C задержками) dispatch("actionName") Ожидания, запросы, таймеры Нет, только через mutations

FAQ / Ошибки

  • В: Где обычно размещать модули?
    О: В папке store/modules/.
  • В: Как получить данные из модуля?
    О: Через this.$store.state.moduleName.property.
  • В: Как вызвать mutation из модуля?
    О: this.$store.commit("mutationName", payload)
  • В: Как вызвать action из модуля?
    О: this.$store.dispatch("actionName", payload)
  • В: Когда использовать actions, а когда mutations?
    О: Actions — для задержек, запросов; mutations — для быстрой смены состояния.

Упражнение

📌 Если нужно сразу изменить state, вызываем commit(),
а если требуется задержка или запрос к серверу — используем dispatch() и затем commit().
