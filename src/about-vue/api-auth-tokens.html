Работа с токенами авторизации во Vue.js и Vuex

Токены авторизации — это современный способ аутентификации и разрешения доступа к API. Они широко используются в веб-приложениях для обеспечения безопасности и управления сессиями пользователей.

Для чего нужны токены авторизации?

  • 🔐 Безопасность: Токены позволяют безопасно передавать информацию о пользователе между клиентом и сервером без необходимости отправлять логин и пароль при каждом запросе.
  • ⚙️ Управление сессиями: Токены помогают управлять сессиями пользователей, позволяя им оставаться авторизованными в течение определённого времени.
  • 🌐 Масштабируемость: Токены облегчают масштабирование приложений, так как серверу не нужно хранить состояние сессий на своей стороне.
  • 🔄 Обновление доступа: С помощью refresh токенов можно обновлять основной токен без необходимости повторной аутентификации пользователя.

Основные понятия

  • Auth Token — основной токен, подтверждающий личность пользователя и дающий доступ к защищённым ресурсам.
  • Refresh Token — вспомогательный токен, позволяющий получить новый auth token без повторного ввода логина/пароля.
  • Хранение токенов — важный аспект безопасности: токены можно хранить в памяти, localStorage, sessionStorage или cookies.
  • Использование токенов в запросах — обычно токены отправляются в заголовке Authorization для аутентификации запросов к API.
  • Интеграция с Vuex — централизованное управление токенами и состоянием авторизации через Vuex store.

🚀 Токены — это ключ к безопасности и управлению сессиями в современных веб-приложениях. Освоив работу с ними, ты сможешь создавать безопасные и масштабируемые приложения на Vue.js!

Что такое токен авторизации?

Токен — это строка, которая подтверждает твою личность и даёт доступ к приватным данным или действиям в приложении.

Типы токенов:

  • Access Token — основной токен, дающий доступ к защищённым ресурсам API.
  • Refresh Token — вспомогательный токен, позволяющий получить новый Access Token без повторного ввода логина/пароля.

Как выглядит токен?

  • Токен выглядит как случайный длинный набор символов, например: eyJhbGciOiJIUzI1NiIsIn...
  • Токен генерируется сервером после успешной авторизации (логина) пользователя.
  • Его нужно отправлять в каждом защищённом запросе к API.

Когда токен есть — не нужно вводить логин/пароль каждый раз. Токен — твой цифровой пропуск!

Где хранить токен в приложении?

Есть 3 основных способа хранить токен на клиенте:

  • В памяти приложения (state компонента, переменные) — исчезает при перезагрузке!
  • В LocalStorage или sessionStorage — токен остаётся между перезагрузками браузера.
  • В Cookies — часто для server-side rendering и автоматической отправки серверу, но требует осторожности.

🔐 Лучше всего хранить токен в sessionStorage, если он не нужен между сессиями, или в cookies с флагом HttpOnly и Secure для серверной части. Так как токены в cookies автоматически отправляются с каждым запросом к серверу.

Пример хранения токена в localStorage:


// Сохраняем токен после логина
localStorage.setItem('auth_token', 'YOUR_TOKEN_HERE');

// Получаем токен из localStorage
const token = localStorage.getItem('auth_token');

Для безопасности не стоит хранить важные токены в localStorage при работе с критичными данными — разбирайся в угрозах XSS!

Использование токена для защищённых запросов к API

После того, как ты получил и сохранил токен, его нужно использовать для аутентификации при каждом запросе к защищённым ресурсам API.
Защищенный запрос - это запрос, который требует наличия действительного токена для доступа к данным или выполнению действий.

Что нужно сделать:

  • При отправке HTTP-запроса добавь токен в заголовок Authorization.
  • Обычно используется схема Bearer, но может быть и другая в зависимости от API.

Пример GET-запроса с токеном в заголовке:

Предположим, у тебя есть токен, сохранённый в localStorage:


fetch('https://api.example.com/data', {
  method: 'GET',
  headers: {
    'Authorization': `Bearer ${token}` // Вставляем токен в заголовок
  }
})
.then(response => response.json())
.then(data => console.log(data))
.catch(error => console.error('Ошибка:', error));
Объяснение кода:
  • Токен берётся из localStorage
  • Он передаётся в заголовке Authorization с префиксом Bearer
  • Токен нельзя «зашивать» в коде — бери всегда из хранилища!
  • О��ычно тип токена — Bearer, но для некоторых API может быть другой.

При каждом запросе к защищённому ресурсу Authorization-заголовок обязателен!

Как обновлять истёкшие токены? (Refresh token)

Токены авторизации часто имеют ограниченный срок действия для безопасности. Когда токен истекает, нужно получить новый, используя refresh token.

Как это работает?

  • Когда основной токен истекает, клиент отправляет запрос на специальный эндпоинт обновления токена.
  • В этом запросе используется refresh token, который был получен вместе с основным токеном при логине.
  • Если refresh token действителен, сервер возвращает новый основной токен (и возможно новый refresh token).

Пример обновления токена:

Предположим, у тебя есть refresh token, сохранённый в localStorage:


const refreshToken = localStorage.getItem('refresh_token');

fetch('https://api.example.com/refresh-token', {
  method: 'POST',
  body: JSON.stringify({ refresh_token: refreshToken }),
  headers: { 'Content-Type': 'application/json' }
})
  .then(response => response.json())
  .then(data => {
    localStorage.setItem('auth_token', data.auth_token);
    localStorage.setItem('refresh_token', data.refresh_token);
  })
  .catch(error => console.error('Ошибка:', error));
  • refresh_token часто обновляется вместе с основным токеном
  • Если истёк и refresh_token — нужен новый вход (логин)

Как интегрировать токены с Vuex?

Vuex отлично подходит для централизованного хранения текущего токена и автоматического обновления UI после авторизации.

Минимальный пример Vuex store для токенов:


// store.js
import { createStore } from 'vuex'

export default createStore({
  state: {
    authToken: localStorage.getItem('auth_token') || null,
    refreshToken: localStorage.getItem('refresh_token') || null
  },
  mutations: {
    setTokens(state, { authToken, refreshToken }) {
      state.authToken = authToken
      state.refreshToken = refreshToken
      localStorage.setItem('auth_token', authToken)
      localStorage.setItem('refresh_token', refreshToken)
    },
    clearTokens(state) {
      state.authToken = null
      state.refreshToken = null
      localStorage.removeItem('auth_token')
      localStorage.removeItem('refresh_token')
    }
  },
  actions: {
    async login({ commit }, credentials) {
      const response = await fetch('https://api.example.com/login', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(credentials)
      })
      const data = await response.json()
      commit('setTokens', {
        authToken: data.auth_token,
        refreshToken: data.refresh_token
      })
    },
    async refreshTokens({ commit, state }) {
      const response = await fetch('https://api.example.com/refresh-token', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ refresh_token: state.refreshToken })
      })
      const data = await response.json()
      commit('setTokens', {
        authToken: data.auth_token,
        refreshToken: data.refresh_token
      })
    },
    logout({ commit }) {
      commit('clearTokens')
    }
  }
})
Объяснение кода:
  • Токены хранятся в состоянии Vuex и синхронизируются с localStorage.
  • Мутации setTokens и clearTokens управляют состоянием токенов.
  • Действия login и refreshTokens выполняют запросы к API для получения и обновления токенов.

Используя Vuex для управления токенами, ты можешь легко интегрировать аутентификацию в своё Vue-приложение и обеспечить централизованный доступ к токенам во всех компонентах!

🔥 Практика с токенами

В этом упражнении мы создадим простое Vue-приложение с формой логина, которое будет сохранять токены авторизации в Vuex store и localStorage. Мы также реализуем автоматическое обновление токенов при их истечении.

Создай 3 файла и используй паттерн, описанный выше:

  • store.js — Vuex store для токенов и авторизации
  • LoginForm.vue — форма входа (логин/пароль), сохраняющая токены при успешном входе
  • main.js — подключение store и запуск приложения

Шаг 1: Vuex Store (store.js)

store.js — здесь мы будем хранить токены и методы для логина, обновления токенов и выхода:


// store.js
import { createStore } from 'vuex'

export default createStore({
  state: {
    authToken: localStorage.getItem('auth_token') || null,
    refreshToken: localStorage.getItem('refresh_token') || null
  },
  mutations: {
    setTokens(state, { authToken, refreshToken }) {
      state.authToken = authToken
      state.refreshToken = refreshToken
      localStorage.setItem('auth_token', authToken)
      localStorage.setItem('refresh_token', refreshToken)
    },
    clearTokens(state) {
      state.authToken = null
      state.refreshToken = null
      localStorage.removeItem('auth_token')
      localStorage.removeItem('refresh_token')
    }
  },
  actions: {
    async login({ commit }, credentials) {
      // credentials { email, password }
      const response = await fetch('https://api.example.com/login', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(credentials)
      })
      const data = await response.json()
      commit('setTokens', {
        authToken: data.auth_token,
        refreshToken: data.refresh_token
      })
    },
    async refreshTokens({ commit, state }) {
      const response = await fetch('https://api.example.com/refresh-token', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ refresh_token: state.refreshToken })
      })
      const data = await response.json()
      commit('setTokens', {
        authToken: data.auth_token,
        refreshToken: data.refresh_token
      })
    },
    logout({ commit }) {
      commit('clearTokens')
    }
  }
})
Объяснение кода:
  • Токены хранятся в состоянии Vuex и синхронизируются с localStorage.
  • Мутации setTokens и clearTokens управляют состоянием токенов.
  • Действия login, refreshTokens и logout выполняют запросы к API для получения, обновления токенов и выхода.

Шаг 2: Компонент LoginForm.vue

LoginForm.vue — форма для ввода email и пароля, которая вызывает действие логина из Vuex store:


<template>
  <form @submit.prevent="login">
    <input v-model="email" placeholder="Email" type="email" required>
    <input v-model="password" placeholder="Пароль" type="password" required>
    <button type="submit">Войти</button>
  </form>
</template>

<script>
export default {
  data() {
    return {
      email: '',
      password: ''
    }
  },
  methods: {
    async login() {
      try {
        await this.$store.dispatch('login', {
          email: this.email,
          password: this.password
        })
        alert('Успешный вход!')
      } catch (error) {
        alert('Ошибка входа')
      }
    }
  }
}
</script>
Объяснение кода:
  • Форма собирает email и пароль пользователя.
  • При отправке формы вызывается метод login, который диспатчит действие login из Vuex store.
  • После успешного входа токены сохраняются в Vuex и localStorage.

Шаг 3: main.js

main.js — подключение store и запуск приложения:


import { createApp } from 'vue'
import App from './App.vue'
import store from './store'

createApp(App)
  .use(store)
  .mount('#app')
Объяснение кода:
  • Импортируем Vue, корневой компонент App и Vuex store.
  • Создаём приложение Vue, подключаем Vuex store и монтируем его к элементу с id app.

Теперь ты можешь безопасно авторизоваться, хранить и обновлять токены в современном Vue/Vuex приложении!
