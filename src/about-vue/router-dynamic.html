Динамические маршруты и параметры во Vue Router

Динамические маршруты позволяют создавать гибкую навигацию, где часть URL может быть переменной (например, id пользователя или товара).

  • Переход на одну страницу для разных сущностей (профиль, товар, категория).
  • Передача данных через адресную строку без query-параметров.
  • Автоматическая подгрузка нужного контента на основе URL.

Vue Router связывает параметры из URL с компонентом, делая работу с данными интуитивно понятной.

Структура динамического маршрута

Вместо статичного пути можно указать переменную с двоеточием:

{ path: '/profile/:id', component: Profile }
  • :id — динамический параметр, который подставляется в маршрут.
  • При переходе на /profile/42 параметр id будет равен 42 .
  • Можно добавить несколько параметров: /category/:categoryId/product/:productId

Важно: Имя параметра после двоеточия должно совпадать с тем, что используется в компоненте для извлечения данных.

Извлечение параметров маршрута внутри компонента

Параметры доступны через this.$route.params . Обычно их извлекают через вычисляемое свойство: 

<template>
  <div>
    <h1>Профиль пользователя</h1>
    <p>ID: {{ userId }}</p>
  </div>
</template>

<script>
export default {
  computed: {
    userId() {
      return this.$route.params.id; // извлекаем id из URL
    }
  }
};
</script>
  • computed — реактивно обновляет значение при изменении маршрута.
  • this.$route.params.id — значение id из URL.

Лучше использовать computed , чтобы компонент автоматически обновлялся при переходе на другой id.

Динамические ссылки с параметрами

Для перехода на страницу с параметром используйте router-link и подставьте id: 

<router-link :to="`/profile/${user.id}`">Посмотреть профиль</router-link>
  • :to="..." — v-bind для динамического пути.
  • \`/profile/${user.id}\` — подставляет id пользователя.

Пример списка пользователей:

<ul>
  <li v-for="user in users" :key="user.id">
    <router-link :to="`/profile/${user.id}`">
      {{ user.name }}
    </router-link>
  </li>
</ul>

Теперь каждый клик по имени ведёт на индивидуальный профиль.

Объектный синтаксис для передачи параметров

Если маршрут задан с name , можно передавать параметры объектом: 

<router-link :to="{ name: 'profile', params: { id: user.id } }">
  Профиль {{ user.name }}
</router-link>
  • name: 'profile' — название маршрута в router.js .
  • params: { id: user.id } — параметры для подстановки в путь.

Настройка маршрута с name: 

const routes = [
  { path: '/profile/:id', name: 'profile', component: Profile }
];

Объектный синтаксис удобен при сложных путях и передаче нескольких параметров.

Передача нескольких параметров в маршруте

Можно использовать сразу несколько параметров в пути:

const routes = [
  { path: '/category/:categoryId/product/:productId', component: ProductPage }
];

Для перехода к конкретному товару:

<router-link :to="`/category/${categoryId}/product/${productId}`">
  Перейти к товару
</router-link>
  • this.$route.params.categoryId
  • this.$route.params.productId

Это удобно для магазинов, форумов, блогов и других приложений с вложенной структурой данных.

Пример: динамический список товаров

Список товаров, где каждая карточка ведёт на /product/:id . 

<template>
  <div>
    <h2>📦 Список товаров</h2>
    <ul>
      <li v-for="product in products" :key="product.id">
        <router-link :to="`/product/${product.id}`">
          {{ product.name }} - {{ product.price }} ₽
        </router-link>
      </li>
    </ul>
  </div>
</template>

<script>
export default {
  data() {
    return {
      products: [
        { id: 1, name: "Ноутбук", price: 50000 },
        { id: 2, name: "Смартфон", price: 30000 },
        { id: 3, name: "Наушники", price: 5000 }
      ]
    };
  }
};
</script>

Такой компонент покажет список товаров, а при клике — откроет страницу товара по его id.

Практика: создайте динамические маршруты

  • Добавьте страницу профиля с маршрутом /profile/:id
  • Выведите список пользователей и ссылки на их профиль через <router-link>
  • Создайте страницу товара с маршрутом /product/:id
  • Передавайте параметры через объектный синтаксис и через строку
  • Добавьте маршрут с двумя параметрами: /category/:categoryId/product/:productId

Минимальный пример динамического роутинга

router.js

// router.js
import { createRouter, createWebHistory } from "vue-router";
import Profile from "./views/Profile.vue";
import ProductPage from "./views/ProductPage.vue";

const routes = [
  { path: "/profile/:id", component: Profile },
  { path: "/product/:id", component: ProductPage },
  { path: "/category/:categoryId/product/:productId", component: ProductPage }
];
const router = createRouter({
  history: createWebHistory(),
  routes
});
export default router;

Profile.vue

<!-- Profile.vue -->
<template>
  <div>
    <h2>Профиль пользователя</h2>
    <p>ID: {{ userId }}</p>
  </div>
</template>
<script>
export default {
  computed: {
    userId() {
      return this.$route.params.id;
    }
  }
};
</script>

ProductPage.vue

<!-- ProductPage.vue -->
<template>
  <div>
    <h2>Товар</h2>
    <p v-if="$route.params.categoryId">Категория: {{ $route.params.categoryId }}</p>
    <p>ID товара: {{ $route.params.id || $route.params.productId }}</p>
  </div>
</template>
<script>
export default {
  computed: {
    // Для универсальности: поддержка /product/:id и /category/:categoryId/product/:productId
    productId() {
      return this.$route.params.id || this.$route.params.productId;
    },
    categoryId() {
      return this.$route.params.categoryId;
    }
  }
};
</script>

App.vue

<!-- App.vue -->
<template>
  <div>
    <h3>Список пользователей</h3>
    <ul>
      <li v-for="user in users" :key="user.id">
        <router-link :to="`/profile/${user.id}`">{{ user.name }}</router-link>
      </li>
    </ul>

    <h3>Список товаров</h3>
    <ul>
      <li v-for="product in products" :key="product.id">
        <router-link :to="`/product/${product.id}`">
          {{ product.name }} (строка)
        </router-link>
        |
        <router-link :to="{ name: 'product', params: { id: product.id } }">
          {{ product.name }} (объект)
        </router-link>
      </li>
    </ul>

    <h3>Товар по категории</h3>
    <router-link :to="`/category/2/product/15`">Категория 2, Товар 15</router-link>

    <router-view></router-view>
  </div>
</template>
<script>
export default {
  data() {
    return {
      users: [
        { id: 1, name: "Алексей" },
        { id: 2, name: "Марина" },
        { id: 3, name: "Владимир" }
      ],
      products: [
        { id: 11, name: "Ноутбук" },
        { id: 22, name: "Смартфон" },
        { id: 33, name: "Наушники" }
      ]
    };
  }
};
</script>

Что здесь происходит?

  • Настраивается роутер с тремя маршрутами, включая маршрут с двумя параметрами.
  • Компонент профиля извлекает id из URL и отображает его.
  • Компонент товара поддерживает оба варианта URL и показывает id и категорию.
  • В App.vue создаётся список пользователей и товаров с динамическими ссылками.
  • Используются оба способа передачи параметров: строка и объект.
Практика с динамическими маршрутами поможет лучше понять принципы работы Vue Router!

Лучшие практики работы с динамическими маршрутами

  • Используйте computed для получения параметров маршрута — они автоматически обновляются.
  • Давайте осмысленные названия параметрам (например, userId , productId ).
  • Используйте name маршрута и объектный синтаксис для сложных переходов.
  • Документируйте структуру маршрута и параметры для команды.
  • Проверяйте, что параметры действительно существуют, чтобы избежать ошибок.
  • Добавляйте обработку 404 для несуществующих id.
Чистая структура маршрутов облегчает поддержку и расширение приложения!
