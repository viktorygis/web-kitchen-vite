Customer Journey Map (CJM)

Customer Journey Map (CJM) — это визуальная карта пути пользователя, которая показывает этапы взаимодействия клиента с продуктом, его цели, действия, эмоции и возникающие проблемы.

Product Manager использует CJM для понимания пользовательского опыта, поиска точек улучшения и формирования продуктовых гипотез.

Зачем нужен CJM

  • Понять реальный путь пользователя.
  • Найти проблемные точки и барьеры.
  • Определить моменты максимальной ценности.
  • Улучшить UX и продуктовые сценарии.
  • Синхронизировать команду вокруг пользователя.

Структура Customer Journey Map

Элемент Описание Вопрос PM
Персона пользователя Описание целевого сегмента клиента. Кто наш пользователь?
Этап пути Последовательность взаимодействия с продуктом. Какие шаги проходит клиент?
Цель пользователя Что пользователь хочет получить. Какую задачу решает клиент?
Действия Что делает пользователь на каждом этапе. Какие действия выполняются?
Боли и проблемы Сложности и негативный опыт. Где пользователь испытывает трудности?
Возможности Идеи улучшений продукта. Что можно изменить?

Как создать Customer Journey Map

Шаг Действие Результат
1. Определить пользователя Выбрать сегмент или persona. Описание целевого клиента.
2. Собрать данные Интервью, аналитика, отзывы пользователей. Реальный пользовательский сценарий.
3. Описать путь Зафиксировать этапы взаимодействия. Карта Journey.
4. Найти проблемы Определить боли и точки оттока. Список гипотез улучшения.
5. Проверить решения Создать эксперименты и улучшения. Новые продуктовые инициативы.
Совет: Не создавайте CJM только на основе предположений. Используйте реальные данные: интервью, аналитику и обратную связь клиентов.

Когда использовать CJM

Ситуация Как помогает CJM
Discovery Помогает понять проблему пользователя перед созданием решения.
Улучшение UX Находит сложные места пользовательского опыта.
Снижение Churn Помогает определить причины ухода пользователей.
Новые функции Показывает, где новая функция создаст ценность.
Командная синхронизация Создаёт единое понимание пользователя.

CJM и другие Product Management фреймворки

Фреймворк Связь с CJM

Service Blueprint Расширяет CJM внутренними процессами компании.

Roadmap Переводит найденные проблемы в инициативы.

FAQ — Customer Journey Map

User Flow показывает последовательность действий пользователя, а CJM включает цели, эмоции, боли и контекст взаимодействия.

Обычно Product Manager работает над CJM вместе с UX-дизайнерами, аналитиками и исследовательской командой.

CJM стоит пересматривать при изменении продукта, аудитории или пользовательского поведения.
